# 项目模块拆解清单

以下是对 `d:/AITestProduct` 项目目录的结构化拆解，按功能子模块归类。

| 目录 | 对应业务子模块 | 说明 |
|------|----------------|------|
| **AITestProduct/** | 主后端 Django 项目（核心配置 & 入口） | 包含 `settings.py`、`urls.py`、`wsgi.py`、`celery.py`、`asgi.py` 等，负责项目整体初始化、路由分发、异步任务调度等。 |
| **frontend/** | 前端 Vue 3 单页面应用 (SPA) | 包含 `src/`、`components/`、`views/`、`router/` 等，负责 UI 渲染与后端 API 交互。 |
| **common/** | 公共库/工具类 | 提供业务无关的通用函数、异常基类、响应包装、序列化工具、Django 中间件及常用服务基类，前后端均可复用。 |
| **data/** | 向量存储 / 知识库数据 | 包含 `chroma/` 目录及 SQLite 数据库，用于存放 AI 语义向量、知识文章等，支持 RAG（检索增强生成）功能。 |
| **defect/** | 缺陷/错误管理模块 | 实现缺陷（Bug）模型、序列化、视图以及 RESTful API，提供缺陷创建、查询、状态流转等功能。 |
| **execution/** | 用例执行引擎 & 性能测试 | 包含调度器、WebSocket 中间件、K6 负载测试集成、执行日志、报告序列化等，支撑自动化测试与性能评估。 |
| **project/** | 测试项目与发布计划管理 | 管理测试项目、发布计划、关联缺陷、pipeline 日志等，提供项目层面的状态追踪与统计。 |
| **redis/** | 本地 Redis 实例 & 配置 | 包含 Windows 版 Redis 可执行文件、配置文件与日志，用于缓存、Celery broker、任务状态存储等。 |
| **scripts/** | 运维/初始化脚本 | 启动脚本、数据库初始化、数据迁移、文档生成等辅助脚本。 |
| **templates/** | 后端模板（Django） | 用于渲染邮件、报表或管理后台的 HTML 模板。 |
| **testcase/** | 用例库 | 存放测试用例的定义文件（可能是 JSON/YAML/Markdown），以及用例编辑/导入导出逻辑。 |
| **tests/** | 单元/集成测试 | 项目自身的 pytest/单元测试代码，验证后端/前端功能。 |
| **user/** | 用户/权限管理 | 包含用户模型、角色、权限校验、登录/注册等相关实现。 |
| **assistant/** | AI 助手与 RAG 服务 | 实现 AI 生成测试用例、知识库检索、语义去重、AI 驱动的 UI 自动化等模块。 |
| **docs/** | 文档资源 | 项目说明、使用手册、接口文档等静态文件。 |
| **开发文档/** | 项目内部开发文档 (中文) | 您提供的需求文档、模块说明等，供参考。 |
| **server_logs/** | 运行时日志目录 | 后端日志、错误日志、访问日志等，便于排查异常。 |
| **htmlcov/** | 覆盖率报告 | `pytest` 生成的 HTML 覆盖率报告，供质量检查。 |
| **UI 脚本** (`start-*.bat`, `start-dev.ps1` 等) | 本地启动/部署脚本 | 用于全栈启动、单独后端或前端调试、Celery 启动等。 |

请确认该清单是否完整，或指出需要补充/修改的部分。
