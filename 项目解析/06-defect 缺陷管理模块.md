# defect 缺陷管理模块

## 模块概述
负责缺陷的创建、追踪、状态管理、缺陷与版本的关联等功能。

## 文件列表

### __init__.py
- **作用**: Python 包初始化

### admin.py
- **作用**: Django Admin 后台配置
- **业务功能**: 配置后台管理界面中显示的缺陷相关模型

### apps.py
- **作用**: 应用配置
- **业务功能**: 定义 DefectConfig

### models.py
- **作用**: 数据模型定义
- **业务功能**:
  - `TestDefect`: 缺陷主模型
    - 缺陷编号、标题
    - 关联版本
    - 严重程度、优先级
    - 状态（新缺陷/处理中/已拒绝/已关闭）
    - 处理人
    - 所属模块
    - 缺陷内容、复现步骤
    - 附件列表
    - 发生环境

### serialize.py
- **作用**: 序列化器定义
- **业务功能**: 缺陷数据序列化/反序列化

### tests.py
- **作用**: 单元测试入口

### urls.py
- **作用**: 缺陷模块路由
- **业务功能**: 定义缺陷管理相关 API 路径

### views.py
- **作用**: 缺陷视图
- **业务功能**:
  - 缺陷 CRUD
  - 缺陷状态流转
  - 缺陷关联版本和模块
  - 缺陷列表查询与筛选
