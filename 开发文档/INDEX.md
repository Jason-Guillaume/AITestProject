## 📊 核心模块地图

| 模块 | 后端目录 | API 前缀 | 关键能力 |
|------|----------|----------|----------|
| 用户/组织/系统消息/审批 | `user/` | `/api/user/`、`/api/sys/`、`/api/change-requests/` | 登录注册、用户中心、组织成员、敏感信息审批、AI 配置管理 |
| 项目管理 | `project/` | `/api/project/` | 项目、任务、发布计划、测试计划 |
| 测试资产 | `testcase/` | `/api/testcase/`、`/api/environments/` | 用例 CRUD、执行入口、导入、方案图片、环境管理 |
| 执行与报表 | `execution/` | `/api/execution/`、`/api/perf/` | 计划、报告、质量指标、性能任务 |
| 缺陷管理 | `defect/` | `/api/defect/` | 缺陷 CRUD、状态流转 |
| AI 助手 | `assistant/` | `/api/ai/`、`/api/assistant/` | 连接校验、同步/流式生成、RAG、去重、配置治理 |
| 服务器日志 | `server_logs/` | `/api/server-logs/` + `/ws/logs/` | 主机加密管理、组织共享、WS 实时 tail、AI 诊断、审计 |
| 公共模块 | `common/` | - | 基础模型、审计服务 |

## 🔄 核心业务流程

### 1. AI 生成用例流程
```
需求文档 → 知识库 RAG → AI 生成 → 去重 → 人工审核 → 入库
```

### 2. 测试执行流程
```
用例选择 → 环境配置 → 触发执行 → WebSocket 推送 → 结果记录
```

### 3. 日志分析流程
```
SSH 采集/ES 检索 → AI 分析 → 工单草稿 → 人工确认 → 创建缺陷
```

### 4. 性能测试流程
```
API 用例 → AI 转 k6 脚本 → 执行压测 → 指标采集 → 报告生成
```
