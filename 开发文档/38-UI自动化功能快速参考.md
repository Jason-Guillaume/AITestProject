# UI自动化脚本上传与执行 - 快速参考指南

## 快速开始

### 1. 上传脚本

```bash
# 上传单个Python文件（LINEAR模式）
curl -X POST http://localhost:8000/api/assistant/ui-scripts/ \
  -F "name=登录测试" \
  -F "script_type=LINEAR" \
  -F "entry_point=test_login.py" \
  -F "file_path=@test_login.py"

# 上传ZIP包（POM模式）
curl -X POST http://localhost:8000/api/assistant/ui-scripts/ \
  -F "name=POM测试套件" \
  -F "script_type=POM" \
  -F "entry_point=tests/test_login.py" \
  -F "file_path=@test_suite.zip"
```

### 2. 执行脚本

```bash
# 执行脚本
curl -X POST http://localhost:8000/api/assistant/ui-script-executions/execute/ \
  -H "Content-Type: application/json" \
  -d '{"script_id": 1, "triggered_by": "manual"}'
```

### 3. 查询日志

```bash
# 获取执行日志
curl http://localhost:8000/api/assistant/ui-script-executions/1/logs/

# 获取最新100条日志
curl http://localhost:8000/api/assistant/ui-script-executions/1/logs/?start=-100&end=-1
```

### 4. 查询状态

```bash
# 获取执行状态详情
curl http://localhost:8000/api/assistant/ui-script-executions/1/status_detail/

# 查询执行历史
curl http://localhost:8000/api/assistant/ui-script-executions/?script_id=1
```

## Python代码示例

### 上传脚本

```python
import requests

# 上传LINEAR脚本
with open('test_script.py', 'rb') as f:
    response = requests.post(
        'http://localhost:8000/api/assistant/ui-scripts/',
        files={'file_path': f},
        data={
            'name': '测试脚本',
            'script_type': 'LINEAR',
            'entry_point': 'test_script.py'
        }
    )

script_id = response.json()['id']
```

### 执行脚本

```python
# 执行脚本
response = requests.post(
    'http://localhost:8000/api/assistant/ui-script-executions/execute/',
    json={'script_id': script_id}
)

execution_id = response.json()['id']
```

### 查询日志

```python
import time

# 等待执行完成
time.sleep(5)

# 获取日志
response = requests.get(
    f'http://localhost:8000/api/assistant/ui-script-executions/{execution_id}/logs/'
)

logs = response.json()['logs']
for log in logs:
    print(f"[{log['type']}] {log['message']}")
```

## 常见问题

### Q1: 脚本执行超时怎么办？
A: 修改 `settings.UI_SCRIPT_TIMEOUT` 配置，或在执行时传入 `timeout` 参数。

### Q2: 日志丢失了？
A: Redis日志默认保留24小时，超时会自动清理。建议及时查询或持久化到文件。

### Q3: 如何查看工作空间文件？
A: 调用 `/api/assistant/ui-scripts/{id}/workspace_info/` 接口。

### Q4: 支持并发执行吗？
A: 当前版本支持，但建议控制并发数量，避免资源耗尽。
