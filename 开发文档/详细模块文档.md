# 详细模块文档

本文件对 **AITestProject** 的核心业务模块进行 **底层实现 → 上层交互 → 可视化展示** 的完整剖析，帮助开发、测试、运维人员快速定位实现细节。

## 📦 公共基础模块 (`common`)

### 数据库模型（抽象基类）
```python
class BaseModel(models.Model):
    creator = models.ForeignKey(settings.AUTH_USER_MODEL, on_delete=models.SET_NULL, null=True, blank=True, related_name="%(class)s_created", verbose_name="创建人")
    updater = models.ForeignKey(settings.AUTH_USER_MODEL, on_delete=models.SET_NULL, null=True, blank=True, related_name="%(class)s_updated", verbose_name="更新人")
    create_time = models.DateTimeField(auto_now_add=True, verbose_name="创建时间")
    update_time = models.DateTimeField(auto_now=True, verbose_name="更新时间")
    is_deleted = models.BooleanField(default=False, verbose_name="是否已删除")

    class Meta:
        abstract = True
```
* **软删实现**：`delete()` 被重写为仅设置 `is_deleted=True` 并保存。
* **审计**：`common/models.AuditEvent` 记录增删改操作，字段包括 `action、object_app、object_model、object_id、before、after、extra`。

### 序列化器、视图基类
* `BaseModelSerializers` 统一实现 `is_valid` 错误信息抽取。
* `BaseModelViewSet` 在 `perform_destroy` 中调用 `instance.delete()`（软删），并在 `list`/`retrieve` 自动过滤 `is_deleted=False`。
* `ApiResponse` 统一返回结构 `{code, msg, data}`，前端只关注 `code`。

## 🐞 缺陷管理模块 (`defect`)

### 数据库模型
```python
class TestDefect(BaseModel):
    defect_no = models.CharField(max_length=255, verbose_name="缺陷编号")
    defect_name = models.CharField(max_length=255, verbose_name="缺陷标题")
    release_version = models.ForeignKey('project.ReleasePlan', on_delete=models.SET_NULL, null=True, verbose_name="关联版本")
    severity = models.IntegerField(choices=[(1,'致命'),(2,'严重'),(3,'一般'),(4,'建议')], default=3, verbose_name="严重程度")
    priority = models.IntegerField(choices=[(1,'高'),(2,'中'),(3,'低')], default=2, verbose_name="优先级")
    status = models.IntegerField(choices=[(1,'新缺陷'),(2,'处理中'),(3,'已拒绝'),(4,'已关闭')], default=1, verbose_name="状态")
    handler = models.ForeignKey('user.User', on_delete=models.PROTECT, related_name='handled_defects', verbose_name="当前处理人")
    module = models.ForeignKey('testcase.TestModule', on_delete=models.SET_NULL, null=True, blank=True, related_name='defects', verbose_name="所属模块")
    defect_content = models.TextField(null=True, blank=True, verbose_name="缺陷内容")
    reproduction_steps = models.JSONField(default=list, blank=True, verbose_name="复现步骤")
    attachments = models.JSONField(default=list, blank=True, verbose_name="附件列表")
    environment = models.CharField(max_length=255, blank=True, default='', verbose_name="发生环境")

    class Meta:
        db_table = 'test_defect'
        constraints = [models.UniqueConstraint(fields=['defect_no', 'is_deleted'], name='uniq_test_defect_no_is_deleted')]
```

### 关键业务流程
1. **创建缺陷** – `defect/views.py` 的 `DefectViewSet.create` 使用 `TestDefectSerializer`，自动填充 `creator`、`create_time`。
2. **状态变更** – 自定义 `@action(detail=True, methods=['patch'])` `set_status`，在保存前创建 `AuditEvent` 记录前后快照。
3. **软删** – 通过继承 `BaseModelViewSet.perform_destroy`，仅将 `is_deleted` 设为 `True`，列表接口已默认过滤。

### API（核心）
| 方法 | 路径 | 序列化器 | 返回字段（列表） |
|------|------|----------|-------------------|
| GET | /api/defects/ | `TestDefectSerializer` | `id, defect_no, defect_name, severity, priority, status, handler, environment, module, create_time` |
| POST | /api/defects/ | `TestDefectSerializer` | 同上（创建） |
| GET | /api/defects/{id}/ | `TestDefectDetailSerializer` | **全部字段**（包括 `defect_content`, `reproduction_steps`, `attachments`） |
| PATCH | /api/defects/{id}/status/ | `TestDefectSerializer` | 仅更新 `status` 并触发审计 |
| DELETE | /api/defects/{id}/ | – | 软删 |

### 前端实现（Vue）
* **列表页面** `src/views/Defect/DefectList.vue` 使用 Element‑Plus Table，调用 `api/defect.ts` 的 `fetchDefects()`。
* **详情抽屉** `DefectDetailDrawer.vue` 调用 `GET /api/defects/{id}/`，展示 `defect_content`、`reproduction_steps`、`attachments`（列表中不显示）。
* **状态按钮** 调用 `PATCH /api/defects/{id}/status/`，成功后 Pinia store 重新请求列表并在 “系统审计” 页面展示对应 `AuditEvent`。

## ⚙️ 执行与调度模块 (`execution`)

### 数据库模型
```python
class ExecutionTask(BaseModel):
    plan = models.ForeignKey('project.ReleasePlan', on_delete=models.CASCADE)
    status = models.CharField(max_length=32, choices=[('pending','待执行'),('running','执行中'),('finished','已完成')])
    result = models.JSONField(null=True, blank=True)  # {case_id: {status:'passed', log:'url/...'}}
```

### 工作流概述
1. 前端 `src/views/Execution/Execute.vue` 调用 `POST /api/executions/`，创建 `ExecutionTask`（状态 `pending`），返回任务 `id`。
2. Celery worker（`scripts/executor.py`）监听任务队列，取出任务后把 `status` 设置为 `running`，执行关联的 **TestCase**（通过内部 `TestRunner`），把每一步结果写入 `result` JSON。
3. 完成后 `status` 改为 `finished`，前端可以通过 **WebSocket** 或轮询 `GET /api/executions/{id}/` 获取实时进度与日志链接。

### API
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/executions/ | 创建执行任务 |
| GET  | /api/executions/{id}/ | 查询任务状态、结果 |

### 前端实现要点
* `Execute.vue` 提交后弹出 **进度弹窗**，内部使用 `setInterval` 每 3 秒轮询任务状态。
* `ExecutionDetail.vue` 根据 `result` 中的 `log` URL 使用 `<a>` 下载或在新窗口打开日志文件。

## 👥 用户、组织、角色模块 (`user`)

### 数据库模型
```python
class Organization(BaseModel):
    name = models.CharField(max_length=255)

class Role(BaseModel):
    name = models.CharField(max_length=64)
    permissions = models.JSONField(default=list)   # 例: ["defect:read", "defect:update"]
```
`User`（Django 自带）通过 `ManyToManyField` 与 `Organization`、`Role` 关联，实现 **细粒度权限**。

### 认证流程
1. 前端登录页面 `src/views/Auth/Login.vue` 调用 `POST /api/token/obtain/`，获取 `access`、`refresh`。
2. `access` 存入 `localStorage`，axios 拦截器在每次请求 Header 中注入 `Authorization: Bearer <access>`。
3. `refresh` 用于 token 过期后自动刷新（`POST /api/token/refresh/`）。

### 权限校验
后端在 `settings.py` 配置 `REST_FRAMEWORK['DEFAULT_PERMISSION_CLASSES']` 为 `IsAuthenticated`，自定义 `HasPermission` 读取 `request.user.roles` 中的 `permissions` 列表，决定是否允许访问对应的 **ViewSet**。

### 前端角色控制
在路由守卫 (`router/index.js`) 中读取 Pinia store 的 `currentUser.permissions`，若路由 `meta.requiresPermission` 不在列表内则跳转到 **403** 页面。

## ❓ 常见问题与排查 (FAQ)
1. **软删记录仍出现在列表** – 检查对应 `ViewSet` 是否继承自 `BaseModelViewSet` 或自行在 `queryset` 添加 `filter(is_deleted=False)`。
2. **前端 401/403** – 检查 JWT 是否过期，`src/store/auth.ts` 是否正确存储并在 axios 拦截器中添加 Header。
3. **Celery 任务不执行** – 确认 Redis 已启动，`config.yaml` 中 `celery.broker_url` 与实际地址匹配。
4. **缺陷状态变更未记录审计** – 确认 `defect/views.py` 中的 `set_status` 动作是否调用 `AuditEvent.objects.create(...)` 保存审计记录。

### 功能概述
提供基于大型语言模型（LLM）的 **自动化脚本生成、代码审查、智能提示** 等功能，供前端或后端通过 HTTP 接口调用。

### 主要入口
* `assistant/views.py` – Django `APIView`，接收 `POST /api/assistant/generate/`，返回生成的脚本内容或审查结果。
* `assistant/urls.py` – 注册路由 `router.register('assistant', AssistantViewSet)`（若使用 ViewSet）或手动 `path('generate/', ...)`。

### 请求/响应示例
```json
POST /api/assistant/generate/
{
  "prompt": "写一个 Python 脚本用于导出缺陷报告为 CSV",
  "language": "python"
}

Response:
{
  "code": 0,
  "msg": "成功",
  "data": {
    "script": "import csv\n..."
  }
}
```

### 前端调用示例 (`src/api/assistant.ts`)
```ts
import axios from '@/utils/axios';

export const generateScript = (prompt: string, language: string) =>
  axios.post('/api/assistant/generate/', { prompt, language });
```

### 使用场景
* 自动生成 **导出报告**、**配置脚本**、**单元测试模板**。
* 对提交的代码进行 **静态审查**，返回潜在问题列表。

## 🗄️ 内存与缓存 (`memory`)

项目使用 **Redis** 作为统一的缓存层，缓存策略分为两类：
1. **短期缓存**（TTL 5‑15 分钟）用于 **查询列表**、**权限检查**，键名 `cache:<model>:list:<query_hash>`。
2. **长期缓存**（TTL 24 小时以上）用于 **字典数据**（如角色权限映射），键名 `cache:dict:<name>`。

在代码中通过封装的 `cache_utils.py` 提供 `get`, `set`, `invalidate` 接口，确保业务层统一使用。

## 📈 调用链图示例（文字描述）

以下为各主要模块的关键业务调用链文字化示例，便于在实际项目中绘制 **UML 时序图** 或 **PlantUML**：

### 1️⃣ 缺陷创建流程
1. 前端 `DefectList.vue` → `api/defect.ts` 的 `createDefect(payload)` 发起 **POST /api/defects/** 请求。
2. Django `DefectViewSet.create` 接收请求，调用 `serializer.save(creator=request.user)`。
3. `BaseModel.save()` 触发 **pre_save** 信号 → `AuditEvent` 记录 `action='create'`（记录 `before=None`、`after=serializer.data`）。
4. 同步 **缓存**：`cache_utils.set('cache:test_defect:list:<hash>', ...)`，更新列表缓存。
5. 响应返回 `ApiResponse.success(data=serializer.data)`，前端更新 Pinia store 并刷新列表。

### 2️⃣ 用例执行任务流程
1. 前端 `Execute.vue` 调用 `api/execution.ts` 的 `runExecution(planId)` → **POST /api/executions/**。
2. `ExecutionViewSet.create` 创建 `ExecutionTask`（status=`pending`），并通过 **Celery** `apply_async` 把任务加入 Redis 队列。
3. Celery worker（`scripts/executor.py`）读取任务，设置 `execution.status='running'` 并保存。
4. 逐条调用 `TestRunner.run(testcase)`，每一步产生日志并写入 `execution.result` JSON。
5. 完成后 `execution.status='finished'`，触发 **post_save** 信号 → `AuditEvent` 记录 `action='execute'`，并通过 **WebSocket** 向前端推送 `execution_finished` 消息。
6. 前端轮询或接收 WebSocket，更新 UI 中的进度条与结果展示。

### 3️⃣ AI 助手脚本生成流程
1. 前端 `AssistantGenerate.vue` 调用 `api/assistant.ts` 的 `generateScript({prompt, language})` → **POST /api/assistant/generate/**。
2. `AssistantViewSet.generate` 接收请求，内部调用 `LLMClient.generate(prompt, language)`（封装的 OpenAI/Claude SDK），获取脚本字符串。
3. 返回 `ApiResponse.success(data={'script': script})`，前端在弹窗中显示并提供复制/下载按钮。

## 💻 示例代码片段

### 1. 缺陷序列化器 (`defect/serialize.py`)

from rest_framework import serializers
from .models import TestDefect
from common.serialize import BaseModelSerializers


class TestDefectSerializer(BaseModelSerializers):

### 2. 缺陷视图集 (`defect/views.py`)
```python
from rest_framework import viewsets, status
from rest_framework.decorators import action
from rest_framework.response import Response
from .models import TestDefect
from .serialize import TestDefectSerializer, TestDefectDetailSerializer
from common.response import ApiResponse
from common.models import AuditEvent


class DefectViewSet(viewsets.ModelViewSet):
    queryset = TestDefect.objects.filter(is_deleted=False)
    serializer_class = TestDefectSerializer

    def retrieve(self, request, *args, **kwargs):
        self.serializer_class = TestDefectDetailSerializer
        return super().retrieve(request, *args, **kwargs)

    @action(detail=True, methods=['patch'], url_path='status')
    def set_status(self, request, pk=None):
        obj = self.get_object()
        old_status = obj.status
        new_status = request.data.get('status')
        obj.status = new_status
        obj.save()
        # 创建审计记录
        AuditEvent.objects.create(
            action=AuditEvent.ACTION_UPDATE,
            object_app='defect',
            object_model='TestDefect',
            object_id=obj.id,
            before={'status': old_status},
            after={'status': new_status},
            extra={'user': request.user.id}
        )
        return Response(ApiResponse.success(data={'status': new_status}), status=status.HTTP_200_OK)
```
