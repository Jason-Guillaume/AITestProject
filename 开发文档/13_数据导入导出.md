# 数据导入与导出模块（data）

## 功能概述
`data` 目录用于存放项目的 **初始化数据、种子文件、导入导出脚本**，帮助快速搭建测试环境、迁移历史数据以及提供供前端/后端使用的静态 JSON/CSV 示例。

## 主要内容
| 文件/子目录 | 说明 |
|-------------|------|
| `seed/` | 项目启动时可通过 `manage.py loaddata` 导入的 Django fixtures（JSON）文件，例如默认组织、权限、系统消息模板。 |
| `sample_json/` | 用于 API 示例请求或前端演示的 JSON 数据片段。 |
| `sample_csv/` | CSV 示例文件，常配合 `tests/` 中的导入测试使用。 |
| `import_scripts/` | Python 脚本（如 `import_projects.py`）实现批量读取 CSV/Excel 并写入对应模型的逻辑。 |
| `export_scripts/` | 导出业务数据为 CSV/Excel 的脚本，常用于运营报表或迁移。 |

## 关键代码示例
### Django fixture 示例 (`data/seed/initial_data.json`)
```json
[
  {"model": "user.organization", "pk": 1, "fields": {"name": "Default Org"}},
  {"model": "user.user", "pk": 1, "fields": {"username": "admin", "is_superuser": true, "is_staff": true, "password": "pbkdf2_sha256$..."}}
]
```

### 导入脚本示例 (`data/import_scripts/import_projects.py`)
```python
import csv
from project.models import TestProject

def import_projects(csv_path: str):
    with open(csv_path, newline='', encoding='utf-8') as f:
        reader = csv.DictReader(f)
        for row in reader:
            TestProject.objects.update_or_create(
                name=row['name'], defaults={
                    'description': row.get('description', ''),
                    'owner_id': row.get('owner_id')
                }
            )

if __name__ == '__main__':
    import_projects('sample_csv/projects.csv')
```

### 导出脚本示例 (`data/export_scripts/export_reports.py`)
```python
import csv
from execution.models import TestReport

def export_reports(csv_path: str):
    reports = TestReport.objects.select_related('plan').all()
    with open(csv_path, 'w', newline='', encoding='utf-8') as f:
        writer = csv.writer(f)
        writer.writerow(['id', 'plan_name', 'status', 'created_at'])
        for r in reports:
            writer.writerow([r.id, r.plan.name, r.status, r.created_at])

if __name__ == '__main__':
    export_reports('reports/report_export.csv')
```

## 为什么使用这些技术
- **Django fixtures**：标准化的 JSON 结构，可直接通过 `manage.py loaddata`/`dumpdata` 进行迁移，保持环境一致性。
- **CSV/Excel 脚本**：业务数据量大时，CSV 更易于编辑和批量处理；使用原生 `csv` 模块避免额外依赖。
- **分离脚本**：将导入/导出逻辑放在 `data` 目录，保持业务代码 (`project`, `execution` 等) 干净，仅在运维或 CI 环境调用。

## 常见错误与排查
1. **fixture 导入冲突**：`loaddata` 时主键冲突导致报错，确保 fixture 中的 `pk` 与现有库不冲突，或使用 `--ignorenonexistent` 参数。
2. **CSV 编码问题**：默认使用 UTF‑8 编码，如出现乱码请检查文件是否为 UTF‑8‑BOM 或使用 `encoding='utf-8-sig'`。
3. **字段映射错误**：导入脚本中字段名必须与模型字段一致，推荐使用 `Model._meta.get_fields()` 动态校验。
4. **导出权限**：导出脚本可能泄露敏感信息，建议在生产环境加入权限检查或仅在受限目录生成。
