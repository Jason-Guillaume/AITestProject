# 01-项目概述与快速启动

## 0. 需求来源与开发动因

- 业务价值摘要：统一认知入口，显著降低项目上手与跨角色协作成本。
- 业务背景：团队需要面向新成员与跨角色协作的统一入口文档，降低“环境怎么起、系统做什么、从哪里看”的沟通成本。
- 现状痛点：项目信息长期分散在代码、口头说明和零散资料中，导致上手慢、联调慢、排查慢。
- 建设目标：通过“项目概述 + 快速启动 + 路线指引”建立标准认知入口，统一协作上下文。
- 预期收益：缩短新人熟悉周期，减少重复沟通，提升跨职能协作效率。

## 1. 项目背景

SmartTest 是一个面向软件测试全流程的 AI 辅助平台，集成了测试用例管理、缺陷跟踪、性能测试、测试计划执行等核心功能，并通过大模型技术提供智能化的测试用例生成与分析能力。

### 核心定位
- **全流程覆盖**：从测试设计、用例编写、计划执行到缺陷管理，形成完整闭环
- **AI 赋能**：基于大模型的测试用例自动生成、RAG 相关性检索、去重优化
- **性能测试**：支持 JMeter/ Locust 场景的压测任务管理与执行
- **组织协作**：多组织架构、角色权限、系统消息与审批流程

## 4. 安装流程

### 4.1 后端安装

```bash
# 克隆项目
cd D:\高四\AITestProduct

# 创建并激活虚拟环境（推荐）
python -m venv venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 执行数据库迁移
python manage.py migrate

# （可选）创建超级用户
python manage.py createsuperuser

# （可选）加载性能任务演示数据
python manage.py seed_perf_tasks
```

**Windows 特殊说明**：
若 `mysqlclient` 安装失败，可改用 `pymysql`：
```bash
pip uninstall mysqlclient -y
pip install pymysql
```
并在 `AITestProduct/__init__.py` 中添加：
```python
import pymysql
pymysql.install_as_MySQLdb()
```

### 4.2 前端安装

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

默认访问地址：`http://localhost:5173`

## 5. 本地启动步骤

### 5.1 启动后端

```bash
# 确保虚拟环境已激活
python manage.py runserver
```

后端服务地址：`http://127.0.0.1:8000`

补充（实时能力 / WebSocket）：

- 若你要使用 **WebSocket**（如 k6 实时指标、服务器日志实时 tail），需要用 **ASGI 进程**承载（`runserver` 仅保证 HTTP 可用）。开发环境示例：

```bash
# 需要安装 daphne（requirements.txt 已包含时可跳过）
daphne -b 127.0.0.1 -p 8000 AITestProduct.asgi:application
```

更多对账信息（WS 路由、鉴权方式、运行拓扑）见：`开发文档/99-项目全局扫描对账索引.md`。

**开发模式特性**：
- `DEBUG = True`：显示详细错误信息
- 静态文件由 Django 自动托管（`/media/`）
- CORS 需额外配置时可安装 `django-cors-headers`

### 5.2 启动前端

```bash
cd frontend
npm run dev
```

前端服务地址：`http://localhost:5173`

**代理配置**（`vite.config.js`）：
- `/api` → `http://127.0.0.1:8000`
- `/media` → `http://127.0.0.1:8000`
- `/ws` → `http://127.0.0.1:8000`（WebSocket 代理；用于 server_logs/k6 等实时通道）

### 5.3 启动 Redis（可选）

```bash
# Windows（若使用 WSL 或服务）
redis-server

# Linux/Mac
redis-server /etc/redis/redis.conf
```

Redis 未启动时，项目会自动降级为内存缓存（LocMem），但验证码等功能会受限。

## 6. 项目打包与部署

### 6.1 前端构建

```bash
cd frontend

# 生产构建
npm run build

# 预览构建产物
npm run preview
```

构建产物目录：`frontend/dist/`

### 6.2 生产环境部署建议

**后端**：
1. 修改 `AITestProduct/settings.py`：
   - `DEBUG = False`
   - 设置 `SECRET_KEY` 为安全的随机字符串
   - 配置 `ALLOWED_HOSTS`
   - 使用环境变量管理数据库密码
2. 使用 Gunicorn/Uvicorn + Nginx 部署
3. 启用 Redis 缓存
4. 配置 HTTPS

**前端**：
1. 构建生成静态文件
2. 通过 Nginx/Apache 提供静态服务
3. 配置反向代理 `/api` 到后端

## 7. 首次使用流程

1. **启动服务**
   ```bash
   # 终端1：启动后端
   python manage.py runserver
   
   # 终端2：启动前端
   cd frontend && npm run dev
   ```

2. **访问平台**
   - 打开浏览器访问 `http://localhost:5173`
   - 默认重定向到 `/dashboard`

3. **登录系统**
   - 若已创建超级用户，使用账号密码登录
   - 或注册新用户（需配置邮件服务）

4. **验证性能任务接口**
   - 访问 `/performance/tasks` 页面
   - 或访问 `http://127.0.0.1:8000/api/perf/tasks/` 查看 JSON 数据

## 8. 常见问题排查

### 8.1 后端启动问题

| 问题 | 解决方案 |
|------|----------|
| `ModuleNotFoundError: No module named 'django'` | 检查虚拟环境是否激活，重新 `pip install -r requirements.txt` |
| `django.db.utils.OperationalError: could not connect to server` | 检查 MySQL 是否启动，配置文件中的数据库名称/用户名/密码是否正确 |
| `Table 'xxx' doesn't exist` | 执行 `python manage.py migrate` |

### 8.2 前端启动问题

| 问题 | 解决方案 |
|------|----------|
| `npm ERR! code ELIFECYCLE` | 删除 `node_modules` 和 `package-lock.json`，重新 `npm install` |
| `EADDRINUSE: address already in use` | 修改 `vite.config.js` 中的 `port` |
| 跨域错误 | 确保使用 Vite 代理，不要直接访问后端域名 |

### 8.3 登录问题

| 问题 | 解决方案 |
|------|----------|
| `401 Unauthorized` | 检查 Token 是否正确设置到 `localStorage.token` |
| `403 Forbidden` | 检查用户是否已激活，或是否缺少权限 |
