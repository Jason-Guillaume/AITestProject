## Design Philosophy

### Why This Approach?
1. **Developer-Centric** - Manual input gives full control, supports any provider
2. **Professional** - IDE-inspired layout feels familiar to technical users
3. **Flexible** - No hardcoded options, works with any OpenAI-compatible API
4. **Secure** - Password field with toggle, local storage only
5. **Extensible** - Advanced section ready for future configuration options

### Removed Features (From Old Design)
- ❌ Hardcoded model dropdown (GLM, GPT, Claude options)
- ❌ Provider-specific logic (iFlytek MaaS detection)
- ❌ Connection status display
- ❌ Disconnect/Reconnect buttons
- ❌ System admin permission checks in UI

### Why Remove These?
The new design focuses on **configuration input** rather than **status management**. Connection testing is still available via the "测试连接" button, but ongoing status monitoring should be handled elsewhere (e.g., a separate dashboard or system status page).
