# 模板文件（templates）

## 功能概述
`templates` 目录存放 **Django 模板**，用于渲染系统邮件、系统消息、报告页面以及前端需要的 HTML 内容。模板采用 Django 自带的模板语言（类似 Jinja），支持变量、条件、循环以及自定义过滤器。

## 常见模板分类
| 子目录 | 用途 |
|--------|------|
| `emails/` | 系统邮件模板，如用户激活、密码重置、缺陷通知等。 |
| `messages/` | 系统消息、站内通知的 HTML 片段。 |
| `reports/` | 质量仪表盘、测试报告等可导出为 PDF 的 HTML 模板。 |
| `partials/` | 可复用的 UI 片段（表格、分页、树形结构等）。 |

## 示例模板片段
### 邮件激活模板 (`templates/emails/activate_account.html`)
```html
<p>亲爱的 {{ user.username }}，</p>
<p>请点击以下链接激活您的账号：</p>
<p><a href="{{ activation_url }}">{{ activation_url }}</a></p>
<p>如果不是您本人请求，请忽略此邮件。</p>
```

### 报告模板 (`templates/reports/quality_report.html`)
```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8" />
  <title>质量报告</title>
  <style>/* 简单样式，WeasyPrint 渲染时使用 */</style>
</head>
<body>
  <h1>项目 {{ data.project_name }} 质量概览</h1>
  <p>通过率：{{ data.pass_rate }}%</p>
  <p>缺陷密度：{{ data.defect_density }} /kLOC</p>
  <div id="chart">{{ chart_svg|safe }}</div>
</body>
</html>
```

## 技术要点
- **布局继承**：多数模板通过 `{% extends "base.html" %}` 统一布局，便于全局样式统一。
- **安全过滤**：使用 `{{ variable|escape }}` 防止 XSS；对 HTML 内容（如图表 SVG）使用 `|safe` 明确标记。
- **邮件发送**：在视图或任务中使用 `render_to_string` 将模板渲染为字符串，再交给 `django.core.mail.EmailMessage` 发送。

## 常见错误与排查
1. **模板未找到**：`TEMPLATES['DIRS']` 未包含项目根目录的 `templates` 路径，检查 `settings.py` 中的配置。
2. **变量未传递**：渲染时忘记在 `context` 中提供必需的变量，模板会报 `VariableDoesNotExist` 错误，确保视图或任务传递完整的上下文。
3. **字符编码**：邮件模板需使用 UTF‑8 编码，若出现乱码请在邮件头部声明 `Content-Type: text/html; charset=utf-8`。
4. **PDF 渲染缺失样式**：WeasyPrint 渲染模板为 PDF 时，外部 CSS 文件需要使用绝对路径或 `static` URL，否则样式丢失。
