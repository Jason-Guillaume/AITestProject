## 一、功能概述

### 1.1 业务背景
在阶段1完成脚本上传功能后，阶段2实现脚本的实际执行能力，包括：
- 脚本执行引擎
- 实时日志收集
- 执行状态管理
- 执行历史记录

### 1.2 核心功能
1. 支持LINEAR和POM两种模式的脚本执行
2. 使用pytest作为测试执行框架
3. 实时日志流式输出到Redis
4. 执行记录持久化到数据库
5. 支持异步执行和状态查询

---

## 二、架构设计

### 2.1 执行流程

```
用户触发执行
    ↓
创建执行记录（pending）
    ↓
启动异步执行线程
    ↓
更新状态为running
    ↓
构建执行环境（PYTHONPATH等）
    ↓
启动pytest子进程
    ↓
流式读取stdout/stderr → Redis
    ↓
等待进程完成
    ↓
更新执行记录（success/failed）
    ↓
返回执行结果
```

### 2.2 技术栈
- **执行框架**: pytest
- **进程管理**: subprocess.Popen
- **日志存储**: Redis List
- **状态存储**: Redis String + MySQL
- **异步执行**: threading

## 四、API接口文档

### 4.1 执行脚本

**接口**: `POST /api/assistant/ui-script-executions/execute/`

**请求参数**:
```json
{
  "script_id": 1,
  "triggered_by": "manual"
}
```

**响应**: HTTP 202 Accepted
```json
{
  "id": 1,
  "execution_id": "1_1714521600",
  "script": 1,
  "script_name": "登录测试脚本",
  "status": "pending",
  "triggered_by": "manual",
  "created_at": "2026-05-01T10:00:00Z"
}
```

### 4.2 获取执行日志

**接口**: `GET /api/assistant/ui-script-executions/{id}/logs/?start=0&end=-1`

**响应**:
```json
{
  "execution_id": "1_1714521600",
  "logs": [
    {
      "timestamp": "2026-05-01T10:00:01Z",
      "type": "system",
      "message": "开始执行脚本"
    }
  ],
  "total": 1
}
```

### 4.3 获取执行状态

**接口**: `GET /api/assistant/ui-script-executions/{id}/status_detail/`

### 4.4 查询执行历史

**接口**: `GET /api/assistant/ui-script-executions/?script_id=1&status=success`

## 七、使用示例

### 7.1 Python代码调用

```python
from assistant.utils.script_runner import execute_ui_script

result = execute_ui_script(
    script_upload_instance_id=1,
    execution_id='custom_id',
    timeout=1800
)

print(result)
# {'success': True, 'execution_id': 'custom_id', 'return_code': 0, 'duration': 12.34}
```

### 7.2 API调用

```bash
# 执行脚本
curl -X POST http://localhost:8000/api/assistant/ui-script-executions/execute/ \
  -H "Content-Type: application/json" \
  -d '{"script_id": 1}'

# 获取日志
curl http://localhost:8000/api/assistant/ui-script-executions/1/logs/
```
