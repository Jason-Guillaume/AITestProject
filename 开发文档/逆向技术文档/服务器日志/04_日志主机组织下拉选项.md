# 04 — 日志主机组织下拉选项（逆向）

## 1. 接口定位

- 类：`server_logs.views.LogServerOrganizationChoicesAPIView`（`APIView`）。
- 路由：`GET /api/server-logs/organization-choices/`（`server_logs/urls.py`）。
- 权限：`IsAuthenticated`。

## 2. 业务含义（与系统 org API 的差异）

- 视图类 docstring 说明：**返回当前用户可绑定到日志主机的组织列表**。
- 与 `/api/user/orgs/` 不同：后者往往仅系统管理员可访问；本接口面向普通登录用户，用于在「创建/编辑远程日志主机」时选择组织。

## 3. 查询逻辑

- 若 `user_is_platform_log_admin(u)`：`Organization.objects.filter(is_deleted=False)` 全量（仍可 `order_by` 截断）。
- 否则：`Organization` 过滤 `creator=u OR members=u`，去重后按 `org_name` 排序。
- 返回形态：JSON 数组，元素为 `{ "id", "org_name" }`，最多取前 500 条（`:500` 切片）。

## 4. 依赖模块

- 懒加载 `from user.models import Organization`，避免循环引用。
- 平台日志管理员判定：`server_logs.access.user_is_platform_log_admin`（超管、staff、`is_system_admin`）。

## 5. 前端使用建议

- 下拉无数据：用户可能不属于任何组织且非平台管理员——此时只能创建「不绑定组织」的主机（仅创建人可见，见 `RemoteLogServer` 模型说明）。
- 绑定组织后，主机可见性扩展到组织成员，与 `remote_log_server_queryset_for_user` 一致。

## 6. 扩展点

- 若未来组织数量超过 500，需要分页或关键词搜索，应新增 query 参数而非静默截断。
