# 03 — 历史检索与错误趋势聚合（Elasticsearch）（逆向）

## 1. `LogHistorySearchAPIView`

- 路由：`GET /api/server-logs/search/`，名称 `server-logs-search`。
- 权限：`IsAuthenticated`。
- 查询参数（内联 Serializer）：`server_id`（可选）、`keyword`（可选）、`limit`（默认 80，上限 500）。
- **空查询防护**：`must` 子句至少要有 `server_id` 或 `keyword` 之一，否则返回 `enabled: True` 但 `message` 提示「请至少提供…」，`results` 为空。
- ES 查询：`bool.must` 可组合 `term server_id` 与 `match message`；排序 `timestamp` 降序。
- 客户端缺失：若未安装 `elasticsearch` Python 库或连接失败，接口返回 **HTTP 200**，`enabled: False`，附带说明文案，**避免前端全局红条**。同时写 `SEARCH` 审计，`meta` 记录错误摘要。

## 2. `LogErrorTrendAPIView`

- 路由：`GET /api/server-logs/agg/error-trend/`。
- 用途：运维/压测趋势图；按时间桶聚合命中数。
- 参数：`server_id`（可选）、`keyword`（默认 `"ERROR"`）、`minutes`（1–1440，默认 60）、`interval`（默认 `30s`，Elasticsearch `date_histogram` 的 `fixed_interval`）。
- 时间范围：当前毫秒时间戳向前推 `minutes`；`extended_bounds` 与 `min_doc_count: 0` 便于补零画折线。
- 响应：`points: [{ ts, count }]`；ES 不可用时同样软失败 JSON。

## 3. 索引与客户端

- 索引名与客户端来自 `server_logs.es_client.get_elasticsearch_client` 与 `get_server_logs_es_index()`（具体环境变量/配置以 settings 为准）。

## 4. 与实时 tail 的关系

- 注释明确：**ES 写入失败不影响实时 WebSocket 推送**；检索接口只「尽力」查 ES。排障时应区分「看不到实时」与「搜不到历史」。

## 5. 安全与多租户

- GET 接口未在视图内按 `server_id` 二次鉴权；若部署在多租户环境，应评估是否需与 `remote_log_server_queryset_for_user` 对齐（当前实现依赖调用方只查询自己有权的 `server_id`，路由层未强制校验）。*若安全基线要求严格，应在后续迭代中补齐 server 维度权限校验。*
