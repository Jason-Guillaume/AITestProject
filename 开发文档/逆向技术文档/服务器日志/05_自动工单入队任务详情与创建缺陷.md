# 05 — 自动工单：入队、任务详情、创建缺陷（逆向）

## 1. 入队 `LogAutoTicketEnqueueAPIView`

- 路由：`POST /api/server-logs/auto-tickets/enqueue/`。
- 权限：`IsAuthenticated`；限流：`server_logs_analyze`。
- 请求：`LogAutoTicketEnqueueSerializer` 校验字段，包含 `server_id`、`anchor_text`、`anchor_ts`（可选）、`window_seconds`、`es_limit`、`create_defect` 及缺陷可选外键（处理人/发布版本/模块）等。
- 权限：`remote_log_server_queryset_for_user` 必须包含目标 `server_id`，否则 **403**。
- 落库：创建 `LogAutoTicketJob`，状态 `PENDING`，**API Key 等凭据不写数据库**（设计见模型 docstring）。
- Celery：`server_logs.tasks.run_log_auto_ticket_job.delay(...)`；若 `_CELERY_AVAILABLE` 为假或 `delay` 抛错，则更新 job 为 `FAILED` 并返回 **503**。
- 成功：返回 **202 Accepted**，`job_id`、`task_id`（回写 `celery_task_id`）。同时 `log_server_log_event`（`AUTO_TICKET`，phase=`enqueue`）与 `record_execute_audit`（`extra` 中带 `job_id`）。

## 2. 任务查询 `LogAutoTicketJobDetailAPIView`

- 路由：`GET /api/server-logs/auto-tickets/jobs/<job_id>/`。
- 可见性：**仅 job 创建人或平台日志管理员**；否则 403；不存在 404。
- 序列化：`LogAutoTicketJobSerializer`，`select_related` 含主机、用户、已创建缺陷及缺陷可选维度。

## 3. 手动建缺陷 `LogAutoTicketCreateDefectAPIView`

- 路由：`POST /api/server-logs/auto-tickets/jobs/<job_id>/create-defect/`。
- 前置条件：`job.status == SUCCESS`；`draft` 为有效 `dict`；尚未 `created_defect_id`（幂等：已创建则直接返回 `already_created: True`）。
- 实现：`server_logs.defect_create.create_test_defect_from_auto_ticket`；请求体可带 `defect_handler` 等覆盖（`LogAutoTicketCreateDefectSerializer` + 原始 dict 键检测）。
- 成功：关联 `job.created_defect`，更新 `meta.defect_create`，记审计与 `AuditEvent.ACTION_CREATE`（针对缺陷对象）。

## 4. 后台任务概要（`run_log_auto_ticket_job`）

- `select_for_update` 将 `PENDING` 置为 `PROCESSING`；拉 ES 上下文 `fetch_es_context_for_anchor`；调用 `analyze_log_ticket_draft` 生成 JSON `draft`；若 `create_defect_requested` 且成功可自动落库缺陷（任务内逻辑，见 `tasks.py` 后半段）。

## 5. 运维要点

- Broker/Worker 不可用表现为入队 503 或 job 长期 `PENDING`（视 worker 情况）。
- 与 `AnalyzeLogWithContextAPIView` 共享分析限流 scope，高峰时需观察 429。
