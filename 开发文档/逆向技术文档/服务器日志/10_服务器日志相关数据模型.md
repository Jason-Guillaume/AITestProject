# 10 — 服务器日志相关数据模型（逆向）

## 1. `RemoteLogServer`

- 表：`server_logs_remote_host`。
- 职责：SSH 主机配置；`password_enc` / `private_key_enc` 存密文；`organization` 可选绑定以实现组织级共享。
- 方法：`set_password` / `get_password`、`set_private_key` / `get_private_key` 封装加解密。

## 2. `ServerType`

- `TextChoices`：`linux` / `windows`，影响远程 tail 命令构造（`ssh_tail._build_remote_command`）。

## 3. `ServerLogAuditEvent`

- 表：`server_logs_audit_event`。
- 非 `BaseModel`：独立轻量审计表，`Action` 枚举含 `ws_connect`、`ws_disconnect`、`ws_stop`、`analyze`、`auto_ticket`、`search`、`host_create/update/delete`。
- 外键：`user`、`remote_log_server`、`organization` 均可空；`meta` JSON；`client_ip`；`created_at` 索引。

## 4. `LogAutoTicketJobStatus`

- `pending` / `processing` / `success` / `failed`。

## 5. `LogAutoTicketJob`

- 表：`server_logs_auto_ticket_job`。
- 字段：`user`、`remote_log_server`、`anchor_text`、`anchor_ts`、`window_seconds`、`es_limit`、状态、 Celery ID、错误信息、`draft` JSON、`meta`。
- 缺陷维度：`create_defect_requested`、`defect_handler`、`defect_release_version`、`defect_module`、`created_defect`（指向 `defect.TestDefect`）。
- **明确注释**：模型凭据不落库，仅任务参数传递 API Key。

## 6. 与发布/缺陷模块关系

- `defect_release_version` → `project.ReleasePlan`；`created_defect` → `defect.TestDefect`；`defect_module` → `testcase.TestModule`。
- 删除/归档发布计划或模块时需考虑外键 `SET_NULL` 或保护策略（以迁移为准）。

## 7. 索引与排序

- `RemoteLogServer`：`ordering = ("-update_time",)`。
- `LogAutoTicketJob`：`ordering = ("-created_at",)`，状态字段 `db_index=True` 便于轮询列表。

## 8. 测试数据建议

- 集成测试可使用 mock ES 与 mock Paramiko，避免真实 SSH；单元测试覆盖加解密与权限 queryset。
