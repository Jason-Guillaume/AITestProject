# 可复用组件：日志自动工单草稿（LogAutoTicketDraft）

## 1. 文件

- **`frontend/src/components/server_logs/LogAutoTicketDraft.vue`**

## 2. 职责

在 **服务器日志** 流程中：**提交 AI 分析片段 → 异步任务入队 → 轮询 job → 展示 draft JSON → 一键创建缺陷**。

## 3. API（`@/api/serverLogs`）

| 函数 | 说明 |
|------|------|
| `enqueueServerLogAutoTicket` | `POST /server-logs/auto-tickets/enqueue/`，超时 60s，可能 **202 Accepted** |
| `getServerLogAutoTicketJob` | `GET /server-logs/auto-tickets/jobs/{jobId}/` |
| `createDefectFromServerLogAutoTicketJob` | `POST .../jobs/{jobId}/create-defect/`，成功 201，已存在 200 `already_created`（注释语义） |

## 4. 轮询模式

- 组件内 **`setInterval` 或 async 循环**（以源码为准）调 **`getServerLogAutoTicketJob`**，直到 **success/failure** 终态。
- **注意**：浏览器隐藏标签页时，仍可能轮询；可结合 **`document.visibilityState`** 暂停（若未实现可作为改进）。

## 5. 与 `defect.js` 的边界

- **不** 直接调 **`createDefectApi`**（`/defect/defects/`）；创建缺陷走后端 **「从工单 job 派生」** 路径，保证 **溯源字段** 一致。

## 6. WebSocket

- **无**。日志 **实时 tail** 在 **`LogTerminal.vue`**；本组件只对 **已选片段/上下文** 做 **异步 HTTP**。

## 7. 主页面嵌入

- `server_logs/index.vue` 在分析区、折叠块或按钮中挂载；传参含 **host、log excerpt、组织** 等（以 props 为准）。

## 8. 超时与错误

- `enqueue` 60s、`create-defect` 60s（`serverLogs.js`）；失败 **`ElMessage.error`** 展示后端 `msg`。

## 9. 环境

- 普通 **Token**；若异步任务在后端队列，与前端环境无关。

## 10. 状态 UI

- **进行中** / **成功草稿** / **失败原因** 三分支；草稿可能为 Markdown 或 JSON（以后端为准）。

## 11. 小结

**异步 AI 工单** = **enqueue + poll**，与 **同步 **`analyzeServerLog`**（长超时）** 区分使用场景。

## 12. 与审计

- 主机侧 **`listServerLogAuditEvents`** 记录人工/自动操作；工单创建是否写审计由后端决定。

## 13. 测试数据

- Mock job 状态机：`queued` → `running` → `success` + `draft` → `create-defect`。

## 14. 安全

- draft 可能含 **敏感堆栈**；展示域需 **权限校验**（后端）。

## 15. 国际化

- 文案可中文；错误码映射可集中表。
