# 可复用组件：当前环境选择器（CurrentEnvironmentSelect）

## 1. 文件

- **`frontend/src/components/CurrentEnvironmentSelect.vue`**

## 2. 职责

- 下拉选择 **测试环境**（`TestEnvironmentOption`），与 **`useCurrentEnvironment`** 共享 **模块级 ref**，全应用 **单例环境上下文**。

## 3. 组合式（`frontend/src/composables/useCurrentEnvironment.ts`）

- **`environmentId`**：`ref<number | null>`，初始化读 **`localStorage`** 键 **`current_test_environment_id`**。
- **`watch(environmentId)`**：写入/删除 localStorage，并 **`dispatchEvent`** —— **`app:current-environment-changed`**，detail 含 **`environmentId`**。
- **`loadEnvironments()`**：**`GET /environments/?page_size=500`**，解析 `results` 或数组根。
- **`setEnvironmentId(id)`**：规范化数字 id。
- **`selectedEnvironment` / `baseUrl`**：`computed`，`baseUrl` 为选中项 **`base_url` trim**。

## 4. Props

- **`compact`**、**`showLabel`**、**`placeholder`**、**`variant`**：`default` | `navbar`（顶栏窄条样式 + tooltip）。

## 5. 使用场景

- **性能任务管理**工具栏：`CurrentEnvironmentSelect :show-label="true" compact`。
- **k6 看板**：脚本里直接 `useCurrentEnvironment()` 调 **`loadEnvironments`** + **`baseUrl`** 预填表单。
- 可在 **`MainLayout`** 顶栏嵌入 `variant="navbar"`（若产品需要统一入口；当前顶栏为自定义 dropdown + `getEnvironments`，两套可并存）。

## 6. API

- **`getEnvironments`**：`frontend/src/api/environment.js`。

## 7. WebSocket 与轮询

- 本组件 **不** 建立 WS；其它页可监听 **`app:current-environment-changed`** 后 **重连** 或 **刷新**（例如看板若未来按环境过滤会话）。

## 8. 与 `resolveApiUrlWithBase`

- **`useApiExecuteConsole`**（`composables/useApiExecuteConsole.ts`）内 **`resolveApiUrlWithBase`** / **`relativizeApiUrlIfUnderBase`** 使用同一 **`baseUrl`** 拼装接口用例真实 URL。

## 9. 空环境

- 若列表无记录，下拉为空；**`environmentId`** 可能在 **`loadEnvironments`** 后被置 **`null`**（选项不包含已存 id 时）。

## 10. SSR / 测试

- 依赖 **`window`** / **`localStorage`**；单测需 jsdom。

## 11. 小结

**一处选择、全局生效**；键名 **`current_test_environment_id`** 是联调关键。

## 12. 事件名称常量建议

- 若多处手写 `'app:current-environment-changed'`，可抽到 `constants/events.ts`（仓库现状为字符串字面量）。

## 13. 与组织/项目过滤

- 当前 `getEnvironments` **未** 带 `project_id`参数（以调用为准）；若后端按租户过滤，依赖 token 即可。

## 14. Element Plus

- **`el-select`** `filterable` `clearable`，`popper-class="dark-tech-dropdown"` 与主皮肤一致。

## 15. 无障碍

- **`aria`** 部分靠 **tooltip**；navbar 变体 `title` 同步 **`tooltipContent`**。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
