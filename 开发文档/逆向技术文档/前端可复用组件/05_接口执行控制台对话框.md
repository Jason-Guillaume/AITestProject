# 可复用组件：接口执行控制台对话框（ApiExecuteConsoleDialog）

## 1. 文件

- **`frontend/src/components/api/ApiExecuteConsoleDialog.vue`**

## 2. 职责

以 **对话框** 形式提供与 **CaseDrawerApiPanel** 同源的 **接口用例执行控制台**（运行、预览、填数、日志展示）。脚本内同样 **`useApiExecuteConsole(caseRowRef)`**。

## 3. Props / 布尔可见性

- 通常 `v-model` 或 **`visible`** 控制 **`el-dialog`**；打开时 **`caseRowRef`** 指向当前行。

## 4. API 与 Composable

- 与 **「用例弹窗与接口抽屉面板」** 文档一致：核心在 **`useApiExecuteConsole.ts`**。
- 关键：`runApiCaseApi`、`previewRunApiCaseApi`、`aiFillTestDataApi`；辅助 **`getPlansApi`、`createReportApi`** 等。

## 5. HTTP 状态短语

- 组合式导出 **`HTTP_STATUS_PHRASE`**（若存在），用于展示 **非整型状态文案**。

## 6. 环境 `baseUrl`

- **`useCurrentEnvironment`**：与抽屉相同，`resolveApiUrlWithBase` 统一解析。

## 7. WebSocket

- **不使用**。

## 8. 与 `TestCase.vue` 关系

- 列表页可在 **行操作** 打开本对话框，避免进入独立 **ApiTest** 页（`/test-case/api`）。

## 9. Prism 代码高亮

- 同 composable：JSON / markup 等；注意大对象 **折叠**。

## 10. cURL 导入（若 UI 暴露）

- composable 内含 **`parseCurlCommand`**，可生成 method/url/headers/body。

## 11. 错误边界

- 解析 cURL 抛错：`ElMessage` 「请输入有效 cURL」类提示。

## 12. 对话框堆叠

- Element Plus **`append-to-body`** 视模板而定；避免嵌套在 `transform` 容器里导致 popper 偏移（若遇到再查具体属性）。

## 13. 小结

**Dialog = 另一种容器**；业务逻辑 **100% 依赖 composable**，维护时改一处即可同步抽屉与弹窗。

## 14. 可访问性

- 对话框需 **焦点陷阱**（Element Plus 默认处理）；日志区 **role** 可后续增强。

## 15. 性能建议

- 关闭对话框时 **清空 execution log ref**，避免泄漏对大对象的引用。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
