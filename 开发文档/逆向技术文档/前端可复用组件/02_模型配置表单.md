# 可复用组件：模型配置表单（ModelConfigForm）

## 1. 文件

- **`frontend/src/components/ModelConfigForm.vue`**

大型表单组件，用于 **OpenAI 兼容** 接口的 **Base URL / API Key / 模型名** 等配置（文案为英文 UI）。系统管理员在 **`AiModelConfig`** 相关页面挂载。

## 2. 关联 API（`@/api/sysAiConfig`）

| 函数 | HTTP | 路径 |
|------|------|------|
| `getAiModelConfigApi` | GET | `/sys/ai-config/` |
| `saveAiModelConfigApi` | PUT | `/sys/ai-config/` |
| `disconnectAiModelConfigApi` | POST | `/sys/ai-config/disconnect/` |
| `reconnectAiModelConfigApi` | POST | `/sys/ai-config/reconnect/` |
| `deleteAiModelConfigApi` | DELETE | `/sys/ai-config/` |

备注：`PUT` 时 **`api_key` 可留空** 表示保留原 Key（注释见 `sysAiConfig.js`）。

## 3. 鉴权与路由

- 系统配置仅 **`is_system_admin`** 用户；路由 **`/system/...`** 在 `beforeEach` 里由 **`SYSTEM_ADMIN_ONLY_PREFIXES`** 拦截，非管理员重定向 **`/system/message`**（`router/index.js`）。
- 普通用户不应看到本组件；若嵌入错误，后端也会 403。

## 4. 与 `assistant.js` 测试连接的区别

- **`verifyAiConnectionApi`**：`POST /ai/verify-connection/`（固定智谱链路，`assistant.js` 注释）。
- **`testAiConnectionApi`**：`POST /ai/test-connection/`（可传 model、api_base_url）。

`ModelConfigForm` 若提供「测试连接」按钮，应对接其一；保存仍以 **`sysAiConfig`** 为真源。

## 5. 表单状态机（概念）

- `isLoading`：拉取 GET。
- `isEditMode`：控制输入是否可编辑。
- `isSaved`：影响 API Key placeholder（「留空保留」文案）。
- `showApiKey`：明文切换。

## 6. 环境变量

- 前端 **无** `VITE_AI_KEY` 之类把 Key 写死在构建里；Key 存后端或由管理员录入。开发环境仅 **`/api` 代理** 指向后端。

## 7. WebSocket / SSE

- 本组件 **不涉及** 长连接；模型配置为一次性 REST。

## 8. 与智能助手、用例生成

- 下游功能（`streamGenerateAiCases`、`generateAiCasesApi`）依赖后端已连通模型；配置失败时这些接口返回业务错误，前端应 **`app:backend-unhealthy`** 或 `ElMessage`（以各页为准）。

## 9. 组件体积与职责

- 单文件超千行：含动画与皮肤。**逆向时抓「methods + API 调用块」即可**，不必通读 CSS。

## 10. 安全

- Key 输入框 **password**；日志中勿 `console.log` 明文 Key。
- 生产 HTTPS 必选，防中间人。

## 11. 国际化

- 当前 UI 文案偏英文；若产品转中文，仅改模板不改 API。

## 12. 相关页面路由示例

- `router/index.js`：`/system/ai-usage`、`/system/ai-quota` 与 AI 配置 adjacent；具体配置页以实际 `views/system/*.vue` 引用本组件为准。

## 13. 小结

`ModelConfigForm` + **`sysAiConfig.js`** 构成 **平台级 LLM 网关设置**；与业务 `assistant` 能力解耦一层（sys vs ai）。

## 14. 联调清单

- GET 空配置 → 首次 PUT → GET 验证 → disconnect → reconnect → delete（慎用）。

## 15. Pinia / Store

- 本组件可不依赖全局 Store；若需「配置变更后广播」，可 **`window.dispatchEvent`** 与 `MainLayout` 同步权限类似。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
