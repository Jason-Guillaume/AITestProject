# 可复用组件：流水线控制台（PipelineConsole）

## 1. 文件与用途

- **路径**：`frontend/src/components/PipelineConsole.vue`。
- **场景**：流水线详情页内嵌的 **日志流面板**，展示 **PIPELINE // LOG STREAM** 风格终端输出。

## 2. Props

| Prop | 类型 | 默认 | 含义 |
|------|------|------|------|
| `pipelineId` | `number` | （必填） | 后端流水线主键 |
| `usePolling` | `boolean` | `true` | **true**：HTTP 轮询；**false**：关闭轮询（预留 WS 模式标签） |
| `pollIntervalMs` | `number` | `1500` | 轮询间隔毫秒 |

`modeLabel` 计算属性显示 **`POLL` 或 `WS`**；当前脚本仅在 **`usePolling` 真值** 时 `setInterval` 拉日志，`WS` 分支未在本文件实现，属产品演进占位。

## 3. 请求的 API

- **`GET /project/pipelines/{pipelineId}/logs/`**
- **Query**：`after_id`（上次最后一条 log id）、`limit`（如 500）。
- **响应**（以组件解析为准）：`items[]`（`log_text`）、`last_log_id`、`pipeline_status`（0–3 映射 Pending/Running/Success/Fail）。

使用统一 **`request`**（`@/utils/request`），自动附带 **Token**。

## 4. 行为要点

- 首次与每轮轮询 **`fetchLogs`**：把新行 **append** 到 **`lines`**，最多保留约 **8000** 行，防内存膨胀。
- **`scrollToBottom`**：`nextTick` 滚到 `logPanelRef` 底部。
- 轮询失败 **静默**，避免刷屏（评论写明适合 **2C2G 无 WebSocket** 部署）。
- **`watch(pipelineId, ...)`** 重置 `lastLogId` 并重启轮询。

## 5. 与路由的关系

- 流水线列表/详情路由：`/pipelines`、`/pipelines/:id`（`router/index.js`），且 **`meta.public: true`**，控制台组件在公开预览场景下仍可拉 **`/api/project/pipelines/...`**（若后端仍校验 token，则需登录；以前端守卫为准）。

## 6. WebSocket 提示

- 注释：**「适合无 WebSocket 的部署」**。若未来接 WS，应：
  - 在 **`usePolling=false`** 时建立 **`ws://.../ws/pipeline/{id}/`**（路径以后端为准）；
  - **`onBeforeUnmount`** 统一 `close`；
  - 与 **`useWebSocket.ts`** 可二选一封装。

## 7. 样式与 a11y

- `role="log"`、`aria-live="polite"`，利于读屏获取新增行。

## 8. 相关接口模块

- **`frontend/src/api/project.js`**：`getPipelinesApi`、`deletePipelineApi`；控制台的 **logs** 子路径不在 `project.js` 顶层导出，组件内直接 **`request.get`**。

## 9. 集成方式示例

```vue
<PipelineConsole :pipeline-id="numericId" :use-polling="true" :poll-interval-ms="1500" />
```

## 10. 小结

`PipelineConsole` 是 **纯轮询日志组件**，依赖 DRF 分页游标 **`after_id`**；与实时 **`EventSource`**、**`WebSocket`** 不同，适合弱实时、资源受限环境。

## 11. 与性能页日志对比

| 能力 | 本组件 | 服务器 `LogTerminal` |
|------|--------|----------------------|
| 协议 | HTTP 轮询 | WebSocket |
| 资源 | 流水线 | SSH 主机文件 |
| URL | `/project/pipelines/.../logs/` | `/ws/server-logs/...` |

## 12. 错误与空状态

- 无数据时展示 **「等待日志输出…」**；后端 `items` 非数组时安全降级为空数组。

## 13. TypeScript

- `<script setup lang="ts">`，与仓库混用 JS 页面并存；`props` 使用 **`withDefaults`**。

## 14. 性能建议

- 若单轮 `limit` 过大，浏览器渲染 `v-for` 压力大；可改为虚拟列表（未实现）。

## 15. 测试注意

- Mock **`request.get`** 返回递增 `last_log_id`，验证 **去重** 与 **截断 8000 行** 逻辑。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
