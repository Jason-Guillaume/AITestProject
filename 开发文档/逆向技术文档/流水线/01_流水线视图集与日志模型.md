# 流水线视图集与日志模型

## 文档目的

从 **REST 视图 + ORM + 异步任务占位**三个层面固定 **`Pipeline` / `PipelineLog`** 与 **`PipelineViewSet`** 的合同；本篇与 **`流水线/02_流水线日志写入与查询链路.md`**（落库机理）及 **`项目任务与发布/06_流水线异步任务与装饰器.md`**（Celery、装饰器）互补，三者合并可还原端到端控制台体验。

## 2. `PipelineViewSet` 行为

源码：**`project/views.py`** 约 473 行起。

共性：

- **`queryset`**：**`Pipeline.objects.all()`**；
- **`serializer_class`**：**`PipelineSerializer`**（`**`Meta`** 默认 **`fields = \"__all__\"`**，含 **`BaseModel` 审计字段**）；
- **解析器**：**`JSONParser, MultiPartParser, FormParser`** —— 允许上传附件型配置（若非使用可忽略）。

### 2.1 `POST .../pipelines/<id>/run/`

|**请求**|**响应**|
|--------|-------|
|**Body**：**`{\"command\":\"bash -lc \\\"pytest\\\"\"}\"`** （字符串，可空）|**`task_id`**：Celery AsyncResult **`id`**|

实现：**`run_pipeline_task.delay(pipeline.id, command)`**；

**语义**：异步 **非阻塞**，HTTP **立即返回** **`status: queued`**；

**安全性**：`**`shell=True`** —— 生产须 **校验调用者权限** + **禁用危险命令前缀**（基线由 **`BaseModelViewSet`** 数据范围兜底，仍建议单独审计）。

### 2.2 `GET .../pipelines/<id>/logs/`

|**查询参数**|**默认**|**语义**|
|-------------|-------|-------|
|**`after_id`**|`**0`**|仅返回 **主键 `id` greater than** `after_pk` 的新行|
|**`limit`**|`**500`**|夹在 **\[1,2000\]**|

流程：

1. **`pipeline = self.get_object()`**；
2. **`pipeline.refresh_from_db(fields=[\"status\"])`** —— Worker 更新了 **`status`** 后 **同一连接内 ORM 缓存需刷新**，否则控制台轮询看到旧 **`RUNNING`**；
3. 查询：**`PipelineLog.objects.filter(pipeline=pipeline, id__gt=after_pk).order_by(\"id\", \"timestamp\")[:limit]`**；
4. 序列化：**`PipelineLogSerializer(many=True)`**；
5. **`last_log_id`**：有新行则为末行 **`id`**，否则 **保持上一轮 `after_id`**；

响应 JSON 键：**`pipeline_status`、`items`、`last_log_id`**。

## 3. 序列化字段（概要）

|**`PipelineSerializer`**|`**`name`、`repo_url`、`status`** + BaseModel (`create_time`…)|
|**`PipelineLogSerializer`**|`**`id`、`log_text`、`timestamp`** ——精简，防 payload 过载|

如需 **关联返回最近 N 日志**，应 **新开嵌套 Serializer**——当前 ViewSet **`list`** 未预取 **`logs`** 以免 N+1。

---

## 4. ORM：`project/models.py`

### 4.1 `Pipeline`

- **表**：**`pipeline`**
- **状态常量**：**`STATUS_PENDING/RUNNING/SUCCESS/FAIL` **→ **`0～3`**（整型）。
- **`repo_url`**：**`URLField`** —— 仅文档化仓储；**不负责 clone** ，真实命令 **`command`** 决定。

### 4.2 `PipelineLog`

- **表**：**`pipeline_log`**
- **FK**：`**`pipeline`**` → **`related_name=\"logs\"`**
- **字段**：`**`log_text`、`timestamp`(auto_now_add)`**
- **无 **`is_deleted`** ——日志 **只追加**；清理走 **离线 retention job**。

**`Meta.ordering = [\"-timestamp\"]`**：与 **`logs` ViewSet **`order_by(\"id\")`** ascending**并存——**REST 视图以 id 递增为准**，避免时间戳漂移造成重复。

## 5. 与 Celery Worker 对齐

|**阶段**|**谁在写 DB**|
|--------|---------------|
|**提交 run**|视图层：**状态仍可能 **`PENDING`****（tasks 入口处改 **`RUNNING`）|
|**执行**|**`run_pipeline_task`**：逐行 **`PipelineLog.objects.create`**|
|**收尾**|同上任务：**更新 **`Pipeline.status`**|

Flower / `inspect active`：**`task_id`** 对齐排查 **卡住 Running**。

## 6. 前端控制台集成要点（Vue）

|**步骤**|**客户端**|
|---------|----------|
|**启动**|保存 **`task_id`（可选用 Toast）|
|**轮询 logs**|`**`setInterval`/requestAnimation`** + **指数退避** |
|**合并行**|前缀 **`[stdout]/[stderr]`** 染色 |
|**收尾**|`**`pipeline_status in (SUCCESS,FAIL)`** 且无新 **`items`** 后停表|

典型组件：**`PipelineConsole.vue`**——axios **`baseURL=/api`**。

## 7. 权限与安全

|**风险**|**备注**|
|--------|-------|
|**任意 Shell**|超级用户 / 项目负责人需分离；可考虑 **只允许预置模版**。
|**日志注入**|**\n** **与 huge lines**——前端虚拟滚动；后端 **单行截断**可选。
