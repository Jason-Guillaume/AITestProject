# 脚本执行中心与浏览器工作台

## 1. 路由结构

| path | name | 组件 |
|------|------|------|
| `/script-hub` | **`ScriptHub`** | `frontend/src/views/ScriptHub.vue` |
| `/script-hub/webui` | **`WebUIScriptWorkbench`** | `frontend/src/views/script/WebUIWorkbench.vue` |

后者 `meta.hiddenInSidebar: true`；从 ScriptHub **卡片跳转**。全局需登录。

## 2. ScriptHub（门户）

**无 API**：`modules` TypeScript **常量数组**：`Web UI 自动化`、`API`、`App`、`性能/H5` 等，每项含 **`available`、`route`、`icon`**。**仅 Web UI** `available: true`，按钮跳转 **`/script-hub/webui`**。  

解释文案（`module.explanation`）在卡片按钮下方居中展示。

## 3. WebUIWorkbench — 能力与布局

面包屑：**脚本执行中心** → **Web UI 工作台**。  

**左侧**：模式 Tab **我的脚本 / 回收站**；导入、批量删/移动/启用禁用、单项更多菜单。  

**列表数据**：`**request.get('/assistant/ui-scripts/', { params: { show_deleted: false } })**`；（回收站 **`/assistant/ui-scripts/trash/`**）。  

**写操作节选**：

- 新建导入：`POST /assistant/ui-scripts/`（`multipart`）  
- `PATCH /assistant/ui-scripts/:id/`、`DELETE ...`、`POST .../toggle_active/`、`POST .../restore/`、`POST .../permanent_delete/`、`POST empty_trash/`

详见文件内 ~~10+~~ 请求点；统一 **`@/utils/request`**。

## 4. Pinia Store

工作台脚本使用 **Composition API refs**（`projects`、`selectedProject`）；**未发现**必选 Pinia。执行参数可能读 **localStorage **环境**。

## 5. Query 与其它参数

`**show_deleted`** 由 UI Tab 切换，不必 URL query。

## 6. 权限

普通登录；脚本资源 **按后端用户/租户隔离**。

## 7. 与 Pipeline 的差异

流水线关注 **CI/CD 日志轮询** `/project/pipelines/`；本平台关注 **`assistant/ui-scripts`** 运行时沙箱。**两者 API 前缀不同**。

## 8. UX 关键点

选中脚本后右侧 **控制台/iframe/终端**区块（以大文件后半为准）用于 **启动浏览器自动化**；卸载前需 **`clearInterval`** 停止日志流。

## 9. 异常

轮询/列表失败 **静默降级**或以 Message 告知； **`fetchLogs`/`loadProjects`** **`catch`** 分支防刷屏。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。
