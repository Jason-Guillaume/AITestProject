# 元素库页

## 1. 路由

| path | name | 组件 |
|------|------|------|
| `/element-library` | **`ElementLibrary`** | `frontend/src/views/ElementLibrary.vue` |

`router/index.js`：`meta: { hiddenInSidebar: true }`，需在 MainLayout 下从智能体链路或外链进入。**需登录**。

## 2. API 模块

**文件**：`frontend/src/api/elementLibrary.js`，统一前缀 **`/ai/`**，经 `axios` **`request`** 实例（`/api` base）。主要资源：**UI 模块、页面、元素、用例（UI）、操作步骤**。

### 模块 `ui-modules`

- `GET /ai/ui-modules/tree/?project_id=`
- `GET /ai/ui-modules/?project_id=`（扁平列表）
- `POST/PATCH`/（实现为 **`PUT`** 更新）、`DELETE /ai/ui-modules/:id/`

### 页面 `ui-pages`

- `GET /ai/ui-pages/?module_id=`
- `POST / PUT / DELETE` 同上模式

### 元素 `ui-elements`

- `GET /ai/ui-elements/?page_id=&search=`
- `GET /ai/ui-elements/by_name/?name=&page_id=`（可选页过滤）
- `POST / PUT / DELETE`

### UI 测试用例与步骤

- `GET/POST /ai/ui-test-cases/`、`PUT/DELETE :id`、`?module_id=`
- `GET/POST /ai/ui-action-steps/`、`PUT/DELETE :id`、`?test_case_id=`

Vue 文件中 import 常为 **具名解构** `{ getUIModulesTree, createUIPageElement, ... }`。

## 3. 上下文：`project_id`

几乎所有树与列表 **`project_id` 必选**——来自 **全局当前项目**。未选项目时页面应禁用操作或给出空提示（以实现为准）。

## 4. 前端结构（概要）

典型 **三段式**：左侧模块树 → 中部页面列表 → 右侧元素/步骤表。**Element Plus**：`el-tree`、`el-table`、`el-dialog` 编辑子实体。  

## 5. Pinia Store

一般用 **本地 state + computed**；未强制 **`elementLibrary`** 专用 store。**与 `appContextStore` 的项目 ID 对齐**。

## 6. Query 参数

无全局约定。**深链若要打开某一 page_id**，可在后续迭代用 **`route.query`** 约定。

## 7. 权限

业务用户默认可访问路径；后端按项目/租户做 **RBAC**。路由层无 `SYSTEM_ADMIN_ONLY`。

## 8. 错误处理

沿用全局 axios 拦截；CRUD **`catch`** 常 **`ElMessage.error`**。

## 9. 与 UI 自动化工作流

元素库输出的 **`*_element`/`locator`** 可被 **`UiAutomationGenerator`/`Workbench`** 生成脚本间接消费；编排关系以后端契约为准。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。



## 11. 调试与追踪建议

建议在 DevTools Network 中过滤 **`/api/`**；并与本文档中的模块前缀交叉核对（如 `/testcase/`、`/execution/`、`/defect/`、`/assistant/`、`/sys/`、`/user/`）。

2. 打开对应 `frontend/src/views/...`，检索 **`from "@/api/`**（或别名 `@/`）；  
3. 若逻辑拆至 `components/` 子组件，亦需对该目录做同样检索。
