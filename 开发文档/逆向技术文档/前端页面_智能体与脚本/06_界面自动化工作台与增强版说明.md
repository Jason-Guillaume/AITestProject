# 界面自动化工作台与增强版说明

## 1. 路由

| path | name | 组件 |
|------|------|------|
| `/ui-automation/workbench` | **`UiAutomationWorkbench`** | `frontend/src/views/UiAutomationWorkbench.vue` |

`meta.hiddenInSidebar: true`；AgentHub 主推入口之一。**需登录**。

## 2. 脚本来源三段模式

控件：`el-radio-group` **`scriptSource`**：

1. **`ai`**（AI 生成）：表单 `targetUrl` + `scenarioSteps`，处理方法 `handleGenerateAI`；请求 **`POST /api/assistant/ui-scripts/`**（`multipart/form-data`，见 `request.post`）。
2. **`linear`**（单 `.py` 上传）：方法 `handleUploadLinear`，仍为 multipart。
3. **`pom`**（ZIP/Git）：ZIP 或 Git URL/分支等字段以实现为准，前缀仍为 **`/assistant/ui-scripts/`**。

`UiAutomationWorkbench.vue` 在后部 **`import request from '@/utils/request'`**，用于上述上传与会话交互。

## 3. 与工作台 UI 文案一致性

页头：**「UI自动化脚本工作台」**，副标题：**支持 AI 生成、单文件上传和 ZIP/Git 导入三种方式**。右侧通常为 **编辑器/预览/脚本树**区域（超长单文件，`read` 未穷尽）。

## 4. 「增强版」在项目中的含义（逆向）

仓库内 **`UiAutomationWorkbench.vue`** 是相对 **`UiAutomationGenerator`** 的 **工程化工作台**：  

- Generator：仅 **`generateUiAutomationScriptApi`** → 返回一段代码展示。  

- Workbench：**真实落库脚本工程**、`Content-Type` 多部分上传、可能与 **执行 Runner**衔接（后端 contract）。  

无单独路由名「enhanced」，文档标题「增强版」指 **能力层级**：**生成 vs 管理与导入**。

## 5. `/assistant/ui-scripts/` 前缀（工作台产生）

与 **`script/WebUIWorkbench.vue`** 列表页共享 REST 前缀（参见 `WebUIWorkbench` 文档）；本文件可能聚焦于 **单次上传 POST** **`/assistant/ui-scripts/`**。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。
