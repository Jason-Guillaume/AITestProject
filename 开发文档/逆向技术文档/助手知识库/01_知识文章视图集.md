# 01 — 知识文章视图集（逆向）

## 1. 路由与类

- `assistant/urls.py`：`router.register(r"knowledge-articles", KnowledgeArticleViewSet)`。
- 基类：`common.views.BaseModelViewSet`（列表/检索/增删改，具体 action 以基类为准）。
- 解析器：`JSONParser`、`MultiPartParser`、`FormParser`（支持 JSON 与带文件表单）。

## 2. `get_queryset` 可见性

- 基础：`KnowledgeArticle.objects.filter(is_deleted=False)`。
- 非特权用户（`user_is_knowledge_document_privileged` 为假）：仅 `(creator=self.request.user) OR (visibility_scope=org AND org_id in user_orgs)`。
- 筛选 query：`category`、`tag`（`tags__contains` JSON 包含），排序 `-create_time`。

## 3. `create` 重写要点

- 支持 `source_file` 上传：超过 `settings.KNOWLEDGE_UPLOAD_MAX_SIZE`（默认 10MB）直接 400。
- 使用 `extract_text_from_uploaded_file` 填充 `markdown_content`；失败返回可读错误。
- `text_source=upload` 且无内容且无文件时 400。
- `tags`：支持 JSON 数组字符串、逗号分隔列表。
- 组织与可见性：`_resolve_org_for_request`；`visibility_scope` 非法时默认 `org`（有组织）或 `private`（无组织）。

## 4. 与 RAG 的关系

- 文章保存后由 `testcase/signals.py` 或知识库任务（视配置）触发 `assistant.rag_chroma.index_test_case` 等——文章侧索引逻辑见 `KnowledgeSearcher`/`knowledge_rag`；本文档仅覆盖 ViewSet 层。

## 5. 权限与审计

- 继承基类权限；组织级共享依赖 `org_id` 与成员关系正确。

## 6. 前端注意

- 创建时若传文件，字段名与 parser 需匹配；大文件应先做前端体积校验。
