# 知识文档上传摄取列表状态重试删除

## 1. 功能概述

**`KnowledgeDocument`** 的上传、统一入库、列表、状态查询、重试、软删由多组 **`APIView`** 承担（非 ViewSet）。路由均在 **`/api/assistant/knowledge/...`**。

| 能力 | 类 | 方法/路径 |
|------|-----|-----------|
| 简单上传 | `KnowledgeDocumentUploadAPIView` | POST `knowledge/documents/upload/` |
| 统一入库（文件/URL） | `KnowledgeDocumentIngestAPIView` | POST `knowledge/documents/ingest/` |
| 列表 | `KnowledgeDocumentListAPIView` | GET `knowledge/documents/` |
| 状态 | `KnowledgeDocumentStatusAPIView` | GET `knowledge/documents/<doc_id>/status/` |
| 重试 | `KnowledgeDocumentRetryAPIView` | POST `knowledge/documents/<doc_id>/retry/` |
| 删除 | `KnowledgeDocumentDeleteAPIView` | DELETE `knowledge/documents/<doc_id>/` |

**涉及文件**：`assistant/views.py`（约 L975–1427）、`assistant/tasks.py`（异步任务）、`assistant/serialize.py`（`KnowledgeDocumentSerializer`）。

## 2. 底层运行逻辑

### 2.1 `POST .../upload/`（旧版直传）

- **`file`** 必填；仅 **`.pdf` / `.md`**；**`title`** 必填。
- **`tags`** 支持 JSON 字符串、逗号列表或数组。
- **`_resolve_org_for_request`** + **`visibility_scope`** 默认逻辑与文章创建类似。
- **`KnowledgeDocument.objects.create`**：`status=processing`，`source_type=upload`，保存 **`file_path`**。
- **`transaction.on_commit`** 后 **`process_knowledge_document_task.delay(doc_id)`** 或线程降级调用 **`process_and_embed_document`**（见 **`assistant/services/rag_service.py`**，与 **`ingest`** 路径的 **`process_document_rag`** 不同）。

响应：**202**，`KnowledgeDocumentSerializer` 数据。

### 2.2 `POST .../ingest/`（推荐：upload / url）

- **`mode`**：`upload`（默认）或 **`url`**。
- **`module_id`**：可选，关联 **`testcase.TestModule`**；无效则 **400**。
- **upload**：校验文件扩展名 **pdf/md**、大小；**`title`** 缺省用文件名。
- **url**：**`url`** 必填且以 **`http://` 或 `https://`** 开头；**`document_type=url`**，**`source_url`** 写入；标题缺省用 URL。
- 创建后 **`status=processing`**，`transaction.on_commit` 触发 **`process_document_rag.delay(document_id)`**（`assistant/tasks.py`：LangChain 切片 + Chroma + 回写 **`vector_db_id` / `semantic_*`**）。

### 2.3 `GET .../documents/`

- **`_mark_stuck_processing_docs_failed`** 先修正卡死任务。
- 分页 **`page` / `page_size`**（1–100）；非特权用户同文章视图的 **creator + org 共享** 过滤。
- 过滤：**`status`、`module_id`、`tag`、`q`**（标题/文件名/source_url icontains）。
- **`purpose_counts`**：按 **`category`** 粗分为 **`all` / `requirement` / `standard` / `template`** 计数（与列表 **`category`** 过滤解耦，供左侧导航）。

### 2.4 `GET .../<doc_id>/status/`

- 权限：**`IsAuthenticated` + `IsAdminOrKnowledgeResourceAccessible`**（对象级 creator/org）。
- 若 **`completed`** 但 **`semantic_*` 为空**，尝试 **`_build_fallback_semantic_payload`** 回填。

### 2.5 `POST .../<doc_id>/retry/`

- **`processing`** 禁止重复提交 → **400**。
- 否则 **`status=pending`**，清 **`error_message`**；若 **`source_type=article`** 且有关联文章 → **`process_knowledge_article_task`**；否则 **`process_knowledge_document_task`** / 线程 **`process_and_embed_document`**。

### 2.6 `DELETE .../<doc_id>/`

- 对象权限同上；**`is_deleted=True`** + 更新 **`updater`**（软删，非物理删文件）。

## 4. 前后端交互

统一前缀 **`/api/assistant/`**。上传/入库成功多为 **202 Accepted**；列表为 **200**，体为 **`{ success, count, page, page_size, purpose_counts, results }`**。

## 5. UI 表现与逻辑

**`KnowledgeCenter.vue`** 文档 Tab：列表轮询或手动刷新 **`status`**；失败展示 **`error_message`**；提供重试、删除。上传区与 **`cyber-table-panel`** 表格风格一致。

---

## 6. 异常处理

| 场景 | HTTP |
|------|------|
| 缺 file/title/mode 非法 | **400** |
| 模块/URL 非法 | **400** |
| 文档不存在 | **404** |
| 无对象权限 | **403**（`_knowledge_document_forbidden_if_no_object_permission`） |
| 异步失败 | 文档行 **`status=failed`** + **`error_message`**（经 **`simplify_vector_error`** 简化） |
