## 2. `get_queryset`：回收站与筛选

### 2.1 回收站

**`recycle=1` / `is_deleted=true`**（不区分大小写，值为 `1/true/yes`）时：

- 基 queryset：**`TestDefect.objects.filter(is_deleted=True)`**；
- 非管理员：**`_apply_member_scope`**（命中 **`module` + `release_version` 双链路**，见 `common/views.py`）。

正常模式：**`super().get_queryset()`** → **`is_deleted=False`** + 成员范围。

### 2.2 列表筛选（AND 关系）

| Query | 行为 |
|--------|------|
| **`severity` / `status` / `priority`** | 转 **int** 失败 → **`qs.none()`** |
| **`release_version`** | **`release_version_id`** |
| **`search`** | **`defect_no` 或 `defect_name` icontains** |

**`select_related`**：`release_version`、`handler`、`creator`、`module`。排序：**`-create_time`**。

## 3. `perform_create`（编号与重复检测）

- **`handler`**：未传则默认 **当前用户**；未登录且无 handler → **400**。
- **重复**：同一 **`release_version` + `module` + `defect_name` + `severity`** 且 **`is_deleted=False`** 已存在 → **`ValidationError`**（带 **`duplicate_id`**）。
- **`defect_no` 空**：事务内 **`select_for_update`** 取最大 id，生成 **`DEF-{id+1:05d}`**，**最多重试 5 次**应对 **`IntegrityError`**。
- **手工编号**：与现存 **`defect_no`**（未删）冲突 → **400**。

---

## 4. `perform_update`（关闭规则）

当 **`validated_data.status == 4`（已关闭）**：

1. **`old_status` 必须为 `2`（处理中）**，否则：**「已关闭只能从处理中状态流转」**。
2. **`request.user.pk` 必须等于 `instance.handler_id`**，否则：**「只有处理人可以将缺陷标记为已关闭」**。

已登录则 **`serializer.save(updater=user)`**。
