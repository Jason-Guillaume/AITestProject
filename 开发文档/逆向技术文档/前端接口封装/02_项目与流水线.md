# 前端接口封装：项目与流水线

## 1. 文件

- **`frontend/src/api/project.js`**

## 2. 统一前缀

- 所有路径相对于 **`request` 的 `baseURL: /api`**，例如 **`/project/projects/`** → **`/api/project/projects/`**。

## 3. 项目

| 导出 | 方法 | 路径 |
|------|------|------|
| `getProjectsApi` | GET | `/project/projects/` |
| `createProjectApi` | POST | `/project/projects/` |
| `updateProjectApi` | PATCH | `/project/projects/{id}/` |
| `deleteProjectApi` | DELETE | `/project/projects/{id}/` |
| 回收站/批量 | 多种 | `/project/projects/recycle/`、`bulk-soft-delete`、`bulk-restore`、`bulk-hard-delete`、`bulk-status` 等 |
| `downloadProjectsCsv` | GET blob | **`downloadAuthedGet('/api/project/projects/export.csv')`** |

## 4. 项目任务

- **`getTasksApi` / `createTaskApi` / `updateTaskApi` / `deleteTaskApi`**：`/project/tasks/`

## 5. 流水线

| 导出 | 路径 |
|------|------|
| `getPipelinesApi` | GET `/project/pipelines/` |
| `deletePipelineApi` | DELETE `/project/pipelines/{id}/` |

**控制台日志**（组件内直接写）：**`GET /project/pipelines/{id}/logs/?after_id=&limit=`**

## 6. 发布计划（Release）

- `getReleasesApi`、`createReleaseApi`、`updateReleaseApi`、`deleteReleaseApi`、`getReleaseDetailApi`
- **`getReleaseRiskBriefApi`**：`GET /project/releases/{id}/risk-brief/`
- **`getReleaseNeverExecutedCasesApi`**：窗口内未执行用例
- **`downloadReleaseNeverExecutedCsv`**：`downloadAuthedGet` 导出 CSV

## 7. 与路由

- **`/pipelines`**、**`/pipelines/:id`**：列表与详情；**`meta.public: true`** + 守卫白名单。

## 8. 环境变量

- 无独立 env；导出 CSV 用 **`downloadAuthedGet`** 附 Token Header。

## 9. WebSocket

- **无**；**`PipelineConsole`** 轮询 REST。

## 10. 与 MainLayout

- **`getProjectsApi`** 填充顶栏项目下拉；与 **`useAppContextStore`** 同步。

## 11. 错误处理

- 删除/批量操作建议 **`ElMessageBox` 确认**（页面层）。

## 12. 小结

**project.js** 覆盖 **项目 + 任务 + 流水线 + 发布**；流水线日志 API **仅** 在组件中字符串引用。

## 13. 权限

- 公开流水线路由下，若删除接口需登录，会 403/401。

## 14. 分页参数

- `params` 透传 DRF：`page`、`page_size`、`search` 等。

## 15. 测试

- Mock `getPipelinesApi` 返回 `results` 数组；断言 **PipelineList** 渲染。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。

## 附：与部署相关

- 前端 **不** 内置后端地址时，依赖 **同源** 或 **反向代理** 将 `/api`、`/ws` 指向同一集群。
- 静态资源与 **History 路由** 需 **fallback** 到 `index.html`。
