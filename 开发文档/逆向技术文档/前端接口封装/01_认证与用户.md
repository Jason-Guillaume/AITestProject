# 前端接口封装：认证与用户

## 1. 模块文件

- **`frontend/src/api/auth.js`**：登录/注册/验证码，**独立 axios**（不走 `@/utils/request`）。
- **`frontend/src/api/user.js`**：登录后业务 **`request`** 实例。

## 2. 认证（auth.js）

| 函数 | 方法 | URL | 说明 |
|------|------|-----|------|
| `loginApi` | POST | **`/api/user/login/`** | 全路径；`Content-Type: application/json`；body `username, password` |
| `captchaApi` | GET | **`/api/user/captcha/`** | 返回 `uuid`、`image` |
| `registerApi` | POST | **`/api/user/register/`** | `username, password, captcha_code, captcha_uuid` |

**注意**：`loginApi` **不用** `baseURL: /api` 的封装，避免与拦截器耦合同环。

## 3. 用户（user.js，`baseURL` 已为 `/api`）

| 函数 | 路径 |
|------|------|
| `getCurrentUserApi` | GET `/user/me/` |
| `changePasswordApi` | POST `/user/change-password/` |
| `getUserProfileApi` | GET `/user/me/profile/` |
| `patchUserProfileApi` | PATCH `/user/me/profile/`（可 FormData 头像） |
| `getSensitiveChangeStatusApi` | GET `/user/me/change-requests-status/` |
| `submitSensitiveChangeApi` | POST `/user/me/sensitive-change/` |
| `getSystemMessagesApi` | GET `/user/system-messages/` |
| `markSystemMessageReadApi` | PATCH `/user/system-messages/${id}/read/` |
| `getAdminChangeRequestsApi` | GET `/user/admin/change-requests/` |
| `approveChangeRequestApi` | POST `/change-requests/${id}/approve/` |
| `rejectChangeRequestApi` | POST `/change-requests/${id}/reject/` |

## 4. 与路由守卫

- 登录成功写 **`localStorage.token`**；**`getCurrentUserApi`** 同步 **`is_system_admin`** 到 `localStorage`（见 `MainLayout` **`syncProfileFromServer`**）。

## 5. 环境变量

- 前端 API 主机由 **Vite 代理** 或 **同源** 提供；**无** 单独 `VITE_AUTH_URL`。

## 6. WebSocket / SSE

- 本模块 **无**；**Dashboard** SSE 用 **`/api/execution/dashboard/stream/`**（见 `dashboard.js`）。

## 7. 下载与 Blob

- 用户模块未包含 CSV；系统审计见 **`audit.js`** 的 **`downloadAuthedGet`**。

## 8. 错误格式

- 后端多为 **`{ code, msg, data }`**；`request` 拦截器统一 **`ElMessage`**。

## 9. 安全

- Token 存 **`localStorage`**；HTTPS 必须；XSS 可窃 token，需 CSP。

## 10. 与 `request.js`

- **`user.js`** 带 **`Authorization: Token ...`**（拦截器注入）；**`auth.js`** 登录前无 token。

## 11. 小结

**auth = 裸 axios 全路径**；**user = 业务封装相对路径**；联调时勿混用。

## 12. 相关页面路由注册

- `/login`、`/register`：`meta.public: true`。

## 13. 组织与用户列表

- **组织/角色/用户 CRUD** 在 **`system.js`**（`/user/orgs/`、`/user/roles/`、`/user/users/`），非本文件。

## 14. 个人审计

- **`audit.js`**：**`getMyAuditEventsApi`** `GET /user/me/audit/events/`。

## 15. 测试建议

- Mock 登录 200 → 保存 token → `getCurrentUserApi` 401 → 拦截器清 token。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。

## 附：与部署相关

- 前端 **不** 内置后端地址时，依赖 **同源** 或 **反向代理** 将 `/api`、`/ws` 指向同一集群。
- 静态资源与 **History 路由** 需 **fallback** 到 `index.html`。
