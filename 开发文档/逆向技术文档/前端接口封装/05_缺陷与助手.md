## 4. 简化问答

- **`askKnowledgeApi`**：`aiKnowledge.js` → **`POST /ai/knowledge/ask/`**

## 5. 路由

- **`/defect/*`**、`/knowledge`、`/knowledge/ask`、`/ai-assistant`、各 **`/ai-*`** 隐藏智能体页。

## 6. WebSocket

- **无**；流式用 **fetch + ReadableStream** 解析 SSE chunk。

## 7. SSE 客户端行为（`streamGenerateAiCases`）

- 401 → 清 token 跳登录；非 ok → **`app:backend-unhealthy`**；超长 delta → 抛错保护前端。

## 8. 环境变量

- 无；模型 Key 走后端 **`sys/ai-config`**。

## 9. 与 `ModelConfigForm`

- 管理员配好 **`sysAiConfig`** 后，本模块 AI 接口才稳定。

## 10. 知识库上传

- **`uploadKnowledgeDocumentApi`**：`multipart`；**`ingestKnowledgeDocumentApi`**：提交解析任务。

## 11. 小结

**defect** 纯 REST；**assistant** 混合 **REST + SSE fetch** + 大量知识库路径。

## 12. 权限

- 知识库写操作通常 **管理员/编辑**；具体 403 以后端为准。

## 13. 超时

- 默认 60s；流式无 axios timeout，靠 **`AbortSignal`**（若传入 options）。

## 14. 测试

- Mock `generateAiCasesNormalized` 返回 `allCovered` 分支。

## 15. 国际化

- `streamParse` 错误日志英文 `console.error`；可本地化。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。

- 前端 **不** 内置后端地址时，依赖 **同源** 或 **反向代理** 将 `/api`、`/ws` 指向同一集群。
- 静态资源与 **History 路由** 需 **fallback** 到 `index.html`。
