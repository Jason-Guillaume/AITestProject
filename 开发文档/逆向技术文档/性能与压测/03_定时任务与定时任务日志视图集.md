# 定时任务与定时任务日志视图集

## 文档目的与读者

说明 **`ScheduledTaskViewSet`** / **`ScheduledTaskLogViewSet`**（`execution/views.py`）与 **`TestScheduler`（APScheduler + `DjangoJobStore`）** 的同步关系、**`pause`/`resume`/`trigger`** 行为，以及日志的 **筛选与按条件批量软删**。适合改 Cron、排查「保存成功但调度未生效」、清理历史日志。

## 2. `ScheduledTask` 与调度器同步

### 2.1 `perform_create`

- **`serializer.save(job_id=f"scheduled-task-{uuid.uuid4().hex}")`**
- 随后 **`TestScheduler.instance().sync_task(instance)`**；失败 → **`ValidationError`**：`任务已创建，但调度同步失败: ...`（**实例已落库**，调度可能不一致）。

### 2.2 `perform_update`

- 保存后同样 **`sync_task`**；失败抛 **`ValidationError`**（文案为「任务已更新，但调度同步失败」）。

### 2.3 `perform_destroy`（软删 + 移除调度）

- **`status=disabled`**、**`is_deleted=True`**，`save` 后 **`TestScheduler.instance().remove_task(instance.id)`**（移除 APScheduler job，**`next_run_time` 置空**）。

> 与 **`BaseModelViewSet.perform_destroy`** 默认仅软删不同，此处 **覆写** 并 **联动调度器**。

### 2.4 `TestScheduler.sync_task`（摘要）

见 `execution/scheduler.py`：

- **`disabled` 或 `is_deleted`**：**`remove_task`**。
- 否则 **`CronTrigger.from_crontab(cron_expression)`**，**`remove_job` + `add_job`**，`func=execute_scheduled_task`，**`args=[str(task.id)]`**，**`id=job_id`**，**`max_instances=1`**，**`coalesce=True`**，**`misfire_grace_time=60`**。
- 写回 **`next_run_time`**；若 **`status=paused`** → **`pause_job`**，否则 **`resume_job`**。

## 3. `@action`：暂停 / 恢复 / 手动触发

| url_path | 行为 |
|----------|------|
| **`pause`** | **`status=paused`**，`save`，**`sync_task`** |
| **`resume`** | **`status=active`**，`save`，**`sync_task`** |
| **`trigger`** | **同步调用** **`execute_scheduled_task(task.id)`**（非 Celery），用于立即跑一次调度入口 |

## 4. `execute_scheduled_task` 与 `run_scheduled_task`

**`scheduler.execute_scheduled_task(task_id)`**（同步）：

1. 创建 **`ScheduledTaskLog`**：`status=running`，`message` 含「已投递异步执行」。
2. **`run_scheduled_task.delay(tid, scheduled_task_log_id=log.id)`**；投递失败则更新 **log** 与 **`ScheduledTask.last_*`** 为失败。

**`run_scheduled_task`**（`execution/tasks.py`，Celery）：

- 过滤 **`test_cases`**，拆 **API / 非 API**；**API** 用 **`group(run_scheduled_api_case.s(...))`** 并行执行。
- 根据失败数决定 **`ScheduledTaskLog`** 状态：**`success` / `partial` / `failed`** 及 **`detail`** JSON（含 **`skipped_non_api_case_ids`、`api_errors`** 等）。
- 更新 **`ScheduledTask.last_run_time`、`last_status`、`last_message`**。

## 5. `ScheduledTaskViewSet` 批量

- **`batch-delete`** / **`batch-update`**（**`patch` 白名单**：`name`、`cron_expression`、`status`、`environment`、`test_case_ids`）— 每条 **`perform_update`** 会触发 **`sync_task`**（见 Serializer 钩子路径）。
- **`batch-copy`**：新任务 **`job_id` 重新生成**，**`status=active`**，复制 cron/环境/用例 M2M，**事务内 `sync_task`**，失败则回滚。

## 6. `ScheduledTaskLogViewSet`

### 6.1 `get_queryset` 筛选

| Query | 行为 |
|--------|------|
| **`scheduled_task`** | int → **`scheduled_task_id`**；非法 → **`qs.none()`** |
| **`status`** | 精确匹配 |
| **`message`** | **`icontains`** |

排序：**`-trigger_time, -id`**。

### 6.2 `batch-delete`

Body **`{ ids }`**，**`update(is_deleted=True)`**。

### 6.3 `batch-delete-by-filter`（POST）

Body 可选：**`scheduled_task`、`status`、`message`、`before_days`（默认 7，≥1）、`max_delete`（默认 500，1–5000）**。  
在 **`get_queryset()` 等价筛选** 上再加 **`trigger_time < now - before_days`**，取 **最早** 的 **`max_delete` 条** id 批量软删。
