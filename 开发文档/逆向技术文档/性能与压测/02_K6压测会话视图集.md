## 3. `get_queryset` 筛选与排序

| Query | 行为 |
|--------|------|
| **`status`** | 须在 **`STATUS_*`** 集合内才过滤 |
| **`run_id`** | **`run_id__icontains`**（前 64 字符） |
| **`created_by_me`** | `1/true/yes` → **`creator=request.user`**；未登录 → **`qs.none()`** |
| **`created_days`** | 正整数 N → **`create_time >= now - N days`** |
| **`created_start` / `created_end`** | ISO 日期或日期时间；**纯日期 `created_end`** 会扩展到当日 **23:59:59** |
| **`ordering`** | **`create_time` / `-create_time` / `status` / `-status`**；二级 **`-id`** 稳定排序 |
| 默认 | **`-create_time, -id`** |

**queryset 基底**：**`K6LoadTestSession.objects.filter(is_deleted=False)`** — 与 **`BaseModelViewSet`** 无关（本 ViewSet **不继承** 该基类），**无** `project` 成员范围；靠 **`created_by_me`** 或后续若加权限中间件收紧。

## 4. `create` 流程

1. **`serializer.is_valid`** → **`perform_create`**：写 **`status=pending`**，填 creator/updater。
2. **`_enqueue_k6_task(instance.pk)`** → **`run_k6_load_test.delay(session_pk)`**。
3. 若投递异常：**`status=failed`**，**`error_message`** 记录原因。
4. 响应 **201**：**`DetailSerializer` 数据** + **`websocket_path`**：`/ws/k6/<run_id>/` + **`websocket_url_hint`**（Vite 代理 + **`?token=`** 提示）。

## 5. 批量动作

| url_path | 方法 | 说明 |
|----------|------|------|
| **`batch-delete`** | POST | Body **`{ ids: number[] }`**；**`update(is_deleted=True)`** |
| **`batch-copy`** | POST | 复制配置字段，**新 `run_id`**，**重新 `_enqueue_k6_task`**；**不**复制脚本/summary |

## 6. WebSocket 与 ASGI

- **路由**：`execution/routing.py` — **`^ws/k6/(?P<run_id>[0-9a-fA-F-]{36})/$`** → **`K6MetricsConsumer`**。
- **ASGI 挂载**：`AITestProduct/asgi.py` 将 **`execution.routing.websocket_urlpatterns`** 与 server_logs 合并，外层 **`TokenAuthMiddleware`**（`execution/middleware_ws.py`）：**`?token=<DRF Token>`** 写入 **`scope["user"]`**。
- **Consumer**：须 **已认证**；**`K6LoadTestSession` 存在且未软删**；加入组 **`k6_run_<run_id>`**；关闭码 **4401/4404**。

Celery 侧 **`_broadcast`** 使用 **`group_send(..., {"type": "k6.metrics", "payload": ...})`**，Consumer 的 **`k6_metrics`** 方法将 **`payload`** JSON 发给客户端。
