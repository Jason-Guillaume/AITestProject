# 性能相关数据模型总览

## 文档目的与读者

汇总 **`perf_task`**、**`k6_load_test_session`**、**`scheduled_task`**、**`scheduled_task_log`** 及与 **`ExecutionTask`** 的并列关系，便于与《01》～《06》及看板文档交叉索引。

---

## 1. 表一览

| 表名 | 模型类 | 主要用途 |
|------|--------|-----------|
| **`perf_task`** | `PerfTask` | JMeter/Locust 场景卡片；**`task_id` 业务主键** |
| **`k6_load_test_session`** | `K6LoadTestSession` | API 用例链转 **k6** 的一次会话 |
| **`scheduled_task`** | `ScheduledTask` | **Cron** + 环境 + 多对多用例 |
| **`scheduled_task_log`** | `ScheduledTaskLog` | 每次触发/执行的轨迹 |
| **`execution_task`** | `ExecutionTask` | 单 HTTP 任务（见看板与质量 §05） |

以上均继承 **`BaseModel`**（`creator`、`is_deleted` 等），**除 `K6LoadTestSession` ViewSet 外** 多数 REST 仍走 **`BaseModelViewSet`** 软删模式。

---

## 2. `perf_task`

| 字段 | 说明 |
|------|------|
| **`task_id`** | **unique**，业务可见 ID |
| **`scenario`** | **`jmeter` / `locust`** |
| **`concurrency` / `duration` / `status` / `executor`** | 配置与状态 |

**无** `project` FK。

---

## 3. `k6_load_test_session`

| 字段 | 说明 |
|------|------|
| **`run_id`** | **UUID**，**unique**，WebSocket 路径参数 |
| **`test_case_ids`** | JSON 数组，有序 |
| **`vus` / `duration` / `use_ai` / `target_base_url`** | 压测参数 |
| **`status`** | pending → generating → running → completed/failed |
| **`script_rel_path` / `script_body` / `generation_source`** | 生成物与来源 **ai/template** |
| **`summary`** | **`k6 summary-export` JSON** |
| **`error_message` / `celery_task_id`** | 排障 |

---

## 4. `scheduled_task`

| 字段 | 说明 |
|------|------|
| **`name` / `cron_expression` / `status`** | **active/paused/disabled** |
| **`job_id`** | **unique**，APScheduler **job id** |
| **`environment_id`** | FK **`TestEnvironment`**，**PROTECT** |
| **`test_cases`** | M2M **`TestCase`** |
| **`next_run_time` / `last_run_time` / `last_status` / `last_message`** | 运行摘要 |

---

## 5. `scheduled_task_log`

| 字段 | 说明 |
|------|------|
| **`scheduled_task_id`** | CASCADE |
| **`trigger_time` / `start_time` / `end_time`** | 时间线 |
| **`status` / `message` / `detail`** | 成功/部分/失败 + JSON 明细 |

---

## 6. 与 `ExecutionTask` 的区分

| 维度 | `ExecutionTask` | `PerfTask` / k6 / `ScheduledTask` |
|------|-----------------|-------------------------------------|
| 执行引擎 | **`APIExecutor`** | **k6 subprocess** 或 **定时里 `run_api_case`** |
| 典型 REST | 无默认注册 | **`/api/perf/*`**、`/api/execution/scheduled-*` |

- 运行单测：`python manage.py test` 针对所列应用可减少回归风险；与环境变量 `.env`、`REST_FRAMEWORK` 默认分页参数一并核对。
- Swagger/OpenAPI：若启用 `drf-spectacular` 或 `swagger`，请对照自动生成 operationId，避免与实际 `url_path` 漂移。
- 兼容 MySQL/PG：`UniqueConstraint(condition=)` 在无部分索引环境下的替代写法常为 `(..., is_deleted)` 三联唯一——迁移文件为最终语义。
- 缓存：短时热点（配置、类目列表）若在 Redis，`cache.clear()` 不会改变 ORM truth；逆向排障前先验证键 TTL。
- 异步：凡是返回 `task_id` 的路径，均需结合 Celery worker 日志与 Flower，防止「接口 200 但任务未认领」。
- 观测：结构化日志建议使用 `logging.getLogger(__name__)` 统一格式；链路追踪可把 `trace_id` 塞进 `AiUsageEvent.meta`。
- 前端状态：Vue/Pinia store 中与「列表筛选 query」绑定的字段名必须与后端 `query_params.get` **大小写完全一致**（如 `testType`）。
- 导出 CSV：`StreamingHttpResponse` 常与 `record_export_audit` 搭配；SOC 归档保留 actor、时间与过滤条件快照。
- 字段级 AES/脱敏：`EnvironmentVariable` 等若在 service 解密，不得在 `serializer.to_representation` 二次泄露明文到日志。
- RBAC：`BaseModelViewSet.enable_data_scope` 与 `_apply_member_scope` 行为以 `common/views.py` 同步更新为准；组织管理员边界勿混用裸 ORM QuerySet。
- `.cursorignore`/`media`：`MEDIA_ROOT` 下二进制不入版本库；文档中如出现上传路径仅存相对片段。
