## 3. `get_queryset` 筛选

在 **`BaseModelViewSet`** 的 **`is_deleted=False`** 与成员范围之后，再按 query 过滤：

| 参数 | 行为 |
|------|------|
| **`name`** | **`task_name__icontains`** |

## 4. 创建：`perform_create`

- 自动生成 **`task_id`**：**`PT-` + `uuid.uuid4().hex[:8].upper()`**（**唯一**）。
- 已登录：**`serializer.save(task_id=..., creator=user)`**；否则无 creator。

## 5. 批量动作

### 5.1 `_parse_task_id_list`

- Body 键：**`task_ids`**（**字符串数组**）；每项 **`strip`** 后取前 **64** 字符；去重保序；空 → **`ValidationError`**。

### 5.2 `POST .../tasks/batch-delete/`

- **`task_id__in`** + **`is_deleted=False`** → **`update(is_deleted=True)`**（软删）。

### 5.3 `POST .../tasks/batch-update/`

- Body：**`{ task_ids, patch }`**。
- **`patch` 白名单**：**`task_name`、`scenario`、`concurrency`、`duration`、`status`、`executor`**。

### 5.4 `POST .../tasks/batch-copy/`

- 新 **`task_id`** 重新生成；**`status=pending`**；复制 **`scenario/concurrency/duration/executor` 展示字段**；名称加后缀（默认 **`（复制）`**，最长 32）。

---

## 6. `POST .../tasks/<task_id>/run/`（`url_path="run"`）

| 前置状态 | 结果 |
|----------|------|
| 已是 **`running`** | **400**：`任务正在运行中，无需重复触发` |
| 已是 **`completed`** | **400**：`已完成任务不可直接执行，请复制后重跑` |
| 其它 | **`status=running`**，**`executor`** 写入当前用户 **`real_name` 或 `username`**，**`updater`** 更新；**200**：`任务已触发执行` |

**重要**：当前 **`run`** 实现 **仅更新数据库状态与执行人**，**仓库内无** `run_perf_task.delay` **等 Celery 调用**；若界面展示「已触发」但无实际压测进程，属**与外部执行器未接线**或**占位逻辑**，扩展时需在本 action 或独立 worker 中补全。
