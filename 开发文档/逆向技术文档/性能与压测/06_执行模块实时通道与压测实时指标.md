# 执行模块实时通道与压测实时指标

## 文档目的与读者

说明 **k6 压测** 的 **Channels 组名**、**事件 `type` 约定**、**Celery 线程内 `_broadcast`** 与 **WebSocket Consumer** 的对应关系，以及 **Token 鉴权**路径。适合前端订阅、排查「连接成功但无 tick」。

---

## 1. 组名与消息类型

| 侧 | 约定 |
|----|------|
| **Celery / `_broadcast`** | **`group_send(..., group=f"k6_run_{run_id_str}", message={"type": "k6.metrics", "payload": <dict>})`** |
| **Consumer** | **`K6MetricsConsumer.k6_metrics`** 读取 **`event["payload"]`**，**`send(text_data=json.dumps(payload))`** |

即：WebSocket 客户端收到的 **顶层 JSON** 即 **`payload`**，内含业务 **`type`** 字段（**`status` / `error` / `tick` / `final`** 等），**不要**与 Channels 的 **`type: k6.metrics`** 混淆（后者为 **Consumer 方法路由名**）。

---

## 2. 常见 `payload.type`（由 `tasks_k6` 发送）

| `payload.type` | 典型字段 | 含义 |
|------------------|-----------|------|
| **`status`** | **`phase`**：`generating` / `running` / `subscribed`；**`message`** | 阶段说明 |
| **`error`** | **`message`** | 链路基类失败、k6 未安装等 |
| **`tick`** | **`qps`、`http_reqs_total`、`p95_ms`、`error_rate`、`ts`** | 从 stderr 解析的实时采样（约 **0.55s** 节流） |
| **`final`** | **`exit_code`、`summary`、`error_message`** | 压测结束一次性结果 |

---

## 3. WebSocket 连接要求

**URL**：**`/ws/k6/<uuid>/`**（见 `execution/routing.py` 正则）。

**鉴权**（`execution/middleware_ws.py` + `consumers.py`）：

1. **Query**：**`?token=<DRF Token>`** → **`scope["user"]`**。
2. **`connect`**：匿名 → **关闭 4401**；**会话不存在或已软删** → **4404**。
3. 成功：**`group_add`**，并下发 **`subscribed`** 状态消息。

---

## 4. 与 Dashboard SSE 的差异

| 通道 | 路径 | 数据 |
|------|------|------|
| **Dashboard** | **`/api/execution/dashboard/stream/`** | 仅 **tick 时钟**，HTTP **SSE** |
| **k6** | **`/ws/k6/<run_id>/`** | **压测指标与阶段**，**WebSocket** |

二者 **协议、鉴权、中间件** 均不同。

---

## 5. 运维检查

| 问题 | 检查 |
|------|------|
| 无 tick | k6 是否输出汇总行到 stderr；**`K6LiveSample.has_metrics`** 是否为假 |
| 连接 4404 | **`run_id`** 拼错或会话已 **`is_deleted`** |
| 连接 4401 | 未带 **`token`** 或 Token 无效 |
| group_send 无效果 | **`CHANNEL_LAYERS`** 配置、Redis 可达性 |

## 7. 与 HTTP 执行实时通道的边界（备忘）

| 能力 | 协议 | 典型路径 |
|------|------|-----------|
| **k6 压测指标** | WebSocket + Channels | **`/ws/k6/<run_id>/`** |
| **服务器日志 tail** | WebSocket | **`/ws/server-logs/...`**（见服务器日志文档） |
| **Dashboard 心跳** | SSE（HTTP） | **`/api/execution/dashboard/stream/`** |
| **流水线控制台** | 当前实现多为 **HTTP 轮询** | 以前端 `PipelineConsole` 为准 |

勿将 **k6 的 `tick`** 与 **SSE `dashboard_tick`** 混在同一前端封装里解析。

## 8. 压测结束后的数据落库

- **`K6LoadTestSession.summary`**：写入 **`k6 run --summary-export`** 的 JSON；失败时可能为 **`null`** 或解析失败保留旧值（以 **`tasks_k6`** 为准）。
- **`error_message`**：非 0 退出码时附 **stderr 尾部** 或退出码说明，便于在 **无 WebSocket** 时仅从 DB 排障。
