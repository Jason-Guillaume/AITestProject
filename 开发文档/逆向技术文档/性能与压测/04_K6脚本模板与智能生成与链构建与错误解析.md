# K6 脚本模板与智能生成与链构建与错误解析

## 文档目的与读者

串联 **`collect_api_chain`** → **（可选）`generate_k6_script_with_ai`** → **`build_k6_script`** 的脚本生成链，以及 **`k6_stderr_parser`** 如何从 **k6 stderr** 抽实时指标。与 **`tasks_k6.run_k6_load_test`** 阅读顺序一致。

---

## 1. 链路基类：`k6_chain_builder.py`

### 1.1 `resolve_request_url(api_url, target_base_url)`

- 已是 **`http(s)://`** → 原样返回。
- 否则 **`target_base_url`** 非空：去尾 `/`，path 无 `/` 则补 **`/`**，拼接。
- **`target_base_url` 空** 且 URL 非绝对 → 返回相对串（后续 **`collect_api_chain`** 会报错）。

### 1.2 `collect_api_chain(case_ids, target_base_url)`

1. **`case_ids` 空** → **`([], "test_case_ids 不能为空")`**。
2. **`TestCase`** 按 id 拉取，**保持 `case_ids` 顺序**；缺失 id → 错误 **`以下用例不存在或已删除`**。
3. 任一非 **API** 类型 → **`用例 {cid} 不是 API 类型，无法用于 k6 链路`**。
4. 每步需 **`ApiTestCase`** 子表；缺失 → 错误 **`用例 {id} 缺少 API 子表数据`**。
5. 最终 URL 仍非 **`http(s)`** → **`相对路径但未提供 target_base_url`**。

**`steps` 元素字段**：`case_id`、`case_name`、`method`、`url`、`headers`、`body`、`expected_status`。

### 1.3 `chain_to_prompt_json`

将 steps 序列化为 JSON 字符串，供 **AI 提示词** 使用。

## 2. AI 生成：`k6_ai_generator.py`

- **`generate_k6_script_with_ai(steps, vus, duration, ...)`** → **`(script|None, err|None)`**。
- 使用 **`chat_completion`** + 固定 **`_SYSTEM`** 提示（要求 **`import http`**、**`options.vus/duration`**、**`check()`** 等）。
- 模型输出经 **`extract_js_from_model_output`**：优先抽取 **```javascript** 围栏内代码。
- 失败时 **`tasks_k6`** 会 **记录 warning** 并 **回退模板**。

---

## 3. 模板生成：`k6_template_generator.py`

- **`build_k6_script(steps, vus, duration)`**：
  - **`vus`** clamp **1–500**；**`duration`** 须匹配 **`^\d+[smh]$`**，否则回 **`30s`**。
  - 生成标准 **k6 JS**：循环 steps，**`http.request(method, url, body, { headers })`**，**`check`** 状态码（有 **`expected_status`** 则等于，否则 **2xx**），步骤间 **`sleep`**。

**特点**：**无外部网络依赖**、输出确定性强，适合 AI 不可用或失败时的兜底。

---

## 4. 实时 stderr 解析：`k6_stderr_parser.py`

- **`K6LiveSample`**：累积 **`qps`、`http_reqs_total`、`p95_ms`、`error_rate`** 等。
- **`feed_line(state, line)`**：用正则匹配 k6 汇总行（如 **`http_reqs ... : N x.xx/s`**、**`http_req_failed ... : xx%`**、**`p(95)=...`**）。
- **`has_metrics`**：任一有值则 **可推送**。
- **`tasks_k6`** 中 **`on_line`**：**约 0.55s 节流** 调 **`_broadcast`**，**`type: tick`**。

## 5. `run_k6_load_test` 中的错误路径（与消息）

| 阶段 | 失败表现 |
|------|-----------|
| **`collect_api_chain`** | **`session.status=failed`**，`error_message=chain_err`，**`_broadcast type:error`** |
| **AI 失败** | 回退模板；若最终仍无脚本（理论上模板总有）— 以代码为准 |
| **无 k6 可执行文件** | **`shutil.which("k6"|"k6.exe")`** 空 → failed + 明确中文错误 |
| **进程退出码非 0** | **`summary` 仍解析**；**`error_message`** 取 stderr 尾部最多约 **30 行** 或退出码说明 |
