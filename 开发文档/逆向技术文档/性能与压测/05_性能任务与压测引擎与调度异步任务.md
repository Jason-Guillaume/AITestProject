## 2. k6：`run_k6_load_test`（`execution/tasks_k6.py`）

| 配置 | 值 |
|------|-----|
| **`soft_time_limit` / `time_limit`** | **3600 / 3660** 秒 |

**阶段**：

1. 加载 **`K6LoadTestSession`**；写 **`generating`**，**`celery_task_id`**。
2. **`_broadcast`**：`phase=generating`。
3. **`collect_api_chain`**；失败则 **failed + broadcast error + return**。
4. **`use_ai`** 为真则 **`generate_k6_script_with_ai`**；成功则 **`generation_source=ai`**，否则 **模板** 并可能带 **`ai_note`**。
5. 写 **`MEDIA_ROOT/k6_runs/<run_id>/script.js`**，更新 **`script_rel_path`、`script_body`、`running`**。
6. **`subprocess.Popen(k6 run --summary-export ...)`**，stderr **线程逐行读**，**`feed_line` + 节流 broadcast tick**。
7. **`proc.wait()`** 后读 **`summary.json`** → **`session.summary`**；**`completed`/`failed`**，失败写 **stderr 尾**。
8. **`_broadcast`**：`type=final`，含 **`exit_code`、`summary`、`error_message`**。

**依赖**：本机 **PATH** 上有 **`k6`/`k6.exe`**；**`MEDIA_ROOT`** 可写；**Channels** 与 **Redis**（channel layer）可用。

## 3. 定时调度：`run_scheduled_task`

| 配置 | 值 |
|------|-----|
| **`name`** | **`execution.run_scheduled_task`** |
| **`max_retries`** | **2**，**软超时 3600s** |

- 取 **`ScheduledTask`** 与可选 **`ScheduledTaskLog`**。
- **`environment_id`** 传入子任务 **`run_scheduled_api_case`**。
- **非 API 用例** 记入 **`detail.skipped_non_api_case_ids`**，不计入 API 失败统计。
- 终态逻辑：**全 API 失败 → failed**；**部分失败 → partial**；**仅非 API → partial**；否则 **success**。

---

## 4. `APIExecutor`（与 k6 的关系）

**`APIExecutor`** 服务于 **`ExecutionTask`** 与接口用例执行链路（**`requests`**），**不是** k6 子进程。k6 路径 **独立** 使用 **subprocess + 自有脚本**。

---

## 5. 调度器进程内入口：`execute_scheduled_task`

- **同步**、由 **APScheduler** 触发；内部 **`run_scheduled_task.delay`** 异步化真正执行。
- 投递异常：同步更新 **log** 与 **`ScheduledTask.last_*`**。

**`TestScheduler.start()`**（应用启动时调用，位置以项目 **`AppConfig.ready`** 或等价为准）会 **`sync_all_active_tasks`**。
