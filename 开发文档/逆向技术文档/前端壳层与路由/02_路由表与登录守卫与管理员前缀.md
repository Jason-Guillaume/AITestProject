# 壳层与路由：路由表、登录守卫、系统管理员前缀

## 1. 文件

- **`frontend/src/router/index.js`**

## 2. History 模式

- **`createWebHistory()`**，无 hash。**`scrollBehavior`**：`to.hash` 长度大于 1 才 `scrollTo`，规避 `getElementById("")` 告警。

## 3. 根重定向

- **`/` → `/dashboard`**。
- **兜底**：与 **`MainLayout` 路由对象并列** 的 **`{ path: "/:catchAll(.*)", redirect: "/" }`**，未知路径回首页（见 `router/index.js` 末尾）。

## 4. 公开路由

- **`/login`、`/register`**：`meta.public: true`。
- **`/pipelines`、`/pipelines/:id`**：`meta.public: true`（CI 预览；守卫中 **额外** `startsWith('/pipelines')` 直接 `next()`）。

## 5. `beforeEach` 逻辑概要

1. **`/pipelines`**：无条件 `next()`（不走 token 检查）。
2. **`token = localStorage.getItem("token")`**，**`isPublic = to.meta.public === true`**。
3. **非公开且无 token** → **`/login?redirect=to.fullPath`**。
4. **已登录访问登录页** → **`/dashboard`**。
5. **非系统管理员**（`localStorage.is_system_admin !== "1"`）访问 **`SYSTEM_ADMIN_ONLY_PREFIXES`** 精确或子路径 → **`/system/message`**。
6. 否则 **`next()`**。

## 6. `SYSTEM_ADMIN_ONLY_PREFIXES`

包含：`/system/org`、`/system/role`、`/system/user`、`/system/messages`、`/system/ai-usage`、`/system/ai-quota`、`/system/audit`。

## 7. 独立布局的性能路由

- `/performance/tasks`、`scheduled-task`、`scheduled-tasks`、`scheduled-task-logs`、`k6-sessions`：**顶层组件**，非 MainLayout 子级。

## 8. `meta.hiddenInSidebar`

- 若干智能体/场景生成页 **`hiddenInSidebar: true`**：路由存在但侧栏不渲染入口（从 Agent Hub 链入）。

## 9. WebSocket 与路由无关

- WS URL 手工拼 **同源** path；不经过 Vue Router。

## 10. `meta.keepAlive`

- **`ServerLogs`**：`keepAlive: true`，配合 `MainLayout` 的 `<keep-alive>`。

## 11. 代码锚点（守卫）

```229:256:d:/AITestProduct/frontend/src/router/index.js
router.beforeEach((to, _from, next) => {
  if (to.path.startsWith('/pipelines')) {
    next();
    return;
  }
```

## 12. 环境变量

- 前端路由 **无** env 控制 feature flag；若需要可用 **`import.meta.env.VITE_*`** 在 `routes` 生成时过滤（当前未用）。

## 13. 与后端 404

- **History 模式** 部署需 **fallback 到 index.html**；否则刷新子路径 404。

## 14. 小结

守卫 = **/token + 系统管理员前缀 + 流水线特例**；公开流水线与登录态分离是刻意设计。

## 15. 测试建议

- 模拟 `localStorage` 清空访问 `/dashboard` → 跳转登录带 redirect。
- 普通用户访问 `/system/audit` → `/system/message`。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
