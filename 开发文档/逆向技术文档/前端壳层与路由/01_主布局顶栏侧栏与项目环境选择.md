# 壳层与路由：主布局顶栏、侧栏、项目与环境选择

## 1. 文件

- **`frontend/src/layouts/MainLayout.vue`**

## 2. 路由挂载

- `router/index.js`：根路径 **`/`** 下 **`component: MainLayout`**，`children` 包含工作台、用例、报告、性能子路径、服务器日志等。
- **例外**：`/performance/tasks`、`/performance/scheduled-*`、`/performance/k6-sessions` **不在** MainLayout 内。

## 3. 顶栏左侧

- **品牌** + **顶栏菜单**：**项目管理**（`/projects`）、**知识中心**（`/knowledge`）、**帮助中心**（`/help`）。
- **`topNavActive`** 由 `route.path` 前缀计算。

## 4. 顶栏中部：项目与环境

- 两个 **`el-dropdown`**：**Project:**、**Env:**，分别绑定 **`selectedProject`**、**`selectedEnv`**。
- 数据源：**`getProjectsApi`**（`/project/projects/`）、**`getEnvironments`**（`/environments/`）。
- 选择后：**`syncProjectContextById` / `syncEnvironmentContextById`** → **`useAppContextStore`**（Pinia）。

## 5. 顶栏右侧

- 搜索框（本地 `searchKw`，具体搜索路由未在片段展开）。
- **消息铃铛**：**`getSystemMessagesApi`** 未读数；跳转消息管理。
- **刷新**：`router.go(0)`。
- **用户下拉**：`profile` / `password` / `logout`；**`getCurrentUserApi`、`getUserProfileApi`** 同步 **`is_system_admin`**、**`avatar_url`**。

## 6. 侧栏 `el-menu`

- 不用 **`router` 模式**，而由 **`onSideMenuSelect`** 手动 **`router.push(index)`**（注释：规避某些环境下 `$router` 取不到）。
- 菜单项与路由对照见逆向文档「路由表」篇；含 **测试用例子菜单**、`/pipelines`、**`/performance/environments`、`/performance/load-monitor`、`/server-logs`**、缺陷子菜单、**`v-if="isSystemAdmin"`** 系统管理块等。

## 7. 面包屑

- **`BREADCRUMB_MAP`**：`route.path` → 字符串数组；部分路径英文/中文混排（如 performance environments）。

## 8. 主内容区

- **`router-view`** + **`keep-alive`**：仅 **`meta.keepAlive`** 为真时缓存（例：**`ServerLogs`**）。
- **`:key`**：`!keepAlive` 用 **`r.path`**，`keepAlive` 用 **`r.name || r.path`**；`/test-case/:type` 强制重建。

## 9. 后端健康

- **`backendHealthMessage`**：`CustomEvent` **`app:backend-unhealthy`** / **`app:backend-healthy`**（与 **`request` 拦截器、`assistant` fetch** 协同）。

## 10. 修改密码弹窗

- **`changePasswordApi`**：`POST /user/change-password/`。

## 11. WebSocket 提示

- **MainLayout 本身不建 WS**；子页面（压测、日志终端）各自连接 **`/ws/k6/`**、**`/ws/server-logs/`**。

## 12. 环境与 `useCurrentEnvironment` 并存

- 顶栏 **Env dropdown** 与 **`CurrentEnvironmentSelect`** 的 **`localStorage` 键不同**；排查问题时两边都要看。

## 14. 小结

MainLayout = **导航 + 全局上下文 + 子路由出口**；性能「任务管理」等独立壳页面刻意 **不** 汇入此处。

## 15. Store

- **`useAppContextStore`**：持久化当前项目/环境 **展示名**，供报表筛选等（以 store 实现为准）。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。

## 附：与部署相关

- 前端 **不** 内置后端地址时，依赖 **同源** 或 **反向代理** 将 `/api`、`/ws` 指向同一集群。
- 静态资源与 **History 路由** 需 **fallback** 到 `index.html`。
