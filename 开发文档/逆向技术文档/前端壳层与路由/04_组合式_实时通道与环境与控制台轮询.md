# 组合式：实时通道、环境事件与控制台轮询

## 1. `useRealtimePolling`

- **文件**：`frontend/src/composables/useRealtimePolling.ts`
- **用途**：**固定间隔** 调 `refreshFn`（默认 5s），并在 **`document.visibilityState === 'visible'`** 时才触发 interval 回调。
- **选项**：`intervalMs`、`runOnMount`、`refreshEvents`（`window` 事件名数组，监听时 **`executeRefresh`**）、`shouldSkip`。
- **生命周期**：`onMounted` 首次刷新（可关）、`startPolling`、`visibilitychange` 时若可见则立即刷新；`onBeforeUnmount` 清理 timer 与监听器。
- **返回值**：`refreshNow`、`startPolling`、`stopPolling`。

## 2. 使用页面

- **`Dashboard.vue`**：**`useRealtimePolling(() => loadDashboardData(false), { intervalMs: 60000, refreshEvents: [...] })`**（参数以文件为准）。
- **`QualityDashboard.vue`**：类似。

## 3. `useSseRefreshTrigger`

- **文件**：`frontend/src/composables/useSseRefreshTrigger.ts`
- **用途**：**`EventSource`** 订阅后端 **SSE**，每条 message **`executeRefresh`**；出错 **`close`** 并按 **`reconnectMs`（默认 3s）** 重连。
- **选项**：`buildUrl()`、`reconnectMs`、`refreshEvents`、`onConnectionChange`。
- **可见性**：隐藏时 **`closeSource`**，可见时 **`connect`** + 刷新。
- **keep-alive**：**`onDeactivated`** 断开，**`onActivated`** 重连，避免缓存页后台长连接。

## 4. Dashboard SSE URL

- **`getDashboardStreamUrl`**（`frontend/src/api/dashboard.js`）：  
  **`/api/execution/dashboard/stream/?token=...&...`**  
  前端用 **`EventSource`** 时需 **完整 origin** 或相对路径（注意 **`/api` 前缀** 与 cookie/token 策略）。

## 5. `useWebSocket`

- **文件**：`frontend/src/composables/useWebSocket.ts`
- **实现**：`new WebSocket(url)` + `onmessage = handler`，**无重连、无鉴权拼接**。
- **实际业务**：**`LoadTestMonitor`**、**`LogTerminal`** 手写 WS URL（含 **token 查询参数**）。

## 6. 环境事件

- **`app:current-environment-changed`**：`useCurrentEnvironment` 在 `environmentId` 变更时派发；轮询/SSE 可在 `refreshEvents` 数组中加入此事件名字符串 **（若需手动注册）** — Dashboard 是否监听以页面为准。

## 7. 后端健康事件

- **`app:backend-unhealthy` / `app:backend-healthy`**：`request` 与 `streamGenerateAiCases` 抛出网络错误时触发；**不**属于组合式，但与轮询共存。

## 8. 小结

| 机制 | 方向 | 典型场景 |
|------|------|----------|
| `useRealtimePolling` | HTTP 拉 | 仪表盘定时摘要 |
| `useSseRefreshTrigger` | SSE 推触发拉 | 看板刷新信号 |
| `WebSocket` | 双向/推 | k6 指标、日志 tail |
| `PipelineConsole` | HTTP 轮询 | 流水线日志 |

## 9. 选型建议

- **防火墙只放行 HTTP**：选轮询 + SSE。
- **强实时、高频**：WS（注意代理 idle timeout）。

## 10. `visibilityState` 意义

- 三种实现均 **避免后台标签空转**（轮询 interval 内判断可见性；SSE/WS 在隐藏时关闭或减频）。

## 11. TypeScript

- `useRealtimePolling`、`useSseRefreshTrigger` 带 **接口选项**；`useWebSocket` 极简无泛型。

## 12. 单元测试

- Mock `setInterval`、`EventSource`、`document.visibilityState`。

## 13. 与 `useSseRefreshTrigger` 的 URL 构建

- 须返回 **绝对 URL** 或 **同源 path**；若 token 仅放 query，注意 **日志勿打印**。

## 14. 错误风暴

- SSE `onerror` 连续重连：已通过 **`reconnectMs`** 节流；仍应服务端修 **5xx** 根因。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
