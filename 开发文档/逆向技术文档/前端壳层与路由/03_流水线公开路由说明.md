# 壳层与路由：流水线公开路由说明

## 1. 路由定义

- **`/pipelines`**：`name: PipelineList`，`component: PipelineList.vue`，**`meta: { public: true }`**。
- **`/pipelines/:id`**：`PipelineDetail.vue`，同样 **`public: true`**。

（见 `frontend/src/router/index.js`。）

## 2. 守卫特判

```231:234:d:/AITestProduct/frontend/src/router/index.js
  if (to.path.startsWith('/pipelines')) {
    next();
    return;
  }
```

- 效果：**跳过 token 检查**，未登录也可进入流水线列表/详情。
- 若后端日志接口仍要鉴权，列表数据可能为空或 401，由 **`request`** 拦截器处理（可能弹登录）。

## 3. 设计意图（注释）

- **CI/CD dashboard preview**：可在只读环境嵌入 iframe 或用无痕访问。

## 4. MainLayout

- 流水线路由 **在 MainLayout `children` 内**，故仍享受 **侧栏 + 顶栏**（与早期注释「Pipeline routes are now defined as children」一致）。

## 5. 组件：PipelineConsole

- 详情页内 **`PipelineConsole`**：`GET /project/pipelines/{id}/logs/` **轮询**；不依赖 WebSocket。

## 6. API 模块

- **`frontend/src/api/project.js`**：**`getPipelinesApi`**、**`deletePipelineApi`**。

## 7. 与 `meta.public` 的「语义差」

- `meta.public: true` **本可** 让守卫跳过 token，但因 **`startsWith('/pipelines')` 提前 return**，两者重复保险。
- 其它 `meta.public` 只有登录注册；流水线额外宽免。

## 8. 安全风险提示

- 流水线日志可能含 **敏感命令**；若需真正公开，应后端 **匿名 Token** 或 **字段脱敏**。

## 9. WebSocket

- 流水线控制台默认 **HTTP**；**`usePolling: true`**。

## 10. 侧栏入口

- **`el-menu-item index="/pipelines"`**，文案「流水线列表」。

## 11. 面包屑

- `BREADCRUMB_MAP` 应包含 pipelines 项（以 MainLayout 为准）。

## 12. 小结

流水线 = **壳内页面 + 守卫白名单** + **轮询日志**；与其它需登录 API 共存时注意 **401 体验**。

## 13. 联调

- 无 token 打开 `/pipelines`：应渲染壳；XHR 若 401，确认产品是否接受 **半屏错误**。

## 14. 与任务性能页对比

- **性能任务** `/performance/tasks` **不在** 白名单，必须登录。

## 15. 静态部署

- 若流水线页需 **完全离线演示**，可 Mock **`getPipelinesApi`** Service Worker（非现成实现）。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
