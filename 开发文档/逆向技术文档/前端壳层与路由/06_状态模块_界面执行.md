# 状态模块：界面执行（Pinia）

## 1. 文件

- **`frontend/src/store/modules/uiExecution.ts`**
- **Store id**：`'uiExecution'`
- **导出**：`useUiExecutionStore`

## 2. 目标

在 **界面自动化工作台**（`UiAutomationWorkbench.vue` 等）**切换 router-view 或折叠面板** 时，保留：

- 当前 **执行 ID**（数字）、**execution_id 字符串**（展示用）
- **是否运行中**
- **日志数组**（带 time/level/message）
- **结果统计** `ExecutionResult`（总量/通过/失败/跳过/pending）

## 3. 启动：`startExecution`

- **`POST /assistant/ui-script-executions/execute/`**  
  Body：`script_id`（`Number(projectId)`）、`triggered_by: 'manual'`、`browser`、`headless`、`parallel`。
- 成功：写 **`currentExecutionId`、`runExecutionIdStr、runHeaderTime`**，并 **`await pollLogs()`** 保证首屏日志。

## 4. 轮询：`pollLogs`

- 拉 **`GET .../ui-script-executions/{id}/logs/`** 与 **`.../status_detail/`**（路径以 store 内为准）。
- **状态** 为 **`success|failed|timeout|cancelled`**：停轮询、写 **`result`**（从 Redis `result_stats` 或 fallback）。
- 否则：**自适应间隔**（前几秒高频 `_pollTickCount`，防日志空转 `_emptyLogStreak`）。

## 5. `_buildLogView`

- **前缀日志**（执行 ID、说明）+ **远程日志**，避免整表替换把提示行冲掉。

## 6. 清理

- **`clearPolling`**：`clearTimeout` + 复位 `_pollTimer`。
- **`reset`**：清空执行态（以 actions 为准）。

## 7. 与 WebSocket

- **无**；UI 自动化执行链路为 **HTTP 轮询**。

## 8. 与环境

- 请求走 **`request`**，**Token** 自动；**base_url** 由后端引擎决定，不在此 store。

## 9. 与路由

- **`/ui-automation/workbench`**（`router/index.js` **`hiddenInSidebar: true`**）；从 Agent Hub 进入。

## 10. 错误处理

- `startExecution` **catch**：**`isRunning=false`、清空 id** 并 **rethrow** 供页面 `ElMessage`。
- 轮询 **404**：提示可能过期仍 **重试**（防抖动，具体以代码 comment 为准）。

## 11. 结果占位

- `resultStatsFromRedisOrFallback`：无 Redis 统计且未结束时 **`pending: true`**，UI 不展示假数字。

## 12. 日志上限

- **`addLog`** 内 **`MAX = 1000`** 条。

## 13. 小结

**Pinia 保存长任务 UI 态**，避免组件销毁丢日志；与 **`useRealtimePolling`**（页面级）层级不同。

## 14. 测试

- Mock `request.post/get` 序列，断言 **终态 stop** 与 **result**。

## 15. 与 pipeline store

- 无共享；流水线用组件内局部状态 + 轮询。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
