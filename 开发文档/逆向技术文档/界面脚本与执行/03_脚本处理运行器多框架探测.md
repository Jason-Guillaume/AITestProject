# 03 — 脚本处理、运行器与多框架探测（逆向）

## 1. `handle_script_upload`（`assistant.utils.script_handler`）

- 由 `UIScriptUploadViewSet.create` 调用：负责解压、目录布局、入口探测、依赖清单等（具体异常转化为 500 回滚）。
- 与 `workspace_path`、`entry_point` 字段写入相关。

## 2. `ScriptRunner`（`assistant.utils.script_runner`）

- `execute_ui_script(script_upload_instance_id, execution_id, execution_config)`：在子进程/解释器中跑测试脚本，采集 stdout/stderr/返回码。
- `get_execution_logs` / `get_execution_status`：为执行视图提供增量日志与运行态（常结合 Redis key=`execution_id`）。

## 3. 多框架探测（概念）

- 上传模型字段含 `script_type`（如 `LINEAR` / `POM`）；handler/runner 据类型选择启动参数或校验目录结构（细节见对应 py 文件）。
- `execution_config.browser/headless/parallel` 传入 runner，以支持 Selenium/Playwright 共存演进（以代码实现为准）。

## 4. 线程与连接

- 执行线程开头 `close_old_connections()`，避免 Django ORM 连接在线程间泄漏。

## 5. 失败诊断

- `error_message` 截断 8000 字符防 DB 过大；完整栈应写应用日志 `logger.error(..., exc_info=True)`。

## 6. 依赖环境

- Worker 机器需安装浏览器驱动或容器化执行环境；当前架构偏「同一 Django 主机本地起线程」——**不适合**高并发重型 UI 回归，应后续抽成任务队列。
