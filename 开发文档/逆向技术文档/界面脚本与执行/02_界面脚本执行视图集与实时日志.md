# 02 — 界面脚本执行视图集与实时日志（逆向）

## 1. `UIScriptExecutionViewSet`

- 路由：`/api/assistant/ui-script-executions/`；解析器与上传视图一致。

## 2. `get_queryset`

- 不按 `BaseModelViewSet.super()`（执行记录无 `is_deleted`）；支持 `script_id`、`status` query 过滤。

## 3. `POST .../execute/`（`detail=False`）

- 必填 `script_id`；脚本须存在、未删除、已启用。
- `execution_config`：`browser`（默认 chrome）、`headless`（多类型解析为 bool）、`parallel`（1–50）。
- 生成 `execution_id = f"{script_id}_{epoch}"`；创建 `UIScriptExecution` 状态 `PENDING`。
- 后台：daemon 线程 `run_script()`，`close_old_connections()` 包裹：
  - 置 `RUNNING` + `started_at`
  - `ScriptRunner().execute_ui_script(script_upload_instance_id, execution_id, execution_config)`
  - 成功/失败写 `return_code`、`duration`、`error_message`（≤8000）、`completed_at`
- 立即响应：**202** + `UIScriptExecutionSerializer`（异步模型）。

## 4. `GET .../<pk>/logs/`

- Query：`start`/`end` 下标；`ScriptRunner.get_execution_logs`（通常 Redis/文件聚合，具体见 runner）。

## 5. `GET .../<pk>/status_detail/`

- 返回 ORM 序列化 + `runner.get_execution_status(execution_id)` 的 `redis_status` 快照。

## 6. 前端轮询模式

- 先拿 202 的 `id`/`execution_id`，再轮询 `status_detail` 或列表 status，配合 `logs` 拉末尾输出。
