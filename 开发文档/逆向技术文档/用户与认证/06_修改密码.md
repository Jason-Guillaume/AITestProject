# 修改密码

## 文档目的

逆向 **`POST /api/user/change-password/`**（**`ChangePasswordAPIView`**，**`user/views.py`**）：字段 **`old_password`/`new_password`/`confirm_password`、正则复杂度门禁、`check_password`、`set_password` 落库**，以及 **当前实现对 `authtoken_token` 不轮换**的安全含义；辅以与 **`UserSensitiveChangeRequestAPIView`**（审批改密）选型对照。

---

## 1. 路由与鉴权

|**项**|**值**|
|-----|-----|
|**Path**|**`/api/user/change-password/`**|
|**Method**|**POST**|
|**视图类**|**`ChangePasswordAPIView`**|

依赖 **全局 DRF**：**`TokenAuthentication` + `IsAuthenticated`**（若以 Session 兜底同理）。匿名：**401**。**CSRF**：Session 模式下需 **`csrftoken` header**。 

---

## 2. 请求体

|字段|约束|
|----|-----|
|`old_password`|必填；与用户当前哈希比对|
|`new_password`|必填；后续复杂度和非空校验|
|`confirm_password`|必填；必须与 `new_password` **恒等**|

前端 **不得在日志打印 body**。**HTTPS** 传输强制。**WAF**：**限速每 IP**。

---

## 3. 校验顺序（以源码为准）

1. **`len(new_password) < 8` → **400**「长度至少 8」
2. 大写字母 **`[A-Z]`**
3. 小写字母 **`[a-z]`**
4. 数字 **`\\d`**
5. **`old/new/confirm`** 组合：**空、`!=`confirm**422 风格 **`400`**
6. **`user.check_password(old_password)`** 失败：**400**「旧密码不正确」（防用户枚举可考虑 **恒定耗时**sleep——未实现则说明风险）

>**注意**：复杂度 **先于** 「非空字段」校验——与设计文档措辞可能不一致，**但以代码顺序为真**。

---

## 4. 落库与安全存储

```python
user.set_password(new_password)
user.save(update_fields=["password"])
```

**Django 默认 PBKDF2**（可调 **`PASSWORD_HASHERS`**）。**argon2 **若启用：**settings 层**。**历史迁移**：参阅 **`upgrade_passwords` command**。

---

## 5. 成功响应

```json
{ "code": 200, "msg": "密码修改成功", "data": null }
```

**无 JWT 轮转**：泄露 token **仍存在风险**。**补丁策略**：

|**方案**|**复杂度**|
|--------|----------|
|**删其他 Token**|**`rest_framework.authtoken` 表里 filter user 批量 delete(savepoint)**|
|**短 TTL**|**OAuth2**/自定义 refresh|
|**服务端黑名单**|Redis **revoked\_jti set**|

产品若 **金融级**：必须在 **本篇文档 +安全基线 **登记 Accepted Risk**或 **roadmap**。**

## 6. 与敏感审批改密分叉

|`UserSensitiveChangeRequestAPIView`**：生成 **`pending`** 工单；**批准后写哈希**。用于 **企业强制双人复核**。  
 **`change-password`**：即时自助。**互斥**：**UI Profile** tabs 需 **产品线决策**。**若 SSO-only tenant**：隐藏本接口 **404**。**Feature flag**。**
