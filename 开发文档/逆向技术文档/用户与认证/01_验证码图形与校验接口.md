# 验证码图形与校验接口

## 文档目的

## 2. CaptchaAPIView（图形验证码）

**文件**：`user/views.py`  
**鉴权**：`authentication_classes = []`、`permission_classes = []` — **匿名可访问**。

### 2.1 客户端 IP

`get_client_ip`：优先 `HTTP_X_FORWARDED_FOR` 逗号分隔取 **第一段**；否则 `REMOTE_ADDR`；皆空则为字符串 **`unknown`**（仍参与限流键）。

### 2.2 按 IP 限流

- 键：`captcha_limit_{ip}`
- 规则：60 秒内计数，**≥10** 次则返回 HTTP **429**，体：`{"code":429,"msg":"请求过于频繁","data":null}`
- 每次合法生成前：`cache.set(limit_key, cur + 1, timeout=60)`

### 2.3 验证码内容与缓存

- 字符集：`string.ascii_uppercase + string.digits`，长度 **4**，使用 **`secrets.choice`**（非弱随机）。
- `captcha_uuid = str(uuid.uuid4())`
- 写入：`cache.set(f"captcha_{captcha_uuid}", code, timeout=300)` — **300 秒**有效。
- 若 `cache.set` 抛异常 → HTTP **500**，提示缓存不可用（常见：Redis 未启动）。

### 2.4 图像生成

1. 尝试 `from captcha.image import ImageCaptcha`；**ImportError** → 500，文案提示 `pip install captcha`。
2. 优先使用 **`user.captcha_image.CleanTechCaptcha`**（清爽科技风：白底、蓝字、弱噪声）；若模块不存在则回退 **`ImageCaptcha`**。
3. `generate` 传入 `bg_color=(255,255,255)`、`fg_color=(37,99,235)`。
4. 图片转 **Base64**，返回 `data:image/png;base64,...`。

### 2.5 成功响应形状

```json
{
  "code": 200,
  "msg": "获取验证码成功",
  "data": {
    "uuid": "<captcha_uuid>",
    "image": "data:image/png;base64,..."
  }
}
```

前端字段名通常为 **`uuid`**（与注册序列化器中的 **`captcha_uuid`** 对应，注意命名差异）。

## 3. 校验发生位置（非本 View）

- **注册**：`UserRegisterSerializer.validate`（`user/serialize.py`）— **强制** `captcha_code` + `captcha_uuid`，比对后 **`cache.delete(captcha_{uuid})`**（一次性消费）。
- **登录**：`UserLoginSerializer.validate` — 若 **同时** 传入 `captcha_code` 与 `captcha_uuid`（均非空）则校验图形码；否则 **不要求** 图形验证码（便于内网或自动化场景）。

---

## 4. CaptchaRecognizeAPIView（OCR，自动化专用）

**路径**：`POST /api/user/captcha/recognize/`  
**鉴权**：匿名；但需 **服务端配置** `CAPTCHA_OCR_SECRET`（Django settings），且请求头 **`X-Captcha-Ocr-Secret`** 与之一致，否则 **403**。未配置密钥 → **503**「未启用 OCR」。  
**Body**：JSON，`image` 为 base64 或 `data:image/...;base64,...`；最大约 **2MB** 解码后长度校验。  
**依赖**：`pip install ddddocr`；返回 `{"code":200,"data":{"text":"..."}}`，文本 **`strip` 后截断 16 字符**。

---

## 7. 显式与静默

| 显式 | 静默 |
|------|------|
| 返回的 `uuid`、Base64 图片 | 明文验证码仅存于缓存，不落库、不返 JSON |
| 429/500 的业务 `code` | 限流计数键过期由缓存 TTL 自动回收 |

---

## 8. 排障清单

1. **500 缓存不可用**：检查 `CACHES`、Redis 进程、网络。
2. **500 未安装 captcha**：`pip install captcha`。
3. **注册总提示验证码错误**：时钟漂移一般不影响；检查前后端 **`uuid` 字段名** 是否与序列化器一致（`captcha_uuid`）。
4. **429**：同 IP 刷新过频；等待 60 秒或换出口 IP。
