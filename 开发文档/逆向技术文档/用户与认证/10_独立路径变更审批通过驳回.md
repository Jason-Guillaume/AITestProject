# 独立路径变更审批通过驳回

## 文档目的

说明 **`AITestProduct/urls.py`** 如何将 **`path("api/change-requests/", include("user.approval_urls"))`** 独立挂载——**前缀不在 `/api/user/`**；并逆向 **`POST .../<pk>/approve/`**、**`POST .../<pk>/reject/`** 与 **`permission_classes = [IsApprovalAdmin]`**，及其与 **`approve_change_request`/`reject_change_request`**、站内信 **`mark_messages_for_request`** 的一致性。

---

## 1. 路由表

|**文件**|**`user/approval_urls.py`**||
|----------|--------------------------|--|
|**pattern**|**`<int:pk>/approve/`**|**`AdminChangeRequestApproveAPIView.as_view()`**|
||**`<int:pk>/reject/`**|**`AdminChangeRequestRejectAPIView.as_view()`**|

|**完整示例**|**`POST https://host/api/change-requests/12/approve/`**|

|**对比**|**`/api/user/admin/change-requests/<pk>/decision/`** REST 动词聚合（**`09`** 篇）|

两套路由 **互不冲突**：网关可对 **`/api/change-requests/`** 单独限速或 **mTLS**，满足 **SOC2 网关分段**诉求。

---

## 2. 权限类

两视图：**`permission_classes = [IsApprovalAdmin]`**。  
语义见 **`09`**：**`superuser/staff/is_system_admin`**。  
若不满足：**403**。  
**JWT/Token**：依赖 **全局 `DEFAULT_AUTHENTICATION_CLASSES`**——无匿名访问。

---

## 3. 处理流程伪代码

1. **`pk` 载入 **`UserChangeRequest`** + **`select_related(\"user\")`**；**`DoesNotExist` → **404**。
2. **approve：** **`approve_change_request(cr, approver=request.user if authenticated else None)`**
3. **reject：** **`reject_change_request`** 对称。
4. **`ValueError` → HTTP **400**：业务规则（工单已结案、用户名占用、密码哈希非法等）。
5. **响应体：** 与 **`09`** **`decision` 成功 envelope**对齐（`**`code`** `200`、`msg`、`data`**）——请以 **`views.py`** 两行 diff 校对。

---

## 4. 通过后副作用摘要（`approve_change_request`）

|**类型**|**要点**|
|--------|-------|
|**状态**|**`pending`→`approved`（或等价枚举常量）|
|**用户名**|目标名未被他人占用——**原子更新 **`User.username`****|
|**密码**|申请体保存的是 **哈希**——**赋值 **`User.password`**** ，**不写明文**|
|**审批人**|**`approved_at`**、`**`processed_by`/ `approver` **字段回填|
|**收件箱**|**`mark_messages_for_request`**：**`SystemMessage.related_request`** 对齐置 **已读/归档**|

`**`reject`**：通常 **不写用户表**，只 **工单关单 + 通知申请人**。

---

## 5. 为何独立于 `/api/user/`

|**原因**|**运维收益**|
|--------|----------|
|**ACL 粒度**|审批专用代理 **仅暴露 change-requests**，降低横向移动面|
|**OpenAPI**|生成 **Swagger tag `Approval`** 更直观|
|**审计**|WAF 规则 **hit count**单独统计|

若 **同域 Cookie CSRF**：**POST** **需 **`CSRF_TRUSTED_ORIGINS`/DRF **`SessionAuthentication`**** 对齐——SPA 若以 Token 则无 CSRF cookie。

---

## 6. 与 **`09` 裁决接口的选型

|**场景**|**偏好**|
|--------|--------|
|**脚本化批量 approve**|**`decision` 单路由**更少 HTTP 往返|
|**可读 RESTful**|**`/approve`、`/reject` **路径自解释|

集成测试应 **任选其一覆盖业务函数**内核即可——**Vitest+E2E** 建议 **approve 路径**，更贴近网关真实流量。

---

## 7. 错误码表（典型）

|**HTTP**|**含义**|
|--------|-------|
|**400**|**`ValueError` 文本**镜像 **`msg`**|
|**403**|**非 ApprovalAdmin**|
|**404**|**id 不属于任何工单或被软删**|

前端 **不应**仅用 **HTTP status**——**仍需 parse body.msg**展示。
