# 个人审计事件查询

## 文档目的

说明 **`GET /api/user/me/audit/events/`** 的分页、筛选参数及返回字段；与 **系统级** **`GET /api/sys/audit/events/`** 的边界划分。

---

## 1. HTTP 路由与权限

- **路径**：`GET /api/user/me/audit/events/`
- **视图**：`UserAuditEventListAPIView`
- **权限**：**`IsAuthenticated`**（显式声明）；匿名 **不允许**。

---

## 2. 查询参数

| 参数 | 默认 | 约束 | 说明 |
|------|------|------|------|
| `page` | 1 | ≥1 | 页码 |
| `page_size` | 50 | 1～200 | 每页条数 |
| `action` | 空 | 精确匹配 | 如 `create`/`update`/`delete` |
| `object_app` | 空 | 精确 | Django app label |
| `object_model` | 空 | 精确 | 模型名 |
| `object_id` | 空 | 精确 | 对象主键字符串 |

---

## 3. 查询集范围

```python
AuditEvent.objects.filter(is_deleted=False, creator=request.user)
```

即：**仅当前登录用户作为 `creator` 写入的审计行**（与 `BaseModelViewSet` 里 `record_audit_event` 传入的 `actor` 一致时才会出现）。

**不包含**：他人操作、系统匿名任务写入的行。全量审计请走 **`/api/sys/audit/events/`**（见系统级智能与审计分册）。

---

## 4. 返回字段（`values` 投影）

每条 `items` 元素包含：

`id`, `action`, `object_app`, `object_model`, `object_id`, `object_repr`, `request_path`, `ip`, `extra`, `create_time`

**不包含**：`before` / `after` 大 JSON（减轻带宽与敏感信息暴露；若产品需要可改视图扩展）。

外层 `data`：

```json
{
  "page": 1,
  "page_size": 50,
  "total": 123,
  "items": [ ... ]
}
```
