# 个人资料维护

## 文档目的

说明 **`UserProfileAPIView`**：`GET` / **`PATCH`** `/api/user/me/profile/` 的字段范围、解析器与头像校验规则。明确 **用户名、登录密码** 不在此接口修改（走敏感变更审批）。

---

## 1. HTTP 路由

- **路径前缀**：`/api/user/me/profile/`（`user/urls.py`）
- **方法**：`GET`（读当前用户资料）、`PATCH`（部分更新）
- **视图**：`UserProfileAPIView`

---

## 2. 解析器

`parser_classes = [JSONParser, MultiPartParser, FormParser]` — 支持 **JSON** 与 **multipart 文件上传**（头像）。

---

## 3. 序列化器 `UserProfileSerializer`

**文件**：`user/serialize.py`  
**模型**：`User`

### 3.1 可读写字段

| 字段 | 说明 |
|------|------|
| `real_name` | 展示名 |
| `phone_number` | 手机 |
| `email` | 邮箱 |
| `avatar` | `ImageField`，上传至 `user_avatars/%Y/%m/` |

### 3.2 只读

- `id`、`username`、`avatar_url`（`SerializerMethodField`）

### 3.3 `avatar_url` 生成

若存在 `obj.avatar`：`request.build_absolute_uri(obj.avatar.url)`，便于前端直接 `<img :src>`。无 request 上下文时退回相对 `url`。

### 3.4 `validate_avatar`

- 最大 **2MB**（`value.size`）。
- 扩展名白名单：**jpg、jpeg、png、webp、gif**（小写比较）。

---

## 4. 响应形状

成功时均为：

```json
{ "code": 200, "msg": "ok|资料已更新", "data": { ...序列化结果... } }
```

`PATCH` 使用 `partial=True`，仅提交需改字段即可。

---

## 5. 数据库

- **表**：`sys_user`（`user.models.User`）
- **头像文件**：`MEDIA_ROOT` 下按 `upload_to` 存储；开发环境可通过根路由 `static(MEDIA_URL, ...)` 访问。

|------|------|
| 返回体中的 `avatar_url` | 不校验邮箱/手机在全局的唯一性（若业务需要应在 `validate_*` 中补） |
