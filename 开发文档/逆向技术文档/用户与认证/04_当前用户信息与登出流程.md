# 当前用户信息与登出流程

## 文档目的

区分 **`GET /api/user/me/`**（轻量当前用户）与 **登出**（当前实现为 **纯前端**）两条路径，避免误以为存在「服务端 session 销毁」接口。

---

## 1. 当前用户：`GET /api/user/me/`

- **视图**：`CurrentUserAPIView`（`user/views.py`）
- **鉴权**：使用全局 DRF 配置 — **需登录**（默认 `IsAuthenticated` + `TokenAuthentication`，以 `settings` 为准）。

### 1.1 未登录

HTTP **401**，体：

```json
{ "code": 401, "msg": "未登录", "data": null }
```

### 1.2 已登录

HTTP 200：

```json
{
  "code": 200,
  "msg": "ok",
  "data": {
    "user_id": 1,
    "username": "...",
    "real_name": "",
    "is_system_admin": false
  }
}
```

用途：前端路由守卫、顶栏展示、与 `localStorage` 缓存的 **二次对齐**（例如刷新页面后恢复 `is_system_admin`）。

---

## 2. 登出流程（现状）

### 2.1 后端

**无** `POST /api/user/logout/` 等路由；**不**在登出时删除 `authtoken_token`。

### 2.2 前端

**`frontend/src/layouts/MainLayout.vue`** — 用户菜单「退出登录」：

1. `ElMessageBox.confirm` 确认；
2. `localStorage.removeItem('token'|'username'|'is_system_admin'|'user_id'|'real_name')`；
3. 清空头像展示用 ref；
4. `router.push('/login')`。

令牌仍保存在数据库中，直至用户再次登录 **`get_or_create`** 复用同一行，或管理员在后台删除 Token。

### 2.3 若需增强安全

可选实现：`POST /api/user/logout/` → `request.user.auth_token.delete()`（需处理匿名）；或轮换 Token；或对接 SSO 统一登出。

---

## 3. 与「修改密码」的关系

修改密码接口见 **`06_修改密码.md`**；成功后 **不** 自动吊销旧 Token（当前代码未 `delete` Token）。若密码泄露后仅改密，建议同步清理 Token 或强制重新登录。
