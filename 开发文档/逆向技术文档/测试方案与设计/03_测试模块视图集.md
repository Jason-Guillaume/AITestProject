# 测试模块视图集

## 文档目的

逆向说明 **`TestModuleViewSet`** 在 Django REST Framework 中的挂载方式、查询参数语义以及与 **`TestCaseViewSet`** 树形筛选函数的协作关系；读者可据此在前端 **`/api/testcase/modules/`** 上正确传递 **`project`、`testType`** 并渲染父子模块树。

---

## 1. 路由与挂载

源码位置：**`testcase/urls.py`**（注意文件头注释可能误写为 project，实为 testcase 路由器）。

行为：

- **`router.register(r"modules", TestModuleViewSet)`**
- HTTP 前缀与项目根 **`AITestProduct/urls.py`** 组合后为 **`/api/testcase/modules/`**
- 标准动作：`list` / `create` / `retrieve` / `update` / `partial_update` / `destroy`，均继承 **`common.views.BaseModelViewSet`**。

基类通常为列表与详情自动施加 **软删除过滤（`is_deleted=False`）** 与 **组织/成员数据范围**，与具体实现以 **`common/views.py`** 为准；新增或修改模块时序列化层会校验与项目、父节点的引用完整性。

---

## 2. `get_queryset` 逐步拆解

实现类：**`testcase/views.py`** 中 **`TestModuleViewSet.get_queryset`**。

1. 以 **`super().get_queryset()`** 为起点：继承全局 queryset 并叠加基类的数据范围与安全过滤。
2. **项目过滤 `?project=`**  
   - 若查询参数 **`project`** 非空：尝试 **`int(project)`**。  
   - 转换失败则返回 **`qs.none()`**，避免异常值导致全表泄露。  
   - 合法时 **`qs.filter(project_id=project_id)`**。
3. **用例子类型过滤 `?testType=`**  
   - 注意参数名驼峰 **`testType`**，与 **`TEST_CASE_TYPE_CHOICES`** 的取值集合求交集；仅当传入值在白名单内才 **`filter(test_type=test_type)`**。  
   - 不传或非法值则不按类型收窄，便于「全部模块」下拉。
4. **稳定排序**：**`order_by("parent_id", "id")`**  
   - 同一父节点下 **`id`** 递增，前端可用一次列表遍历 O(n) 构建树；**`parent_id` 为空的节点**最先出现（在同一父键分组规则下）。

## 3. 与用例树的交叉：` _module_self_and_descendant_ids`

**`testcase/views.py`** 模块级函数 **`_module_self_and_descendant_ids(root_id, project_id=None)`** 在 **`TestCaseViewSet`** 筛选「某模块节点及其子孙」时使用：

- BFS / 逐级查询 **`TestModule`** 子节点，`**is_deleted=False**`；
- 可选 **`project_id`** 收窄到同一测试项目，避免跨项目误关联；
- 返回 **去重后的 id 列表**，供 **`filter(module_id__in=...)`** 一类查询。

本篇视图集本身是 **资源的 CRUD 与列表**；用例侧的「选了树节点要带子孙」在阅读 **`01_测试用例视图集与分页筛选.md`** 时需对照此函数语义。

## 4. 模型摘要（`TestCase.models.TestModule`）

- **表名**：通常为 **`test_module`**（以迁移为准）。
- **外键 `project`**：指向 **`project.models.TestProject`**（或等价项目模型）， **`related_name` 常为 `modules`**，保证「一个项目多套模块树」隔离。
- **字段**：**`name`**、**`parent`**（自引用，可空）、**`test_type`**（与 **`TestCase`、`TestModule`** 共用 **`TEST_CASE_TYPE_CHOICES`** 字符串）。
- **软删除**：沿用 **`BaseModel`** 上的 **`is_deleted`** 及审计字段，删除操作在 ViewSet 中走 **`perform_destroy`** 语义而非物理 **`DELETE`**。

序列化请参考 **`testcase/serialize.py`** 中 **`TestModuleSerializer`**：**`fields`/`read_only`** 决定前端可写字段（如 **`name`、`parent`、`project`、`test_type`**）。

## 5. 典型联调要点

- 切换项目时必须 **重新请求 `modules?project=<id>`**；旧缓存会导致模块树与用例列表不一致。
- **`testType`** 与用例列表的 **`test_type`** 应同源，以免 UI 上出现「选了接口模块却列表筛功能用例」的错位。
- 若列表为空，先用浏览器或 curl 核查 **`project` 是否为 int**、 **`testType`** 是否与后端常量完全一致（区分连字符 **`ui-automation`** 等）。
