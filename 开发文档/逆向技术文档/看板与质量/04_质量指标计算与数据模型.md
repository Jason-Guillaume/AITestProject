# 质量指标计算与数据模型

## 文档目的与读者

说明 **`MetricCalculator`** 三个 **`calc_*`** 的 **SQL 口径**、**`TestQualityMetric`** 表结构及 **`update_or_create`** 维度；与《03》质量看板接口联读。

## 1. `TestQualityMetric`（`execution.models`）

| 字段 | 说明 |
|------|------|
| **`metric_date`** | `DateField`，指标归属日 |
| **`metric_type`** | **`pass_rate` / `defect_density` / `requirement_coverage`** |
| **`metric_value`** | **`Decimal(12,4)`** |
| **`dimension`** | JSON；计算器写入 **`{"project_id": <int>}`** 或 **`{"project_id": "all"}`**（全平台） |

**表名**：**`test_quality_metric`**。  
**索引**：**`(metric_date, metric_type)`**。  
**ordering**：**`-metric_date`, `-id`**。

---

## 2. `_upsert_metric` 唯一性

**`update_or_create`** 查找键：

- **`metric_date`**
- **`metric_type`**
- **`dimension`**（完整 JSON 对象相等匹配）

**`defaults`**：仅 **`metric_value`**。  
同一项目、同一日、同一类型 **重复计算会覆盖** 旧值。

---

## 3. `calc_pass_rate(project_id, metric_date)`

- **数据源**：**`ExecutionLog`**，`**is_deleted=False**`，**`create_time__date=metric_date`**。
- **项目过滤**：`project_id is not None` 时 **`test_case__module__project_id=project_id`**。
- **计算**：**`passed = Count(id, filter=is_passed=True)`**，**`total=Count(id)`**；若 total>0：**`(passed/total)*100`**，**`quantize(0.0001)`**。
- **副作用**：**`_upsert_metric`** 写入 **`METRIC_PASS_RATE`**。

---

## 4. `calc_defect_density(project_id, metric_date)`

- **缺陷**：**`TestDefect`**，`is_deleted=False`，**`create_time__date=metric_date`**；有 **`project_id`** 时 **`module__project_id=project_id`**（缺陷须挂模块项目链）。
- **用例分母**：**`TestCase`**，`is_deleted=False`；同样按 **`module__project_id`** 过滤。
- **计算**：**`defect_cnt / case_cnt`**（case 为 0 则密度 0），**`quantize(0.0001)`**。
- **副作用**：**`METRIC_DEFECT_DENSITY`**。

---

## 5. `calc_requirement_coverage(project_id, metric_date)`

- **数据源**：**`TestPlan`**，`is_deleted=False`，**`create_time__date=metric_date`**；有项目时 **`version__project_id=project_id`**。
- **聚合**：按 **`version_id`** **`values().annotate(req=Avg("req_count"), cov=Avg("coverage_rate"))`**。
- **公式**：  
  **`total_req`** = 各版本 **`req`** 之和（转 int）；  
  **`covered_req`** = Σ **`req * cov / 100`**；  
  若 **`total_req>0`**：**`coverage = covered_req / total_req * 100`**。
- **副作用**：**`METRIC_REQUIREMENT_COVERAGE`**。

> 该指标依赖 **`TestPlan`** 上 **`req_count`/`coverage_rate`** 的维护质量；若计划未日增，则当日曲线可能为 0。

---

## 6. 与看板接口的维度对齐问题

计算器写入 **`project_id: "all"`** 时，看板查询使用 **`dimension__project_id=None`** 可能 **无法命中**（见《03》）。扩展全平台大盘时需统一 **dimension 契约**。
