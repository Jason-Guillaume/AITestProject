# 执行任务模型与引擎入口

## 文档目的与读者

说明 **`ExecutionTask`** 模型字段语义、**`run_execution_task`** Celery 任务（锁、状态机、**`ENV_UNHEALTHY`** 分支）与 **`APIExecutor.execute`** 的关系。适合对接「单 URL 探测」类后台任务或扩展 **ViewSet 暴露 CRUD**。

---

## 1. 模型 `ExecutionTask`（表 **`execution_task`**）

| 字段 | 说明 |
|------|------|
| **`task_name`** | 展示名 |
| **`method`** | HTTP 方法，默认 **GET** |
| **`url`** | 请求 URL，最长 2048 |
| **`headers` / `body`** | JSON；body 可 **null** |
| **`timeout_seconds`** | 默认 **30** |
| **`expected_status`** | 可选期望状态码 |
| **`expected_body_contains`** | 可选响应体子串断言 |
| **`status`** | **`pending` / `running` / `completed` / `failed`** |
| **`started_at` / `finished_at` / `duration_ms`** | 执行窗口与耗时 |
| **`celery_task_id` / `error_message` / `report`** | 运行痕迹；**`report`** 为引擎返回结构 |
| **`project`** | 可选 FK **`TestProject`**，`SET_NULL`，注释说明用于权限冗余 |

**ordering**：**`-create_time`**。

## 2. 引擎 `APIExecutor`（`execution/engine.py`）

- **`execute(self, task: ExecutionTask)`**：构造 **`ExecutionTask`** 维度、**`HealthChecker.check_before_task`**（见环境与变量文档）、**`requests.Session.request`**、断言、脱敏日志。
- **失败**：环境不健康 → **`RuntimeError("ENV_UNHEALTHY: ...")`**；非法方法/URL 等 → **`ValueError`**。

返回 dict 含 **`passed`、`duration_ms`、`request/response` 摘要、`assertions`** 等（以引擎实现为准）。

## 3. Celery：`run_execution_task`（`execution/tasks.py`）

| 配置 | 值 |
|------|-----|
| **`name`** | **`execution.run_execution_task`** |
| **`autoretry_for`** | **`(Exception,)`** |
| **`max_retries`** | **3** |
| **`soft_time_limit` / `time_limit`** | **60s / 75s** |

### 3.1 分布式锁

- **Key**：**`execution_task_lock:{id}`**  
- **`cache.add`**，**timeout=180**；未拿到锁 → **`{"skipped": True, "reason": "任务正在执行中"}`**（**不**改 DB）。

### 3.2 事务段

**`select_for_update`** 取 **`is_deleted=False`** 任务：

- 若已是 **`STATUS_RUNNING`**：**删锁** → **`skipped: 任务已在运行态`**。
- 否则置 **`running`**，写 **`started_at`**，清空 **`finished_at`/`duration_ms`/`error_message`**，**`celery_task_id=self.request.id`**。

### 3.3 执行与异常

- **`APIExecutor().execute(task)`** 成功：写 **`report`**（注入 **`trace_id`**）、**`duration_ms`/`finished_at`**，**`status=completed` 若 passed 否则 failed`**；失败且非环境：**`error_message="断言未通过"`**（未通过时）。
- **`except Exception`**：
  - **`ENV_UNHEALTHY` 前缀**：**`status=failed`**，**`report.cancelled=True`**，**`error_message` 固定中文「环境健康检查失败…」**，**`return task.report`（不 retry）**。
  - **其它**：写失败 **`report`** 后 **`raise self.retry(exc=exc)`**。
- **`finally`**：**`cache.delete(lock_key)`**。

## 4. 与 REST 的关系

**`execution/urls.py`** 的 **DefaultRouter** **未** 注册 **`ExecutionTaskViewSet`**。任务通常由 **其它模块、管理命令或未来 API** **`run_execution_task.delay(execution_task_id)`** 触发。若新增 REST，建议：

- 继承 **`BaseModelViewSet`** 并挂 **`project`** 成员范围；
- 创建后置 **`pending`**，由前端或队列触发 **`delay`**。
