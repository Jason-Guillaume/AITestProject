# 看板流式输出接口

## 文档目的与读者

说明 **`GET /api/execution/dashboard/stream/`**（**`DashboardStreamView`**）的 **SSE** 语义、**异步实现动机**、**Token 透传**方式及与 **`dashboard/summary`** 的配合方式。适合排查 EventSource 401、Nginx 缓冲导致断流等问题。

---

## 1. 路由与响应类型

| 项 | 值 |
|----|-----|
| URL | **`/api/execution/dashboard/stream/`** |
| 注册 | `execution/urls.py` → `path("dashboard/stream/", DashboardStreamView.as_view(), ...)` |
| 响应 | **`StreamingHttpResponse`**，`Content-Type: text/event-stream; charset=utf-8` |

---

## 2. 认证策略（`_resolve_user`）

顺序：

1. **`request.user.is_authenticated`** → 直接返回用户。
2. 否则读取 **`?token=<DRF Token 字符串>`**。
3. 若无 query token，再解析 **`Authorization: Token <key>`**（大小写不敏感前缀 **`token `**）。
4. **`Token.objects.select_related("user").get(key=token_key)`**；不存在 → **`None`**。

**失败**：返回 **`JsonResponse({"detail": "认证失败"}, status=401)`**，**体为 JSON**，**非** `text/event-stream`。浏览器原生 **`EventSource`** 无法读取 HTTP 状态码时，需在封装层用 **`fetch`+ReadableStream** 或统一错误页处理。

---

## 3. 异步与线程池（实现要点）

视图使用 **`async def get`**，循环内 **`await asyncio.sleep(3)`**。注释说明：在 **ASGI** 下若用同步 view + **`time.sleep`** 会长期占用同步线程池，导致其它 API 超时；故改为 **async sleep** 释放线程。

**注意**：`get` 为 async 时，需确认部署栈（Django ASGI + uvicorn 等）对 **`StreamingHttpResponse` + async view** 的兼容性；若出现异常可回查框架版本说明。

---

## 4. SSE 帧格式

| 顺序 | 内容 |
|------|------|
| 首帧 | **`retry: 3000\n\n`**（客户端重连间隔提示，单位 ms） |
| 循环体 | **`data: {JSON}\n\n`** |

**JSON 负载**（逻辑）：

```json
{
  "type": "dashboard_tick",
  "ts": "<ISO8601>",
  "project_id": "<来自 GET 的原始字符串，可为 null>"
}
```

**`project_id`** 来自 **`request.GET.get("project_id")`**，**不做 int 校验**，原样放入 JSON（可能为 **`"1"`** 或 **`None`**），前端若需数值应自行转换。

## 5. 响应头（防缓存与反向代理）

- **`Cache-Control: no-cache, no-store, no-transform`**
- **`Pragma: no-cache`**
- **`X-Accel-Buffering: no`**（Nginx 关闭缓冲，降低 SSE 延迟）

---

## 6. 与摘要接口的分工

| 接口 | 数据 |
|------|------|
| **`/dashboard/stream/`** | 仅 **tick 信号**，不含业务聚合 |
| **`/dashboard/summary/`** | 卡片、折线、饼图等 **完整聚合**（见《02》） |

推荐模式：**EventSource 收到 tick** → 防抖后 **`GET .../dashboard/summary/`**（可带 **`project_id`**）刷新界面。
