# 组织角色用户管理

## 1. 路由（系统管理员专享）

`frontend/src/router/index.js` 中常量 **`SYSTEM_ADMIN_ONLY_PREFIXES`** 包含：**`/system/org`、`/system/role`、`/system/user`**。

当用户已登录但 **`localStorage.is_system_admin !== '1'`** 时访问上述任一精确路径（或以它们为前缀的路径），守卫会 **`next({ path: '/system/message' })`**。

| Path | Vue |
|------|-----|
| `/system/org` | `views/system/OrgManagement.vue` |
| `/system/role` | `views/system/RoleManagement.vue` |
| `/system/user` | `views/system/UserManagement.vue` |

均需 **MainLayout + token**。

## 2. Org API（`api/system.js`）

- `GET/POST/PATCH/DELETE /user/orgs/` — `getOrgsApi`、`createOrgApi`、`updateOrgApi`、`deleteOrgApi`  

**OrgManagement** 还可能 **`import { getUsersApi }`** — 选人挂到组织。

## 3. Role API

- **角色**：`frontend/src/views/system/RoleManagement.vue` — `getRolesApi` 等与 **`createRoleApi` / `updateRoleApi` / `deleteRoleApi`**（前缀 **`/user/roles/`**）。若表单含菜单权限 JSON，以组件内绑定为准。

## 4. User API

**UserManagement.vue**：  

- **`getUsersApi(params)`**
- **`createUserApi`、`updateUserApi`、`deleteUserApi`**  

接口前缀 **`/user/users/`**。表格列：用户名、邮箱、所属组织等（以后端 serializer 为准）。

## 5. Login 与用户态钩子

**`is_system_admin`**：`Login`、`MainLayout` 等若在登录后调用 **`GET /user/me/`**，应将标志写入 **`localStorage`**（常见为字符串 **`'1'`**），守卫据此放行系统路由。

## 6. Pinia Store

三块页面一般用 **表格 + dialog refs**；未发现统一 **`systemManagementStore`**。

## 7. Query Params

通常为 **服务端分页**：`page`/`page_size`/`search`。无强制 path query。

## 8. UX 协同

Cyber 表单；危险操作需二次确认。角色权限若为 JSON/YAML，需在前端或服务端校验格式后再提交。

## 9. Related

站内信 **`/system/messages`** 与同前缀策略；与 **配额/审计** 相邻菜单。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。

建议在 DevTools Network 中过滤 **`/api/`**；并与本文档中的模块前缀交叉核对（如 `/testcase/`、`/execution/`、`/defect/`、`/assistant/`、`/sys/`、`/user/`）。

3. 若逻辑拆至 `components/` 子组件，亦需对该目录做同样检索。
