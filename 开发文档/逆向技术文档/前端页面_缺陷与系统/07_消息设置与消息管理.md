# 消息设置与消息管理

## 1. 路由

| path | 组件 |
|------|------|
| `/system/message` | `frontend/src/views/system/MessageSetting.vue` |
| `/system/messages` | `frontend/src/views/system/MessageManage.vue` |

均位于 **MainLayout**；需登录。**消息管理页**被列入 **`SYSTEM_ADMIN_ONLY_PREFIXES`** → **仅 **`localStorage.is_system_admin === '1'`** **可访问**，否则 **`beforeEach`** 重写到 **`/system/message`**。**消息设置页**是普通用户 **默认落点（被守卫 redirect）**。

## 2. MessageSetting API（`frontend/src/api/system.js`）

| API | HTTP |
|-----|------|
| `getMessageSettingsApi` | `GET /user/message-settings/` |
| `createMessageSettingApi` | `POST /user/message-settings/` |
| `updateMessageSettingApi` | `PATCH /user/message-settings/:id/` |

配置项：告警渠道开关、SMTP/Webhook placeholder（具体字段随 serializer）。  

## 3. MessageManage API（`frontend/src/api/user.js`）

| API | HTTP |
|-----|------|
| `getSystemMessagesApi` | **`GET /user/system-messages/`** 收件箱列表 |
| `markSystemMessageReadApi` | **`PATCH /user/system-messages/:id/read/`** |

管理视图可能 **分页**、`el-table` 展示 **已读标记**。**MainLayout** 若在 **`/system/messages`** listen **badge** unread count，会与 **watch `$route`** 同步（见 **`MainLayout`** 片段）。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。
