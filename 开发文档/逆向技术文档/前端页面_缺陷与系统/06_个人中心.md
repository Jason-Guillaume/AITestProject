# 个人中心

## 1. 路由

- **路径**：`/user/center`
- **组件**：`frontend/src/views/UserCenter.vue`
- **布局**：MainLayout，`meta.public: false`，需 **`token`**。

侧边栏通常为 **用户信息/个人中心** 入口。

## 2. API（`frontend/src/api/user.js`）

| API | HTTP |
|-----|------|
| `getCurrentUserApi` | `GET /user/me/`（同步 **`is_system_admin`** 常用）|
| `getUserProfileApi` | `GET /user/me/profile/` |
| `patchUserProfileApi` | `PATCH /user/me/profile/`（可 **FormData 带头像**）|
| `getSensitiveChangeStatusApi` | `GET /user/me/change-requests-status/` |
| `submitSensitiveChangeApi` | `POST /user/me/sensitive-change/` |
| `getAdminChangeRequestsApi`、`approve`、`reject` | 仅在 **管理员审敏感变更** 场景 |

`UserCenter.vue` 当前实际导入：`getUserProfileApi`、`patchUserProfileApi`、`getSensitiveChangeStatusApi`、`submitSensitiveChangeApi`。表中其它方法（改密、管理员审批）在同文件 `user.js` 中保留给其它页面复用。

## 3. Profile 表单

分区：**基本信息**（昵称、手机、头像上传）、**安全**（改密）、**敏感操作**（改用户名触发审批占位）。  

若 `PATCH /user/me/profile/` 携带头像文件，应使用 **`FormData`** 并交给 axios 设定 **`multipart/form-data`**（避免手写错误 boundary）。

## 4. SYSTEM_ADMIN_ONLY

个人中心路由 **不属于** **`SYSTEM_ADMIN_ONLY_PREFIXES`**，任何登录用户可达。  

`GET /user/me/`（若由布局拉取）返回的 **`is_system_admin`** 会驱动 MainLayout **系统管理菜单是否展示**。

## 5. Pinia Store

可选用 **组件本地状态** hydrate profile；未发现强制 store id。

## 6. Query 参数

无必选 query。

## 7. Axios 封装

**`request` 封装**（业务 axios）附带 Authorization；登录独立 **`auth.js`**。

## 8. Avatar 预览

头像可先 **`URL.createObjectURL(file)`** 本地预览，`patchUserProfileApi` 成功后再 **`revokeObjectURL`** 并刷新 profile。

## 9. UX

Cyber Glass 卡片 + `el-alert` 敏感变更审批提示样式。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。
