# 登录与注册

## 1. 路由

| path | name | `meta.public` |
|------|------|---------------|
| `/login` | `Login` | **`true`**（匿名可访问） |
| `/register` | `Register` | **`true`** |

组件：`frontend/src/views/Login.vue`、`Register.vue`。根路由 `/` redirect `/dashboard`。

## 2. 全局守卫交互

**`frontend/src/router/index.js`**：

1. **`/pipelines`** 特例：先于 token 校验 **放行**。  
2. **`!meta.public && !token`** → **`next({ path: '/login', query: { redirect: to.fullPath } })`**。  
3. **`/login` 且已有 token** → **`/dashboard`**，避免重复登录表单。

## 3. Login API（`frontend/src/api/auth.js`）

- **`loginApi`**：独立 `axios.post('/api/user/login/', payload)`——**不使用** `@/utils/request`，避免与业务拦截器耦合。载荷常见为 `username`、`password`。成功后在视图内写入 `localStorage.token`（键名以实现为准），并跳转 `route.query.redirect` 或默认 `/dashboard`。

## 4. Register API（`auth.js`）

- **`captchaApi`**：**`GET /api/user/captcha/`** → `{ uuid, image }`。  
- **`registerApi`**：**`POST /api/user/register/`** → body 含 **`captcha_code` + captcha_uuid**。

注册页：**图形验证码刷新**、`Base64`/URL 绑定 `<img>`。

## 5. Login 表单要点

错误信息来自 **`loginApi` 的 Axios 报错体**（`response.data`）。成功后由 MainLayout/App 钩子请求 **`GET /user/me/`** 并写入 **`localStorage.is_system_admin`**（常见为 **`'1'` / `''`**）。

## 6. SYSTEM_ADMIN_ONLY 影响

当 **`localStorage.is_system_admin !== '1'`** 且目标是 **`SYSTEM_ADMIN_ONLY_PREFIXES`**（含 org/role/user/messages/**ai-usage**/ai-quota/audit）时，守卫改写为 **`/system/message`**。  

登录成功路径选择需考虑该策略。

## 7. Pinia Store

登录表单 **一般不用 store**；用户态可能 **later** hydrate 到 **`appContext`/profile**。

## 8. Layout

Login/Register **不在 MainLayout**（顶层独立 component），因而 **无双侧栏**。

## 9. HTTPS / Token 存储

当前 **`localStorage` plain text** token；SSE/下载工具复用。**XSS** 风险评估属安全基线。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。
