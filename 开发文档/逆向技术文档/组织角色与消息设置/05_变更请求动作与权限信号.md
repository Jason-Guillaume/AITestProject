# 变更请求动作与权限信号

## 文档目的

将本章涉及的 **横切机制** 收束到一篇：**敏感变更审批函数**、**权限类**、**组织↔项目成员同步信号**、**敏感变更提交后的站内信 `on_commit`**。便于与 **`用户与认证`** 分册交叉引用而不重复粘贴大段代码。

## 1. 审批与站内信：`user/change_request_actions.py`

| 符号 | 作用 |
|------|------|
| **`admin_recipients_queryset`** | `Q(is_staff=True) \| Q(is_superuser=True) \| Q(is_system_admin=True)` 去重，作为 **待审批站内信** 接收人集合 |
| **`create_inbox_messages_for_change_request`** | 对每位管理员 **`bulk_create(SystemMessage)`**，`related_request` 指向申请 |
| **`mark_messages_for_request`** | 审批结束后把关联消息 **`is_read=True`** |
| **`approve_change_request` / `reject_change_request`** | 状态机 + 写回用户名字段或密码哈希 + 标记消息已读；`ValueError` 表示业务失败 |

详见 **`用户与认证/07`**、**`09`**、**`10`** 各篇。

## 3. 组织信号：`user/signals.py`

### 3.1 组织成员 → 项目成员

- **`@receiver(m2m_changed, sender=Organization.members.through)`**
- **`@receiver(m2m_changed, sender=Organization.projects.through)`**

当 `action in ("post_add", "post_remove", "post_clear")` 时调用 **`_sync_org_project_members(org)`**：

- 取当前组织 **`members`**；若为空则 **直接 return**。
- 对 **`org.projects.all()`** 中每个项目 **`proj.members.add(*members)`**。

**边界**（与 **`01`** 一致）：**只增不删**；组织清空成员 **不会** 从项目移除用户。

### 3.2 与 `User.projects` 的关系

`TestProject.members` 的 **`related_name="projects"`** 在用户侧字段名为 **`user.projects`**（`UserViewSet.get_queryset` 用其做成员可见范围）。信号维护的是 **项目成员表**，与 **用户对象上的 `projects` M2M** 为同一关系反向，需避免文档中混用「组织 projects」与「用户 projects」两个概念。
