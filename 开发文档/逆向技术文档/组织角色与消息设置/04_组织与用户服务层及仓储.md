# 组织与用户服务层及仓储

## 文档目的

说明 **`user`** 应用内 **Repository + Service** 的分层职责、主要方法及 **与 HTTP ViewSet 的关系**。当前仓库中 **`OrganizationViewSet` / `UserViewSet` 并未直接注入这些 Service**，REST 仍以 **序列化器 + ORM** 为主路径；Service/Repository **主要供单元测试与后续扩展复用**。

---

## 1. 目录结构

| 路径 | 职责 |
|------|------|
| `user/repositories/organization_repository.py` | `Organization` 表查询、名称唯一性、用户所属组织列表 |
| `user/repositories/user_repository.py` | `User` 表查询、用户名/邮箱唯一、活跃用户、系统管理员等 |
| `user/services/organization_service.py` | 组织创建/更新/成员增删的领域规则 + 事务 |
| `user/services/user_service.py` | 用户创建/更新/激活/停用及密码哈希 |
| `user/services/__init__.py` | 导出 `UserService`、`OrganizationService` |

**引用情况**（`grep` 业务代码）：除 **`user/tests/`** 外，**未发现** `views.py` 等生产视图直接 `import OrganizationService` / `UserService`。若要将校验统一到 Service，需在 ViewSet 中显式注入并处理 **`common.exceptions`** 到 HTTP 响应的映射。

---

## 2. `OrganizationRepository`（节选行为）

- 继承 **`BaseRepository[Organization]`** — 自带 `get_by_id` 缓存、`delete` 软删等（见工程根 **`05_基础仓储与服务基类.md`**）。
- **`get_all(filters)`**：基础 `is_deleted=False` + `prefetch_related('members')`；若 `filters` 含 **`search`**，对 `org_name`、`description` **`icontains`**；排序 **`-create_time`**（与 ViewSet 中 `order_by("-create_time")` 的列表体验一致）。
- **`get_by_name` / `exists_by_name`**：名称唯一校验，支持 **`exclude_id`** 更新场景。
- **`get_user_organizations(user_id)`**：`members__id=user_id` 过滤未删除组织。
- **`count_organizations`**：未删除组织计数。

---

## 3. `OrganizationService`

- 继承 **`BaseService[Organization]`**，构造时 **`OrganizationRepository()`**。
- **`create_organization`**：校验 `org_name` 非空；**`exists_by_name`** 防重；`repository.create(org_data, creator=creator)`（creator 走仓储 `create` 的 kwargs）。
- **`update_organization`**：存在性；若改名则再次查重；**`repository.update`**。
- **`add_member` / `remove_member`**：`get_by_id` 取组织；用户必须 **`is_deleted=False`**（`add` 分支）；`org.members.add/remove`；**不** 在此处触发项目同步 — 仍依赖 **`m2m_changed` 信号**（见 **`01`**）。

抛错类型：**`ValidationException`**、**`DuplicateException`**、**`ResourceNotFoundException`**（来自 `common.exceptions`，与 DRF `ValidationError` 不同，视图若调用需自行转换 HTTP 状态码）。

---

## 4. `UserRepository`

- **`get_all(filters)`**：支持 `is_active`、`is_system_admin`、**`search`**（用户名/姓名/邮箱/手机 `icontains`），默认 **`-create_time`**。
- **`get_by_username` / `get_by_email`**、**`exists_by_username` / `exists_by_email`**（含 `exclude_id`）。
- **`get_active_users` / `get_system_admins`**：封装常用过滤。

---

## 5. `UserService`

- **`create_user`**：校验用户名/密码/真实姓名；用户名/邮箱重复 → **`DuplicateException`**；密码 **`make_password`** 后 **`repository.create`**。
- **`update_user`**：存在性；用户名/邮箱变更查重；若含 **`password`** 字段则再次 **`make_password`**。
- **`activate_user` / `deactivate_user`**：更新 **`is_active`**。
- 其它方法多委托 **`repository`**。

---

## 6. 与 ViewSet 的并行关系（重要）

| 路径 | 当前实现倾向 |
|------|----------------|
| HTTP CRUD 组织 | **`OrganizationViewSet`** + **`OrganizationSerializer`** + 信号 |
| HTTP CRUD 用户 | **`UserViewSet`** + **`UserSerializer`** |
| 领域规则复用 / 单测 | **`OrganizationService` / `UserService`** |

两套逻辑 **可能重复**（例如名称唯一）：修改业务规则时需 **同时检查** ViewSet 与 Service，或逐步收敛到 Service。
