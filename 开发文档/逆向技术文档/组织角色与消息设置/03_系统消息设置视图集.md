# 系统消息设置视图集

## 文档目的

区分 **`SystemMessageSetting`**（**渠道偏好配置**，每人至多一条）与 **`SystemMessage`**（**站内信收件箱**）。说明 **`SystemMessageSettingViewSet`** 的查询过滤、**`perform_create` 的 `update_or_create`** 语义，以及 **`?latest=1`** 的切片行为对 DRF 的影响。

---

## 1. 数据模型

**表**：`sys_message_setting`（`user/models.py`）

| 字段 | 说明 |
|------|------|
| `recipient` | FK → `User`，**可空**（历史/兼容）；业务上由视图绑定当前用户 |
| `in_app_enabled` | 站内消息，默认 `True` |
| `email_enabled` | 邮件，默认 `False` |
| `sms_enabled` | 短信，默认 `False` |
| `digest_enabled` | 日报汇总，默认 `False` |
| `digest_time` | `HH:mm`，可空 |
| 继承 `BaseModel` | 常规审计与软删字段 |

**约束**：`UniqueConstraint(fields=["recipient"], name="unique_user_message_setting")` — 每个 **`recipient_id`** 最多一行（`recipient` 为 NULL 时 PostgreSQL/MySQL 对唯一约束行为可能不同，生产应保证绑定用户写入）。

---

## 2. HTTP：`SystemMessageSettingViewSet`

**前缀**：`/api/user/message-settings/`（注意 **复数** 与连字符）

- **父类**：`BaseModelViewSet`（列表带软删过滤、管理员数据范围等默认逻辑）。
- **分页**：`SystemPagination`（默认 `page_size=10`，最大 100）。

### 2.1 `get_queryset`

- 未登录 → **`qs.none()`**。
- 否则 **`filter(recipient=request.user)`** — 普通用户只能读写 **自己** 的配置。
- 若 **`?latest=1`**：`return qs.order_by("-create_time")[:1]`  
  - 返回的是 **切片后的列表**，不是 `QuerySet`；DRF 在部分版本/场景下对列表 + 分页可能表现异常，**建议前端优先用列表接口不带 `latest` 或自行取首条**。

### 2.2 `perform_create`

- 未登录 → **`ValidationError({"detail": "未登录"})`**。
- 已登录：  
  **`SystemMessageSetting.objects.update_or_create(recipient=user, defaults=serializer.validated_data)`**  
  然后将 **`serializer.instance = instance`**，保证响应与后续钩子拿到正确对象。

即：**POST 创建** 在业务上等价于 **「有则更新、无则插入」**，与唯一约束配合实现 **幂等保存偏好**。

## 3. 与站内信 `SystemMessage` 的区别

| 实体 | 用途 |
|------|------|
| **SystemMessageSetting** | 用户是否接收某类渠道、日报时间等 **配置** |
| **SystemMessage** | 具体 **收件箱消息**（标题、正文、已读、关联审批单等），见 **`用户与认证/08_系统消息列表与已读标记.md`** |

二者 **无** 外键关联；产品文案上需避免都叫「消息」导致混淆。

---

## 4. 前后端交互

- 前端 **消息设置** 页（如 `/system/message`）。
- **GET**：拉当前用户配置列表（通常 0～1 条）。
- **POST/PATCH**：提交开关与时间；若走 `POST` 且已存在行，依赖 **`update_or_create`** 合并。

---

## 5. 显式与静默

| 显式 | 静默 |
|------|------|
| 各布尔开关、`digest_time` | `recipient` 由后端绑定，前端不应伪造他人 `recipient_id`（即使传入也会被过滤或覆盖，以序列化器为准） |
