# 角色视图集

## 文档目的

说明 **`RoleViewSet`** 如何将 Django 内置 **`Group`** 暴露为 REST「角色」资源；与 **`User`** 上 **`groups`** M2M（`related_name="custom_user_set"`）的关系；以及 **未** 使用 `BaseModelViewSet` 带来的 **无软删/无统一审计** 影响。

---

## 1. HTTP 路由

**前缀**：`/api/user/roles/`（`router.register(r"roles", RoleViewSet)`）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/user/roles/` | 列表 |
| POST | `/api/user/roles/` | 创建组 |
| GET | `/api/user/roles/<pk>/` | 详情 |
| PUT/PATCH | `/api/user/roles/<pk>/` | 更新（常用 PATCH） |
| DELETE | `/api/user/roles/<pk>/` | **物理删除** `auth_group` 行 |

---

## 2. 视图与序列化

**类**：`RoleViewSet`（`user/views.py`）

- **父类**：`viewsets.ModelViewSet` — **不是** `BaseModelViewSet`。
- **`queryset`**：`Group.objects.all()`（含 Django 默认组，若有）。
- **`serializer_class`**：`RoleSerializer`
- **`get_permissions`**：追加 **`IsSystemAdmin()`**。

**`RoleSerializer`**（`user/serialize.py`）：

```python
class Meta:
    model = Group
    fields = ["id", "name"]
```

即 API 仅暴露 **id** 与 **name**；与 **`Permission`** 的 M2M 不在此序列化器维护（需在 Admin 或其它接口扩展）。

---

## 3. 数据库

**表**：Django 内置 **`auth_group`**

- `id`：主键  
- `name`：**唯一**（`CharField` max_length 150，以 Django 版本为准）

用户与组关系表 **`auth_user_groups`**（用户模型使用自定义 `related_name`，但底层仍关联到 `Group`）。

---

## 4. 与用户管理的配合

**`UserViewSet`** / **`UserSerializer`** 在创建用户时可 `pop` **`groups`** 并 **`user.groups.set(groups)`**（见用户分册 **`12_用户资源视图集增删改查.md`**）。  
角色名变更或删除组，可能影响已绑定用户的权限展示，需在管理端约束「禁止删除仍被引用的组」或前端校验。

---

## 5. 显式与静默

| 显式 | 静默 |
|------|------|
| 接口 JSON 中的 `id`、`name` | 删除组时 **不** 自动清理用户-组关系（Django 默认行为以版本为准，多对多可能 CASCADE 或阻止删除，需实测） |
| 403 非系统管理员 | **无** `record_audit_event` 钩子（相对 `BaseModelViewSet`） |

---
