# 环境与变量数据模型

## 文档目的与读者

汇总 **`test_environment`**、**`environment_variable`**、**`environment_health_check`** 三表字段、约束与级联行为，便于与《01》《02》《03》及执行模块对照。

---

## 1. ER 关系（逻辑）

```
test_environment (1) ──< (N) environment_variable
        ↑
        │ 无外键；仅业务上被「计划/报告 environment 字符串」「报告执行 environment_id」引用

environment_health_check   （独立表；由 HealthChecker 等写入，与环境表无 FK）
```

---

## 2. 表：`test_environment`

| 字段 | 类型要点 | 说明 |
|------|-----------|------|
| `name` | `varchar(100)` **`unique=True`** | **全局唯一**，非按项目隔离；命名需避免与兄弟团队冲突 |
| `env_type` | choices：`dev/test/staging/prod/custom` | 默认 `test` |
| `base_url` | `varchar(2048)`，可空 | 供接口探测、执行拼接等 |
| `auth_config` | JSON，默认 `{}` | 鉴权模板（Bearer、API Key 等），**执行引擎消费**，本表不校验 schema |
| `health_check_path` | `varchar(512)` | 相对路径；空则 validate 时对 **base_url 根** GET |
| `db_config` | JSON，默认 `{}` | 典型键 `host, port, user, password, db_name`；**validate 仅消费 host+port** |
| `description` | text | 备注 |
| BaseModel | `creator_id` 等 | 软删 `is_deleted` 等 |

**Meta**：`db_table = "test_environment"`，`ordering = ("-create_time",)`。

---

## 3. 表：`environment_variable`

| 字段 | 类型要点 | 说明 |
|------|-----------|------|
| `environment_id` | FK → `test_environment`，**`on_delete=CASCADE`** | 删环境则变量一并删除 |
| `key` | `varchar(128)` | 与 `environment_id` **联合唯一** |
| `value` | text | 非敏感存明文；敏感存 **`enc::`** 前缀密文 |
| `is_secret` | bool | 控制加密与 `display_value` 掩码 |
| `description` | `varchar(255)` | 可选说明 |

**Meta**：`db_table = "environment_variable"`，`ordering = ("id",)`，约束 **`uniq_env_var_key`**。

**方法（节选）**：

- `encrypt_text` / `decrypt_text` / `get_decrypted_value` — 见《02》。
- **`save` 重写**：敏感值自动加密后再 `super().save()`。

---

## 4. 表：`environment_health_check`

| 字段 | 说明 |
|------|------|
| `check_type` | `db` / `api` / `redis` |
| `status` | `healthy` / `unhealthy` |
| `response_time_ms` | 毫秒耗时 |
| `error_log` | 文本错误 |
| `target` | 探测目标摘要 |
| `dimension` | JSON 扩展维度（如 `execution_task_id`） |

**Meta**：`db_table = "environment_health_check"`，组合索引 **`(check_type, status, -create_time)`**。

**写入方**：当前以 **`execution.services.health_checker.HealthChecker`** 为主；**`validate_environment` 不写此表**。

## 5. 与业务其他表的「弱关联」

| 使用方 | 关联方式 |
|--------|-----------|
| `TestPlan.environment` / `TestReport.environment` | **字符串**，与 `TestEnvironment` 无强制 FK |
| `run_test_report_execute(..., environment_id=)` | 整数 ID 指向 **`TestEnvironment`**（若存在），用于执行期配置 |

---

## 6. 迁移与密钥轮换

- 环境变量密文依赖项目 **`SECRET_KEY`**（或模型文件内密码学材料，以 `testcase/models.py` 为准）。
- **轮换密钥**后：旧 `enc::` 数据若未重加密，运行态 **`get_decrypted_value`** 会失败。
