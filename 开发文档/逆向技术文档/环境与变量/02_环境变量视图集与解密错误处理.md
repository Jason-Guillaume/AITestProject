# 环境变量视图集与解密错误处理

## 文档目的与读者

说明 **`EnvironmentVariable`** 的 **入库加密**、**列表脱敏展示**、**运行态解密失败** 三类行为差异，以及 ViewSet 路径与数据范围。适合安全评审、前端展示改造、以及执行失败报「解密」类错误的定位。

---

## 1. API 与 ViewSet

| 项 | 值 |
|----|-----|
| ViewSet | `EnvironmentVariableViewSet`（`testcase/views.py`） |
| 路由 | **仅** `GET/POST/PATCH/DELETE /api/testcase/environment-variables/`（`testcase/urls.py` 中 `router.register(r"environment-variables", ...)`） |
| queryset | `EnvironmentVariable.objects.select_related("environment").all()` |
| Serializer | `EnvironmentVariableSerializer` |

**未** 实现自定义 `get_queryset`：过滤条件依赖 **DRF 默认**（无额外 query 参数时返回基类 `get_queryset` 结果）。  
数据范围仍走 **`BaseModelViewSet`**：`EnvironmentVariable` 无 `project` FK，非管理员同样 **仅 `creator=user`**（与《01》环境本体一致）。**不**按「能否访问该 environment」二次放宽——若产品上要「环境成员共享变量」，需后续改 `_apply_member_scope` 或增加 `environment` 链路策略。

## 2. 序列化：`display_value` 与原始 `value`

```82:100:testcase/serialize.py
class EnvironmentVariableSerializer(BaseModelSerializers):
    creator_name = serializers.CharField(source="creator.real_name", read_only=True)
    display_value = serializers.SerializerMethodField(read_only=True)

    class Meta:
        model = EnvironmentVariable
        fields = "__all__"
        read_only_fields = ("creator", "updater", "create_time", "update_time")

    def get_display_value(self, obj: EnvironmentVariable):
        """敏感变量默认返回脱敏值，避免前端误展示明文。"""
        if obj.is_secret:
            return "******"
        return obj.value

    def to_representation(self, instance):
        data = super().to_representation(instance)
        data["display_value"] = self.get_display_value(instance)
        return data
```

| 字段 | `is_secret=False` | `is_secret=True` |
|------|-------------------|------------------|
| `display_value` | 等于 `value`（明文） | 固定 **`******`** |
| `value`（`fields="__all__"`） | 明文 | **密文** `enc::...`（FERNET 类 token） |

**安全提示**：当前实现 **仍把 `value` 序列化进 JSON**。前端应 **列表/详情优先渲染 `display_value`**；敏感场景不要在表格中直接绑 `value` 列。若合规要求「响应体不出现密文」，需在 Serializer 层剔除或改写 `to_representation`。

## 3. 模型层：加密、解密与异常类型

### 3.1 前缀与 `save` 钩子

- 密文统一前缀 **`ENV_VALUE_PREFIX = "enc::"`**。
- **`save`**：当 `is_secret=True` 且 `value` 非 `enc::` 开头时，在写库前调用 **`encrypt_text`**；否则原样保存（支持已加密内容重复保存）。

### 3.2 `decrypt_text` 与 `get_decrypted_value`

| 方法 | `errors="ignore"`（默认） | `errors="raise"` |
|------|---------------------------|------------------|
| `decrypt_text` | 非法 token 打日志后返回 **`""`** | 抛 **`EnvironmentVariableDecryptionError`** |
| `get_decrypted_value` | 非敏感直接返回 `value` | 敏感走 `decrypt_text(..., errors="raise")`，失败即抛出 |

执行引擎在注入变量时应调用 **`get_decrypted_value()`**（或等价逻辑）；若服务器 **`SECRET_KEY`/加密盐** 轮换且旧密文未迁移，会触发 **`EnvironmentVariableDecryptionError`**，表现为用例执行前/执行中失败。

### 3.3 数据库约束

- **`UniqueConstraint(environment, key)`**：同一环境下变量名唯一。
- 删除环境：**`CASCADE`** 删除子变量（见《04》）。

## 4. 与「环境校验」接口的区别

| 能力 | `POST .../environments/:id/validate/` | 本 ViewSet CRUD |
|------|----------------------------------------|-----------------|
| 目的 | 探测 HTTP/DB 连通性 | 维护 KV |
| 是否读写 `environment_variable` 表 | 否 | 是 |
| 是否解密变量 | 否 | 读详情时会带出 `value`（见 §2） |

## 5. 前后端交互建议

| 场景 | 建议 |
|------|------|
| 列表展示 | 列绑定 **`display_value`** |
| 编辑敏感变量 | PATCH 传新明文即可；`save` 会再次加密（勿把 `enc::` 当用户输入重复加密除非有意） |
| 按环境筛选 | 若需服务端筛选，应在 ViewSet 增加 `get_queryset` 对 `?environment=` 解析（当前代码未内置，前端可能仅客户端过滤） |

（若仓库后续为 `EnvironmentVariableViewSet` 增加了 `environment` query 过滤，以当时源码为准。）

## 6. 异常与日志

| 情况 | 表现 |
|------|------|
| 同名 key 重复 | 创建/PATCH 违反唯一约束 → 400 |
| 解密失败（运行态） | `EnvironmentVariableDecryptionError`；需检查密钥与密文是否匹配 |
| `decrypt_text` ignore 模式 | 返回空串，业务可能把「空密码」当合法值继续请求，需结合执行日志排查 |
