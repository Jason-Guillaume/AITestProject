# 测试环境视图集双路由挂载说明

## 文档目的与读者

说明 **`TestEnvironmentViewSet`** 为何挂在 **两条 URL 前缀** 下、与 **`BaseModelViewSet`** 数据范围如何叠加、以及 **`validate`** 动作在两条树上的等价性。适合排查「前端调了 `/api/environments/` 但 404」「列表只有本人数据」等问题。

---

## 1. 路由总览

| 前缀 | Django `include` | Router 注册 | 典型 list URL |
|------|------------------|-------------|----------------|
| `/api/testcase/` | `AITestProduct/urls.py` → `testcase.urls` | `router.register(r"environments", TestEnvironmentViewSet)` | `GET /api/testcase/environments/` |
| `/api/environments/` | `AITestProduct/urls.py` → `testcase.environment_urls` | `router.register(r"", TestEnvironmentViewSet, basename="environment")` | `GET /api/environments/` |

两条树指向 **同一 ViewSet 类**，故 **list / create / retrieve / update / destroy** 及 **`POST .../validate/`** 行为一致，仅路径前缀不同。短路径便于环境管理页独立配置 `axios` base（例如 `frontend` 中直接写 `/environments/`）。

**根路由证据**（节选）：

```14:16:AITestProduct/urls.py
    path("api/testcase/", include("testcase.urls")),
    path("api/environments/", include("testcase.environment_urls")),
    path("api/execution/", include("execution.urls")),
```

**专用子路由**（`basename` 避免与主 router 匿名冲突）：

```1:8:testcase/environment_urls.py
from rest_framework.routers import DefaultRouter

from testcase.views import TestEnvironmentViewSet

router = DefaultRouter()
router.register(r"", TestEnvironmentViewSet, basename="environment")

urlpatterns = router.urls
```

## 2. ViewSet 与序列化

- **类**：`testcase.views.TestEnvironmentViewSet`，继承 **`BaseModelViewSet`**。
- **queryset**：`TestEnvironment.objects.all()`（是否 `select_related` 未做，列表量通常不大）。
- **serializer**：`TestEnvironmentSerializer`（`testcase/serialize.py`），`fields = "__all__"`，`read_only_fields` 含 `creator/updater/create_time/update_time`，并附带 **`creator_name`**（`creator.real_name`）。

自定义动作仅 **`validate_environment`**（`url_path="validate"`，**detail=True**），见《03_环境健康检查记录.md》。

---

## 3. 数据范围（为何「非本人环境」列表里看不到）

`TestEnvironment` **无** `project` / `module` 等 FK，`_apply_member_scope` 在 `common/views.py` 中会一路不匹配关系策略，最终兜底为：

```96:96:common/views.py
        return qs.filter(creator=user)
```

因此：**非超级用户 / 非 staff / 非 `is_system_admin` 的普通用户**，在 **`enable_data_scope=True`（默认）** 时，列表与详情 **仅能访问自己创建的环境**；**不能**通过「项目成员」关系看到别人创建的全局同名环境。

| 角色 | 列表语义 |
|------|-----------|
| 管理员（上表三种之一） | `is_deleted=False` 全量 |
| 普通已登录用户 | `creator_id = self.request.user.id` |
| 未认证 | 基类不套成员过滤，但仍受 **DRF 权限类** 约束（若接口要求登录则 401/403） |

> 与 **`TestPlan.environment` / `TestReport.environment`（字符串快照）** 无 ORM 级外键关联；计划里填的环境名可与 `TestEnvironment.name` 人工对齐，系统不做一致性校验。

## 4. 标准 REST 行为摘要

| 操作 | 方法 | 双路由等价示例 |
|------|------|----------------|
| 列表 | GET | `/api/environments/` ≡ `/api/testcase/environments/` |
| 新建 | POST | Body 与 Serializer 字段一致 |
| 详情 | GET | `.../:id/` |
| 部分更新 | PATCH | 常用字段：`name`、`env_type`、`base_url`、`health_check_path`、`auth_config`、`db_config`、`description` |
| 删除 | DELETE | **软删**：`BaseModelViewSet.perform_destroy` 置 `is_deleted=True`（非物理删） |

**`name` 全局唯一**（`TestEnvironment.name` `unique=True`）：跨用户、跨项目也不能重名；重复创建会触发 DB 或校验层错误（通常 400）。

## 5. 与环境变量的挂载差异（避免混路径）

**环境变量** `EnvironmentVariableViewSet` **仅** 注册在 `testcase/urls.py`：

- `GET /api/testcase/environment-variables/`
- **没有** `/api/environment-variables/` 短路径

若前端把变量请求写到 `/api/environments/...` 会错。详见《02_环境变量视图集与解密错误处理.md》。

---

## 6. 与执行/计划模块的衔接

| 概念 | 说明 |
|------|------|
| `validate` | 对 **`TestEnvironment`** 上配置的 `base_url` + `health_check_path` 做 **GET** 探测；对 `db_config` 做 **TCP** 探测；**不落库** `EnvironmentHealthCheck`（与执行引擎那条链路不同） |
| `execution` 报告执行 | `batch-execute` 可传 **`environment_id`**，由 Celery 任务拉取对应 `TestEnvironment` 做变量注入等（见执行模块文档） |
| `HealthChecker` | 在 **`APIExecutor.execute`** 前写 **`EnvironmentHealthCheck`** 表，与本文 `validate` 是 **两套机制**（见《03》） |

## 7. 排障速查

| 现象 | 可能原因 |
|------|-----------|
| 短路径 404 | 根 `urls.py` 未 `include("testcase.environment_urls")` 或前缀写错 |
| 列表为空 | 当前用户非创建者且非管理员；或数据全被软删 |
| 创建报唯一约束 | `name` 与他人冲突 |
| validate 总失败 | `base_url` 为空、网络不通、TLS 自签证书、`health_path` 拼错（见 `_environment_http_probe_url`） |

**HTTP 拼接规则**（`testcase/views.py`）：`base` 无尾斜杠时先加 `/` 再 `urljoin`，相对路径会正确拼在 host 下。
