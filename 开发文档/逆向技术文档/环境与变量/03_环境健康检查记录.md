# 环境健康检查记录

## 文档目的与读者

厘清两类「健康」相关能力：**（A）** 测试环境管理里的 **即时校验接口**（不落 `environment_health_check` 表）；**（B）** 接口执行任务执行前 **`HealthChecker`** 写库的 **DB/API/Redis** 检查。适合区分「点了环境页检测」与「跑任务时多了一堆 health 记录」。

---

## 1. 数据模型 `EnvironmentHealthCheck`

表名 **`environment_health_check`**（`testcase/models.py`）。

| 字段 | 说明 |
|------|------|
| `check_type` | `db` / `api` / `redis` |
| `status` | `healthy` / `unhealthy` |
| `response_time_ms` | 非负整数，代码里 `max(int(...), 0)` |
| `error_log` | 失败原因，**截断至 4000 字符**（`_record` 内切片） |
| `target` | 如 `host:port`、完整 API URL、`default-db` |
| `dimension` | 任意 JSON；执行器写入 **`execution_task_id`、`task_name`** 等 |
| BaseModel | `creator` 等字段存在，但 **`HealthChecker._record`** 使用 `objects.create` **未显式填 creator**（依项目 `BaseModel` 默认值而定） |

索引：`(check_type, status, -create_time)`，便于按类型与状态做报表查询。

**重要**：当前代码库中 **不存在** 以 `EnvironmentHealthCheck` 为 queryset 的 **List API**（无专门 ViewSet）；查数据请走 **Django Admin / 报表任务 / ORM**。

## 2. （A）即时校验：`POST .../validate/`

- **路径**（两前缀等价）：  
  `POST /api/testcase/environments/:id/validate/`  
  `POST /api/environments/:id/validate/`
- **实现**：`TestEnvironmentViewSet.validate_environment`（`testcase/views.py`）。

### 2.1 请求体

| 字段 | 说明 |
|------|------|
| `timeout` | 可选；默认 **5**；非法值回退 5； clamp 到 **[1, 30]** 秒 |
| `health_path` | 可选；若提供则 **仅本次** 覆盖模型上的 `health_check_path` |

### 2.2 HTTP 探测

- 使用 **`requests.get(full_url, timeout=timeout)`**（非 HEAD）。
- `full_url` = `_environment_http_probe_url(base_url, probe_path)`：`urllib.parse.urljoin` 规则，根 URL 无 `/` 时会补 `/` 再拼相对路径。
- **`base_url` 为空**：不发起请求，返回 `http.ok=false`，错误文案 **「未配置 Base URL，无法执行 HTTP 探测」**，总 **`ok=false`**。

### 2.3 DB 探测

- 读取 **`env_obj.db_config`**（须为 `dict`）。
- 仅当 **`host` 与 `port` 均存在** 时：`socket.create_connection((host, port), timeout=timeout)`。
- 若 `db_config` 非空但缺 host/port：返回 `db.ok=false`，错误 **「db_config 缺少 host/port」**，总 **`ok=false`**。
- **不验证** MySQL 协议/账号密码，仅 **TCP 可达性**。

### 2.4 响应形状（逻辑）

```json
{
  "ok": true,
  "http": { "ok": true, "status_code": 200, "url": "..." },
  "db": { "ok": true, "host": "...", "port": 3306 }
}
```

失败分支在 `http` / `db` 子对象中带 `error` 字符串。**不写入** `EnvironmentHealthCheck`。

## 3. （B）任务前检查：`HealthChecker` + `environment_health_check` 表

**位置**：`execution/services/health_checker.py`。  
**调用链**：`execution/engine.py` → `APIExecutor.execute` 在发真实业务请求前：

```27:35:execution/engine.py
            dimension = {
                "execution_task_id": task.id,
                "task_name": task.task_name,
            }
            checker = HealthChecker()
            summary = checker.check_before_task(api_url=task.url, dimension=dimension)
            if not summary.get("ok"):
                checker.send_alert_email(task_name=task.task_name, summary=summary)
                raise RuntimeError(f"ENV_UNHEALTHY: {summary.get('unhealthy')}")
```

### 3.1 `check_before_task` 行为

依次执行（**每项都会尽量 `_record` 一行**，成功或失败各一行）：

| 步骤 | `check_type` | 逻辑摘要 |
|------|--------------|----------|
| DB | `db` | 若调用时 **未传 `db_config`**：`connections["default"].cursor()` 执行 `SELECT 1`；`target` 为 **`default-db`**。若传 `config`：`socket` 连 `host:port` |
| API | `api` | **`requests.head(url)`**，5s，`allow_redirects=True`；**2xx–3xx** 视为 healthy |
| Redis | `redis` | 读环境变量 **`REDIS_HOST`/`REDIS_PORT`**（默认 `127.0.0.1:6379`），`socket` 2s 连接探测 |

与（A）的差异一览：

| 维度 | validate（A） | HealthChecker（B） |
|------|---------------|---------------------|
| HTTP 动词 | GET | HEAD |
| 是否写表 | 否 | 是 |
| Redis | 不测 | 测 |
| DB | 用户配置的 host:port TCP | 默认连 Django `default` 或自定义 host:port |
| 失败后续 | 仅返回 JSON | 发邮件（若配置了管理员邮箱）+ 抛 **`ENV_UNHEALTHY`** 中断执行 |

### 3.2 邮件告警

`send_alert_email`：`USER_CENTER_ADMIN_EMAIL` 逗号分隔优先，否则 **`settings.ADMINS`**；无收件人则静默返回。

## 4. 运维与排障

| 现象 | 建议 |
|------|------|
| 表里突然大量 `api` 记录 | 正常：每次 **`APIExecutor.execute`** 都会跑 **`check_before_task`** |
| `api` 一直 unhealthy | 目标 URL 对 **HEAD** 返回 4xx/5xx 或网络拒绝；与（A）的 **GET** 探测结果可能不一致 |
| `redis` 失败 | Worker 环境 **`REDIS_HOST/PORT`** 与真实 broker 不一致 |
| 想做「环境页历史曲线」 | 需自建查询或后续加只读 API；当前无现成 REST |
