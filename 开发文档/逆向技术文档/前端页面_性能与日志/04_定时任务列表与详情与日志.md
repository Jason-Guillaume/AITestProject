# 前端页面：定时任务（列表 / 详情 / 调度日志）

## 1. 路由一览（`frontend/src/router/index.js`）

| 路径 | 名称 | 组件 |
|------|------|------|
| `/performance/scheduled-tasks` | `ScheduledTaskList` | `ScheduledTaskList.vue` |
| `/performance/scheduled-task/:id?` | `ScheduledTaskDetail` | `ScheduledTaskDetail.vue`（`id` 可选：无 id 新建） |
| `/performance/scheduled-task-logs` | `ScheduledTaskLog` | `ScheduledTaskLog.vue` |

以上三条均在 **主布局之外** 定义，与 `/performance/tasks` 相同，**不挂 MainLayout**。

## 2. API 模块（`frontend/src/api/scheduledTask.js`）

**定时任务 CRUD 与动作**

- `GET/POST /execution/scheduled-tasks/`
- `GET/PATCH/DELETE /execution/scheduled-tasks/{id}/`
- `POST .../pause/`、`resume/`、`trigger/`
- 批量：`batch-delete`、`batch-update`、`batch-copy`

**调度日志**

- `GET /execution/scheduled-task-logs/`（`getScheduledTaskLogs`）
- `POST .../batch-delete/`、`batch-delete-by-filter/`

## 3. 列表页（`ScheduledTaskList.vue`）

- 关键字筛选、分页表格、多选批量删除/改状态/复制。
- 行操作：跳转 **`/performance/scheduled-task/{id}`**。
- 「新建」：**`/performance/scheduled-task`**（无 id）。

## 4. 详情 / 新建页（`ScheduledTaskDetail.vue`）

- 表单：`name`、`cron_expression`、`status`（active/paused/disabled）、`environment`（选项来自 **`getEnvironments`**）、`test_case_ids`（多选）。
- **用例选项**：`request.get("/testcase/cases/", { params: { page_size: 200, testType: "api" } })`，即仅拉 API 类用例为例，路径以 **`/api` 为前缀的相对 URL** 由 `request` 处理。
- 提交：`createScheduledTask` / `updateScheduledTask`；编辑态支持 **`pauseScheduledTask`、`resumeScheduledTask`、`triggerScheduledTask`**。
- 保存成功后示例跳 **`/performance/tasks`**（性能任务管理首页）。

## 5. 日志页（`ScheduledTaskLog.vue`）

- 按任务、状态、消息关键字筛选；支持选中批量删除、按筛选批量清理、导出当前页 JSON。
- 数据来自 **`getScheduledTaskLogs`** 等。

## 6. 环境与 WebSocket

- **环境**：详情页用 **`GET /environments/`**；执行语义上可与全局 **`useCurrentEnvironment`** 并存。
- **WebSocket**：定时任务三页均为 **HTTP 轮询/一次性请求**，无前端 WS；与 k6 看板 **`/ws/k6/`**、服务器日志 **`/ws/server-logs/`** 不同。

## 7. 侧栏入口说明

- 主布局侧栏**未**单独列出上述路径；入口多在 **性能任务管理页工具栏** 或书签直达。

## 8. 调度日志批量删除语义

- **`batchDeleteScheduledTaskLogs`**：`POST /execution/scheduled-task-logs/batch-delete/`，body 含 id 列表。
- **`batchDeleteScheduledTaskLogsByFilter`**：`POST .../batch-delete-by-filter/`，与列表筛选条件一致清空（具体字段以后端验证为准）。日志页有「按筛选清理」按钮适配。

## 9. 详情页环境字段类型

- 表单里 **`environment`** 绑定为 **环境下拉** 的 `value`（多为 id）。若后端 PATCH 期望 slug 或名称，需在 **`updateScheduledTask`** 前转换（当前以 `ScheduledTaskDetail.vue` 为准）。

## 10. Cron 与立即执行

- **`triggerScheduledTask`** 用于手动触发一次，不改变 Cron；**`pause`/`resume`** 影响调度器是否派发。

## 11. 错误处理模式

- 保存/暂停等 **`catch`** 内统一 `error?.response?.data?.msg`，与 Django 统一包裹格式一致。

## 12. 与 `useRealtimePolling` 无关

- 三页均**未**使用 `useRealtimePolling`；日志列表靠用户点击「查询」或分页触发刷新。

## 13. 测试清单建议

- 新建 → 列表可见 → 编辑 Cron → 触发 → 日志页出现 running/success → 批量删除 → 按筛选清理。
