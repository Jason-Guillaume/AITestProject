# 前端页面：服务器日志（主页面 + 终端子组件）

## 1. 路由

- **路径**：`/server-logs`，名 **`ServerLogs`**。
- **布局**：`MainLayout` 子路由；`meta.keepAlive: true`，配合 **`useSseRefreshTrigger` 的 keep-alive 断开逻辑**时需注意其它页，本页主逻辑为 HTTP + WS。
- **侧栏**：`index="/server-logs"`。

- **主页面**：`frontend/src/views/server_logs/index.vue`。
- **终端**：`frontend/src/views/server_logs/LogTerminal.vue`。

## 3. 主机与 REST API（`@/api/serverLogs`）

| 函数 | 说明 |
|------|------|

主页面：左侧主机列表（勾选、新增、编辑、删除），右侧路径输入、「连接实时日志」、检索、趋势图、审计折叠区等。

## 4. WebSocket：实时日志（`LogTerminal.vue`）

- **URL**（`wsUrl()`）：
  - `const proto = window.location.protocol === "https:" ? "wss:" : "ws:"`
  - **`${proto}//${host}/ws/server-logs/${serverId}/?token=...&path=...`**
  - `path` 为 **`encodeURIComponent`** 的远程日志路径。
- **消息**：JSON；`type === "log"` 推进行文；`status` / `error` 更新状态 `ElMessage`。
- **断开**：发送 **`{ type: "stop" }`** 再 `close`；连接按钮由父页 **`streaming`** 控制。

## 5. 环境与多租户

- 主机卡片可展示 **`organization_name`**；若后端需要组织筛选，有 **`listLogServerOrganizationChoices`**。

## 6. 与 `useWebSocket` 组合式

- 终端内 **直接使用 `new WebSocket`**；全局 **`useWebSocket.ts`** 仅简单封装，未附带 token/path，故实时日志未采用该封装。

## 7. 鉴权

- `token` 从 **`localStorage`** 读取；与 REST 的 **`Token`** Header 一致，供 WS 查询参数传递（网关需校验）。

## 8. 主页面：检索与趋势

- **历史检索**：`searchServerLogs` → ES 关键字；结果以纯文本 `pre` 拼接行展示（见 `index.vue`）。
- **错误趋势**：`getServerLogErrorTrend`，ECharts 折线；默认 ERROR、近 60 分钟（文案以页面为准）。
- **审计**：`listServerLogAuditEvents` 分页表格，折叠区内刷新。

## 9. `LogTerminal` 行为细节

- **`streaming` 为 false** 时不连接 WS；父组件切换主机时若仍在推流，会有 **alert** 提示是否切换连接（见主模板 `streaming && activeHost.id !== selected.id`）。
- 行渲染 **HTML 高亮**（ERROR/WARN/Exception），注意 **`escapeHtml` 防 XSS** 后再替换关键字。

## 10. `keepAlive` 影响

- 路由 **`meta.keepAlive: true`**，`MainLayout` 对 `keepAlive` 子视图包一层 `<keep-alive>`。从日志页切走再回来，组件实例保留；**`LogTerminal` 应在 `streaming` 变 false 时 `disconnectWs`**，避免后台仍占连接。

## 11. AI 分析接口超时

- `analyzeServerLog` **120s**、`analyzeServerLogWithContext` **180s**（`serverLogs.js` 内 `timeout`）；前端应显示 loading，防重复提交。

## 12. 与缺陷模块联动

- **自动化工单草稿**走 **`/server-logs/auto-tickets/`**；成功后 **`createDefectFromServerLogAutoTicketJob`** 与 **`defect.js`** 的 **`createDefectApi`** 不同路径，勿混用。

- 若 WS 频繁断开，检查反向代理 **read timeout**、后端 channel layer、以及浏览器睡眠后唤醒的重连策略（当前由用户手动「连接实时日志」）。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
