# 前端页面：k6 压测会话列表

## 1. 路由

- **路径**：`/performance/k6-sessions`，名 **`K6SessionList`**。
- **布局**：与任务管理、定时任务相同，**顶层组件**、非 `MainLayout`。

## 2. 视图

- `frontend/src/views/performance/K6SessionList.vue`。

## 3. 功能概要

- **工具栏**：跳转 **`/performance/load-monitor`**（去压测看板）；批量删除；批量复制并执行（`batchCopyK6Sessions`）。
- **筛选**：`run_id`、**status**（pending/generating/running/completed/failed）、排序、创建日期范围、**仅看我创建**（`created_by_me`）。
- **表格列**：`id`、`run_id`、`status`、`vus`、`duration`、`use_ai`、`create_time`；操作「查看」跳转 **`/performance/load-monitor?run_id=...`**。

## 4. API（`@/api/perfK6`）

- **`getK6Sessions(params)`**：`GET /perf/k6-sessions/`，查询参数与后端 FilterSet 一致（含分页、ordering、`created_by_me` 等，以后端为准）。
- **`batchDeleteK6Sessions(data)`**：`POST /perf/k6-sessions/batch-delete/`。
- **`batchCopyK6Sessions(data)`**：`POST /perf/k6-sessions/batch-copy/`（复制并触发执行，语义以后端为准）。

## 5. 与压测看板的关系

- 列表页负责 **历史会话检索与批量运维**；实时曲线与 **WebSocket `/ws/k6/{run_id}/`** 在 **`LoadTestMonitor.vue`**。
- 若看板支持 query **`run_id`**，可从列表一键深入监控同一运行。

## 6. 环境与鉴权

- 列表请求走统一 **`request`**（`baseURL: /api` + `Authorization: Token ...`）。
- 会话创建时的 **Base URL** 在看板页通过 **`useCurrentEnvironment`** 预填；列表页不直接展示环境字段时，仍以 **`run_id`** 关联后端记录。

## 7. WebSocket 提示

- **本会话列表页不使用 WebSocket**；仅 REST。需要实时指标时请打开压测看板页。

## 8. `getK6Sessions` 查询参数（典型）

- `page` / `page_size`（若后端分页类为 `LimitOffset` 或 `PageNumber`，与 `TestManagement` 其它列表一致）。
- `ordering`：前端提供 `-create_time`、`create_time`、`status`、`-status`。
- `status`：过滤运行状态。
- `run_id`：模糊或精确由后端决定。
- `created_by_me`：布尔，仅当前用户会话。

## 9. 批量复制语义

- **`batchCopyK6Sessions`** 返回新会话并可能触发 **再次执行**（以后端 `batch-copy` 实现为准）；前端 loading 防重复点击。

## 10. 国际化与时间

- **`formatDate`** 截取显示为 `YYYY-MM-DD HH:mm` 样式（字符串 `slice`）；若需时区，应改为 `dayjs`/`Intl`。

## 11. 与看板 deep-link

- 「查看」使用 **`router.push(\`/performance/load-monitor?run_id=${row.run_id}\`)`**；看板 **`onMounted`** 若读取 `route.query.run_id`，可自动 **`getK6Session`** 并重连 WS（若当前实现未做，属后续增强点）。

## 12. 鉴权失败

- 401 时全局拦截器清 token；列表会空且控制台 axios 报错，用户需重新登录。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
