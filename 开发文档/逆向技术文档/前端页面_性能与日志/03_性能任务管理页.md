# 前端页面：性能任务管理页

## 1. 路由与布局特点

- **路径**：`/performance/tasks`，名 **`PerformanceTaskManagement`**。
- **重要**：该路由在 **`router/index.js` 中独立于 `MainLayout`**，整页自带左侧菜单 + 顶栏（`TaskManagement.vue` 内联布局），**不经过**主应用侧栏，但仍受 **全局 `beforeEach`** 的 token 校验约束。

## 2. 视图

- `frontend/src/views/performance/TaskManagement.vue`。

## 3. 页面能力概要

- **筛选**：任务名、状态、执行人等表单 + 查询/重置。
- **表格**：多选、分页、`TableActionGroup` 行内操作（编辑、删除、执行等，以文件内 `taskTableActions` 为准）。
- **批量**：批量删除、批量改状态、批量复制。
- **快捷入口**：工具栏按钮跳转环境管理、k6 看板、k6 会话列表、定时任务新建/列表/日志等（`router.push` 到 `/performance/...` 或 `/performance/scheduled-task` 等）。

## 4. API（`@/api/perfTask`）

| 函数 | HTTP | 路径 |
|------|------|------|
| `getPerfTasks` | GET | `/perf/tasks/` |
| `createPerfTask` | POST | `/perf/tasks/` |
| `runPerfTask` | POST | `/perf/tasks/{id}/run/` |
| `deletePerfTask` | DELETE | `/perf/tasks/{id}/` |
| `batchDeletePerfTasks` | POST | `/perf/tasks/batch-delete/` |
| `batchUpdatePerfTasks` | POST | `/perf/tasks/batch-update/` |
| `batchCopyPerfTasks` | POST | `/perf/tasks/batch-copy/` |

## 5. 环境组件

- 嵌入 **`CurrentEnvironmentSelect`**（`@/components/CurrentEnvironmentSelect.vue`），依赖 **`useCurrentEnvironment`**：从 **`GET /environments/`** 拉选项，选中写入 **`localStorage`**（`current_test_environment_id`）并派发 **`app:current-environment-changed`**。任务执行若需环境 ID，应与该组合式或表单字段对齐。

## 6. WebSocket

- 本管理页自身**无** WebSocket；实时压测曲线在 **`LoadTestMonitor`** 通过 **`/ws/k6/`** 完成。

## 7. 与定时任务的关系

- 工具栏「新建定时任务」指向 **`/performance/scheduled-task`**（无 id 为新建）；列表与日志为其它 `/performance/scheduled-*` 路由，API 前缀为 **`/execution/scheduled-tasks/`** 系列（见 `scheduledTask.js`）。

## 8. 鉴权与跳转

- 未登录访问重定向 **`/login?redirect=...`**。
- 保存定时任务详情成功后示例代码会 **`router.push('/performance/tasks')`** 回到本页（见 `ScheduledTaskDetail.vue`）。

## 10. 侧栏为何无「任务管理」

- `MainLayout` 侧栏仅挂载「环境管理」「k6 压测」等项，**未**挂 `/performance/tasks`；用户通常从收藏夹或内部链接进入。若产品需要，可新增 `el-menu-item index="/performance/tasks"`。

## 11. 表格与操作键

- **`TableActionGroup`** 通过 `actions` 配置 `key`、`icon`、`tooltip`、**`loading`**，由父组件在 handler 内根据 `row` 调 **`runPerfTask`** / **`deletePerfTask`** 等。

## 12. 状态枚举

- 前端展示用 **`statusText` / `statusTagType`**（以 `TaskManagement.vue` 内映射为准）应与后端 `PerfTask.status` 对齐；新增状态需同步标签颜色。

## 13. 分页与导出

- 若后续增加导出 CSV，可仿 `project.js` 的 **`downloadAuthedGet`**，避免丢 Token。

## 14. 与「当前环境」的测试建议

- 切换 **`CurrentEnvironmentSelect`** 后立刻创建/运行任务，确认后端收到的是**最新** `environment_id` 或目标 URL（视 API 合同而定）。

## 15. 小结

性能任务管理页是 **独立壳 + 业务表** 的聚合入口，API 前缀 **`/perf/tasks/`**；实时能力让位于 k6 看板 WS。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。

## 附：术语

- **REST**：与 **`request` axios 实例** 相关的 JSON HTTP 接口（`baseURL /api`）。
- **WS**：**`WebSocket`**，用于 k6 指标、SSH 日志 tail 等。
- **SSE**：**`EventSource`**，看板可用于触发摘要刷新。
