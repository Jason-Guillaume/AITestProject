# 前端页面：环境管理页

## 1. 路由与布局

- **路径**：`/performance/environments`，路由名 **`EnvironmentManagement`**（见 `frontend/src/router/index.js`）。
- **挂载**：嵌套在 **`MainLayout`** 子路由下，与侧栏「环境管理」菜单项 **`index="/performance/environments"`** 一致（`frontend/src/layouts/MainLayout.vue`）。
- **鉴权**：非 `meta.public`，需 **`localStorage.token`**；非流水线例外路径。

## 2. 视图与组件

- **视图**：`frontend/src/views/performance/EnvironmentManagement.vue`。
- **表格操作**：`TableActionGroup`（`@/components/common/TableActionGroup.vue`），用于单行编辑、删除、校验等。
- **无 WebSocket**：本页为 CRUD + 校验 HTTP，不涉及实时长连接。

## 3. 调用的 API（`@/api/environment`）

| 能力 | 方法 | 相对路径（`request` 已带 `/api` 前缀） |
|------|------|----------------------------------------|
| 列表 | GET | `/environments/` |
| 新建 | POST | `/environments/` |
| 更新 | PATCH | `/environments/{id}/` |
| 删除 | DELETE | `/environments/{id}/` |
| 健康校验 | POST | `/environments/{id}/validate/` |

列表参数常见 `page_size` 等由后端分页约定；侧栏顶栏拉环境时可用较大 `page_size`（如 `MainLayout` 内拉取下拉选项）。

## 4. 环境与全局上下文

- **本页数据**：表格绑定 `envList`，来自 `getEnvironments`。
- **「当前执行环境」组合式**：`useCurrentEnvironment`（`frontend/src/composables/useCurrentEnvironment.ts`）使用同一 **`GET /environments/`** 拉全量选项，并把选中 ID 落在 **`localStorage`** 键 **`current_test_environment_id`**，同时派发 **`app:current-environment-changed`**。压测看板等页用 **`baseUrl`** 预填相对路径用例的目标地址。
- **顶栏 Project/Env**：`MainLayout` 另有独立 **`selectedEnv`** / **`getEnvironments`** 同步到 `appContextStore`（Pinia），与组合式环境可并存；文档上区分「列表管理」与「执行上下文」即可。

## 5. 表单与校验逻辑概要

- 弹窗字段：`name`、`env_type`（dev/test/staging/prod/custom）、`base_url`、`health_check_path`、`db_config`（JSON 文本编辑）、`description`。
- **校验**：创建/更新走 `createEnvironment` / `updateEnvironment`；「校验」按钮走 `validateEnvironment`，用于触发后端健康检查路径探测。

## 6. 错误与交互

- 删除前可 `ElMessageBox` 确认（以实际实现为准）。
- 请求失败时依赖统一 **`request`** 拦截器（`frontend/src/utils/request.js`）：超时 60s、`Token` Header、401 清理 token 等。

## 7. 相关页面跳转

- 性能任务管理页工具栏有「环境管理」跳转同路径；侧栏也可直接进入。

## 8. 小结

环境管理页是性能/接口执行链路的**配置入口**之一；执行时真正拼 URL 依赖 **`useCurrentEnvironment` 的 `baseUrl`** 与各页面表单中的 `target_base_url` 覆盖策略。

## 9. 与 `MainLayout` 顶栏环境下拉的关系

- `MainLayout` 在挂载时请求 **`getProjectsApi`**、**`getEnvironments`**，把结果塞进 **`projectOptions` / `envOptions`**，并写入 **`useAppContextStore`**（`appContextStore`）。
- **`CurrentEnvironmentSelect`** 与顶栏使用**不同**存储键：组合式用 **`current_test_environment_id`**，顶栏可能用本地 `selectedEnv`（以 `MainLayout` 脚本为准）。联调时若发现「顶栏选了环境、压测页仍无 baseUrl」，应检查是否需在切换顶栏时同步调用 **`setEnvironmentId`** 或监听同一自定义事件。

## 10. `request` 与原始路径

- 业务 axios 实例 **`baseURL: "/api"`**，故 `environment.js` 中写 **`/environments/`** 实际请求 **`/api/environments/`**。
- 登录/注册在 **`auth.js`** 中使用**裸 axios** 写全路径 **`/api/user/...`**，勿混淆两套客户端。

## 11. 侧栏与面包屑

- `MainLayout` 内 `BREADCRUMB_MAP` 对 **`/performance/environments`** 有英文面包屑片段（Workspace / Performance / Environments），与中文菜单并列存在，属产品设计取舍。

## 12. 常见问题（排查）

- **校验一直失败**：确认 `health_check_path` 与后端探测逻辑、目标网络策略。
- **列表空**：检查 token、项目/组织过滤参数（若后端按租户裁剪）。
- **与其它页 base_url 不一致**：以数据库中最新记录为准，必要时在环境页修改后让 **`loadEnvironments`** 重新拉取。
