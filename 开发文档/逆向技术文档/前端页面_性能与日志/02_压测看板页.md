# 前端页面：k6 压测看板页

## 1. 路由

- **路径**：`/performance/load-monitor`，名 **`LoadTestMonitor`**。
- **布局**：`MainLayout` 子路由；侧栏「k6 压测看板」对应 **`index="/performance/load-monitor"`**。

## 2. 视图文件

- `frontend/src/views/performance/LoadTestMonitor.vue`。

## 3. 核心流程

1. 用户填写 **API 用例 ID 列表**（逗号/换行分隔）、可选 **目标 Base URL**、`vus`、`duration`（如 `30s`）、是否 **AI 生成脚本**。
2. **`onStart`** 调用 **`createK6Session`**（`@/api/perfK6` → **`POST /perf/k6-sessions/`**），拿到 **`run_id`** 等。
3. 使用 **ECharts** 初始化 QPS、P95（及错误率等相关序列）折线图。
4. 建立 **WebSocket** 订阅实时指标；结束或用户断开时关闭连接。

## 4. WebSocket 约定

- **URL 拼装**（脚本内 `buildWsUrl`）：
  - 协议：`https` → `wss`，否则 `ws`。
  - 路径：**`/ws/k6/{run_id}/?token={encodeURIComponent(token)}`**
  - `token` 来自 **`localStorage.getItem("token")`**。
- 消息解析后更新图表与 **`sessionStatus`**；连接状态用 **`wsConnected`** 展示。
- 项目中另提供极简 **`useWebSocket(url, onMessage)`**（`frontend/src/composables/useWebSocket.ts`），本页当前为 **手写 `new WebSocket`**，便于附带 token 与业务状态。

## 5. REST API（`@/api/perfK6`）

| 函数 | 说明 |
|------|------|
| `createK6Session(data)` | POST `/perf/k6-sessions/`，创建会话并触发异步执行 |
| `getK6Session(runId)` | GET `/perf/k6-sessions/{runId}/`，含脚本快照与 summary |
| `getK6Sessions(params)` | 列表/筛选 |
| `batchDeleteK6Sessions` / `batchCopyK6Sessions` | 批量删除/复制 |

看板页主要用 **`createK6Session`**；详情(summary)可在折叠区 **`lastFinal`** 中展示轮询或 WS 结束后的数据（以组件内逻辑为准）。

## 6. 环境与 Base URL

- `useCurrentEnvironment()`：**`loadEnvironments`** + **`baseUrl`**。`applyBaseUrlFromCurrentEnv` 在本地 **`target_base_url` 为空** 时用当前环境 **`base_url`** 预填，便于相对路径用例。

## 7. 与会话列表联动

- `K6SessionList` 行操作「查看」跳 **`/performance/load-monitor?run_id=...`**（见会话列表页），看板可根据 query 恢复会话与 WS（若实现）。

## 8. 注意点

- WS 与页面同 host，依赖反向代理将 **`/ws/`** 转到 ASGI/WebSocket 服务。
- 压测为长任务；浏览器刷新会丢前端图表状态，**`run_id`** 可从列表页再次进入恢复。

## 9. 图表与性能

- ECharts 实例在 **`onMounted` / `onBeforeUnmount`** 中创建与 `dispose`，避免内存泄漏。
- 若 WS 推送频率极高，应注意 **`setOption` 节流**（实现若已有 `requestAnimationFrame` 或批量缓冲，以仓库为准）。

## 10. AI 脚本开关

- 表单字段 **`use_ai`** 传入 **`createK6Session`** body；后端决定是否生成/调整 k6 脚本。失败时 **`sessionStatus`** 与 REST 错误体应通过 `ElMessage` 展示。

## 11. 与定时任务、性能任务页的差异

- **性能任务**（`/perf/tasks/`）偏「平台内任务单/工单」模型；**k6 会话**（`/perf/k6-sessions/`）偏「一次压测运行 + 实时指标」。
- **定时任务**走 **`/execution/scheduled-tasks/`**，可编排 API 用例批次，与 k6 链路解耦，勿混用路径。

## 13. 代理与部署

- 开发环境 **Vite** 需将 **`/api` 与 `/ws`** 一并代理到后端，否则 WS 握手会 404。
- 生产环境 **HTTPS** 下必须使用 **`wss`**，与脚本中协议选择逻辑一致。

## 14. 小结

压测看板是前端 **REST + WebSocket** 结合的典型页：REST 负责 **创建运行**，WS 负责 **实时指标**；环境 baseUrl 降低相对路径用例的配置成本。

## 附：联调自检（通用）

1. 浏览器 **Network** 中请求 URL 是否带 **`/api`** 前缀（`auth.js` 全路径除外）。
2. **`Authorization: Token …`** 是否随 `request` 发出；401 后是否退回登录页。
3. **Vite / Nginx** 是否同时代理 **`/api`** 与 **`/ws`**（若页面使用 WebSocket）。
4. 生产 **HTTPS** 下压测与日志是否使用 **`wss`**。
