## 2. `get_queryset` 与 `scenario_id`

当 Query 参数 **`scenario_id`** 存在且可解析为 **正整数** 时：`qs.filter(scenario_id=sid)`。

## 3. 模型字段语义（执行相关）

| 字段 | 说明 |
|------|------|
| **`scenario`** | 所属场景；**`CASCADE`** 删除场景时步骤一并删除 |
| **`order`** | 正整数顺序；**`scenario_runner`** 按 **`order`, `id`** 排序执行 |
| **`name`** | 步骤展示名 |
| **`test_case`** | FK **`TestCase`**，**`PROTECT`** — 禁止删除仍被步骤引用的用例（需先解绑或删步骤） |
| **`is_enabled`** | 为假或软删时：**不计入 total**，**skipped++**，不创建失败型 StepRun（与禁用跳过逻辑一致，见《05》） |
| **`failure_strategy`** | 空串则继承 **`ApiScenario.failure_strategy`**；否则 **`abort`/`continue`** |
| **`extraction_rules`** | JSON 数组；默认注入 **`run_api_case` overrides** 的 **`extraction_rules`**（若 `step_overrides` 未覆盖） |
| **`step_overrides`** | JSON 对象；运行前 **拷贝为 overrides 基底**，可含 **`environment_id`、`variables`、`extraction_rules`/`variable_extractions`** 等（见 `_merge_step_overrides`） |

**`extraction_rules` 示例**（模型 `help_text`）：  
`[{"var_name":"token","source":"body","expression":"$.data.token"}]`

## 4. `_merge_step_overrides` 与步骤覆盖（摘要）

运行器逻辑（`execution/services/scenario_runner.py`）：

1. **`overrides = dict(step.step_overrides)`**。
2. 若 **`overrides` 内未显式 `environment_id`**：依次尝试 **`run.environment_id`** → **`scenario.environment_id`**，有则写入 **`overrides["environment_id"]`**。
3. **`overrides["variables"]`** = 当前 **运行时变量池** `runtime_vars`（场景默认 ∪ run 初始 ∪ Celery overrides）。
4. 若 **`overrides` 未含 `extraction_rules` 且未含 `variable_extractions`**：补 **`step.extraction_rules`** 列表拷贝。

因此：**步骤级 `step_overrides.variables` 会覆盖同名全局运行时变量**（因整包 `overrides` 交给 `run_api_case`，具体优先级以实现为准）；**提取规则**可被 **`step_overrides`** 完全替换。

---

## 5. 标准 CRUD 注意点

- **创建步骤**：须指定合法 **`scenario`**、**`test_case`**；`test_case` 建议为 **API 类型**，否则运行时会 **跳过** 并记 **`ApiScenarioStepRun` skipped**（见《05》）。
- **顺序调整**：PATCH **`order`**；运行器每次执行前重新 **`order_by("order","id")`**。

---

## 6. 排障

| 现象 | 方向 |
|------|------|
| 步骤列表空 | **`scenario_id`** 是否传错；权限是否看不到该项目下场景 |
| 执行总跳过某步 | **`is_enabled`**、**`test_case.is_deleted`**、**`test_type` 非 api** |
| 变量提取不生效 | 检查 **`extraction_rules`** 与 **`step_overrides`** 是否互相覆盖 |
