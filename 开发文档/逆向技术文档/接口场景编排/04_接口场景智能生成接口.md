# 接口场景智能生成接口

## 文档目的与读者

说明 **`POST /api/execution/api-scenarios/generate/`**（`ApiScenarioGenerateAPIView`）的两阶段契约：**仅生成草稿**（`confirm_create=false`）与 **落库创建场景+步骤+用例**（`confirm_create=true`）。适合对接「OpenAPI 导入为场景」向导、理解 **项目/模块权限** 与 **审计** 记录。

---

## 1. 路由与权限

| 项 | 值 |
|----|-----|
| URL | **`POST /api/execution/api-scenarios/generate/`** |
| 注册 | `execution/urls.py` 中 **`path("api-scenarios/generate/", ...)`**（挂在 **`api/execution/`** 下） |
| **`permission_classes`** | **`[IsAuthenticated]`**（显式要求登录；与部分仅 Token 全局默认可能不同） |

---

## 2. 请求体总览

| 字段 | 必填 | 说明 |
|------|------|------|
| **`project_id`** | 是 | 整数 |
| **`module_id`** | 是 | 整数；须属于 **`project_id`** 下 **`TestModule`** |
| **`environment_id`** | 否 | 整数；**仅 `confirm_create=true` 落库** 时写入 **`ApiScenario.environment_id`** |
| **`base_url`** | 否 | OpenAPI 模式：优先于 spec 内 **`servers[0].url`** |
| **`scenario_name`** | 否 | 草稿/场景名 |
| **`max_steps`** | 否 | 默认 **30**，clamp 到 **[1, 100]** |
| **`openapi_spec`** | 与 `curl_list` 二选一 | **字符串**：JSON 或 YAML 文本 |
| **`curl_list`** | 与 `openapi_spec` 二选一 | **字符串数组** |
| **`confirm_create`** | 否 | 默认 **false**；**仅 Python 真值 `True`** 触发落库（`is True`） |

**400 类错误**：统一多 **`{"success": false, "message": "..."}`**（HTTP 状态码见各分支：`400`/`403`/`404`）。

---

## 3. 权限与模块校验

1. **`TestProject`**：`pk=project_id` 且 **`is_deleted=False`**；不存在 → **404**。
2. **项目访问权**：当前用户须为 **`is_system_admin` / superuser / staff** 或 **`proj.members`**（成员也须 **`is_deleted=False`**）；否则 **403** `无权限访问该项目`。
3. **`TestModule`**：存在、`is_deleted=False`、**`project_id` 与请求一致**；否则 **400** `module_id 不属于该项目`。

> **`module.test_type` 是否为 API** 在本接口 **未二次校验**；落库时用例固定 **`TEST_CASE_TYPE_API`**。若模块类型不符，可能与用例列表筛选预期不一致，宜在前端限制模块选择。

---

## 4. 草稿生成（`confirm_create=False`）

- 调 **`build_scenario_draft_from_openapi`** 或 **`build_scenario_draft_from_curl_list`**（`execution/services/scenario_generator.py`）。
- **`ValueError`** → **400**，`message` 为异常字符串（如「openapi_spec 未发现 paths」）。
- 成功：**`record_execute_audit`**（`action=scenario_generate`，`source`、`project_id`、`module_id`、`steps` 数量）。
- 响应：**`{"success": true, "draft": draft}`**，**不写库**。

**draft 形状**（逻辑上）：

```json
{
  "scenario": { "name": "...", "default_variables": { } },
  "steps": [
    {
      "order": 1,
      "name": "...",
      "request": { "method": "GET", "url": "...", "headers": {}, "body": {}, "expected_status": 200 },
      "extraction_rules": []
    }
  ]
}
```

详见《05》生成器细节。

---

## 5. 落库创建（`confirm_create=True`）

### 5.1 前置

- **`draft.steps`** 须为非空 **list**；否则 **400** `草稿 steps 为空，无法创建`。

### 5.2 事务内操作

1. **`ApiScenario.objects.create`**：`project_id`、`environment_id`、`name`（来自 draft，截断 255）、`default_variables`、`failure_strategy=STRATEGY_ABORT`、`is_active=True`、`creator/updater`。
2. 对每一步 **`st in steps`**（非法项 **continue** 跳过）：
   - 解析 **`request`**：`method`、`url`（空则跳过）、`headers`、`body`、`expected_status`。
   - **`create_typed_case`** 创建 **API 用例**（`API_DEFAULTS` + 各子类型默认 dict 占位）。
   - **`TestCaseStep.objects.create`**：单步描述、断言空数组。
   - **`ApiScenarioStep.objects.create`**：`order=idx`（enumerate 从 1）、`extraction_rules` 来自 draft（须为 list）、`step_overrides={}`。

3. 事务外（成功后）：**`record_audit_event(ACTION_CREATE, instance=scenario, extra=...)`**，`extra` 含 **`scenario_id`、`source`、`created_case_ids`（截断前 200）、`cases_count`**。

### 5.3 响应

```json
{
  "success": true,
  "scenario_id": 10,
  "created_case_ids": [101, 102],
  "steps_count": 2
}
```

## 6. 与「测试用例 import-api-spec」的差异

| 维度 | `import-api-spec`（testcase） | `api-scenarios/generate`（execution） |
|------|-------------------------------|----------------------------------------|
| 输入 | LLM 解析 swagger/curl → items | **确定性**解析 OpenAPI / 正则解析 cURL 列表 → draft |
| 输出 | 多条 **`TestCase`** | **`ApiScenario` + `ApiScenarioStep` + `TestCase` + `TestCaseStep`** |
| 权限 | 模块写权限 `_user_can_write_module_cases` | **项目成员** + 模块归属 |
| 认证 | 依全局默认 | **强制 `IsAuthenticated`** |
