## 1. 生成器：`execution/services/scenario_generator.py`

### 1.1 公共能力

| 函数 | 作用 |
|------|------|
| **`_safe_load_yaml_or_json`** | 文本以 `{`/`[` 开头则 **`json.loads`**；否则尝试 **`yaml.safe_load`**；失败再兜底 JSON |
| **`_pick_first_server_base`** | 取 OpenAPI **`servers[0].url`** |
| **`_iter_openapi_ops`** | 遍历 **`paths`** 下 **get/post/put/delete/patch/head/options** |
| **`_guess_is_login`** | 路径/operationId/summary/description 中含 login、token 等关键字 → 登录类接口 |
| **`_request_body_template`** | 从 **application/json** schema **properties** 取前 10 个键，值模板 **`${key}`** |
| **`_expected_status`** | 优先 **200/201/204**；否则任取一个 **2xx** |
| **`_detect_bearer_auth`** | `components.securitySchemes` 或 global **`security`** 是否 Bearer |

### 1.2 `build_scenario_draft_from_openapi`

- **`base_url`** 参数非空则优先；否则用 spec 的 server；再无则可能与 path 拼接出错，依赖 **`url` 拼接分支**。
- **`ops`** 排序：**登录类接口优先**，再按 path；截取 **`max_steps`**（1–100）。
- 若检测到 Bearer：**`default_vars["token"]=""`**；每步 headers 默认 **`Authorization: Bearer ${token}`**。
- 对 **推测为登录** 且 Bearer 的步骤：附加 **两条默认 `extraction_rules`**（`$.data.token` 与 **`$.token`**），便于后续步骤引用。

### 1.3 `build_scenario_draft_from_curl_list`

- 每项字符串用正则抽取 **`-X METHOD`**、URL、**`-H 'k:v'`**、**`--data`/`--data-raw`/`-d`**。
- body 尝试 **`json.loads`**；失败则 **`{"raw": 截断}`**。
- **`expected_status`** 恒为 **`None`**（与 OpenAPI 分支不同）。
- 无有效步骤 → **`ValueError("curl_list 无有效 curl")`**。

> 正则写在源码中；复杂 cURL（续行、引号嵌套）可能解析不全，宜在文档/产品上提示用户简化。

## 2. 运行器：`execution/services/scenario_runner.py`

### 2.1 入口签名

```python
def run_api_scenario(*, run_id: int, runtime_overrides: Dict[str, Any] | None = None) -> Tuple[ApiScenarioRun, Dict[str, Any]]:
```

**`runtime_overrides`**：当前仅 **`run_api_scenario_run` 任务**传入 **`{"variables": {...}}`**，在 **`runtime_vars`** 上 **第三次 update**（覆盖 run 初始与场景默认中的同名键）。

### 2.2 并发与幂等

- 开头 **`select_for_update`** 取 **`ApiScenarioRun`**：若状态已是 **`running` 或 `success`**，**直接返回** **`{"skipped": True, "reason": "already_running_or_done"}`**，**不重复执行**。
- 将状态置 **`running`**，补 **`started_at`**，无 **`trace_id`** 则 **`uuid.uuid4()`**。

### 2.3 步骤循环

1. 加载 **`ApiScenarioStep`**：`scenario` 匹配、**`is_deleted=False`**，**`order_by("order","id")`**。
2. 跳过：**`not is_enabled`**、**`is_deleted`**、**非 API test_type`**（记 **`skipped`** + **`ApiScenarioStepRun` STATUS_SKIPPED`**）。
3. **用例已软删** / **无 API profile**：记失败；若策略 **`abort`** 则 **`fatal_error`** 并 **break**。
4. 正常路径：**`run_api_case(case, api_prof, overrides=..., user=None, write_legacy_apilog=False)`**。
5. 将 **`result.extracted_variables`** 合并进 **`runtime_vars`**（忽略值为 **`None`** 的键）。
6. **`ApiScenarioStepRun`**：`execution_log_id`、`passed`、`extracted_variables`、`message`（≤255）。
7. 断言失败且 **`abort`**：**`fatal_error`** 并 **break**；**`continue`** 则进入下一步。

### 2.4 终态与 `summary`

| 条件 | `ApiScenarioRun.status` |
|------|-------------------------|
| 有 **`fatal_error`** | **`failed`** |
| **`failed>0` 且 `passed>0`** | **`partial`** |
| **`failed>0` 且 `passed==0` 且 `total>0`** | **`failed`** |
| 否则 | **`success`** |

**`summary`**：`total_steps`、`passed_steps`、`failed_steps`、`skipped_steps`、`step_results`（每步 case_id、execution_log_id、passed）、**`failure_reason`**。  
**`ApiScenarioRun`** 批量 **`update`**：**`final_variables`**、`**error_message**`=fatal_error`、耗时等。

## 3. Celery 包装：`run_api_scenario_run`

**`execution/tasks.py`**：`@shared_task` **`execution.run_api_scenario_run`**，`autoretry_for=(Exception,)`，`max_retries=2`，**软超时 3600s**。  
返回 JSON 含 **`ok`、`run_id`、`scenario_id`、`status`、`trace_id`、`summary`**。
