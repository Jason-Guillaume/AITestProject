# 接口场景运行与步骤运行视图集

## 文档目的与读者

说明 **`ApiScenarioRun`**、**`ApiScenarioStepRun`** 两个只读为主的资源在 REST 上的用法、**`get_queryset` 过滤参数**，以及它们与 **Celery 任务**、**`ExecutionLog`** 的关联。适合构建「运行历史」「步骤时间线」类 UI。

---

## 1. 路由对照

| ViewSet | 注册前缀 | 典型用途 |
|---------|-----------|-----------|
| `ApiScenarioRunViewSet` | **`/api/execution/api-scenario-runs/`** | 某场景多次执行记录 |
| `ApiScenarioStepRunViewSet` | **`/api/execution/api-scenario-step-runs/`** | 单次 run 内各步结果 |

---

## 2. `ApiScenarioRunViewSet`

### 2.1 queryset 与 Serializer

- **queryset**：`ApiScenarioRun.objects.all().select_related("scenario")`
- **Serializer**：`ApiScenarioRunSerializer`，`fields = "__all__"`，只读 **`creator/updater/时间`**

### 2.2 `get_queryset` 过滤

| Query 参数 | 行为 |
|-------------|------|
| **`scenario_id`** | 可转 int → **`filter(scenario_id=sid)`**；非法 → **`qs.none()`** |
| **`trace_id`** | 非空字符串 → **`filter(trace_id=trace_id)`**（精确匹配，已 `strip`） |

二者可 **叠加**。**数据范围**：`ApiScenarioRun` 无直接 **`project`** 字段；`_apply_member_scope` 兜底为 **`creator=user`**（非管理员仅看自己创建的 Run 记录）。若产品需要「同项目成员看他人触发的 run」，需扩展模型或自定义 `get_queryset`。

### 2.3 核心字段（运行生命周期）

| 字段 | 说明 |
|------|------|
| **`status`** | `pending` / `running` / `success` / `failed` / `partial`（由 **`run_api_scenario`** 结束时写回） |
| **`trace_id`** | UUID 字符串；run 开始时若空则生成 |
| **`started_at` / `finished_at` / `duration_ms`** | 起止与耗时 |
| **`environment_id`** | **`TestEnvironment` 主键快照**（整型，无 FK 跨 app 强约束） |
| **`initial_variables` / `final_variables`** | 运行前合并变量 vs 结束后变量池（含步骤提取写入） |
| **`summary`** | JSON：含 **`total_steps`、`passed_steps`、`failed_steps`、`skipped_steps`、`step_results`、`failure_reason`** |
| **`celery_task_id`** | 异步任务 id |
| **`error_message`** | 致命中止原因（如断言失败中止） |

## 3. `ApiScenarioStepRunViewSet`

### 3.1 queryset 与 Serializer

- **queryset**：`ApiScenarioStepRun.objects.all()`（无 `select_related`，列表大时可考虑优化）
- **Serializer**：`ApiScenarioStepRunSerializer`

### 3.2 `get_queryset` 过滤

| Query 参数 | 行为 |
|-------------|------|
| **`run_id`** | 可转 int → **`filter(run_id=rid)`**；非法 → **`qs.none()`** |

用于：`GET /api/execution/api-scenario-step-runs/?run_id=456` 拉一条场景运行的全部步骤结果。

### 3.3 核心字段

| 字段 | 说明 |
|------|------|
| **`run`** | FK → **`ApiScenarioRun`**，`CASCADE` |
| **`step`** | FK → **`ApiScenarioStep`**，`PROTECT` |
| **`order` / `test_case_id`** | 快照 |
| **`execution_log_id`** | **`testcase.ExecutionLog`** 主键，无外键 |
| **`status`** | `completed` / `failed` / `skipped` |
| **`passed`** | 是否通过（与 **`ExecutionLog.is_passed`** 对齐） |
| **`extracted_variables`** | 本步 **`run_api_case`** 返回的提取结果 |
| **`message`** | 摘要，**最长 255**（异常信息在写入时截断） |

---

## 4. 与 Celery 的衔接

- **`run_scenario`** 创建 **`ApiScenarioRun`** 后调用 **`run_api_scenario_run.delay`**。
- 任务体（`execution/tasks.py`）调用 **`run_api_scenario(run_id=..., runtime_overrides=overrides)`**，返回 dict 含 **`ok`、`run_id`、`scenario_id`、`status`、`trace_id`、`summary`**。

轮询 **`ApiScenarioRun`** 的 **`status`/`summary`** 即可展示进度（若需实时流，另需 WebSocket/SSE 设计，当前未在场景模块内置）。

---

## 5. 排障

| 现象 | 方向 |
|------|------|
| run 一直 `pending` | Worker 未消费、任务抛错重试耗尽 |
| `summary.failure_reason` 有值 | 断言失败且 **`abort`**、用例已删、API profile 缺失等 |
| `step_runs` 无记录 | run 未跑到 **`run_api_scenario`** 或早期异常 |
| 列表只能看到自己的 run | **`creator` 兜底数据范围**（见 §2.2） |
