# 根路由与接口前缀聚合

## 文档目的与读者

本文说明 **Django HTTP API 的单一入口**：所有业务 REST 接口如何被拆分到各子应用的 `urls.py`，以及前端 `axios` 如何与之对齐。适合新成员做 **接口定位、联调排障、新增模块挂载** 时阅读。

---

## 1. 配置锚点

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `ROOT_URLCONF` | `AITestProduct.urls` | 见 `AITestProduct/settings.py` |
| 根路由文件 | `AITestProduct/urls.py` | 全站 `urlpatterns` 唯一定义处（HTTP） |
| 媒体 URL | `MEDIA_URL = "/media/"` | 仅 `DEBUG=True` 时在根 urls 中 `static()` 挂载本地文件 |

WebSocket 不在本文件注册，见 **`02_异步网关与上下行及实时通道总路由.md`**。

---

## 2. 请求解析顺序（概念）

1. 请求到达 WSGI/ASGI 的 HTTP 分支后，由 Django URL 解析器处理。
2. 按 `urlpatterns` **自上而下** 匹配；`path("api/user/", include(...))` 会剥掉已匹配前缀，把 **剩余子路径** 交给 `user.urls`。
3. 子应用中再继续匹配 `path`、`re_path` 或 `DefaultRouter` 生成的路由。
4. 未命中任何规则 → **404**（具体响应形态受 DRF / 中间件影响）。

## 4. 前缀与子模块职责速查

| HTTP 前缀 | 包含模块 | 主要职责（摘要） |
|-----------|-----------|------------------|
| `/admin/` | Django 内置 | 管理后台 |
| `/api/sys/` | `user.sys_urls` | 全局 AI 模型配置、配额策略、用量统计/导出、**系统级**审计列表 |
| `/api/change-requests/` | `user.approval_urls` | 敏感变更审批的 **独立** `approve` / `reject` 路径（见下节） |
| `/api/user/` | `user.urls` | 验证码、注册登录、`/me/`、组织/角色/用户 ViewSet、系统消息、个人审计等 |
| `/api/project/` | `project.urls` | 测试项目、任务、发布、流水线等 |
| `/api/testcase/` | `testcase.urls` | 用例、模块、方案、**含** `environments` 与 `environment-variables` 等路由 |
| `/api/environments/` | `testcase.environment_urls` | **仅** `TestEnvironmentViewSet` 的「短前缀」入口（与上条双挂） |
| `/api/execution/` | `execution.urls` | 计划、报告、执行记录等 |
| `/api/perf/` | `execution.perf_urls` | 性能压测子路由（任务、K6 会话、定时任务等） |
| `/api/defect/` | `defect.urls` | 缺陷、版本、看板等 |
| `/api/assistant/` | `assistant.urls` | 助手主路由（脚本、知识库等，以该文件为准） |
| `/api/ai/` | `assistant.ai_urls` | AI 生成流、预览、补丁、知识问答、UI 元素库等 **独立聚合** |
| `/api/server-logs/` | `server_logs.urls` | 日志主机、ES 检索、审计、自动工单等 |

## 5. 测试环境「双入口」说明

`TestEnvironmentViewSet` 被注册 **两次**，便于网关或旧客户端使用不同前缀：

1. **`/api/testcase/environments/`** — 来自 `testcase/urls.py` 中 `router.register(r"environments", TestEnvironmentViewSet)`；与用例域其它资源同前缀。
2. **`/api/environments/`** — 来自 `testcase/environment_urls.py`：`router.register(r"", TestEnvironmentViewSet, basename="environment")`，即列表为 `/api/environments/`，详情为 `/api/environments/<pk>/`。

**行为**：同一 ViewSet、同一权限与序列化逻辑；仅 URL 前缀不同。前端 `frontend/src/api/environment.js` 等需与之一致（若混用两套路径，注意反向代理缓存策略）。

---

## 6. 审批独立前缀示例

`user/approval_urls.py` 仅包含：

- `POST/GET ...` **`/api/change-requests/<pk>/approve/`**
- **`/api/change-requests/<pk>/reject/`**

与 `user.urls` 内 `admin/change-requests/` 列表/裁决接口 **路径不同**，避免与 Router 生成的 `users/`、`orgs/` 等混淆；网关层可按前缀单独做限流或审计。

---

## 7. 前端如何拼接 URL

- **`frontend/src/utils/request.js`** 中 axios 实例：`baseURL: "/api"`，`timeout: 60000`。
- 各 `frontend/src/api/*.js` 内路径一般写 **去掉 `/api` 后的段**，例如 `user/login/` 实际请求为 **`/api/user/login/`**。
- 部署时若 Django 与前端 **不同源**，需在反向代理把 `/api` 与 `/media`（若需）转发到后端，或改为环境变量中的绝对 `baseURL`。

后端健康与错误提示：`request.js` 内会对异常分发 **`app:backend-healthy` / `app:backend-unhealthy`** 自定义事件，供布局层统一提示（与根路由无强耦合，联调时需保证 **前缀未被代理改写错误**）。

## 8. 显式与静默行为归纳

| 显式（可观测） | 静默（默认不暴露给调用方） |
|----------------|----------------------------|
| 浏览器 Network 中的完整 URL | `include()` 内部 `urlpatterns` 的导入顺序对「同路径多规则」的影响 |
| 404 响应体 | `APPEND_SLASH` 触发的 301（若开启且路径缺尾斜杠） |
| `DEBUG` 下直接访问 `/media/...` 静态文件 | 生产环境通常由 Nginx 提供媒体，不再走 Django `static()` |

---

## 9. 排障清单

1. **404 且路径以 `/api` 开头**：先对照本文表格检查 **第二段**（`user`/`project`/…）是否写错；再查子应用 `urls.py`。
2. **启动即 ImportError**：某个 `include("app.urls")` 的模块或其依赖导入失败；根 urls 不会捕获，进程直接退出。
3. **前端 200 但数据不对**：可能是 **双环境入口** 或 **多版本代理** 指到不同后端实例。
4. **媒体文件 404**：确认 `DEBUG`、 `MEDIA_ROOT` 与 `static(MEDIA_URL, ...)` 是否生效；生产应改用 Web 服务器别名。

---

## 10. 新增业务子系统时的建议步骤

1. 新建 Django app（或复用现有 app 的 `urls.py`）。
2. 在 **`AITestProduct/urls.py`** 增加一行 `path("api/<your-prefix>/", include("yourapp.urls"))`。
3. 在 `yourapp/urls.py` 中注册 `ViewSet` / `APIView`。
4. 前端新增 `src/api/yourModule.js`，路径以 **`/<your-prefix>/`** 开头（无 `/api`，由 axios `baseURL` 补全）。
5. 若需 WebSocket：在 **`asgi.py`** 合并 `websocket_urlpatterns`（见下一篇文档）。
