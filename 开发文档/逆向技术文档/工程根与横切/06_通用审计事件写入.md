# 通用审计事件写入

## 文档目的与读者

本文说明 **`record_audit_event`** 及相关辅助函数：在 **不阻断主业务** 的前提下向 **`audit_event`** 表追加一行操作记录，并对敏感字段脱敏、对大文本截断。适合对接 **合规审计、排障「谁删了数据」**、或扩展新 `action` 时阅读。

---

## 1. 实现位置

**文件**：`common/services/audit.py`  
**模型**：`common.models.AuditEvent`（表名 **`audit_event`**，见下一篇文档）。

---

## 2. `record_audit_event` 签名与参数语义

```67:76:common/services/audit.py
def record_audit_event(
    *,
    action: str,
    actor,
    instance: Optional[models.Model],
    request=None,
    before: Any = None,
    after: Any = None,
    extra: Optional[Dict[str, Any]] = None,
) -> None:
```

| 参数 | 说明 |
|------|------|
| `action` | 建议使用 `AuditEvent` 上常量：`create` / `update` / `delete` / `export` / `execute` |
| `actor` | 一般为 `request.user`；未认证时 `creator`/`updater` 不写 |
| `instance` | 用于推导 `object_app`、`object_model`、`object_id`、`object_repr`；可为 `None` |
| `request` | 提取 `path`、`HTTP_USER_AGENT`、`X-Forwarded-For` 或 `REMOTE_ADDR` |
| `before` / `after` | 显式传入模型或 dict；缺省且存在 `instance` 时由 **`_model_payload(instance)`** 生成 |
| `extra` | 附加 JSON，原样写入（仍建议避免放密钥） |

**返回值**：`None`；整体包在 **`try/except`** 内，**任意异常直接 `return`**，主事务不受影响。

---

## 3. 快照生成与脱敏

### 3.1 `_model_payload(instance)`

- 遍历 `instance._meta.get_fields()`，跳过无 `attname`、`id`/`pk`、**M2M / 反向一对多**。
- 标量字段 `getattr(instance, name)` 写入 dict。
- 结果经 **`_redact`**。

### 3.2 `_redact`

- **dict**：键名转小写后若在敏感集合中则值为 **`***`**：含 `password`、`token`、`access_token`、`refresh_token`、`secret`、`api_key`、`authorization`、`cookie`、`set-cookie` 等（见源码 `_SENSITIVE_KEYS`）。
- **list**：递归处理，**最多前 200 条** 元素。
- 其它类型经 **`_truncate`**：字符串超长截断并追加 `...(truncated,total=...)`，默认最大 **2000** 字符。

### 3.3 写入前再次脱敏

`AuditEvent.objects.create(..., before=_redact(before), after=_redact(after), ...)` — 即显式传入的 `before`/`after` 也会过一遍 `_redact`。

## 4. 辅助入口

| 函数 | action | 典型调用场景 |
|------|--------|----------------|
| **`record_export_audit`** | `export` | CSV/报表导出 |
| **`record_execute_audit`** | `execute` | 批量执行、任务触发、异步入队 |

二者内部均调用 **`record_audit_event`**，并传入 `extra` 可选扩展。

---

## 5. 与 `BaseModelViewSet` 的配合

在 **`perform_create` / `perform_update` / `perform_destroy`** 末尾调用（见 **`03_公共视图基类与统一模型视图集.md`**）：

- **Create**：`before=None`；`after` 默认由 `instance` 推导。
- **Update**：传入 `before=serializer.instance`（保存前快照）。
- **Delete（软删）**：`before` 为删除前对象；实例仍带 `pk` 便于追溯。

非 ViewSet 流程（如管理命令、Celery 任务）也可直接调用本函数，**`request` 可传 `None`**（则路径/IP/UA 为空串）。

## 6. 数据库表 `audit_event`（字段摘要）

完整定义见 **`07_基础模型与审计事件数据模型.md`**。本节强调 **业务常用列**：

| 字段 | 用途 |
|------|------|
| `action` | 动作枚举 |
| `object_app` / `object_model` / `object_id` | 定位被操作对象 |
| `object_repr` | `str(instance)` 截断 255 |
| `request_path` / `ip` / `user_agent` | 请求上下文 |
| `before` / `after` | JSON 快照（已脱敏/截断） |
| `extra` | 扩展 JSON |
| `creator_id` | 操作人（继承 `BaseModel`） |

索引：`(action, -create_time)`、`(object_app, object_model, object_id)` — 利于列表筛选与对象维度的历史查询。

## 7. 前端可读接口（交叉引用）

- **个人维度**：`GET /api/user/me/audit/events/`（仅 `creator=request.user`，见用户分册）。
- **系统维度**：`GET /api/sys/audit/events/`、`export.csv`（见系统级智能与审计分册）。

## 8. 显式与静默

| 显式（管理端/审计页） | 静默 |
|------------------------|------|
| 动作、对象摘要、时间、路径 | 写入失败：无日志、无抛错、主业务已成功提交 |
| `before`/`after` 在详情接口中可能裁剪字段 | 完整快照体积与敏感信息仍需谨慎开放给前端 |

---

## 9. 扩展建议

- 新增动作：在 **`AuditEvent.ACTION_*`** 与 `ACTION_CHOICES` 中增加常量，再在业务点调用 `record_audit_event`。
- 强合规场景：可在 `extra` 中写入 **审批单号、工单号**，便于与外部 ITSM 对账。
