# 异步网关与上下行及实时通道总路由

## 文档目的与读者

本文描述 **ASGI 应用入口**：HTTP 仍走 Django；**WebSocket** 由 Django Channels 的 `ProtocolTypeRouter` 分流，并在进入业务 Consumer 前完成 **Token 鉴权**。适合排查 **压测实时曲线不更新、服务器日志终端连不上、WS 401** 等问题。

---

## 1. 与 HTTP 根路由的关系

| 通道 | 配置文件 | 说明 |
|------|-----------|------|
| HTTP(S) | `AITestProduct/urls.py` | 见 **`01_根路由与接口前缀聚合.md`** |
| WebSocket | `AITestProduct/asgi.py` | 本文；路径 **不** 带 `/api` 前缀，以 `ws/...` 为模式根 |

部署时必须使用 **ASGI 服务器**（如 Daphne、Uvicorn + ASGI）处理 WebSocket；若仅用 Gunicorn+WSGI，WS 握手会失败。

---

## 2. ASGI `application` 装配顺序

源码 **`AITestProduct/asgi.py`** 逻辑可归纳为：

1. 设置 `DJANGO_SETTINGS_MODULE`。
2. `django_asgi_app = get_asgi_application()` — 内嵌原有 Django HTTP + 同步中间件链。
3. 合并两个应用的 WebSocket 路由：  
   `_ws_all = execution_ws_urlpatterns + server_logs_ws_urlpatterns`
4. 最终：  
   `ProtocolTypeRouter({"http": django_asgi_app, "websocket": TokenAuthMiddleware(URLRouter(_ws_all))})`

```14:31:AITestProduct/asgi.py
django_asgi_app = get_asgi_application()

from execution.middleware_ws import TokenAuthMiddleware  # noqa: E402
from execution.routing import (
    websocket_urlpatterns as execution_ws_urlpatterns,
)  # noqa: E402
from server_logs.routing import (
    websocket_urlpatterns as server_logs_ws_urlpatterns,
)  # noqa: E402

_ws_all = execution_ws_urlpatterns + server_logs_ws_urlpatterns

application = ProtocolTypeRouter(
    {
        "http": django_asgi_app,
        "websocket": TokenAuthMiddleware(URLRouter(_ws_all)),
    }
)
```

**注意**：`TokenAuthMiddleware` 的 import 放在 `get_asgi_application()` 之后，是为避免 Django 初始化顺序问题（文件中 noqa 已标注）。

## 3. Token 鉴权中间件

**文件**：`execution/middleware_ws.py`

- 仅当 `scope["type"] == "websocket"` 时解析查询串。
- 使用 `urllib.parse.parse_qs` 读取 **`token`** 参数（DRF `Token` 的 key 字符串）。
- `@database_sync_to_async` 查询 **`rest_framework.authtoken.models.Token`**；命中则 `scope["user"] = token.user`，否则 **`AnonymousUser()`**。

```24:35:execution/middleware_ws.py
class TokenAuthMiddleware:
    """ASGI 中间件：在路由前解析 token。"""

    def __init__(self, inner):
        self.inner = inner

    async def __call__(self, scope, receive, send):
        if scope["type"] == "websocket":
            qs = parse_qs(scope.get("query_string", b"").decode())
            token = (qs.get("token") or [None])[0]
            scope["user"] = await _user_from_token(token or "")
        return await self.inner(scope, receive, send)
```

**数据库**：鉴权阶段读取表 **`authtoken_token`**（字段含 `key`、`user_id`、`created` 等）。

## 4. WebSocket 路由表与 Consumer

### 4.1 压测 K6 实时指标

**文件**：`execution/routing.py`

| 正则模式 | Consumer | 说明 |
|----------|-----------|------|
| `^ws/k6/(?P<run_id>[0-9a-fA-F-]{36})/$` | `K6MetricsConsumer` | `run_id` 为 UUID 格式的 K6 会话运行 ID |

**`K6MetricsConsumer.connect`（摘要）** — `execution/consumers.py`：

- 若 `AnonymousUser` 或未认证 → **`close(code=4401)`**（自定义语义：未授权）。
- 若数据库中不存在对应 `K6LoadTestSession`（`is_deleted=False`）→ **`close(code=4404)`**。
- 否则加入 Channels 组 **`k6_run_<run_id>`**，`accept()` 后发送 JSON：`type: status, phase: subscribed`。

下行推送通过 **`k6_metrics`** 事件从 channel layer 广播到组内连接。

### 4.2 服务器日志 SSH 流

**文件**：`server_logs/routing.py`

| 正则模式 | Consumer | 说明 |
|----------|-----------|------|
| `^ws/server-logs/(?P<server_id>[0-9]+)/$` | `LogViewerConsumer` | `server_id` 为日志主机主键 |

**`LogViewerConsumer.connect`（摘要）** — `server_logs/consumers.py`：

- 未登录 → **4401**。
- `server_id` 非法 → **4400**。
- 通过 **`remote_log_server_queryset_for_user(user)`** 加载主机；无权限或不存在 → **4404**。
- 查询串可带 **`path=`**（经 `urllib.parse.unquote`），否则用主机 `default_log_path`；路径经 **`validate_remote_log_path`** 校验（与操作系统类型相关）。

前端完整 URL 形态示例（概念）：  
`wss://<host>/ws/server-logs/<server_id>/?token=<key>&path=<urlencoded_log_path>`

## 5. 上下行数据流（简图）

```text
浏览器 WS 握手
    → ProtocolTypeRouter [websocket]
    → TokenAuthMiddleware（query ?token= → scope["user"]）
    → URLRouter(_ws_all)
    → K6MetricsConsumer / LogViewerConsumer
         ↑ 下行：channel layer group_send / send(text_data=...)
         ↓ 上行：receive 文本帧（如 ping、stop 等，以各 Consumer 实现为准）
```

HTTP API 的「上行请求体 / 下行 JSON」仍在 **`django_asgi_app`** 路径处理，与 WS **并行互不替换**。

---

## 6. 依赖与运维注意

- **Channel Layer**：生产需在 `settings` 中配置 Redis 等作为 backend，否则多进程间 **`group_send`** 无法跨 worker 送达（开发环境可能用内存层，表现为仅单机有效）。
- **Token 泄露**：WS URL 常带明文 `token`，务必使用 **WSS**、控制 Referer/日志脱敏。
- **关闭码**：4401/4404 等为应用层约定，前端 `useWebSocket` 或原生 `onclose` 可根据 `code` 做重试或提示。

---

## 7. 显式与静默

| 显式给前端 | 静默 |
|------------|------|
| 文档或代码中写明的 `ws(s)://.../ws/k6/<run_id>/?token=` | Token 查库失败时中间件静默设为匿名；具体是否断开由 Consumer 决定 |
| 连接成功后首条 `status/subscribed` JSON | channel layer 内部路由键、组名 |

---

## 8. 排障

1. **一直 401 / 立即断开**：查 `token` 是否过期、是否错用 Session；WS 不走 Session Cookie 除非额外实现。
2. **K6 404**：`run_id` 与后端会话表不一致或会话已软删。
3. **日志 404**：当前用户不在该主机组织成员范围内（见 `remote_log_server_queryset_for_user`）。
4. **能连上无数据**：查 Redis channel layer、以及后台是否向 `k6_run_*` 组 **`group_send`**。
