# 05 — 补丁：列表、应用、回滚、从执行生成（逆向）

## 1. `POST /api/ai/patches/from-execution/`（`AiPatchFromExecutionAPIView`）

- Body：`execution_log_id`、可选 `hint`。
- 内部实例化 `AiSuggestCaseFixAPIView().post(request)`；非 200 原样返回。
- 再读 `ExecutionLog` 校验权限与未通过状态。
- `before_steps = _case_steps_snapshot(tc)`（未删步骤 ≤120）。
- `after_steps = payload["suggested_steps"]`；根据前后步数变化估 `risk_level`（空或变化过大 → `high`）。
- 创建 `AiPatch`：`status=draft`，`target_type="testcase.TestCase"`，`target_id=str(tc.id)`，`source_execution_log_id`，`before/after/changes` JSON。
- `record_audit_event(ACTION_EXECUTE, instance=patch)`。

## 2. `POST /api/ai/patches/<id>/apply/`（`AiPatchApplyAPIView`）

- 需 `confirm: true` JSON；仅 `draft` 可应用。
- 目标 `TestCase` 存在且 `_can_user_access_test_case`。
- 事务：`TestCaseStep` 全部软删 → 根据 `after.steps` 至多 60 条重建；`patch.status=applied`，`applied_at=now`，并把应用前快照写回 `patch.before.steps`。
- 审计：`record_audit_event(ACTION_UPDATE, instance=testcase)`。

## 3. `POST /api/ai/patches/<id>/rollback/`（`AiPatchRollbackAPIView`）

- 仅 `applied`；`confirm: true`。
- 用 `before.steps` 覆盖当前步骤（软删 + 重建）；`status=rolled_back`。

## 4. `GET /api/ai/patches/`（`AiPatchListAPIView`）

- Query：`target_type`、`target_id`、`status`、`page`、`page_size`（最大 200）。
- 返回精简字段列表（id、状态、风险、summary、时间戳等）。

## 5. 权限小结

- 补丁列表当前 **未**按用户过滤——如暴露于生产需补 creator/项目维度筛选。
