# 02 — Phase1 预览与批量生成用例（逆向）

## 1. `POST /api/ai/phase1-preview/`（`AiPhase1PreviewAPIView`）

- 先做 `_guard_ai_request_or_429(..., action="phase1_preview")`。
- `_prepare_ai_generate_context(request)`：组装 requirement、module、api_spec、test_type、dedup 等上下文；失败 400 + usage 记失败。
- 仅执行 `_run_phase1_intent_analysis(client, ...)`：输出推导模块名、关键测试点列表等（原始 JSON 在 `data.raw`）。
- 成功返回 `data.module_name`、`key_test_points`、`model` 等；写 `write_ai_usage_event`。

## 2. `POST /api/ai/generate-cases/`（`AiGenerateCasesAPIView`）

- `action="generate_cases"` 治理槽位。
- Phase1：可用 body `phase1_override` dict 跳过模型阶段（便于前端确认后重放）。
- RAG：`rag_enabled` 真时 `build_rag_system_prompt`，`RAG_TOP_K` 来自 settings。
- Phase2：`_run_phase2_parse` 优先 `response_format=json_object`，失败再降级普通文本；`parse_batch_json_array` / `_parse_ai_cases_json` 双解。
- 归一化：`_normalize_generated_case`、`normalize_batch_case_item`；API 类型用 `enrich_normalized_case_with_api_fields`。
- 去重：`_deduplicate_cases_with_guard` + `_apply_dedup_pipeline`（`dedup_mode` 支持 drop/highlight 等，含 debug 回传）。
- 行数兜底：不足 `AI_GENERATE_MIN_CASES` 时 `_expand_cases_to_minimum_if_needed` 多轮补生成；最终 `_enforce_cases_cap`。
- **all_covered**：若模型输出语义为「已有用例覆盖无需新增」，`is_all_covered_output(raw)` 触发 200 + 空 cases + `run_id`。

## 3. 超时守护

- `_timeout_guard` 对整体 wall clock 限制 `AI_GENERATE_TOTAL_TIMEOUT_SECONDS`。

## 4. 出站载荷

- `run_id` 由 `_write_ai_case_generation_run` 写入生成运行表，前端可用于关联导入/审计。
