# 09 — 提示词策略与错误码解析（逆向）

## 1. 领域提示词拼装

- `dispatch_ai_generation(ctx)`：按 `test_type` 选择系统提示模板与用户消息骨架。
- `apply_test_type_domain_strategy(system_prompt, ctx)`：在给定测试类型上叠加领域策略（功能/安全/API 等分支在实现内）。
- `build_engine_addon(ctx)`：附加引擎级说明（版本/约束），拼到 system 尾部。

## 2. Phase1 / Phase2 文本块

- `_phase1_analysis_block(phase1)`：把意图识别结果结构化注入 system，供 Phase2 引用。
- `_PHASE2_MULTI_CASE_MANDATE`：强制多案例、JSON 结构约束等（常量在同文件）。

## 3. RAG 提示

- `build_rag_system_prompt(domain_system_with_phase1, phase1_require, module_name, api_spec, top_k)`：把检索到的知识 article chunks 插入系统提示。

## 4. 错误码（同步生成 `AiGenerateCasesAPIView`）

- `INVALID_INPUT`：上下文预处理失败。
- `SDK_NOT_AVAILABLE`：未安装 openai SDK。
- `GENERATION_TIMEOUT`：总时长超限。
- `INVALID_JSON` / `INVALID_CASES`：Phase2 无法解析或无有效用例。
- `PHASE2_TIMEOUT` / `PHASE2_FAILED`：第二阶段异常。
- 外层 OpenAI 错误交给 `resolve_ai_generate_cases_outer_openai_error` 映射，避免泄露令牌。

## 5. 流式错误 `type=error`

- 含 `AUTH_ERROR`、`parse_failed`、`code`（HTTP status from APIError）、429 文案替换等。

## 6. 连接测试

- `resolve_ai_test_connection_openai_error`：把 SDK 异常转前端可读中文/英文 message。

## 7. 调试

- `settings.DEBUG` 下 `_debug_log_openai_target` 打印 scene/model/base_url。
