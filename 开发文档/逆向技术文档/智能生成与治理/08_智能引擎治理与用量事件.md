# 08 — 智能引擎治理与用量事件（逆向）

## 1. 并发与配额入口 `_guard_ai_request_or_429`

- 在生成、流式、Phase1、连接测试、知识 autofill/ask、安全生成/分析、用例修复等路径统一调用。
- 拒绝时返回 HTTP 429（或包装 JsonResponse），具体字段以实现模块 `assistant.services.ai_governance` 为准。
- 与之配对：`release_ai_concurrency_slot(user_id=...)`，通常在 `finally` 调用，避免异常路径泄漏槽位。

## 2. `AiUsageEvent` 模型（表 `ai_usage_event`）

- 字段：`user`、`action`（枚举含 generate_cases、generate_cases_stream、phase1_preview、test_connection、verify、knowledge_autofill、knowledge_ask、security_* 等）、`endpoint`、`success`、`status_code`、`error_code/message`（短文本）、`model_used`、`test_type`、`module_id`、`streamed`、`all_covered`、`latency_ms`、`prompt_chars`、`output_chars`、`cases_count`、`meta` JSON、`created_at`。
- 设计注释：**不落敏感原文**，适合给运营侧做统计。

## 3. 写入入口 `write_ai_usage_event`

- 各视图在成功/失败分支调用，携带脱敏 `error_message` 上限（常见 512）。

## 4. 生成运行 `_write_ai_case_generation_run`

- 与 `AiUsageEvent` 并行存在，侧重「一次生成的结构化产物关联」（`run_id` 回流给前端同步/流式接口）。

## 5. 排障

- 若用户抱怨「总被拒绝」：查治理配置（每用户并发、日配额、IP 限制等，位置在 ai_governance/settings）。
- 若统计缺失：视图 `except Exception: pass` 会吞掉 telemetry ——应结合服务器日志。
