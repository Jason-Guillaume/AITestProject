# 10 — 智能补丁与生成运行数据模型（逆向）

## 1. `AiPatch`（表 `ai_patch`，节选）

- 状态：`draft` / `applied` / `rolled_back` / `cancelled`。
- 风险：`low` / `medium` / `high`（由生成逻辑根据步骤变化幅度估算）。
- 字段：`creator`、`target_type`（当前补丁流固定 `testcase.TestCase`）、`target_id` 字符串、`source_execution_log_id`。
- JSON：`before`、`after`、`changes`（如 `replace_all_steps` 元数据）、`summary`、`risks`（长度受限以视图层截断为准）。
- 时间：`applied_at`、`rolled_back_at`、`created_at` 等（完整字段见模型）。

## 2. 补丁生命周期（与 API 对应）

1. draft：`from-execution` 创建。
2. applied：`apply` 将 `after.steps` 写入 `TestCaseStep`。
3. rolled_back：`rollback` 用 `before.steps` 还原。
4. cancelled：预留枚举（具体端点若未暴露则仅数据层）。

## 3. `AiCaseGenerationRun`（若存在，命名以模型为准）

- `_write_ai_case_generation_run` 用于绑定 Phase1 结果、是否流式、是否 all_covered、cases_count、dedup_debug 标记、`prompt_version` 等 meta。
- 前端 `run_id` 回传后可在「智能生成」历史页展示。

## 4. 与 `AiUsageEvent`

- `AiUsageEvent` 重放统计；`AiCaseGenerationRun`（如存在）偏重业务追踪与 Debug。

## 5. 审计

- 补丁应用/回滚：`record_audit_event` 打在 `TestCase` 上，`extra` 附带 `ai_patch_id`。

## 6. 数据膨胀控制

- `summary`/`risks` 在视图层 `[:512]`、`[:1000]` 截断；步骤快照最多 60 条应用——防止极端 LLM 输出撑爆事务。
