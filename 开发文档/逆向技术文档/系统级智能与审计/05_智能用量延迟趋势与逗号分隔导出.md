# 智能用量延迟趋势与逗号分隔导出

## 文档目的

说明 **`GET /api/sys/ai-usage/metrics/`**（趋势 + 分位数 + avg/max）、**`GET /api/sys/ai-usage/latency-trend/`**（按日 P95），以及 **`GET /api/sys/ai-usage/export.csv`** 流式导出与 **`record_export_audit`**。

---

## 1. 综合指标：`AiUsageMetricsAPIView`

**路径**：`GET /api/sys/ai-usage/metrics/`

| 参数 | 默认 | 范围 | 说明 |
|------|------|------|------|
| `days` | 7 | 1～31 | `since = now - days` |
| `max_samples` | 5000 | 200～20000 | 每个 **action** 拉取最近 latency 样本在 **Python** 算 **p50/p95** |

### 1.1 返回结构

- **`xAxis`**：日期字符串列表（与 `TruncDate` 得到的自然日对齐）
- **`trend`**：每个 action 的 `total` / `success` / `failed` 与 `xAxis` 同序数组
- **`metrics`**：每个 action 的 **`avg_ms`、`max_ms`、`total`**（DB 聚合）+ **`p50_ms`、`p95_ms`**（样本分位数）

### 1.2 实现注意

- 分位数在 **内存** 中计算，避免各数据库 **`percentile_cont`** 兼容问题。
- `latency_ms<=0` 的记录在分位数样本中 **排除**。

---

## 2. 延迟趋势：`AiUsageLatencyTrendAPIView`

**路径**：`GET /api/sys/ai-usage/latency-trend/`

| 参数 | 默认 | 说明 |
|------|------|------|
| `days` | 7 | 无自定义日期时滑动窗口 |
| `max_samples` | 500 | 50～5000；**每个自然日、每个 action** 抽样条数 |
| `start_date` / `end_date` | 可选 | 若指定：生成 **`day_list`**，且窗口 **最多 62 天** 防超大计算 |

### 2.1 返回

```json
{ "success": true, "days": 7, "xAxis": ["..."], "series": [ { "action": "...", "p95_ms": [ ... ] } ] }
```

每日 P95：对该日该 action 的 `latency_ms` 倒序取 `max_samples` 条后在 Python 算 `_quantile(vals, 0.95)`。

---

## 3. CSV 导出：`AiUsageExportCsvAPIView`

**路径**：`GET /api/sys/ai-usage/export.csv`

| 参数 | 默认 | 范围 |
|------|------|------|
| `limit` | 2000 | 1～200000 |

- 使用 **`_apply_ai_usage_filters`**（与 events 一致，**含逗号 action**）。
- **`StreamingHttpResponse`** + **`iterator(chunk_size=1000)`** 流式写行。
- 列：`created_at, action, success, status_code, user_id, model_used, test_type, module_id, latency_ms, prompt_chars, output_chars, cases_count, error_code, error_message, endpoint`。
- 结束后 **`record_export_audit(..., extra={"export":"ai_usage_events","rows": limit})`**。

---

## 4. 「逗号分隔」含义

在 **`_apply_ai_usage_filters`** 中，**`action`** 查询参数若含 **逗号**，则拆分为列表 **`action__in`**，用于多 action 联合筛选；**`export.csv` / `events` / `top-errors` / `latency-trend`** 等共用该逻辑。
