## 2. 用量摘要：**`AiUsageSummaryAPIView`**

### 2.1 Query **`days`**

```python
days = clamp(int(query or 7), low=1, high=31)
since = timezone.now() - timedelta(days=days)
qs.filter(created_at__gte=since)
```

### 2.2 SQL 聚合

**`values(\"action\",\"success\").annotate(count=Count(\"id\"))`**  
Python 二层循环组装：

```python
by_action[act] = { \"success\", \"failed\", \"total\" }
```

**布尔 **`success`** 分行计数**——**饼图**：`**`failed/(success+failed)`** SLA。

### 2.3 响应

|**键**|**示例**|
|-----|-------|
|**`success`**|外层 bool（非 HTTP）|
|**`since`**|datetime ISO|
|**`days`**|int|
|**`by_action`**|`**`\"generate_cases\": {\"success\":120,\"failed\":3,\"total\":123}`|

**不含 user 维度 slicing**——要按人拆 **改用 export + pandas**。

## 3. 错误排行：**`AiUsageTopErrorsAPIView`**

### 3.1 时间与 **`base_qs` 分歧**

|**条件**|**base_qs 起点**|
|----------|----------------|
|**`start_date` 或 `end_date` 任一存在**|**`AiUsageEvent.objects.filter(success=False)`**——**丢弃默认 `created_at>=since(days)`**|
|**两者皆空**|**近 `days`(1..31)** + **`success=False`**|

**动机**：日历筛选要做 **全局历史失败 TopN**——否则 **七日窗会截断老错误码**。产品 UI 若在 **datepicker** 填入值需 **仔细阅读此分支**，避免 **误判接口 bug**。

### 3.2 二次筛选

**`_apply_ai_usage_filters(base_qs, request, allow_multi_action=True)`** —— **`action`** 支持 **逗号多选**、**`user_id`**、**`error_code`**、`success`(恒 false 仍允许重复 filter True? 已实现无冲突)。

### 3.3 聚合键与排序

**`values(\"action\",\"error_code\",\"error_message\")`**  
**.annotate(count, max_latency=Max(latency_ms))**  
**.order_by(\"-count\")[: limit*3 ]` → **`results[:limit]`**

`**`limit`** 默认 **10**，clamp **\[1,50\]**。**`*3`** 缓冲区 **吃掉并列**，防 **结果被截断层**——历史注释如此。

|**输出字段**|**说明**|
|--------------|-------|
|**`max_latency_ms`**|本条聚合桶内 **`latency`** 最大值，用于 **长尾识别**|
