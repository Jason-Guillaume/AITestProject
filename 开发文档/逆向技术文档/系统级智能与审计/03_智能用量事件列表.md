## 2. **`page_size` 语义 ≠ 分页 offset**

源码 **`page_size`**（query）：

```python
page_size = request.query_params.get("page_size", 50)
# int 校验后 clamp：max(1, min(page_size, 200))
rows = qs.order_by("-created_at")[:page_size]
```

|**结论**：**无 **`page`** 参数**——实际是 **「时间倒序截取 head」**。前端若照搬 **分页器 total count**会得到 **失真**。**正确用法**：仪表盘「最近告警」信息流。

## 3. **`_apply_ai_usage_filters` 详解**

|**参数**|**语义**|
|---------|--------|
|**`action`**|`**`allow_multi_action=True`**：**包含 **`,` → **`action__in`_split(parts)**；否则 **`action=` 单值**|
|**`success`**|`**`1/true/yes`→ **`success=True`**；`**`0/false/no`→ False**|
|**`user_id`**|`**`int`** 过滤；解析失败 **静默跳过**|
|**`error_code`**|`**`strip` 精确匹配 **`error_code` 字段|
|**`start_date`/`end_date`**|`**`YYYY-MM-DD`** → **`created_at__date` 含边界**|**`_parse_date_yyyymmdd` 容错返回 None**|

与其它报表接口 **共用**该方法，避免出现 **两套日期解析**。修改过滤规则时请 **跑一次 `summary`/`export` Golden Test**。

## 4. 响应信封

```json
{ "success": true, "results": [ { ...row } ] }
```

**非 `code/msg/data`** 风格——SPA **axios 拦截器若统一判 `code==200`** 需 **特例分支 `success`** 键。

单行字段手动组装（节选）：**`id`、`created_at`、`user_id`、`action`、`endpoint`、`success`、`status_code`、`error_code`、`error_message`、`model_used`、`test_type`、`module_id`、`streamed`、`all_covered`、`latency_ms`、`prompt_chars`、`output_chars`、`cases_count`、`meta`**。**`meta`** 为 **`JSONField`**：**可嵌套模型名、temperature、租户**。

## 6. **`created_at` 与时区**

**`TruncDate`、`date__lte`**依赖 **Django `USE_TZ` 与本番 **DB session timezone****。若运维 **夏令时跳变**，**日界统计偏移 1 天**——**`timescale`/`clickhouse`**  offload 时需 **UTC-normalize**。
