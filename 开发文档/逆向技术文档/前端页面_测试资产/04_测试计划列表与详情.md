# 测试计划列表与详情

## 1. 路由

| 路径 | 视图 |
|------|------|
| `/test-plan` | `frontend/src/views/TestPlan/index.vue` |
| `/test-plan/:id` | `frontend/src/views/TestPlan/Detail.vue` |

位于 `MainLayout` 内，访问需登录。

## 2. 列表页（index）

**路径**：`views/TestPlan/index.vue`

核心能力：**新增计划**、**回收站**、**选择模式 + 批量软删/恢复/硬删**、**批量改状态**、**批量复制**、按名称 **`searchKw`** 过滤、`el-table` 行链到 **`/test-plan/${row.id}`**。

**下拉数据**：可能拉取 **`getReleasesApi`**（`project.js`：`/project/releases/`）与 **`getUsersApi`**（`system.js`：`/user/users/`）用于表单/指派类字段。

**计划 API**：`@/api/execution` **`plans`**。

## 3. 详情页（Detail）

**路径**：`views/TestPlan/Detail.vue`

- **`getPlanDetailApi(id)`**、`updatePlanApi`、`deletePlanApi`。
- **`getUsersApi`** 等辅助系统用户列表。

路由参数：**`route.params.id`**。

## 4. Execution API：`plans`（`frontend/src/api/execution.js`）

- `GET /execution/plans/`、`POST`、`PATCH`、`DELETE`、`GET :id`
- `POST batch-delete`、`batch-update`、`batch-copy`
- 回收：`GET recycle`、`restore`、`hard-delete`、`bulk-soft-delete`、`bulk-restore`、`bulk-hard-delete`

## 5. Store / 上下文

无专用 store； **`current_project_id`** 常作为列表请求的 **`project`** 查询参数。

## 6. 权限与守卫

与非系统管理页一致；无 `SYSTEM_ADMIN_ONLY`。**系统管理员前缀**不适用本模块。

## 7. Query 与其它跳转

列表无必选 query。**测试报告详情**会通过 **`plan_link_id`** 类字段链回 **`/test-plan/:id`**（见报告逆向文档）。

## 8. 状态与表格列

前端将 **`plan_status`** 映射为 **`el-tag` 文案/颜色**；列含 **`iteration`、`environment`、`coverage_rate`、`req_count`、`case_count`** 等业务字段，以后端 serializer 为准。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。
