# 接口测试独立页

## 1. 路由与设计意图

| 路径 | name | 组件 |
|------|------|------|
| `/test-case/api` | **`ApiTest`** | `frontend/src/views/ApiTest.vue` |

`ApiTest.vue` **不复制**列表实现，仅用 **`<script setup>`**：

- `provide('testCaseForcedType', 'api')`
- 模板 **单根 `<TestCase />`**`（同名导入 `./TestCase.vue`）

从而在 **路径恒为 `/test-case/api`** 时，强制 **`activeTestCaseType === 'api'`**，避免因 **`route.params.type`** 缺失导致类型推导失败。

## 2. 与 `/test-case/:type` 的等价性

当 `:type=api` 时，行为与本子页面一致；区别在于 **provide 优先级**：子页 **`testCaseForcedType` 优先于 params**。

## 3. SideBar 与菜单

`MainLayout` **「接口测试」**菜单项 **`index="/test-case/api"`**，与用户文档中独立页描述一致。

## 4. 权限与上下文

与普通用例页相同：**需登录**、需 **当前项目 ID**（模块树 / 列表）。无系统管理员专享 gate。

## 5. Store

**无单独 store**；逻辑全在 **`TestCase.vue`** 内（注入 + 共用 composable）。

## 6. API

**完全等同** **`TestCase` 在 `api` 类型下**的全部调用：**`getCasesApi` 带 `test_type=api`**、`run-api`、`preview-run-api`、批量预检、`ai-fill-test-data`、`assistant.suggest-case-fix`，以及 **AI SSE 导入**。

路径前缀仍为 **`/api/testcase/`**、`/api/ai/`**，经 `frontend/src/utils/request.js` BaseURL。

## 7. 关键 UX

- **「调试」**打开 **`ApiExecuteConsoleDialog`**（在 `TestCase` 表格 API 模版内）。
- 列宽约 **720px**（相对功能用例 **480px** 等，见 `TestCase` 类型分支）。

## 8. Query 参数

本子组件**不产生**必选 query。

## 9. 注意

若在 **`TestCase` 的子树重复 provide 同名 key**，会覆盖父级注入——**禁止**在非设计场景下这么做；当前仅 **`ApiTest` 顶层** provide 一次。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。

建议在 DevTools Network 中过滤 **`/api/`**；并与本文档中的模块前缀交叉核对（如 `/testcase/`、`/execution/`、`/defect/`、`/assistant/`、`/sys/`、`/user/`）。

### 11.2 从路由反查视图与 API

1. `frontend/src/router/index.js` 由 path → `component`；  
2. 打开对应 `frontend/src/views/...`，检索 **`from "@/api/`**（或别名 `@/`）；  
3. 若逻辑拆至 `components/` 子组件，亦需对该目录做同样检索。
