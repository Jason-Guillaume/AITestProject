# 测试方案

## 1. 路由与入口

| 路由路径 | 视图 | 说明 |
|----------|------|------|
| `/test-approach` | `frontend/src/views/TestApproach.vue` | `MainLayout` 子路由，需登录 |

侧栏归入测试资产相关菜单链路；无动态 `:id`，仅单列表/卡片工作台形态。

## 2. 权限与守卫

- 需 **`localStorage.token`**；非 `SYSTEM_ADMIN_ONLY`。
- 数据一般按 **`current_project_id`**（或等价请求参数）与后端租户/项目权限过滤，以后端校验为准。

## 3. 关键组件与前端结构

- **`TestApproach.vue`**：卡片网格 + **搜索、回收站模式、批量选择模式、更多筛选 Drawer、智能新增**，与测试设计/计划列表在交互上同属「运营管理」范式。
- 依赖 **`@/api/testcase`** 中 **方案（approaches）** 全集接口（详见下一节）。

## 4. API 调用（`/api/testcase/...`，经 `frontend/src/api/testcase.js`）

**列表与 CRUD**

- `GET /testcase/approaches/` — `getApproachsApi(params)` — 分页/筛选
- `POST /testcase/approaches/` — `createApproachApi`
- `PATCH /testcase/approaches/:id/` — `updateApproachApi`
- `DELETE /testcase/approaches/:id/` — `deleteApproachApi`
- `POST /testcase/approaches/batch-delete/`、`batch-copy/` — 批量

**回收站语义**（与后端统一）

- `GET /testcase/approaches/recycle/`、`POST .../:id/restore/`、`POST .../:id/hard-delete/`
- `bulk-soft-delete`、`bulk-restore`、`bulk-hard-delete` 等系列

**附件图片**（若启用）

- `GET .../:id/images/`、`POST .../:id/images/upload/`（multipart）、`DELETE .../:id/images/:imageId/`

具体请求字段（body/query）以实现代码与 OpenAPI 为准；列表加载函数内通常会带 **`project`** 或页面级过滤器。

## 5. URL 查询参数

- 路由本身**无必选** `:id`/query。

## 6. Pinia / 上下文

通常通过 **顶层项目选择器**写入 `localStorage.current_project_id` 并由 MainLayout/App 同步；页面内筛选器可能读取同一上下文，**不要求**单独的 `stores`。

## 7. 与用户相关的交互要点

1. **回收站**：切换后列表数据源改为 recycle API，批量操作语义与正常列表区分。
2. **选择模式**：卡片角标选择与表格多选类似，批量软删/恢复/硬删。
3. **智能新增**：打开创建向导/对话框后会走 `POST approaches`。

## 8. 异常处理

列表与批量操作一般用 **`ElMessage`/`ElMessageBox`**（以组件内实现为准）；网络错误展示后端 **`detail`** 或 **msg**。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。

3. 若逻辑拆至 `components/` 子组件，亦需对该目录做同样检索。
