# 测试用例多类型主页面

## 1. 路由与子入口

**主组件**：`frontend/src/views/TestCase.vue`。

| 路径 | name | `params` |
|------|------|----------|
| `/test-case` | — | `redirect → /test-case/functional` |
| `/test-case/:type` | `TestCase` | **`:type` 必须为下列之一** |

**合法 `:type`**（代码常量 `TEST_CASE_ROUTE_TYPES`）：

`functional`、`api`、`performance`、`security`、`ui-automation`。

侧栏在 `MainLayout.vue` 为五种类型各自绑定上述路径；**`/test-case/*` 在布局内用 `:key="$route.fullPath"` 强制重挂载**，避免类型切换遗留状态。

**独立包裹页**：`views/ApiTest.vue` 等价于 **`/test-case/api`**，见 **`06_接口测试独立页.md`**。

## 2. `activeTestCaseType` 决策链

```ts
inject('testCaseForcedType') ?? route.params.type
```

- **`ApiTest.vue`**：`provide('testCaseForcedType', 'api')`，与 URL 是否为 `/test-case/api` **解耦**。
- **非法 `:type`**：逻辑内会告警并处理为安全默认（以实现为准）。

列表请求合并 **`test_type`**（或与后端约定的 `testType`/`test_type` 字段）：**`activeTestCaseType`** 驱动筛选、对话框宽度、批量导入、API **execute/run/preview** 等分支。

## 3. 权限

需登录 token；非系统独占路由。**项目必选**：多处提示「请先选择项目」再加载模块树/用例表。

## 4. Pinia / 上下文

- **`useAppContextStore`**：部分子流程读本 store（与脚本/场景模块一致）。
- **`localStorage.current_project_id`**：模块树、用例列表核心过滤。

## 5. API 概要（`@/api/testcase.js`）

与用例密集相关：**`cases`、`modules`、`steps`**；含 **回收站** `recycle`、`restore`、`hard-delete`、批量；**接口类**专有：

- `POST /testcase/cases/:id/execute-api/`、`.../run-api/`、`preview-run-api/`
- `POST batch-preview-run-api/`、`batch-execute/`

**AI 协同**：

- **`@/api/assistant`**：`suggestCaseFixFromExecutionLogApi` → **`POST /api/ai/suggest-case-fix/`**
- **`TestCase.vue`** 内还通过 **原生 fetch** 调 **`/api/ai/generate-cases-stream/`**（流式导入，参见 `assistant.js` 注释）

其它：`apply-ai-suggested-steps`、`ai-import`、`versions`、`execution-logs` 等均以 `api/testcase.js` 导出为准。

## 6. 关键 UI 块

1. **左侧模块树**：`el-tree`、`getModulesApi`；宽窄屏 docking。
2. **主表**：类型列模版区分 **API（调试打开 `ApiExecuteConsoleDialog`）**、性能、安全、UI 自动化。
3. **导入/导出/批量执行/AI**：按 `activeTestCaseType` 显隐。
4. **步骤编辑、版本、执行日志**：单用例侧边或弹层。

## 7. Query 与其它路由跳转

主列表一般**无必选 query**；部分分享或深链若以 query 传入需读代码。**`route.params`** 为类型主轴。

## 8. 异常

接口失败、`ElMessage` 展示 **`response.data.detail|msg`**；SSE/流式需处理 **401** 时在全局 axios 拦截器统一登出。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。



## 11. 调试与追踪建议

### 11.1 浏览器 Network 过滤

建议在 DevTools Network 中过滤 **`/api/`**；并与本文档中的模块前缀交叉核对（如 `/testcase/`、`/execution/`、`/defect/`、`/assistant/`、`/sys/`、`/user/`）。

### 11.2 从路由反查视图与 API

1. `frontend/src/router/index.js` 由 path → `component`；  
2. 打开对应 `frontend/src/views/...`，检索 **`from "@/api/`**（或别名 `@/`）；  
3. 若逻辑拆至 `components/` 子组件，亦需对该目录做同样检索。
