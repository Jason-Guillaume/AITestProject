# 测试设计列表与详情

## 1. 路由映射

| 路径 | 组件 | name |
|------|------|------|
| `/test-design` | `views/TestDesign/index.vue` | 默认子路由 |
| `/test-design/:id` | `views/TestDesign/Detail.vue` | 动态 `:id` 设计详情 |

两者均在 **`MainLayout`** 下，`meta.public: false`。

## 2. 列表页（index）能力摘要

**文件**：`frontend/src/views/TestDesign/index.vue`

- **工具栏**：新增测试设计弹窗、`回收站`、`选择` 模式及相关批量：**软删、恢复、硬删、批量归档、批量评审状态**。
- **搜索**：前端 `searchKw` 过滤 `filteredList`（或对后端分页结果本地过滤）。
- **表格行**：名称列跳转 **`router.push(\`/test-design/${row.id}\`)`**。
- **API**：主要来自 **`@/api/testcase`** 的 **`designs`** 资源族（见下一节）。

## 3. 详情页（Detail）能力摘要

**文件**：`frontend/src/views/TestDesign/Detail.vue`

- `:id` 来自 **`route.params.id`**，`getDesignDetailApi(id)` 拉详情。
- **Tab**：「设计测试内容」「评审测试设计」「执行测试用例」（部分占位 `el-empty`）。
- 工具条含占位 AI 控件、跳转测试计划意图等；删除走 **`deleteDesignApi`**。

## 4. API 一览（designs，`api/testcase.js`）

- `GET /testcase/designs/` — `getDesignsApi`
- `POST /testcase/designs/` — `createDesignApi`
- `PATCH /testcase/designs/:id/` — `updateDesignApi`
- `DELETE /testcase/designs/:id/` — `deleteDesignApi`
- `GET /testcase/designs/:id/` — `getDesignDetailApi`
- `POST batch-delete`、`batch-update`

**回收站**

- `GET /testcase/designs/recycle/`、`restore`、`hard-delete`、`bulk-soft-delete`、`bulk-restore`、`bulk-hard-delete`

列表/回收切换与测试计划交互模式一致。

## 5. 查询参数（query）

- 列表：**无全局约定 query**，筛选在组件 state。
- 详情：**仅用路径参数 `:id`**。

## 6. Store

**无强制 Pinia**；项目维度由顶部栏 `current_project_id` 注入到请求 params（若在 `load()` 中带 `project`）。

## 7. 权限

普通登录用户；无 `SYSTEM_ADMIN_ONLY`。**组织/租户隔离由后端 enforcement**。

## 8. UI 与设计树

详情「设计测试内容」使用 **`el-tree`** 展示 **`treeData`**（类型 tag + 加减编辑图标）；部分内容仍为静态/占位，演进时需对照后端结构设计实体字段。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。
