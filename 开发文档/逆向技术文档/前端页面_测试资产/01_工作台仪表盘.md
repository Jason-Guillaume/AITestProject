# 工作台仪表盘

## 1. 路由与入口

| 路由路径 | Vue 视图 | `router` meta |
|----------|-----------|---------------|
| `/dashboard` | `frontend/src/views/Dashboard.vue` | 挂载在 `MainLayout` 子路由，`meta.public: false`，需登录 |

侧栏：**「工作台 → 仪表盘」** 对应 `/dashboard`（见 `layouts/MainLayout.vue`）。全局根路径 `/` 会重定向到 `/dashboard`。登录后若在 `/login` 且已有 token，会被重定向到 `/dashboard`。

## 2. 权限与守卫

- 与普通业务页一致：无 `meta.public`，**必须** `localStorage.token`（见 `frontend/src/router/index.js`）。
- **无**额外 `SYSTEM_ADMIN_ONLY`；所有已登录用户可访问。
- 流水线 `/pipelines` 特例与本页无关。

## 3. 关键依赖与前端文件

| 角色 | 文件 |
|------|------|
| 页面主体 | `views/Dashboard.vue` |
| Summary API | `api/dashboard.js` → `getDashboardSummaryApi` |
| SSE URL 构造 | `getDashboardStreamUrl`（附带 `token` 及可选额外 query） |
| 轮询 | `composables/useRealtimePolling`（`intervalMs: 5000`） |
| SSE 触发刷新 | `composables/useSseRefreshTrigger`（`reconnectMs: 3000`） |

轮询在遇到 **SSE 已连接**（`shouldSkip: () => streamConnected.value`）时跳过，减轻重复请求。监听自定义事件：**`app:current-project-changed`**、**`app:current-environment-changed`**（由 `stores/appContextStore.js` 在切换项目/环境时 `dispatch`）。

## 4. API 与时序（摘要）

**首屏与刷新**

- `GET /api/execution/dashboard/summary/`（`getDashboardSummaryApi()`）→ 后端聚合 **统计卡片、折线（周/月）、饼图、柱状图、`activities`** 等。
- 成功后在 `Dashboard.vue` 中映射 `statCards`、渲染三组 ECharts，并更新 **`activities`**、`lastUpdatedAt`。

**实时通道**

- `getDashboardStreamUrl()` → 拼装 **`/api/execution/dashboard/stream?token=...`**（可加其它参数）。
- SSE 联通后 `streamConnected=true`，右上角「最近测试任务」文案显示 **实时通道已连接**；断开则 **实时通道重连中**。
- 收到服务端事件或通过连接状态变化会触发 **`loadDashboardData`** 等非首屏报错模式刷新。

注：工作台摘要请求**当前实现未**向 `summary` 传 `project_id`；切换项目主要靠轮询/SSE + 监听事件触发重新拉取（以后端聚合范围为准）。

## 5. 关键 UI 与子区域

1. **统计卡片**：四套指标（总数、今日执行、未解决缺陷、通过率等字段由后端填充），含较前日箭头与渐变进度条样式。
2. **测试执行情况**：折线图 Tab **周 / 月** 切换数据源 `dashboardData.lineChart.week|month`。
3. **AI 测试分析**：饼图 + 图例来自固定主题枚举（安全、合规等）。
4. **缺陷统计分析**：柱状图。
5. **最近测试任务**：列表 + 空状态「暂无最近任务数据……」占位。

图表在 `onBeforeUnmount` 中 `dispose` ECharts 实例；窗口 resize 监听用于 `resize`。

## 6. 查询参数与环境

无路径参数；SSE URL 仅用 **query** 拼装 token。上下文依赖：**`localStorage.token`**。

## 7. Store 与其它模块

-dashboard 视图本身**不直接使用 Pinia**，项目/环境变更通过 **`window.dispatchEvent(CustomEvent)`** 驱动。
- MainLayout 与全局 header 仍会维护 `current_project_id`（与 quality 模块不同）。

## 8. 异常与降级

- `loadDashboardData` 在静默刷新（`showError=false`）时出错只打控制台；显式报错路径可弹出「获取首页统计数据失败」（以实际调用为准）。
- SSE 失败后由 `useSseRefreshTrigger` 按 `reconnectMs` 重连；在此期间轮询仍可补充数据（在未连接 SSE 时）。

## 10. 与前端的通用约定（跨页）

### 10.1 `request` 与鉴权

- 业务接口通过 `frontend/src/utils/request.js`（或等价封装）发出，默认附带 **`Authorization: Bearer <localStorage.token>`**（以实现为准）。
- **登录 / 注册 / 图形验证码** 使用 `frontend/src/api/auth.js` 的独立 `axios`，避免与业务拦截器循环依赖。
- **401**：一般在响应拦截器中清理 token、跳转登录；个别页面（如流水线控制台）静默失败时需结合后端是否真的允许匿名访问。

### 10.2 项目与环境上下文

- 全局项目与环境 ID 常存于：`localStorage.current_project_id`、`localStorage.current_test_environment_id`，并由 **`stores/appContextStore.js`** 在变更时派发 **`CustomEvent`**：
  - `app:current-project-changed`
  - `app:current-environment-changed`
- 依赖这些事件的页面常与 **轮询 / SSE** 组合，以降低「切换项目却仍显示旧数据」的概率。

### 10.3 路由器守卫摘要

摘自 `frontend/src/router/index.js`：

1. **以 `/pipelines` 开头**：先于 token 校验直接放行。
2. **其余非 `meta.public` 且无 token**：重定向 **`/login?redirect=<fullPath>`**。
3. **访问 `/login` 且本地已有 token**：重定向 **`/dashboard`**。
4. **路径匹配系统管理员前缀表且本地 `is_system_admin !== '1'`**：重定向 **`/system/message`**（普通用户可被引导至消息设置页）。
