# 流水线异步任务与装饰器

## 文档目的

说明 **`Pipeline`** / **`PipelineLog`** 模型、**`PipelineViewSet`** 的 **`run`** 与 **`logs`** 接口，以及 Celery 任务 **`run_pipeline_task`** 如何将子进程输出写入日志表；另记 **`project/decorators.py`** 中 **`ensure_memory`** 供低内存环境选用（**当前 `tasks.py` 未引用**）。

---

## 1. 模型

### 1.1 `Pipeline`（表 `pipeline`）

| 字段 | 说明 |
|------|------|
| `name` | 流水线名称 |
| `repo_url` | 仓库 URL |
| `status` | `0` Pending / `1` Running / `2` Success / `3` Fail |

继承 **`BaseModel`**（含软删等）。

### 1.2 `PipelineLog`（表 `pipeline_log`）

| 字段 | 说明 |
|------|------|
| `pipeline` | FK → `Pipeline`，`related_name="logs"` |
| `log_text` | 单行或多行文本（任务内会加 `[stdout]`/`[stderr]` 前缀） |
| `timestamp` | `auto_now_add` |

**非** `BaseModel` — 无软删；删除流水线会 **CASCADE** 删日志。

## 2. `PipelineViewSet`

**前缀**：`/api/project/pipelines/`  
**序列化器**：`PipelineSerializer`；解析器含 multipart（与项目其它接口一致）。

### 2.1 `POST .../run/`

- Body：`{"command": "<shell 字符串>"}`，缺省时任务内会用占位 echo。
- **`run_pipeline_task.delay(pipeline.id, command)`** 返回 **`task_id`**（Celery AsyncResult id）与 **`status: queued`**。

### 2.2 `GET .../logs/`

| 参数 | 默认 | 范围 |
|------|------|------|
| `after_id` | `0` | 只取 **`id > after_pk`** 的增量 |
| `limit` | 500 | 1～2000 |

响应：`pipeline_status`（从 DB **refresh**）、**`items`**（`PipelineLogSerializer` 列表）、**`last_log_id`**（无新行则仍为 `after_id`）。

与前端 **`PipelineConsole`** 轮询契约一致（见 **`前端可复用组件/01`**）。

## 3. Celery：`run_pipeline_task`

**文件**：`project/tasks.py`  
**装饰器**：`@app.task(bind=True, name="run_pipeline_task")`（`AITestProduct.celery.app`）

流程摘要：

1. 取 **`Pipeline`**，不存在则 return。
2. **`status=RUNNING`** 保存。
3. **`subprocess.Popen(..., shell=True, stdout=PIPE, stderr=PIPE, text=True)`**；两路读线程写入 **`queue.Queue`**，主循环 **`_drain_queue_to_db`** 批量 **`PipelineLog.objects.create`**。
4. 进程结束后根据 **return_code==0** 置 **Success** 否则 **Fail**。
5. 异常路径：写一条日志文本为异常字符串，**`status=FAIL`**；`finally` 中 **terminate/kill** 仍在运行的子进程。

**安全提示**：`shell=True` + 用户传入 `command` 存在 **命令注入** 风险；仅应对可信管理员开放 UI。

## 4. `project/decorators.py`：`ensure_memory`

- 装饰器在运行被包装函数前循环检测 **可用内存 MiB**（优先 **psutil**；Unix 可 `sysconf`；**无 psutil 的 Windows 回退为假定内存充足**）。
- 低于 **`threshold_mb`**（默认 500）则 **sleep `check_interval` 秒** 再测，并 **`print`** 提示。
- **可选**：用于包装重任务；**`run_pipeline_task` 当前未使用**，若集成需避免与 Celery worker 阻塞策略冲突。
