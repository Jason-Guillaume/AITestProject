# 测试项目视图集

## 文档目的

说明 **`TestProjectViewSet`**（`/api/project/projects/`）的标准 CRUD、**回收站/恢复/硬删**、**批量软删/恢复/硬删/改状态** 及 **`export.csv`** 导出。模型 **`TestProject`**（表 `test_project`）含父子项目、成员 M2M、封面图与进度字段。

## 1. 路由与基类

- **注册**：`project/urls.py` → `router.register(r"projects", TestProjectViewSet)`
- **完整前缀**：`/api/project/projects/`（根路由见 **`工程根与横切/01`**）
- **基类**：`BaseModelViewSet` — 默认列表 **软删过滤**、成员数据范围、**`perform_destroy` 软删** + 审计

## 2. 模型要点（`project.models.TestProject`）

| 字段 | 说明 |
|------|------|
| `project_name` / `description` | 名称与描述 |
| `parent` | 自关联 FK，`PROTECT`；**`clean()`** 防自指与环 |
| `icon` | `ImageField`，`upload_to=project_covers/` |
| `project_status` | `1` 进行中 / `2` 已完成 |
| `progress` | 0～100 |
| `members` | M2M → `user.User`，`related_name="projects"` |

**`update_progress`**：按 **`execution.TestPlan`** 中 `version` 关联本项目且 `plan_status=3`（完成）占比写回 `progress`（供其它模块调用）。

## 4. 自定义 `@action` 一览

| url_path | 方法 | 说明 |
|----------|------|------|
| `recycle/` | GET | **`is_deleted=True`** 列表；`?q=` 搜名称/描述/父名称 |
| `<pk>/restore/` | POST | 软删恢复 |
| `<pk>/hard-delete/` | POST | **物理删除**单条（仅 `is_deleted=True`）；写审计 **`ACTION_DELETE`** |
| `bulk-soft-delete/` | POST | Body `{ "ids": [] }` |
| `bulk-restore/` | POST | 批量恢复 |
| `bulk-hard-delete/` | POST | 逐条 `delete()` + 每条审计 |
| `bulk-status/` | POST | Body `{ "ids":[], "project_status":1|2 }`；完成时 **`progress=100`** |
| `export.csv` | GET | 流式 CSV，见下节 |

---

## 5. `export.csv`

**查询参数**：

- **`mode`**：`filtered`（默认）| `selected`
- **`q`**：`filtered` 模式下关键词（名称/描述/父名 `icontains`）
- **`ids`**：`selected` 模式下逗号分隔 id 列表（必填）
- **`include_deleted`**：`0|1`，默认 0

**响应**：`StreamingHttpResponse`，`record_export_audit` 带 `mode`、`include_deleted`、`q`、`ids_count`。
