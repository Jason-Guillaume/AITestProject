# 智能填充测试数据

## 文档目的与读者

说明 **`POST /api/testcase/ai-fill-test-data/`** 的请求约束、LLM 提示词意图、成功/失败响应形态，以及与本平台「模型连接配置」的关系。适合对接用例编辑器、排查 400「模型输出非合法 JSON」等问题。

## 1. 功能概述

根据 **字段元数据列表**（名称、类型、可选描述），调用 **`ai_fill_test_data`**（`testcase/services/ai_openai.py`）经 **`chat_completion`** 生成 **一条 JSON 对象**，键与字段 `name` 对齐，值符合类型与常见业务语义（如中国大陆手机号 11 位）。**不落库**；前端将 `data` 回填表单后再走正常用例 **PATCH/POST**。

## 2. HTTP 接口

| 项 | 值 |
|----|-----|
| URL | **`POST /api/testcase/ai-fill-test-data/`** |
| View | `AiFillTestDataAPIView`（`testcase/views.py`） |
| 认证 | 继承全局 DRF 配置（若路由在需登录的 namespace 下，则须带 Token） |

### 2.1 请求体（JSON）

| 字段 | 必填 | 说明 |
|------|------|------|
| **`fields`** | **是** | 非空数组；元素建议为对象，至少含 **`name`**、**`type`**；可选 **`description`** |
| `api_key` | 否 | 透传给 `chat_completion`；用于 **单次请求覆盖** 默认模型配置 |
| `base_url` / `api_base_url` | 否 | 二者等价取其一；OpenAI 兼容网关地址 |
| `model` | 否 | 模型名覆盖 |

**校验失败**：缺少 `fields` → **`{"msg": "缺少 fields"}`** 与 **400**。

### 2.2 成功响应

```json
{ "data": { "phone": "13800138000", "...": "..." } }
```

`data` 为 **单个 JSON 对象**（由服务端解析 LLM 输出后得到）。

### 2.3 失败响应

统一形态 **`{"msg": "<原因>"}`**，HTTP **400**。典型原因：

| `msg` 含义来源 | 说明 |
|----------------|------|
| `fields 须为非空数组` | `ai_fill_test_data` 入参校验 |
| `chat_completion` 错误 | 网络、鉴权、模型不可用等（以 `ai_openai` 实现为准） |
| `模型输出非合法 JSON` | `parse_llm_json` 失败，**msg 内可能含 `raw=` 截断预览**（防日志爆炸） |
| `模型应返回 JSON 对象` | 解析结果是数组等非 dict |
