# 字段抽取建议

## 文档目的与读者

说明 **`POST /api/testcase/suggest-extractions/`** 如何将 **任意 JSON** 结构递归扫描为 **变量提取建议列表**（含 `json_path`、`var_name`、`reason`），以及过滤规则。适合接口用例「从响应 body 一键生成提取规则」类功能。

---

## 1. 功能概述

**纯算法、无 LLM、无 DB**。`SuggestExtractionsAPIView` 将请求体归一成 `data`，调用 **`suggest_extractions(data)`**（`testcase/services/variable_runtime.py`），返回 **`{"suggestions": [...]}`**。

---

## 2. HTTP 接口

| 项 | 值 |
|----|-----|
| URL | **`POST /api/testcase/suggest-extractions/`** |
| View | `SuggestExtractionsAPIView` |

### 2.1 Body 兼容形态

解析顺序（`request.data` 为 `dict` 时）：

1. 若含 **`json_data`** → 取其值作为根数据；
2. 否则若含 **`body`** → 取其值；
3. 否则 **把整个 dict 当作根数据**（允许直接 POST `{"token":"..."}` 这种扁平 JSON）。

非 dict 的 `request.data`（少见）则 **整体作为 `data`**。

**400**：`data is None` 时返回 **`{"msg": "请提供 json_data 或 body"}`**。

### 2.2 成功响应

```json
{
  "suggestions": [
    {
      "field_name": "token",
      "json_path": "$.data.token",
      "reason": "键名命中关键词 token",
      "var_name": "token",
      "source": "body",
      "expression": "$.data.token"
    }
  ]
}
```

字段含义以 **`suggest_extractions`** 实现为准（见 §3）。

---

## 3. 算法摘要：`suggest_extractions`

### 3.1 遍历

- 自 **`$`** 起 DFS：对象拼 **`$.child`**，数组拼 **`$[i]`**。
- **仅叶子**（非 dict、非 list）参与匹配；对象键名作为 **`key_name`** 传入。

### 3.2 命中规则 `_match_business_signal`

- **关键词子串**（`key_name` 转小写后）：`token`, `auth`, `sid`, `id`, `uuid`, `orderno`, `code`, `sn` 等（见源码 `_KEYWORD_HINTS`）。
- **字符串值**：长度 **>20** 视为疑似令牌；或匹配 **UUID** 正则。

### 3.3 过滤 `_should_filter_common_status`

避免把通用状态字段当业务变量，例如 **`code:200`**、**`msg:"ok"`**、**`status:true`** 等（`_COMMON_STATUS_FILTERS`）。

### 3.4 去重

同一 **`json_path`** 只保留一条（`seen_paths`）。

### 3.5 变量名 `_to_var_name`

由键名或路径片段生成 **`var_name`**，供后续 **`VariableExtractor`** 或前端表单使用；兜底 **`extracted_var`**。

## 4. 限制与边界

| 情况 | 行为 |
|------|------|
| 大 JSON | 全量在内存遍历；极端大 body 注意 worker 内存 |
| 嵌套数组 | 路径带 `[idx]`，前端若只支持简化 JSONPath 需转换 |
| Header/Cookie | **不在此接口分析范围**；仅 body 结构 |
