# 开放规范与交互文档及命令行导入

## 文档目的与读者

说明 **`POST /api/testcase/import-api-spec/`** 从 **Swagger/OpenAPI JSON** 或 **cURL** 文本 **批量创建 API 用例** 的完整链路：**权限 → LLM 解析 → 门禁归一化 → ORM 写入**。适合评估导入条数上限、403 场景与 **`gate_ai_api_cases`** 报错结构。

---

## 1. HTTP 接口总览

| 项 | 值 |
|----|-----|
| URL | **`POST /api/testcase/import-api-spec/`** |
| View | `ApiImportFromSpecAPIView`（`testcase/views.py`，约 L1886 起） |

---

## 2. 请求体字段

| 字段 | 必填 | 说明 |
|------|------|------|
| **`source_type`** / **`type`** | 逻辑必填 | **`swagger` / `openapi` / `curl`**（大小写不敏感，以 `ai_import_api_cases` 为准） |
| **`content`** / **`text`** | 必填（非空串） | 规范 JSON 文本或 cURL 多行文本 |
| **`module`** / **`module_id`** | **必填** | 目标 **`TestModule` 主键**，须为 **整数** |

可选（透传 LLM）：

- `api_key`、`base_url`/`api_base_url`、`model`

---

## 3. 权限与业务前置校验

### 3.1 模块存在性

```python
module = TestModule.objects.filter(pk=mid, is_deleted=False).first()
```

不存在 → **`404`**，`{"msg": "模块不存在"}`。

### 3.2 写权限 `_user_can_write_module_cases`

满足任一视为可写：

- 超级用户 / staff / **`user.is_system_admin`**
- **项目创建者**（`proj.creator_id == user.pk`）
- **项目成员**（`proj.members` 含当前用户）

否则 → **`403`**，`{"msg": "无权在该模块下创建用例"}`。

### 3.3 模块测试类型

```python
if module.test_type != TEST_CASE_TYPE_API:
    return 400 {"msg": "目标模块的测试类型须为 API 测试"}
```

---

## 4. LLM 解析：`ai_import_api_cases`

实现：`testcase/services/ai_openai.py`。

| 要点 | 说明 |
|------|------|
| 支持类型 | `swagger` / `openapi` / `curl` |
| 内容长度 | 传入模型前整体 strip；**超过 120000 字符截断**（带 `…(truncated)` 提示） |
| 模型输出 | **必须是 JSON 数组**；元素字段含 **`case_name`、`api_url`、`api_method`、`api_headers`、`api_body`、`api_expected_status`**（部分可选，由提示词约束） |
| 失败 | 返回 `(None, err_msg)` → API **400** `{"msg": err_msg}` |

常见 **`msg`**：`source_type 须为 swagger / openapi / curl`、`content 不能为空`、模型输出非 JSON 数组等。

---

## 5. 门禁：`gate_ai_api_cases`

**文件**：`testcase/services/ai_case_gate.py`。

- 对 **`items`** 逐条 **`validate_and_normalize_ai_api_case`**。
- 返回 **`(gated_list, errors)`**，其中 **`errors`** 元素形如 **`{"index": i, "error": "...", "raw": {...}}`**（`raw` 仅含部分键便于排障）。

**视图行为**：**仅对通过门禁的 `gated` 建用例**；错误项 **不中断** 整个请求（除非你想改为「有 error 则 400」——当前不是）。

---

## 6. 持久化与 `api_source_curl`

对每条 **`item in gated`**：

1. 组装 **`api_payload`**：`api_url`、`api_method`、`api_headers`、`api_body`、`api_expected_status`。
2. 若 **`source_type` 为 `curl`**：把原始 `content` **strip 后截断至 120000** 写入 **`api_source_curl`**，便于日后还原/审计。
3. **`base` 字段**：`module`、`case_name`、`test_type=API`、`level=P2`、`is_valid=True`，登录用户填 **`creator`/`updater`**。
4. 调用 **`create_typed_case(TEST_CASE_TYPE_API, base, api_payload, {}, {}, {})`** 创建多表继承下的 **`TestCase` + `ApiTestCase`**。

### 6.1 响应形状

与 `testcase/views.py` 中 `return Response` 一致：

```json
{
  "created": 12,
  "ids": [101, 102],
  "errors": [
    {
      "index": 3,
      "error": "...",
      "raw": {"case_name": "...", "api_url": "...", "api_method": "GET"}
    }
  ]
}
```

键名固定为 **`created`**（整数）、**`ids`**（新建用例主键列表）、**`errors`**（门禁或后续步骤错误列表，可为空数组）。

---

## 7. 与「命令行导入」的关系

本 View **仅 HTTP**。若仓库另有 **管理命令 / CI** 导入 OpenAPI，应复用 **`ai_import_api_cases` + `gate_ai_api_cases` + `create_typed_case`** 三件套以保持行为一致；本文档标题中的「命令行」指这种 **工程内复用路径**，非本 View 自带 CLI。

---

## 8. 显式与静默

| 显式 | 静默 |
|------|------|
| 新建用例 ID 列表、条数、门禁错误索引 | LLM 原始长文本 **默认不落库**（curl 源除外截断存储） |
| `errors` 中 `raw` 小片段 | 未通过门禁的整条 **不插入** |
