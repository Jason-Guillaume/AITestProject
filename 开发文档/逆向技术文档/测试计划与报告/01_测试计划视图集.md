# 测试计划视图集

## 文档目的与读者

拆解 **`TestPlanViewSet`**（`execution/views.py`）：标准 CRUD、**回收站双模式**、**批量复制/软删/更新**、**恢复与硬删**、以及与 **`TestPlanSerializer`** / **`TestPlan`** 模型的字段对应。适合改权限、对齐前端批量接口、或排查「回收站列表为空」。

---

## 1. 路由与类

| 项 | 值 |
|----|-----|
| 前缀 | **`/api/execution/plans/`**（`AITestProduct/urls.py` → `path("api/execution/", include("execution.urls"))`） |
| Router | `execution/urls.py`：`router.register(r"plans", TestPlanViewSet)` |
| queryset 默认 | `TestPlan.objects.all().prefetch_related("testers")` |

---

## 2. `get_queryset` 与回收站

与多数继承 **`BaseModelViewSet`** 的资源一致：

- **正常列表**：`is_deleted=False`；若 **`enable_data_scope`** 且用户为 **普通成员**，走 **`_apply_member_scope`**。  
  `TestPlan` 含 **`version`**（FK → **`ReleasePlan`**），命中策略为 **`version__project__members` 或 `creator`**（见 `common/views.py`）。

- **回收站**：Query 参数 **`recycle=1`** 或 **`is_deleted=true`**（大小写不敏感，值为 `1/true/yes`）时：
  - 基 queryset 改为 **`TestPlan.objects.filter(is_deleted=True).prefetch_related("testers")`**；
  - 非管理员再 **`_apply_member_scope`**；
  - 排序 **`-update_time, -id`**。

**快捷入口**：`GET /api/execution/plans/recycle/` 在动作内 **临时注入** `recycle=1` 后转调 **`list`**，等价于带 query 的列表。

---

## 3. 标准 REST 与 Serializer

- **Serializer**：`TestPlanSerializer`（`execution/serialize.py`）。
- **特点**：
  - **`testers`**：`PrimaryKeyRelatedField(many=True)`，queryset 为 **`User.objects.filter(is_deleted=False)`**；`create`/`update` 时 **`pop` 后 `set`**。
  - **`testers_display`**：只读，拼接 `real_name` 或 `username`。
  - **`read_only_fields`**：`creator/updater/create_time/update_time`（创建走 `perform_create` 填 creator）。

## 4. `@action` 一览（`url_path` 均为相对 `plans/`）

| url_path | 方法 | 作用 |
|----------|------|------|
| **`batch-copy`** | POST | Body `{ "ids": number[], "name_suffix?": string }`；复制计划；新计划 **`plan_status` 重置为字段 default**（通常为 **未开始**）；**`pass_rate` 置 0**；复制 **`testers` M2M** |
| **`batch-delete`** | POST | 批量 **软删**（`is_deleted=True`） |
| **`batch-update`** | POST | Body `{ "ids", "patch" }`；`patch` 键 **白名单**：`plan_name, iteration, version, environment, req_count, case_count, coverage_rate, plan_status, pass_rate, defect_count, testers, start_date, end_date` |
| **`recycle`** | GET | 回收站列表 |
| **`restore`** | POST | 单条恢复（`pk` 须在软删集合） |
| **`bulk-soft-delete`** | POST | 与 `batch-delete` 语义接近（实现路径略不同，均以当前 `get_queryset` 为准） |
| **`bulk-restore`** | POST | 批量恢复；对 **`TestPlan.objects.filter(is_deleted=True)`** 再套成员范围 |
| **`hard-delete`** | POST | **物理删除** + `record_audit_event(DELETE)` |
| **`bulk-hard-delete`** | POST | 批量物理删 + 逐条审计（失败收集 `errors`） |

### 4.1 `_parse_id_list` 规则

- `ids` **必填**、须为 **数组**；元素转 **`int`**，非法跳过；**≤0 丢弃**；去重且 **保序**。
- 全空 → **`ValidationError`**（业务包装：`msg`/`code`/`data`）。

## 5. 数据模型要点（`execution/models.py`）

| 字段 | 说明 |
|------|------|
| `plan_name` | 名称 |
| `iteration` | 可空字符串 |
| **`version`** | FK **`ReleasePlan`**，`PROTECT` |
| **`environment`** | **CharField** 名称快照，**非** `TestEnvironment` FK |
| 统计字段 | `req_count`、`case_count`、`coverage_rate`、`pass_rate`、`defect_count` |
| **`plan_status`** | **1 未开始 / 2 进行中 / 3 已完成** |
| **`testers`** | M2M `User` |
| `start_date` / `end_date` | 可空时间 |

---

## 6. 与 `TestPlanService` 的关系

`execution/services/test_plan_service.py` 提供 **`create_test_plan` / `update_test_plan`** 等：含 **名称唯一**、必填校验、**`tester_ids`** 批量绑定。  
**DRF ViewSet 主路径**仍可直接用 Serializer 落库；Service 供 **管理命令、其它模块、事务封装** 复用（以仓库引用为准）。

---

## 7. 排障

| 现象 | 方向 |
|------|------|
| 列表缺某计划 | 用户不在 **`version.project.members`** 且非创建者；或计划已软删 |
| `batch-update` 无效果 | `patch` 键不在白名单或 Serializer 校验失败（见返回 `errors`） |
| 硬删失败 | `PROTECT`：若版本仍被其它实体强依赖，可能触发 DB 错误（以实际关联为准） |
