# 测试报告视图集与追溯载荷

## 文档目的与读者

说明 **`TestReportViewSet`** 的 REST 与批量动作，重点 **`trace_id` / `execution_log_id` / request_payload / response_payload`** 如何与 **接口执行** 闭环；并展开 **`batch-refresh-from-plan`** 与 **`batch-execute`**（Celery）契约。适合对接报告详情页、执行状态轮询与 Celery 运维。

| 项 | 值 |

## 3. 核心 `@action`

### 3.1 `POST .../reports/batch-refresh-from-plan/`

- Body：`{ "ids": number[] }`。
- 对每个 **未软删** 报告：`select_related("plan")`，从 **`plan`** 同步 **`environment`、`req_count`、`case_count`、`coverage_rate`、`pass_rate`、`defect_count`** 到报告，再走 **`perform_update`**（含审计）。
- 缺 **`plan`**：该 id 记入 **`errors`**，不中断其它 id。

### 3.2 `POST .../reports/batch-execute/`

- Body：`{ "ids": number[], "environment_id?": number }`。
- 对每个报告 id：`run_test_report_execute.delay(rid, environment_id=environment_id)`（`execution/tasks.py`）。
- 响应 **`triggered_items`** 含 **`celery_task_id`**；并尽力 **`record_execute_audit`**（失败吞掉）。

**任务语义摘要**（详见 tasks 内文档字符串）：

1. 取 **`report.plan.version`** 下 **`ReleasePlanTestCase`** 关联的用例 id；
2. 过滤 **`TestCase`** 中 **`test_type=API`**；
3. **`group(run_report_api_case.s(...))`** 并行跑每条；
4. 汇总 **通过/失败**，计算 **`pass_rate`**，并回写 **`trace_id` / `execution_log_id` / 快照`**（以任务内实现为准）。

**跳过原因示例**（任务返回 dict）：`report_not_found`、`plan_or_release_missing`、`no_cases`、`no_api_cases`。

### 3.3 其它批量与回收站

与计划侧命名一致：`batch-delete`、`batch-update`、`batch-copy`、`recycle`、`restore`、`bulk-soft-delete`、`bulk-restore`、`hard-delete`、`bulk-hard-delete`。  
**`batch-copy`**：新报告 **`create_method=1`**，**清空** **`trace_id`**，**`execution_log_id=None`**，**`request_payload/response_payload=None`**，名称加后缀（默认 **`（复制）`**）。

## 4. Serializer

**`TestReportSerializer`**：`fields = "__all__"`，`creator_name` 只读；**`create/update` 无 M2M 特殊逻辑**（与计划不同）。

---

## 5. 与执行引擎 `HealthChecker` 的关系

单条 API 用例执行仍走 **`APIExecutor.execute`**，内含 **任务前 `check_before_task`**（写 **`environment_health_check`** 表，失败可邮件 + 中止）。  
**报告批量执行**在子任务 **`run_report_api_case`** 内是否再次触发，以 **`execution/tasks.py`** 实现为准（若每条 case 独立 `APIExecutor`，则每条前都会健康检查——须结合负载评估）。

---

## 6. 排障

| 现象 | 方向 |
|------|------|
| `batch-execute` 立即成功但报告无变化 | Celery worker 未消费 / 任务 `skipped`（无 API 用例） |
| `triggered_items` 有 id 但一直旧数据 | 子任务异常；查 worker 日志与 **`ExecutionLog`** |
| 追溯字段为空 | 尚未有成功执行写回；或复制报告被清空引用 |
