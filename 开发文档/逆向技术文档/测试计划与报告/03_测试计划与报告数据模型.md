# 测试计划与报告数据模型

## 文档目的与读者

从 **ER、删除策略、权限链路、统计口径** 四个维度汇总 **`test_plan`** 与 **`test_report`** 两张表，便于与 ViewSet 文档及项目进度统计对照。

---

## 1. ER 关系

```
TestProject (implicit via ReleasePlan)
    └── ReleasePlan (project.TestProject 侧具体类名以 project 应用为准)
            └── TestPlan (version_id → ReleasePlan, PROTECT)
                    └── TestReport (plan_id → TestPlan, PROTECT)
```

**`TestReport.project`**：冗余 **`TestProject`** FK，**`null=True, SET_NULL`**，注释标明用于 **权限与查询加速**，值通常来自 **`plan.version.project`**。

---

## 2. 表：`test_plan`（`execution.models.TestPlan`）

| 字段 | 说明 |
|------|------|
| `plan_name` | 计划名称 |
| `iteration` | 可空迭代名 |
| **`version_id`** | 关联发布版本；**`PROTECT`** — 版本不可被随意物理删若仍被计划引用 |
| **`environment`** | **字符串**；与 **`TestEnvironment.name`** 或任意标签对齐，**无 DB 级约束** |
| 统计 | `req_count`、`case_count`、`coverage_rate`、`pass_rate`、`defect_count` |
| **`plan_status`** | **1 未开始 / 2 进行中 / 3 已完成** |
| **`testers`** | 与 `User` 多对多 |
| `start_date` / `end_date` | 计划周期 |
| BaseModel | 软删、审计字段等 |

**项目进度**：业务上可用 **`plan_status=3`** 作为「计划完成」口径（参见项目模块文档中 **`TestProject.update_progress`** 若存在）。

---

## 3. 表：`test_report`（`execution.models.TestReport`）

| 字段 | 说明 |
|------|------|
| **`plan_id`** | 所属计划；**`PROTECT`** |
| `report_name` | 报告标题 |
| **`create_method`** | `1` 新建 / `2` 定期 |
| **`environment`** | 字符串快照 |
| 统计 | 与计划侧类似：`req_count`、`case_count`、`coverage_rate`、`pass_rate`、`defect_count` |
| **`start_time` / `end_time`** | **必填**（模型层非空）；创建时若未填，服务层常用 **当前时间** 默认 |
| **`trace_id`** | 执行追踪 |
| **`execution_log_id`** | 指向 **`ExecutionLog.id`**，无外键 |
| **`request_payload` / `response_payload`** | JSON 快照 |
| **`project_id`** | 冗余项目 |

---

## 4. 权限与查询链路（ORM 视角）

| 模型 | `_apply_member_scope` 典型命中 |
|------|----------------------------------|
| `TestPlan` | **`version__project__members`** 或 **`creator`** |
| `TestReport` | **`project__members`** 或 **`creator`** |

管理员（superuser/staff/`is_system_admin`）跳过成员过滤，但仍默认 **`is_deleted=False`**（回收站模式除外）。

---

## 5. 软删与物理删

- **业务删除**：`BaseModelViewSet.perform_destroy` → **`is_deleted=True`**。
- **回收站硬删**：`hard-delete` / `bulk-hard-delete` → **`QuerySet.delete()`** 物理移除；若仍有 **`PROTECT`** 引用，可能抛 **`ProtectedError`**。

---

## 6. 与「环境表」的边界

| 对象 | 类型 | 风险 |
|------|------|------|
| `TestEnvironment` | 独立表 + 双路由 API | 全局唯一 `name`、变量加密 |
| `plan.environment` / `report.environment` | 字符串 | 改名不同步、无 FK 校验 |

执行报告时传 **`environment_id`** 会走 **`TestEnvironment` 主键**（见 `run_test_report_execute`），与字符串字段是 **两条线**。
