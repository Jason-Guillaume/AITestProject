# 用例子类型（功能 / 接口 / 性能 / 安全 / UI 自动化）

逆向梳理 **`TestCase.test_type`** 与 **`testcase/models.py`** 中各 **OneToOne 子类型表**（如接口、性能、安全、界面自动化扩展）之间的对应关系，并解读 **`TestCaseViewSet.get_queryset`** 中 **`select_related`** 策略以降低 N+1 查询；前端与编排层应使用与后端 **完全一致的字符串常量**。

## 1. 枚举定义与数据源

常量位置：**`testcase/models.py`** 文件顶部：

- **`TEST_CASE_TYPE_*`**：各类型 Python 常量，便于源码引用；
- **`TEST_CASE_TYPE_CHOICES`**：Django **`choices`** 元组列表，写入 **`CharField`**。

对外暴露的稳定字符串取值（节选，以代码为准）：

| 取值 | 业务含义 |
|------|----------|
| **`functional`** | 通用功能／手工／无强制扩展子表 |
| **`api`** | 接口测试（REST 请求、断言脚本等） |
| **`performance`** | 性能脚本 / 场景扩展 |
| **`security`** | 安全专项字段扩展 |
| **`ui-automation`** | UI 自动化（含脚本绑定、可选元素库 FK） |

**`TestCase.test_type`**：**`CharField(max_length=32, choices=TEST_CASE_TYPE_CHOICES)`**。  
**`TestModule.test_type`** 与白名单共用同一套值，以保证 **模块树上的类型与列表筛选项对齐**。

## 2. OneToOne 子类型 ORM

各子类型在 **`testcase/models.py`** 中以 **一对一** 方式扩展 **`TestCase`**（具体类名、 **`related_name`** 请参阅模型定义，如 **`apitestcase`、`perftestcase`、`securitytestcase`、`uiautomationtestcase`** 等）。

语义要点：

- **主键侧**：每条子表行至多对应一行 **`TestCase`**；
- **`create_typed_case` 等业务服务**（**`testcase/services/case_subtypes.py`**）在创建时用 **`test_type`** 决定 **是否创建对应子表行**；
- **列表序列化**：**`TestCaseSerializer`**（及嵌套 Serializer）按需 **` SerializerMethodField`** 或嵌套读出子表，未命中子表时使用默认空结构。

快照与回溯：**`_get_subtype_snapshot_data` / `_apply_subtype_snapshot`** 将子表字段打散到快照 JSON，再回放时逆向填充；若类型切换则涉及 **孤儿子表删除** 与新子表创建的规则（以实现代码为准）。

## 3. `TestCaseViewSet.get_queryset` 的 select_related

**目标**：列表混合多类型或在「仅看某一类型」时仍避免对每个用例触发重复 SQL。

源码策略（口述层次，以实现为准）：

- 当 **`test_type`** 查询参数 **明确等于某一子类型** 时：**仅 **`select_related` 与该类型匹配的 FK**（如仅接口型时 join **`apitestcase`**）。
- 当 **未限定 `test_type`**（或需要展示混合卡片）时：对 **所有已知子类型外键同时使用 `select_related`**，单次 SQL 拉出主表与各扩展表的 LEFT JOIN。

**注意**：`select_related` 仅适用于 **单值 FK / OneToOne**；若有 **反向外键集合**，需 **`prefetch_related`**（若存在此类关联）。

## 4. API 与普通功能用例差异（执行侧）

- **接口**：**`run_api_case`**（**`testcase/services/api_execution.py`**）解析 URL、headers、body、断言与变量运行时；环境与 **`EnvironmentVariable`** 联动。
- **性能 / 安全 / UI**：由 **execution / assistant** 等应用中的 runner 消费各自子表字段；逆向时需跳转对应 **`execution/views.py`** 与任务模块。

本篇不展开断言 DSL，参阅 **`07_接口执行引擎与断言变量认证运行时.md`**。

## 5. 前端路由与 Tabs

SPA 一般用 **`test_type`** 切换详情页 Tabs（接口配置 / 脚本 / 元素库）；若 URL 仅用 **`case id`**，则 **首期请求 `retrieve`** 后以返回的 **`test_type`** 决定渲染分支。错误做法：仅凭模块 **`test_type`** 渲染子表单，而用例可能被 AI 批量改为另一类型而未同步刷新。

## 8. 维护提示

若在 **`TEST_CASE_TYPE_CHOICES`** 增删枚举：

1. 迁移 **`test_type`/`test_module.test_type`** 合法值；
2. 同步前端 TypeScript union、国际化文案；
