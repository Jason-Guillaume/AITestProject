# 测试用例视图集与分页筛选

## 文档目的

说明 **`TestCaseViewSet`**（`/api/testcase/cases/`）的分页类、**`get_queryset`** 在 **正常 / 回收站** 下的 **`project` / `module` / `search` / `testType`** 组合逻辑、**`perform_destroy` 软删写 `deleted_at`**，以及 **`create`** 时对 **`RECYCLE_CONFLICT`** 的 **409** 处理与 **`knowledge_recommendations`** 附加字段。

## 2. `get_queryset` 要点

### 2.1 回收站模式

- **`_recycle_mode_from_params`** 为真时：基础 **`TestCase.all_objects.filter(is_deleted=True)`**（需含已删行）。
- **数据范围**：非管理员 **`_apply_member_scope`**；管理员不过滤成员（`pass` 分支）。
- **`?project=`**：回收站下对 **孤儿用例**（`module` 空）有特殊 **`Q(module__isnull=True, creator=...)`** 与管理员 **`creator__projects`** 组合，避免跨项目泄露（见源码注释）。
- **排序**：**`-deleted_at, -id`**。

### 2.2 正常模式

- **`super().get_queryset()`** → 默认仅 **`is_deleted=False`**。
- **`module`**：支持 **子树** **`_module_self_and_descendant_ids`** 过滤。
- **`search`**：`case_name__icontains`。
- **`testType`**：合法枚举则 **`filter(test_type=...)`**。
- **`select_related`**：按 `test_type` 仅关联需要的子类型（`apitestcase` 等），功能型不额外 join。

---

## 3. 软删与创建

- **`perform_destroy`**：**禁止物理 delete**；`is_deleted=True` 且 **`deleted_at=now`**。
- **`create`**：校验失败若 **`detail.code == RECYCLE_CONFLICT`** → **HTTP 409**；成功则尝试 **`KnowledgeSearcher.search_similar`** 填充 **`knowledge_recommendations`**（异常吞掉）。
