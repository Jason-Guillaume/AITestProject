# 接口执行引擎与断言变量认证运行时

## 文档目的

将 **`testcase/services/`** 目录下的 **`api_execution.py`、`assertions.py`、`variable_runtime.py`、`auth_runtime.py`**（及 **`preview_resolved_request`、`run_api_case`**）与 **`TestCaseViewSet`** 暴露的 **`execute-api`/`run-api`/`preview-run-api`/`batch-preview-run-api`** 对齐，说明 **单次执行、预览解析、批量预检、`${var}` 未解析巡检**；便于对接前端 **`useApiExecuteConsole`** 与测试报告落库链路。

---

## 1. 概念流水线

```text
TestCase(API) + Profile（ApiTestCase 等）
       ↓ get_api_profile_for_execute
preview_resolved_request 或 run_api_core
       ↓ environment base_url（overrides.environment_id）
       ↓ EnvironmentVariable / 运行时 dict（variable_runtime）
       ↓ auth_runtime（Bearer/Basic/自定义头）
       ↓ HTTP 客户端（requests 或 urllib ——以 api_execution 为准）
       ↓ assertions（期望状态码、JSONPath…）
       ↓ ExecutionLog（+ 可选 ApiTestLog 兼容路径）
```

**`execute-api`**：**`_run_api_core(..., write_legacy_apilog=True)`** —— 写 **`ApiTestLog`**。  
**`run-api`**：同核心，**不写 **遗留 **`ApiTestLog`**，仅以 **`ExecutionLog`** 为新一代事实源。

源码锚点：**`testcase/views.py`** 786 行附件。

---

## 2. `_run_api_core` 关注点（逆向层次）

共性参数：**`request`**、**`overrides`**（来自 POST JSON）：

- **`environment_id`**：解析 **`TestEnvironment.base_url`** 与 **`health_check_path`** 不参与执行拼接（除非预览实现引用）；
- **URL / method / headers / body** 运行时覆盖：**用于「控制台试跑不改库」**，最终仍受序列化合法性约束；
- 异常路径：Profile 缺失、环境未授权、断言抛错等均应 **结构化响应** `{msg}` 或 `{error}`（读 `_run_api_core` 实测）。

超时、TLS 校验、跳转跟随等——**以服务实现为准**，本文不臆造默认值。

---

## 3. `preview-run-api`（不发网络）

**`preview_resolved_request(api_prof, overrides)`**：

- **仅拼装**解析后的 **`request` 视图**（URL、verb、头部、正文）；
- 用于 UI「执行前预览」与安全审计复制 cURL；

**约束**： **`test_type` 必须为 API**；否则 **400**。  
**`get_api_profile_for_execute`** 返回 **`None`** 时提示 **「请先编辑保存该用例」**——常见于草稿未落子类型表。

---

## 4. `batch-preview-run-api`

**路径**：**`POST /api/testcase/cases/batch-preview-run-api/`**  
**Body**：

```json
{ "ids": [1, 2], "overrides": { "environment_id": 3 } }
```

行为摘要（**`views.py`**）：

1. **`ids`** 必填数组；逐项保留 **传入顺序** 输出；
2. **QuerySet**：**`test_type=API` 且未删`**；
3. 每条：**`preview_resolved_request`**；
4. **正则 `\$\{[^}]+\}`** 在未解析占位符扫描 **`request`** 子 JSON；若有残留 → **`ok:false` 的子原因 `unresolved` 变量列表（截断 50）**。

用于 **门禁 / 批量导入 QA**：与 **`gate_ai_api_cases`、`precheck`** 同属「质量闸门」家族。

---

## 5. 断言模块 `assertions`

将 **声明式断言清单**转为 **逐项 pass/fail** 结构与 **可读消息**；断言失败不一定抛 HTTP 500 ——常以 **200 + `assertions[].passed=false`**（以 **`run_api_case` 返回值包装**为准）。

断言类型：**状态码 / 响应体 substring / JSONPath / header**（读 **`assertions.py`** 注册的 handler）。

---

## 6. 变量 `variable_runtime`

职责：

1. **`${var}` 占位符替换**：来源 **请求体 **`overrides.variables`**、`EnvironmentVariable`**、前置步骤 **`ExecutionLog` 抽出链**；
2. **`suggest_extractions`**：根据响应 **推荐可抽取字段** ——挂载在单独的 APIView **`SuggestExtractionsAPIView`**（**`testcase/urls`** 顶层 path）。

与其它用例的步骤编排（Scenario）对接时：**变量生命周期**超出单用例则可能走 **`execution`** 应用 ——本篇限定 **tc 单次 API**。

---

## 7. 认证 `auth_runtime`

输入：**用例配置的 auth 片段 + 环境中敏感变量**；

输出：**增补 `Authorization` 或等价头**；

与 **环境与变量分册**：若变量 **加密-at-rest**，读取路径须 **解密钩子统一**——否则 **Bearer 明文落日志**合规风险。

---

## 8. 日志双写兼容

| 表/模型 | 触发条件 | 用途 |
|---------|----------|------|
| `ExecutionLog` | 新式执行主链 | 统一编排、报告、回看 |
| `ApiTestLog` | `execute-api` | 历史兼容字段、迁移期报表 |

长期应以前端 **仅依赖 ExecutionLog**，但逆向旧接口需知 **`execute-api`** 仍存在。

参阅 **`05_接口测试日志与执行日志模型.md`**。

---

## 9. 前端 `useApiExecuteConsole`（契约）

- **环境下拉** → **`environment_id` 装入 overrides**；
- **Run** → **`run-api` 优先**（或可配置）；
- **Preview** → **`preview-run-api`**；
- **批量检查** → 调 **`batch-preview-run-api`** 展示 **未闭合 `${}`**；

WebSocket/SSE ——若非 testcase 视图提供则 **不适用**。
