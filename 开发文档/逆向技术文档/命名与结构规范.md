# 逆向技术文档 — 命名与结构规范

本文约定「全量子功能逆向文档」的文件与目录命名方式，**文件名与目录名均以中文为主**（可辅以数字前缀便于排序）。

## 1. 放置位置

- 所有本规范下的逆向文档统一放在目录：**`开发文档/逆向技术文档/`**（与本文件同级及下级各中文子目录）。
- 与仓库内已有的 `01_公共基础模块.md` 等历史文档**并存**；新写的逆向还原以本目录为准，避免与旧英文名文档混用。

## 2. 文件命名规则

| 规则 | 说明 |
|------|------|
| 语言 | **主名称为中文**，例如 `验证码与登录接口.md` |
| 排序 | 同一目录下建议使用 **`NN_`** 两位数字前缀（01–99），保证目录排序与阅读顺序一致 |
| 后缀 | 统一 **`.md`**，编码 **UTF-8** |
| 禁止字符 | 不使用 Windows 非法字符：`\ / : * ? " < > \|` |
| 粒度 | 每个文件对应**一个可独立说明的子功能**（单一视图集、独立接口、独立实时通道、或单一前端业务页及其调用面） |
| 内容结构 | 每篇须覆盖：底层运行逻辑、数据模型、前后端交互、界面表现与逻辑、异常处理（与项目既定文档标准对齐） |
| 技术专名 | 业界通用缩写（如 **K6**、**cURL**）可出现在文件名中；能译为中文的优先用中文（如「接口」优于单独使用 `API` 作文件名主体）。 |

## 3. 目录结构规则

- **一级子目录**：按**领域**划分，使用中文，例如 `用户与认证/`、`测试用例与步骤/`。
- **可选二级**：仅当单目录下文件超过约 15 个时，再拆中文子目录。
- **正文**：类名、路径、英文标识符可保留在正文与小标题中。

## 4. 与索引的关系

- 同目录下的 **`子功能文档全量索引.md`** 为权威清单：其中列出的「规划路径 + 中文文件名」即为应撰写与应维护的文档列表。
- 新增子功能时：先改索引，再按相同规则建文件。

## 5. 示例

- `开发文档/逆向技术文档/用户与认证/01_验证码与图形接口.md`
- `开发文档/逆向技术文档/接口场景编排/03_场景运行与步骤运行视图集.md`
