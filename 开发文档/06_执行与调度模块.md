# 执行与调度模块（execution）

## 功能概述
负责测试计划、报告、性能任务、定时任务以及 API 场景化执行的完整生命周期管理。从模型定义、任务调度、实际执行到结果持久化、实时推送，都在本子系统内完成。

## 使用技术
- **Django ORM**：`TestPlan`, `TestReport`, `PerfTask`, `ScheduledTask`, `ExecutionTask`, `ApiScenario` 等模型统一管理业务数据。
- **Execution Engine (`engine.py`)**：`APIExecutor` 使用 `requests` 发起实际 HTTP 调用，统一处理请求构造、Header 清洗、正文截断以及断言结果包装。
- **Django Channels**：`consumers.py` 中的 `K6MetricsConsumer` 示例实现 WebSocket 实时推送（如性能指标、任务日志）。
- **Celery**：`tasks.py`、`tasks_k6.py` 提供异步执行、重试、超时控制，保证耗时任务不阻塞 API。
- **调度器 (`scheduler.py`)**：基于 **APScheduler** 的单例 `TestScheduler` 将数据库中的 `ScheduledTask` 转化为实际的 cron 调度。
- **DRF ViewSet & APIView**：`views.py`、`views_k6.py` 暴露 CRUD、批量操作、任务触发、结果查询等接口。

### 执行引擎 (`execution/engine.py`)
```python
class APIExecutor:
    def execute(self, task: ExecutionTask) -> Dict[str, Any]:
        request_cfg = self._build_request_payload(task.payload)
        resp = requests.request(**request_cfg, timeout=30)
        return self._build_assertion_result(resp)

    def _build_request_payload(self, payload: dict) -> dict:
        # 统一处理 method、url、headers、body
        return {
            "method": payload.get("method", "GET"),
            "url": payload["url"],
            "headers": self._sanitize_headers(payload.get("headers", {})),
            "data": self._sanitize_body(payload.get("body")),
        }
```

## 为什么使用这些技术
- **统一模型**：所有调度相关对象都有统一的审计字段、软删行为，便于权限与回收站管理。
- **Celery**：可靠的异步任务框架，支持重试、超时、结果存储，适合对外部接口的调用。
- **APScheduler**：将数据库中的 cron 表达式转为真实调度，无需手写定时脚本。
- **Channels**：实时推送执行日志或性能指标，提高用户体验，避免前端轮询。
- **DRF**：提供一致的 RESTful 接口，前端只需调用标准 CRUD 与触发 endpoint。

## 常见错误与排查
1. **任务未被调度**：确认 `TestScheduler.add_task` 已被调用，并且 `scheduler.start()` 正在运行；检查 Celery Beat 是否启动 (`celery -A AITestProduct beat -l info`)。
2. **执行超时或网络错误**：`APIExecutor.execute` 默认 30 秒 timeout，若目标服务响应慢可在代码中调大 `timeout`，并在 `settings.py` 中配置 `REQUEST_TIMEOUT`。
3. **WebSocket 断连**：`TokenAuthMiddleware` 验证失败，确保前端在建立连接时通过 query 参数或 `Sec-WebSocket-Protocol` 传递 JWT。
4. **Celery 任务失败**：查看 worker 日志，常见原因包括序列化错误（模型实例未能 pickle）、数据库连接失效或第三方服务不可达。
5. **定时任务 cron 表达式错误**：`cron_to_kwargs` 需要合法的字段（minute、hour、day_of_week 等），错误会在 scheduler 添加时抛出，需要在创建或编辑 `ScheduledTask` 时前端做校验。
