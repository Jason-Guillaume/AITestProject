# 前端概述（frontend）

## 功能概述
前端负责与后端 RESTful API 交互，展示知识库、AI 生成、UI 自动化、缺陷管理、执行调度等业务数据，并提供可视化的操作界面。整体采用单页面应用（SPA）架构，提升用户体验和响应速度。

## 使用技术
- **框架**：Vue 3（或 React 18）配合 **Vite**/**Webpack** 进行快速构建。
- **状态管理**：Pinia（Vue）或 Redux（React），统一管理认证、分页、筛选等全局状态。
- **UI 组件库**：Ant Design Vue / Material‑UI，提供丰富的表格、表单、树形、图表等组件。
- **路由**：Vue Router / React Router，分模块路由（知识库、AI 生成、UI 自动化、缺陷、调度、仪表盘）。
- **图表**：ECharts、Chart.js，用于质量仪表盘、性能趋势等数据可视化。
- **WebSocket**：配合后端 **Django Channels** 实时推送执行日志、K6 负载测试指标。
- **认证**：JWT + HttpOnly Cookie，前端在每次请求中自动携带 `Authorization: Bearer <token>`。

## 关键代码文件及片段
### `frontend/src/api/index.js`
```javascript
import axios from 'axios';
const instance = axios.create({
    baseURL: process.env.VUE_APP_API_BASE_URL,
    withCredentials: true, // 发送 Cookie
});

export const getKnowledgeArticles = (params) =>
    instance.get('/api/knowledge/articles/', { params });

export const generateAiCases = (payload) =>
    instance.post('/api/ai/generate/', payload);
```

### `frontend/src/store/user.js`
```javascript
import { defineStore } from 'pinia';
export const useUserStore = defineStore('user', {
    state: () => ({ token: null, userInfo: {} }),
    actions: {
        async login(credentials) {
            const res = await instance.post('/api/auth/login/', credentials);
            this.token = res.data.token;
            // token 存在 HttpOnly Cookie，前端仅保存少量信息
        },
        logout() {
            this.token = null;
            // 调用后端 logout 清除 Cookie
        },
    },
});
```

### `frontend/src/views/Knowledge.vue`
```html
<template>
  <a-table :columns="columns" :data-source="articles" row-key="id" />
</template>

<script setup>
import { ref, onMounted } from 'vue';
import { getKnowledgeArticles } from '@/api';
const articles = ref([]);
onMounted(async () => {
  const resp = await getKnowledgeArticles({ page: 1, pageSize: 20 });
  articles.value = resp.data.results;
});
const columns = [
  { title: '标题', dataIndex: 'title' },
  { title: '创建时间', dataIndex: 'created_at' },
];
</script>
```

## 为什么使用这些技术
- **SPA + Vite**：极快的热更新和构建速度，提升开发效率。
- **Pinia/Redux**：统一的状态管理避免页面之间的重复请求和数据不一致。
- **Ant Design**：成熟的企业级 UI 组件库，减少自行实现表格、树形、分页等复杂交互的工作量。
- **WebSocket + Channels**：实现实时日志推送，用户可以在页面实时看到脚本执行进度，而无需轮询。
- **JWT + HttpOnly Cookie**：安全的身份验证方式，防止 XSS 攻击窃取 token。

## 常见错误与排查
1. **API 跨域错误（CORS）**：后端 `settings.py` 中 `CORS_ALLOWED_ORIGINS` 未包含前端地址，检查并添加 `http://localhost:3000`（或实际端口）。
2. **Token 失效**：前端收到 `401` 后自动跳转登录页，确保后端的 `ACCESS_TOKEN_LIFETIME` 配置与前端刷新策略匹配。
3. **WebSocket 连接被拒**：`TokenAuthMiddleware` 未通过，前端在 `new WebSocket(url, ["token", token])` 时需要在 query 参数中带上 token，或在 `Sec-WebSocket-Protocol` 中声明。
4. **表格分页不生效**：后端分页字段名称不一致（`page` vs `pageNo`），对齐前端请求参数或后端 `PageNumberPagination` 配置 `page_query_parameter = 'page'`。
5. **样式冲突**：全局 CSS 与组件库样式冲突导致 UI 显示异常，使用 `scoped` 样式或在 `vite.config.js` 中配置 `css.preprocessorOptions`。
