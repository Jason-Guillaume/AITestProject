# UI自动化工作台上传认证问题修复文档

## 📋 问题描述

**现象**: 上传文件时返回 404 错误  
**根本原因**: 使用了原生 `axios` 而不是项目的 `request` 实例，导致请求未携带认证 Token  
**修复日期**: 2026-05-01

---

## 🔍 问题分析

### 错误表现
从浏览器 Network 面板可以看到：
- POST `/api/assistant/ui-scripts/` 返回 404
- 请求头缺少 `Authorization: Token xxx`

### 根本原因
```javascript
// ❌ 错误：直接使用 axios，没有认证拦截器
import axios from 'axios';
await axios.post('/api/assistant/ui-scripts/', formData);
```

项目使用了自定义的 `request` 实例（`@/utils/request.js`），该实例配置了：
1. **baseURL**: `/api`
2. **请求拦截器**: 自动添加 `Authorization: Token xxx`
3. **响应拦截器**: 统一错误处理
4. **超时设置**: 60秒

直接使用原生 `axios` 会绕过这些配置，导致：
- 请求路径错误（需要手动加 `/api` 前缀）
- 缺少认证 Token
- 错误处理不统一

## 🔧 修复方案

### 1. 修改导入语句

**修复前**:
```javascript
import axios from 'axios';
```

**修复后**:
```javascript
import request from '@/utils/request';
```

### 2. 修改上传函数

**单文件上传 - 修复前**:
```javascript

  headers: { 'Content-Type': 'multipart/form-data' }
});
```

**单文件上传 - 修复后**:
```javascript
await request.post('/assistant/ui-scripts/', formData, {

```

**ZIP/Git导入 - 修复前**:
```javascript

  headers: { 'Content-Type': 'multipart/form-data' }
});
```

**ZIP/Git导入 - 修复后**:
```javascript
await request.post('/assistant/ui-scripts/', formData, {
  headers: { 'Content-Type': 'multipart/form-data' }

```

### 关键变化
1. `axios` → `request`
2. `/api/assistant/ui-scripts/` → `/assistant/ui-scripts/`（baseURL 已包含 `/api`）
3. 自动添加认证 Token

## 📚 项目 request 实例说明

### 配置详情 (frontend/src/utils/request.js)

```javascript
const request = axios.create({
  baseURL: '/api',           // 基础路径
  timeout: 60000,            // 60秒超时
});

// 请求拦截器：自动添加 Token
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Token ${token}`;
  }
  return config;
});

// 响应拦截器：统一错误处理
request.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error?.response?.status === 401) {
      ElMessage.error('登录已过期，请重新登录');
      localStorage.removeItem('token');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```

### 使用规范

**✅ 正确用法**:
```javascript
import request from '@/utils/request';

// GET 请求
await request.get('/assistant/ui-scripts/');

// POST 请求
await request.post('/assistant/ui-scripts/', data);

// 文件上传
await request.post('/assistant/ui-scripts/', formData, {
  headers: { 'Content-Type': 'multipart/form-data' }
});
```

**❌ 错误用法**:
```javascript
import axios from 'axios';

// 缺少认证，会返回 401 或 404
await axios.post('/api/assistant/ui-scripts/', data);
```

## 📝 经验总结

### 问题根源
在 Vue 项目中直接使用原生 `axios` 而不是项目配置的 `request` 实例。

### 最佳实践
1. **统一使用项目的 request 实例**
2. **不要直接导入 axios**
3. **URL 路径不需要 `/api` 前缀**（baseURL 已配置）
4. **文件上传需要设置 Content-Type**

### 类似问题排查
如果遇到 401/404 错误：
1. 检查是否使用了 `request` 而不是 `axios`
2. 检查 URL 路径是否正确（不要重复 `/api`）
3. 检查 localStorage 中是否有 token
4. 检查后端路由是否正确注册
