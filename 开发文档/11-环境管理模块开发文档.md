# 11-环境管理模块开发文档

## 0. 需求来源与开发动因

- 业务价值摘要：集中治理测试环境与变量配置，提升联调效率与安全性。
- 业务背景：自动化测试覆盖多环境（开发/测试/预发/生产镜像），需要统一环境信息与变量治理。
- 现状痛点：环境参数分散、敏感信息明文存储，易引发配置错误、联调失败与安全风险。
- 建设目标：建设环境管理模块，集中管理连接参数与密文变量。
- 预期收益：提升环境治理效率、配置一致性与审计可追踪性。

## 1. 模块目标

环境管理模块用于统一管理自动化测试所依赖的运行环境（开发、测试、预发、生产镜像等），并支持环境级变量配置与密文存储，避免敏感信息明文泄漏。

核心能力：

- 测试环境管理（环境类型、Base URL、数据库连接配置）
- 环境变量管理（普通变量与敏感变量）
- 敏感变量对称加密存储与按需解密
- 环境连通性验证接口（HTTP / DB）

## 2. 数据模型设计

### 2.1 TestEnvironment

建议表名：`test_environment`

关键字段：

- `name`: 环境名称（唯一）
- `env_type`: 环境类型（dev/test/staging/prod/custom）
- `base_url`: 接口基础地址
- `db_config`: 数据库配置（JSON，包含 host/port/user/password/db_name 等）
- `description`: 环境描述
- 继承 `BaseModel`：`creator/updater/create_time/update_time/is_deleted`

约束建议：

- `name` 全局唯一，避免重复命名导致调度选择歧义

### 2.2 EnvironmentVariable

建议表名：`environment_variable`

关键字段：

- `environment`: 关联环境（FK -> TestEnvironment）
- `key`: 变量名
- `value`: 变量值（密文或明文）
- `is_secret`: 是否敏感（敏感时加密存储）
- `description`: 变量说明
- 继承 `BaseModel`

约束建议：

- `environment + key` 联合唯一

## 3. 敏感变量加密方案

### 3.1 技术选型

- 使用 `cryptography.fernet.Fernet` 实现对称加密
- 存储格式采用前缀标识：`enc::<cipher_text>`

### 3.2 密钥来源

- 优先读取环境变量：`AITEST_ENV_SECRET_KEY`
- 若未配置，使用 `SECRET_KEY` 派生 32 字节 key（开发兜底）
- 生产环境建议强制注入独立密钥，不使用默认派生模式

### 3.3 加解密时机

- 写入：`EnvironmentVariable.save()` 中，当 `is_secret=True` 时自动加密
- 读取：提供 `get_decrypted_value()` 方法，业务按需调用

### 3.4 安全建议

- 避免在日志中打印敏感变量明文
- 配置导出接口应默认脱敏
- 密钥轮换时可增加离线迁移脚本，批量重加密

## 4. API 设计（DRF）

基础 CRUD：

- `GET /api/testcase/environments/`
- `POST /api/testcase/environments/`
- `GET /api/testcase/environments/{id}/`
- `PUT/PATCH /api/testcase/environments/{id}/`
- `DELETE /api/testcase/environments/{id}/`

变量 CRUD：

- `GET /api/testcase/environment-variables/`
- `POST /api/testcase/environment-variables/`
- `GET /api/testcase/environment-variables/{id}/`
- `PUT/PATCH /api/testcase/environment-variables/{id}/`
- `DELETE /api/testcase/environment-variables/{id}/`

连通性校验：

- `POST /api/testcase/environments/{id}/validate/`

请求体（可选覆盖）：

```json
{
  "timeout": 5
}
```

响应体示例：

```json
{
  "ok": true,
  "http": {
    "ok": true,
    "status_code": 200
  },
  "db": {
    "ok": true,
    "host": "127.0.0.1",
    "port": 3306
  }
}
```

## 5. 异常处理与日志

- 使用 `logging.getLogger(__name__)` 记录关键流程
- 对加密失败、解密失败、连通性异常进行捕获并输出可定位日志
- API 层返回结构化错误，不暴露堆栈细节

## 6. 与调度模块协同

- `ScheduledTask` 通过 FK 关联 `TestEnvironment`
- 调度执行时读取环境 `base_url` 与变量，构建运行上下文
- 环境删除前应限制：若被调度任务引用，建议先解绑或禁用任务
