# 99-项目全局扫描对账索引（Single Source of Truth）

> 本文档用于把“当前仓库代码实现”固化为一份**可对账、可维护**的索引。  
> 建议把它当作**单一事实源**：当路由、启动方式、环境变量、脚本发生变化时，优先更新本文件；其他分册只保留“读者最需要的最小信息 + 链接”。

---

## 1. 全局入口与分层（代码为准）

- **后端主路由入口**：`AITestProduct/urls.py`
  - 所有 HTTP API 统一挂在 `/api/*`
- **ASGI/WS 入口**：`AITestProduct/asgi.py`
  - WebSocket 由 `TokenAuthMiddleware(URLRouter(...))` 承载
  - WS 路由合并：`execution/routing.py` + `server_logs/routing.py`
- **前端请求封装**：`frontend/src/utils/request.js`
  - `baseURL="/api"`，请求头注入 `Authorization: Token <token>`
- **前端路由入口**：`frontend/src/router/index.js`
- **前端开发代理**：`frontend/vite.config.js`（`/api`、`/media`、`/ws`）

## 5. 环境变量总表（按域分组）

### 5.1 Django 基础

来源：`AITestProduct/settings.py`（含本地 `.env` loader）。

- `DJANGO_DEBUG`：默认 `1/True`；当为 `0` 时触发**生产安全启动校验**
- `DJANGO_SECRET_KEY`：生产必填（`DJANGO_DEBUG=0` 时缺失会启动失败）
- `DJANGO_ALLOWED_HOSTS`：逗号分隔；为空则 `ALLOWED_HOSTS=[]`

### 5.2 MySQL（主库）

来源：`AITestProduct/settings.py`。

- `DB_HOST`（默认 `127.0.0.1`）
- `DB_PORT`（默认 `3306`）
- `DB_NAME`（默认 `ai_test_product`）
- `DB_USER`（默认 `root`）
- `DB_PASSWORD`（默认空；**生产必填**，否则启动失败）

### 5.3 Redis / Cache / Channels

来源：`AITestProduct/settings.py`，以及环境健康检查 `execution/services/health_checker.py`。

- `REDIS_HOST`（默认 `127.0.0.1`）
- `REDIS_PORT`（默认 `6379`）
- `USE_REDIS_CACHE`（默认 `1`）
- `CHANNEL_REDIS_HOST`（默认继承 `REDIS_HOST`）
- `CHANNEL_REDIS_PORT`（默认继承 `REDIS_PORT`）

行为约定（重要）：

- Redis 不可用时：
  - Cache 自动降级为 `LocMemCache`
  - Channels 自动降级为 `InMemoryChannelLayer`（仅适合单机开发）

### 5.4 邮件（敏感变更审批通知 / 健康检查告警）

来源：`AITestProduct/settings.py`。

- `EMAIL_BACKEND`（默认 `django.core.mail.backends.console.EmailBackend`）
- `DEFAULT_FROM_EMAIL`（默认 `webmaster@localhost`）
- `EMAIL_SUBJECT_PREFIX`（默认 `[AITest]`）
- `USER_CENTER_ADMIN_EMAIL`（逗号分隔；为空则回退 `ADMINS`）

### 5.5 登录兼容开关（历史迁移窗口用）

来源：`AITestProduct/settings.py`（默认全关）。

- `LEGACY_LOGIN_MD5_HEX`
- `LEGACY_LOGIN_SHA256_HEX`
- `LEGACY_LOGIN_BOTH_SIDES_MD5_HEX`
- `LEGACY_LOGIN_PLAINTEXT`

### 5.6 Celery（执行/异步任务）

来源：`AITestProduct/settings.py`。

- `CELERY_BROKER_URL`
  - 默认优先 Redis：`redis://<REDIS_HOST>:<REDIS_PORT>/1`
  - Redis 不可用时回退 MySQL broker（复用 `DATABASES['default']`）
- `CELERY_RESULT_BACKEND`
  - 默认：若安装 `django_celery_results` 则 `django-db`；否则 `rpc://`

启动脚本（Windows）：`scripts/start-celery.ps1`、`scripts/start-celery-solo.ps1`

### 5.7 知识中心 / RAG（文档向量化）

来源：`AITestProduct/settings.py`（部分）与 `assistant/tasks.py`（对未配置项有默认值）。

### 5.10 服务器日志（server_logs）

来源：`AITestProduct/settings.py` + 专题：`开发文档/22-服务器日志监控模块开发文档.md`。

## 6. 运行进程拓扑（dev / prod）

### 6.1 开发环境（最常见）

```mermaid
flowchart LR
  Browser[Browser(Vite_5173)] -->|"/api"| DjangoHTTP[Django_8000_HTTP]
  Browser -->|"/ws"| DjangoASGI[Daphne_8000_WS]
  DjangoHTTP --> MySQL[(MySQL)]
  DjangoHTTP --> Redis[(Redis_optional)]
  DjangoASGI --> Redis
  CeleryWorker[Celery_worker_optional] --> Redis
  CeleryWorker --> MySQL
```

要点：

- 仅跑 `python manage.py runserver`：HTTP 可用；**WS 需要 ASGI（daphne）**。
- Celery worker 非必需，但涉及“异步执行/定时/队列”类能力时需要启动（脚本见 §7）。

### 6.2 生产环境（建议形态）

- HTTP/ASGI：Gunicorn/Uvicorn + Daphne（或统一 ASGI server）
- 反向代理：Nginx（静态前端 + `/api` + `/media` + `/ws`）
- Redis：生产建议强制启用（cache + channel layer + broker）
