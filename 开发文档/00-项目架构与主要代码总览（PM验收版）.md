# 00-项目架构与主要代码总览（PM 验收版）

## 0. 文档目的（给项目经理）

本文件用于项目经理快速核对：

- **整体架构**是否符合预期（前端、后端、异步任务、实时通道、AI/RAG）
- **主要模块职责与边界**是否清晰、是否存在需求缺口
- **AI 模块**（重点）是否实现了“连接校验 / 同步生成 / 流式生成 / 去重 / RAG / 配置治理”的预期能力
- **服务器日志模块**是否满足“组织共享 / 实时查看 / AI 诊断 / 审计 / 安全”的预期能力

> 本文档以“需求核对 + 关键代码入口”为主，不追求涵盖所有实现细节。详细分册见 `开发文档/` 其他章节。

## 1. 系统总体架构（宏观视角）

### 1.1 技术栈与关键组件

- **前端**：Vue 3 + Vite + Element Plus + Axios
- **后端**：Django + DRF（REST API）
- **实时通道**：Django Channels（WebSocket）
- **异步任务**：Celery（用于执行引擎、AI/知识库类异步任务等）
- **存储**：
  - MySQL：主数据
  - Redis：缓存/验证码/Channels layer（开发环境支持降级）
  - ChromaDB：向量检索（RAG）
- **外部依赖**：
  - OpenAI 兼容接口（可配置 base_url/model/api_key）
  - 可选 Grafana Loki（历史日志检索）

### 1.2 系统分层（前后端）

```text
浏览器（Vue3）
  ├─ 页面（views）+ 组件（components）
  ├─ 路由与布局（router/layouts）
  └─ API 封装 + 请求拦截（src/api + utils/request.js）
           │
           │  HTTP: /api/*
           │  WS:  /ws/*
           ▼
后端（Django/DRF + Channels）
  ├─ HTTP 路由：AITestProduct/urls.py → 各业务 app
  ├─ REST：ViewSet/APIView + Serializer + ORM
  ├─ WebSocket：AITestProduct/asgi.py → TokenAuthMiddleware → Consumers
  └─ 异步：Celery worker（执行引擎等）
           │
           ▼
MySQL / Redis / Chroma / Media
```

## 3. 关键全链路（验收关注点）

### 3.1 登录与鉴权（Token）

- 登录：`POST /api/user/login/` → 返回 `data.token`
- 前端：`frontend/src/utils/request.js` 自动注入 `Authorization: Token <token>`
- 后端：DRF 默认 `TokenAuthentication + IsAuthenticated`（除登录/注册/验证码等显式开放接口）

### 3.2 WebSocket 鉴权（服务器日志 / 执行监控通用）

- WS URL 通过 querystring 传 `token`：`/ws/.../?token=<token>`
- 后端在 `AITestProduct/asgi.py` 对 websocket 应用 `execution/middleware_ws.py:TokenAuthMiddleware`，将 token 映射为 `scope["user"]`

## 4. AI 模块（重点：需求一致性核对）

### 4.1 PM 视角：AI 已实现的能力清单

- **连通性校验**：
  - `/api/ai/verify-connection/`：固定模型快速校验
  - `/api/ai/test-connection/`：自定义 base_url/model/api_key 校验
- **生成用例（同步）**：`POST /api/ai/generate-cases/`
- **生成用例（流式 SSE）**：`POST /api/ai/generate-cases-stream/`
- **RAG（知识库/相似用例检索）**：按模块检索 Top-K，相似片段拼入 System Prompt
- **去重**：
  - 生成结果字符串去重
  - （可选）语义去重（Embedding + cosine，相似度阈值可配置）
- **配置治理**：
  - 系统级 AI 配置：`/api/sys/ai-config/`（系统管理员）
  - 支持请求体临时覆盖 `api_key/base_url/model`
- **失败与降级**：
  - 上游 Key 无效/过期 → 后端返回 4xx/5xx（并给出可读错误信息）
  - 流式接口避免 DRF 内容协商导致的 406（采用标准 Django `View`）

### 4.2 主要代码入口（开发/验收定位）

- 路由：
  - `assistant/ai_urls.py`：`/api/ai/*`
  - `user/sys_urls.py`：`/api/sys/ai-config/*`
- 核心实现：
  - `assistant/views.py`：上下文解析、模型调用、SSE 输出、JSON 容错解析
  - `assistant/ai_generate_strategies.py`：按 `test_type` 分发策略与 prompt 结构
  - `assistant/rag_pipeline.py`、`assistant/rag_chroma.py`：RAG 检索与拼装
  - `assistant/generated_case_dedup.py` + `assistant/services/semantic_dedup.py`：去重

### 4.3 AI 流式协议（前后端约定）

- 后端持续输出 SSE `data: {...}\n\n`
- 前端以事件类型消费（connected/meta/delta/ping/done），最终以 `done` 事件收敛

### 4.4 AI 模块风险与边界（PM 需要确认）

- **外部依赖风险**：上游模型平台不可用会影响生成与诊断
- **成本与配额**：需明确默认模型、默认限流与配额策略（尤其生成/诊断高频场景）
- **结果一致性**：AI 输出存在非确定性，需要产品侧确认“可接受的波动范围”和“失败提示口径”

## 5. 服务器日志模块（新增重点：需求一致性核对）

### 5.1 PM 视角：已实现能力

- **远程主机配置**（REST CRUD）
  - 密码/私钥加密存储（Fernet 密文）
  - 支持绑定组织（组织创建人 + members 可共享）
- **实时日志查看**（WebSocket）
  - 后端发起 SSH，执行 `tail -f` / `Get-Content -Wait`
  - 前端终端 UI 虚拟滚动，支持 stop、复制路径、自动滚动
- **AI 诊断**
  - 前端划选日志 → `POST /api/server-logs/analyze/` 返回 Markdown
  - 限流：`server_logs_analyze` scope
- **审计**
  - WS connect/disconnect/stop、主机 CRUD、AI 诊断、历史检索均落库
- **安全**
  - 路径校验防注入（禁止换行/分号/反引号/命令替换片段等）
  - WS/REST 权限范围一致（创建人/组织成员/管理员）
- **稳定性（关键修复点）**
  - SSH 失败会推送 `error` 并主动关闭 WS
  - 线程结束会推送 `done` 并主动关闭 WS
  - WS 场景缩短 SSH connect 超时，优先快速失败，避免“连接挂住”

### 5.2 主要代码入口

- REST：`server_logs/views.py`、`server_logs/urls.py`、`server_logs/serializers.py`
- WS：`server_logs/consumers.py`、`server_logs/routing.py`
- SSH：`server_logs/ssh_tail.py`
- 权限：`server_logs/access.py`
- 路径校验：`server_logs/validators.py`
- 审计：`server_logs/audit.py`

## 6. 本轮修复汇总（影响需求/稳定性的“事实更新”）

为保证“实现与需求一致”以及“可稳定联调”，近期已落地的关键修复包含：

- **数据隔离 join 安全**：避免字段同名但类型不同导致 500（见 `common/views.py` 关系字段判定）
- **本地启动稳定性**：`start-dev.ps1` 增强端口 8000 残留进程清理，减少端口占用/假死
- **服务器日志 WS 收敛**：在 `error/done` 时主动 close，避免 dev reload 后全站 500
- **SSH 快速失败**：WS 场景缩短 connect 超时，提升用户反馈速度与资源释放速度

- [ ] 用户能注册/登录并正常访问受保护接口（401/403 行为符合预期）
