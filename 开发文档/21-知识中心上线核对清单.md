# 21-知识中心上线核对清单

## 1. 环境与依赖

- [ ] Python 依赖已安装：`sqlalchemy`、`pymysql`、`django-celery-results`
- [ ] LangChain 相关依赖可用（`langchain`、`langchain-community`、`langchain-openai`，若使用 Ollama 需 `langchain-ollama`）
- [ ] MySQL 服务可访问，账号具备建表与读写权限
- [ ] （可选）OpenAI/Ollama embedding 配置已准备

## 2. Django 配置检查

- [ ] `DATABASES["default"]` 指向正确 MySQL
- [ ] `CELERY_BROKER_URL` 为 `sqla+mysql+pymysql://...`
- [ ] `CELERY_RESULT_BACKEND` 为 `django-db`
- [ ] 已启用 `django_celery_results`（在 `INSTALLED_APPS` 中）
- [ ] `KNOWLEDGE_CHROMA_PERSIST_DIR`（可选）已按部署路径配置
- [ ] `KNOWLEDGE_CHROMA_COLLECTION`（可选）已按环境命名

## 3. 数据库迁移

- [ ] 已执行：`python manage.py migrate`
- [ ] `knowledge_document` 新字段已存在（`file_name/module/document_type/source_url/vector_db_id`）
- [ ] `django_celery_results_taskresult` 等结果表已创建

## 4. 进程启动

- [ ] Django 服务已启动
- [ ] Celery Worker 已启动并连接到 MySQL Broker
- [ ] Worker 日志无持续报错（连接、权限、导入失败等）

建议命令（示例）：

```bash
python -m celery -A AITestProduct worker -l info
```

## 5. 功能验收（后端）

- [ ] 调用 `POST /api/assistant/knowledge/documents/ingest/`（upload）返回 `202`
- [ ] 调用 `POST /api/assistant/knowledge/documents/ingest/`（url）返回 `202`
- [ ] `KnowledgeDocument.status` 流转正确：
  - `pending -> processing -> completed`（成功）
  - 异常时进入 `failed` 且有 `error_message`
- [ ] 成功后 `vector_db_id` 已写入
- [ ] Chroma 持久化目录有数据文件生成

## 6. 功能验收（前端）

- [ ] “知识文档入库”区可上传 PDF/MD
- [ ] URL 模式可提交 `http(s)` 链接
- [ ] 提交成功后状态列表自动刷新
- [ ] 轮询在有任务时自动开启，无处理中任务自动停止
- [ ] 状态颜色符合预期：
  - 处理中（蓝）
  - 已完成（绿）
  - 失败（红）

## 7. 异常回归

- [ ] 上传空文件/不支持格式时，前后端提示清晰
- [ ] embedding 服务不可用时，任务失败信息可定位
- [ ] MySQL 短暂断连后，服务可恢复并继续处理
- [ ] 重试按钮可重新投递失败任务

## 8. 上线后观测项（建议）

- [ ] 每日新增文档量、失败率、平均处理时长
- [ ] 失败原因 TopN（解析失败、向量化失败、连接失败）
- [ ] Chroma 存储增长趋势与磁盘占用
- [ ] Celery 队列积压情况与 worker 吞吐
