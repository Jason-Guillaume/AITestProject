# UI 自动化模块（assistant UI 子模块）

## 功能概述
管理 UI 元素的树形结构（模块、页面、元素、用例、步骤），并提供脚本上传、执行、实时日志的完整工作流。支持 AI 根据需求自动生成 UI 测试用例。

## 使用技术
- **Django ORM**：定义 `UIModule`, `UIPage`, `UIPageElement`, `UITestCase`, `UIActionStep` 五大模型，实现层级关联。
- **Django REST Framework (DRF) ViewSet**：`ui_element_views.py` 提供标准 CRUD + 自定义 `tree`、`by_name` 路由。
- **Serializers**：`ui_element_serializers.py` 包含 `UIModuleTreeSerializer`，实现嵌套序列化。
- **AI Prompt**：`ui_automation_views.py` 调用统一的 Prompt 系统自动生成 UI 用例。
- **文件存储**：`UIScriptUpload` 使用 `FileField` 保存压缩脚本包。
- **Celery**：`UIScriptExecutionViewSet` 通过 `run_script_async` 异步执行脚本，避免阻塞请求。
- **Django Channels**：类似 `K6MetricsConsumer` 的实现，用于推送脚本执行日志到前端。

## 关键代码文件及片段
### 模型 (`assistant/ui_element_models.py`)
```python
class UIModule(BaseModel):
    name = models.CharField(max_length=100)
    description = models.TextField(blank=True)

class UIPage(BaseModel):
    module = models.ForeignKey(UIModule, related_name="pages", on_delete=models.CASCADE)
    url = models.URLField()

class UIPageElement(BaseModel):
    page = models.ForeignKey(UIPage, related_name="elements", on_delete=models.CASCADE)
    selector = models.CharField(max_length=255)

class UITestCase(BaseModel):
    page = models.ForeignKey(UIPage, related_name="test_cases", on_delete=models.CASCADE)
    name = models.CharField(max_length=255)

class UIActionStep(BaseModel):
    test_case = models.ForeignKey(UITestCase, related_name="steps", on_delete=models.CASCADE)
    action_type = models.CharField(max_length=50)
    target = models.CharField(max_length=255)
    value = models.TextField(blank=True, null=True)
```

### 视图集 (`assistant/ui_element_views.py`)
```python
class UIModuleViewSet(BaseModelViewSet):
    queryset = UIModule.objects.all()
    serializer_class = UIModuleSerializer

    @action(detail=True, methods=["get"], url_path="tree")
    def tree(self, request, pk=None):
        module = self.get_object()
        return Response(UIModuleTreeSerializer(module).data)
```

### AI 生成入口 (`assistant/ui_automation_views.py`)

```python
class UiAutomationGenerateAPIView(APIView):
    def post(self, request):
        requirement = request.data.get("requirement")
        ctx = get_ai_prompt_context(requirement)
        result = dispatch_ai_generation(ctx)
        return ApiResponse.success(result.generated_cases)
```

```python
class UIScriptUpload(BaseModel):
    file = models.FileField(upload_to="ui_scripts/")
    is_active = models.BooleanField(default=False)

class UIScriptExecution(BaseModel):
    script = models.ForeignKey(UIScriptUpload, on_delete=models.CASCADE)
    status = models.CharField(max_length=20, default="pending")
    log = models.TextField(blank=True)
```

### 执行视图 (`assistant/views.py` 片段)
```python
class UIScriptExecutionViewSet(BaseModelViewSet):
    queryset = UIScriptExecution.objects.all()
    serializer_class = UIScriptExecutionSerializer

    @action(detail=True, methods=["post"], url_path="execute")
    def execute(self, request, pk=None):
        exec_obj = self.get_object()
        run_id = run_script_async(exec_obj.id)  # Celery task
        return ApiResponse.success({"run_id": run_id})
```

## 为什么使用这些技术
- **层级模型** 让 UI 元素可复用、组合，降低维护成本。
- **DRF + Action** 将标准 CRUD 与自定义树形/执行接口统一，前端只需一次请求即可获取完整结构。
- **Celery + Channels** 实现异步执行和实时日志推送，提升用户体验并避免阻塞。
- **文件存储** `FileField` 自动管理上传路径和磁盘空间。

## 常见错误与排查
1. **关联外键错误**：`UIPage.module_id` 或 `UIPageElement.page_id` 填写错误导致查询为空，检查外键是否正确设置。
2. **脚本未激活**：`UIScriptUpload.is_active=False` 时执行会被拒绝，确保脚本已勾选激活状态。
3. **Celery 任务未调度**：`run_script_async` 未注册或 worker 未启动，检查 `celery -A AITestProduct worker -l info` 是否在运行。
4. **WebSocket 日志未收到**：`TokenAuthMiddleware` 验证失败，确认前端在连接时携带有效 `Authorization` token。
