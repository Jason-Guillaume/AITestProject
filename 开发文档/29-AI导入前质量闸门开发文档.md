# 29-AI 导入前质量闸门开发文档

## 0. 需求来源与开发动因

- **业务价值摘要**：把明显坏数据（缺模块、缺步骤、API 缺 url/method、未替换变量）在导入前拦住，减少回收站与返工。
- **现状痛点**：AI 生成质量存在波动，导入后再发现问题成本高。
- **建设目标**
  - 前端预览表对“问题行”可视化标红 + tooltip 提示
  - 支持一键“阻断导入/允许导入（仅提示）”
  - 后端 `ai-import` 增加 `strict=true` 校验，避免绕过前端

---

## 1. 能力范围（本期）

### 1.1 前端质量闸门（默认阻断）

位置：`AI 生成测试用例` 弹窗 → 生成预览表

能力：
- 行级问题识别（示例）
  - 缺少用例名称
  - 未指定目标模块（`_target_module_id` 为空）
  - 缺少步骤（`steps` 为空且 `steps_list` 为空）
  - 存在未替换变量（`unresolved_vars` 非空）
  - API 类型缺少 `api_url/api_method`
- 预览表对问题行标红，并在“用例名称”处 hover 展示问题原因
- 开关：`发现问题则阻断导入`（默认开启）

### 1.2 后端严格校验（strict）

接口：`POST /api/testcase/cases/ai-import/`

新增参数：
- `strict: true/false`（默认 false；前端导入默认传 true）
- `precheck_overrides`（可选，对象）：与 `ai-import-precheck` 的 `overrides` 同形（`environment_id`、`variables`）。当 **`strict=true` 且 `test_type=api`** 时，后端对每条 item **复用同一套静态预检**（URL 合法性、`validate_before_request`、未替换 `${...}`），防止仅绕过前端仍写入坏 API 用例。

严格校验项（当前实现）：
- API 用例必须有 `api_url` 与 `api_method`
- 必须有可导入步骤（解析后的 steps 非空）
- **`strict=true` + API**：通过 `precheck_api_draft_item`（`testcase/services/ai_import_precheck_core.py`）与导入前预检对齐

---

## 2. 代码变更清单

- 前端：
  - `frontend/src/components/TestCaseModal.vue`
    - `rowIssues()`：问题识别
    - `blockImportOnIssues`：阻断开关
    - 表格 `row-class-name` 标红与 tooltip
    - 导入请求携带 `strict: true`
- 后端：
  - `testcase/views.py`
    - `TestCaseViewSet.ai_import` 支持 `strict` 并进行关键字段校验；`strict`+API 时读取 `precheck_overrides` 并调用预检核心
  - `testcase/services/ai_import_precheck_core.py`
    - `precheck_api_draft_item`：`ai-import-precheck` 与 `ai-import` strict 共用

---

## 3. 验收清单（最小）

- [x] 生成预览表中存在问题的行会标红，hover 可看到原因
- [x] 开启“阻断导入”时，勾选问题行会阻断导入并提示
- [x] 关闭“阻断导入”时仍可导入，但后端 `strict=true` 会让问题条目在 `failed[]` 中返回
- [x] API 缺少 url/method 时会被识别并阻断/失败
- [x] `strict=true` + API：直连 `ai-import` 且 body 含未替换 `${var}` 时，条目进入 `failed[]`（与 `precheck_overrides` 下预检一致）

最小复测路径：按 `开发文档/31` §5 步骤 3（P2）与步骤 6（P5）走一遍即可。

---

## 4. 回滚方案

- 前端：移除 `rowIssues` 与阻断开关即可（不影响生成与导入主流程）
- 后端：移除 `strict` 参数解析与校验逻辑即可（接口兼容）

---

## 5. 实现状态与全链路索引

- **结论**：已按设计落地（`TestCaseModal.vue` 行级 `rowIssues`、阻断开关、`ai-import` 传 `strict: true`；后端 `TestCaseViewSet.ai_import` 校验 API 字段与步骤）。
- **增补（P5）**：`strict=true` 且 API 类型时，后端使用 `precheck_api_draft_item` 与导入前预检同源校验；前端导入请求携带 `precheck_overrides`（环境 + 变量），与预检面板一致。
- **在本全链路中的优先级**：**P2**（文档主线）；**P5** 为 P2/P3 的后端加固（见总册 **31**）。
- **总览**：`开发文档/31-AI用例导入全链路-优先级与实现状态.md`。
