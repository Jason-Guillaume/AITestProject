# 环境管理与加密存储模块（Env‑01）

## I. 业务深度逻辑 (Business Logic Deep Dive)
1. **加密工具实现**
   - 新增 `utils/cryptography_helper.py`（若不存在），使用 **AES‑GCM** 对称加密。
   - 密钥通过环境变量 `ENCRYPTION_KEY`（Base64 编码的 32 bytes）在 `settings.py` 中读取：`ENCRYPTION_KEY = os.getenv('ENCRYPTION_KEY')`。
   - 简化实现示例：
     ```python
     from Crypto.Cipher import AES
     import base64, os
     from django.conf import settings

     def encrypt(text: str) -> str:
         key = base64.b64decode(settings.ENCRYPTION_KEY)
         nonce = os.urandom(12)
         cipher = AES.new(key, AES.MODE_GCM, nonce=nonce)
         ct, tag = cipher.encrypt_and_digest(text.encode())
         return base64.b64encode(nonce + tag + ct).decode()

     def decrypt(cipher_b64: str) -> str:
         data = base64.b64decode(cipher_b64)
         nonce, tag, ct = data[:12], data[12:28], data[28:]
         key = base64.b64decode(settings.ENCRYPTION_KEY)
         cipher = AES.new(key, AES.MODE_GCM, nonce=nonce)
         return cipher.decrypt_and_verify(ct, tag).decode()
     ```
2. **模型层自动加密**
   - 在 `common/models.py` 中提供 `EncryptedFieldsMixin`：
     ```python
     class EncryptedFieldsMixin(models.Model):
         encrypted_fields = []  # 子类声明需要加密的字段名列表
         def save(self, *args, **kwargs):
             for f in self.encrypted_fields:
                 val = getattr(self, f)
                 if val and not str(val).startswith('enc:'):
                     setattr(self, f, 'enc:' + encrypt(val))
             super().save(*args, **kwargs)
         def decrypt(self, field):
             val = getattr(self, field)
             if val and str(val).startswith('enc:'):
                 return decrypt(val[4:])
             return val
         class Meta:
             abstract = True
     ```
   - `AIModelConfig` 与 `AiQuotaPolicy` 继承该 Mixin 并声明 `encrypted_fields = ['api_key']`、`['quota_secret']`，实现写入自动加密、读取时通过 `instance.decrypt('api_key')` 获得明文。
3. **环境切换**
   - `settings.py` 增加 `ENVIRONMENT = os.getenv('APP_ENV', 'dev')`，支持 `dev / test / prod` 三套配置。
   - 启动脚本 `start-all.bat` 通过 `set APP_ENV=prod` 决定使用的数据库、缓存、日志级别。
   - 提供管理 API `POST /api/sys/switch-env/`（`SwitchEnvAPIView`），仅系统管理员可调用，内部更新 `settings.ENVIRONMENT` 并使用 `os.execv` 重启进程，实现运行时切换。
4. **密钥轮转**
   - 新增 `POST /api/sys/rotate-key/`（`RotateEncryptionKeyAPIView`），步骤如下：
     1. 读取当前 `ENCRYPTION_KEY`。
     2. 生成新密钥 `new_key = base64.b64encode(os.urandom(32)).decode()`。
     3. 在事务中遍历所有继承 `EncryptedFieldsMixin` 的模型表，先使用旧密钥 `decrypt` 获得明文，再用 `new_key` 加密并写回。
     4. 更新 `settings.ENCRYPTION_KEY`（写入 `.env` 或配置中心）。
   - 若任一步骤抛异常，事务回滚，确保业务不受影响。

## II. 数据库全解析 (Database Schema & Visibility)

| 表 | 加密字段 | 加密方式 | 可见性 |
|---|----------|----------|--------|
| `user_ai_modelconfig` | `api_key` | AES‑GCM (Base64) | **后端私有** |
| `user_ai_quotapolicy` | `quota_secret` | AES‑GCM (Base64) | **后端私有** |
| `assistant_ai_usageevent` | `model_type` | — | 前端只读 |

> 所有加密列标记 **[后端私有]**，通过 `EncryptedFieldsMixin` 在 ORM 层透明加解密，API 永不返回密文。

## III. 交互生命周期 (Interaction Lifecycle)

| 前端操作 | API | 后端业务 | 响应 |
|----------|-----|----------|------|
| **新增/更新 AI 配置** | `POST /api/sys/ai-config/` | `serializer.save()` → `EncryptedFieldsMixin.save()` 自动加密 `api_key` | `ApiResponse.success(message='已保存')` |
| **轮转密钥** | `POST /api/sys/rotate-key/` | 读取旧密钥 → 生成新密钥 → 事务遍历加密模型解密后再加密 → 提交 | `ApiResponse.success(message='密钥已成功轮转')` |
| **获取配置（管理员）** | `GET /api/sys/ai-config/`（管理员） | `instance.decrypt('api_key')` 返回明文，仅在后台页面使用 | `{"model_type":"openai","api_key":"sk-...","base_url":"..."}` |
| **切换运行环境** | `POST /api/sys/switch-env/` | 权限校验 → 修改 `settings.ENVIRONMENT` → `os.execv` 重启进程 | `ApiResponse.success(message='已切换至 prod')` |

## IV. 前端呈现与 UI 规范 (Frontend & UI Spec)
- **系统设置 – 环境切换**：使用 **Tabs**（开发、测试、生产），每个 Tab 展示对应的日志级别、数据库 URL、缓存地址。切换后弹出确认对话框，调用 `api/switchEnv`，成功后页面自动刷新。
- **加密配置编辑**：管理员面板使用 `a-input-password`，右侧提供眼睛图标切换明文/密文，保存时直接 `POST`，后端负责加密。
- **密钥轮转按钮**：红色 `a-button type="primary" danger`，二次确认后调用 `api/rotateKey`，使用 `nprogress` 显示进度条，完成后弹出绿色 toast。
- **主题统一**：卡片背景 `#0a0f1f`，边框 `#1f8ef1`，按钮渐变 `linear-gradient(45deg, #1f8ef1, #00bfff)`，保持 **Cyberpunk Blue** 风格。
