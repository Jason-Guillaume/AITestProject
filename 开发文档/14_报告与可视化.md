# 报告与可视化模块（reports）

## 功能概述
`reports` 目录用于保存 **自动化生成的 PDF/HTML 报告、质量仪表盘快照、统计图表**，这些报告往往由后端的 `GeneratedTestArtifact`、`QualityDashboardView` 或 `TestReport` 等接口生成，供运维、产品或客户查看。

## 主要文件类型
| 文件后缀 | 说明 |
|----------|------|
| `.pdf` | PDF 格式的完整报告，适合离线阅读与分享。 |
| `.html` | HTML 报告，可直接在浏览器打开，通常包含交互式图表。 |
| `.png` / `.jpg` | 报告中的图表或截图。 |

## 生成流程概述
1. **数据聚合**：后端视图（如 `QualityDashboardView`）统计缺陷密度、通过率、性能指标等，并返回 JSON 数据。
2. **渲染模板**：使用 Django `templates/` 中的 Jinja‑like 模板渲染为 HTML。
3. **转换为 PDF**：通过 **WeasyPrint**、**wkhtmltopdf** 或 **ReportLab** 将 HTML 转为 PDF，写入 `reports/` 目录。
4. **存储与访问**：生成的文件路径写入对应模型（如 `GeneratedTestArtifact.file_path`），前端通过下载接口获取。

## 示例代码片段（后端生成 PDF）
```python
from django.template.loader import render_to_string
from weasyprint import HTML
import os

def generate_quality_report(user, data):
    html_content = render_to_string('reports/quality_report.html', {'data': data, 'user': user})
    output_path = os.path.join(settings.BASE_DIR, 'reports', f'quality_{user.id}_{int(time.time())}.pdf')
    HTML(string=html_content).write_pdf(output_path)
    return output_path
```

## 常见错误与排查
1. **字体缺失**：WeasyPrint 渲染 PDF 时找不到系统字体导致文字缺失，确保在服务器上安装常用字体（如 `DejaVuSans`）。
2. **相对路径失效**：模板中引用图片时使用 `static` 或绝对路径，否则在 PDF 渲染阶段找不到资源。
3. **文件权限**：生成的 PDF 写入目录需可写权限，检查 `reports/` 目录的 `chmod 775` 并确保运行用户拥有写入权限。
4. **性能瓶颈**：大量并发生成报告时建议使用 Celery 异步任务，避免阻塞 API 响应。
