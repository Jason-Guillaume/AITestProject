# 00-项目总览

## 项目简介

SmartTest 是一个面向软件测试全流程的 AI 辅助平台，集成了测试用例管理、缺陷跟踪、性能测试、测试计划执行等核心功能，并通过大模型技术提供智能化的测试用例生成与分析能力。

### 核心定位

- **全流程覆盖**：从测试设计、用例编写、计划执行到缺陷管理，形成完整闭环
- **AI 赋能**：基于大模型的测试用例自动生成、RAG 相关性检索、去重优化
- **性能测试**：支持 JMeter/Locust/k6 场景的压测任务管理与执行
- **组织协作**：多组织架构、角色权限、系统消息与审批流程

## 快速启动

### 后端安装

```bash
# 克隆项目
cd D:\AITestProduct

# 创建并激活虚拟环境（推荐）
python -m venv venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 执行数据库迁移
python manage.py migrate

# （可选）创建超级用户
python manage.py createsuperuser

# （可选）加载性能任务演示数据
python manage.py seed_perf_tasks
```

**Windows 特殊说明**：
若 `mysqlclient` 安装失败，可改用 `pymysql`：
```bash
pip uninstall mysqlclient -y
pip install pymysql
```
并在 `AITestProduct/__init__.py` 中添加：
```python
import pymysql
pymysql.install_as_MySQLdb()
```

### 前端安装

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

默认访问地址：`http://localhost:5173`

### 本地启动步骤

#### 启动后端

```bash
# 确保虚拟环境已激活
python manage.py runserver
```

后端服务地址：`http://127.0.0.1:8000`

#### 启动前端

```bash
cd frontend
npm run dev
```

前端服务地址：`http://localhost:5173`

**代理配置**（`vite.config.js`）：
- `/api` → `http://127.0.0.1:8000`
- `/media` → `http://127.0.0.1:8000`
- `/ws` → `http://127.0.0.1:8000`（WebSocket 代理）

## AI 模块能力

### 已实现能力

- **连通性校验**：
  - `/api/ai/verify-connection/`：固定模型快速校验
  - `/api/ai/test-connection/`：自定义 base_url/model/api_key 校验
- **生成用例（同步）**：`POST /api/ai/generate-cases/`
- **生成用例（流式 SSE）**：`POST /api/ai/generate-cases-stream/`
- **失败执行修订建议**：`POST /api/ai/suggest-case-fix/`
- **RAG（知识库/相似用例检索）**：按模块检索 Top-K
- **去重**：字符串去重 + 语义去重
- **配置治理**：系统级 AI 配置 + 请求体临时覆盖

---

## 服务器日志模块能力

- **远程主机配置**（REST CRUD）
  - 密码/私钥加密存储（Fernet 密文）
  - 支持绑定组织（组织创建人 + members 可共享）
- **实时日志查看**（WebSocket）
  - 后端发起 SSH，执行 `tail -f` / `Get-Content -Wait`
  - 前端终端 UI 虚拟滚动
- **AI 诊断**
  - 前端划选日志 → `POST /api/server-logs/analyze/` 返回 Markdown
- **审计**
  - WS connect/disconnect/stop、主机 CRUD、AI 诊断均落库
- **安全**
  - 路径校验防注入
  - WS/REST 权限范围一致

## 核心业务流程

### 1. AI 生成用例流程
```
需求文档 → 知识库 RAG → AI 生成 → 去重 → 人工审核 → 入库
```

### 2. 测试执行流程
```
用例选择 → 环境配置 → 触发执行 → WebSocket 推送 → 结果记录
```

### 3. 日志分析流程
```
SSH 采集/ES 检索 → AI 分析 → 工单草稿 → 人工确认 → 创建缺陷
```

### 4. 性能测试流程
```
API 用例 → AI 转 k6 脚本 → 执行压测 → 指标采集 → 报告生成
```
