# 内存与缓存模块（memory）

## 功能概述
提供运行时缓存、分布式锁、任务状态存储等功能，支撑 Celery 调度、K6 实时指标以及跨进程数据共享。主要使用 **Redis** 作为后端存储，也提供 **本地内存（LRUCache）** 作为轻量 fallback。

## 使用技术
- **Redis**：键值存储，支持 TTL、发布/订阅（PubSub）用于 WebSocket 推送与任务锁。
- **django-redis**：在 Django `CACHES` 配置中集成，统一使用 `cache` 接口。
- **python‑lru‑cache**（或自实现的 `functools.lru_cache`）：用于函数级缓存，避免重复计算。

## 关键代码文件及片段
### 缓存配置 (`settings.py` 示例)
```python
CACHES = {
    "default": {
        "BACKEND": "django_redis.cache.RedisCache",
        "LOCATION": "redis://127.0.0.1:6379/1",
        "OPTIONS": {
            "CLIENT_CLASS": "django_redis.client.DefaultClient",
        },
    }
}
```

### 分布式锁 (`memory/lock.py` 示例)
```python
import redis
from contextlib import contextmanager

_client = redis.StrictRedis.from_url("redis://127.0.0.1:6379/1")

@contextmanager
def distributed_lock(name: str, timeout: int = 30):
    lock = _client.lock(name, timeout=timeout)
    acquired = lock.acquire(blocking=True)
    try:
        if acquired:
            yield
        else:
            raise TimeoutError("Unable to acquire lock")
    finally:
        if acquired:
            lock.release()
```

### 任务状态缓存 (`memory/task_state.py` 示例)
```python
from django.core.cache import cache

def set_task_state(task_id: str, state: str, ttl: int = 3600):
    cache.set(f"task_state:{task_id}", state, timeout=ttl)

def get_task_state(task_id: str) -> str | None:
    return cache.get(f"task_state:{task_id}")
```

## 为什么使用这些技术
- **Redis** 提供持久化、TTL、原子操作，非常适合分布式锁和跨进程状态同步。
- **django-redis** 让缓存直接通过 Django `cache` 接口使用，统一编码风格。
- **LRUCache** 对于不需要持久化的轻量计算结果，能够在内存中快速命中，降低 Redis 访问频率。

## 常见错误与排查
1. **Redis 连接异常**：检查 `settings.py` 中 `CACHES['default']['LOCATION']` 是否可达，使用 `redis-cli ping` 验证。
2. **锁未释放**：业务异常导致 `finally` 未执行，建议在 Celery `task_failure` 信号中手动清理锁。
3. **缓存失效**：TTL 设置过短导致频繁重新计算，依据业务需求适当延长 TTL。
