## 🔍 发现的问题

### 问题1: 认证问题
- **现象**: 返回 404 错误
- **原因**: 使用原生 `axios` 而不是项目的 `request` 实例
- **影响**: 请求未携带认证 Token

### 问题2: 字段验证问题
- **现象**: 返回 `file_path: ["该路径没有任何文件。"]` 和 `entry_point: ["该字段是必填项。"]`
- **原因**: 
  1. 模型字段 `file_path` 和 `entry_point` 未设置 `blank=True, null=True`
  2. FormData 字段名错误（使用了 `file` 而不是 `file_path`）
  3. LINEAR 类型不需要 entry_point 但模型要求必填

### 问题3: 序列化器验证逻辑不完善
- **原因**: 缺少针对不同脚本类型的验证逻辑
- **影响**: 无法正确处理 LINEAR 和 POM 两种模式的不同需求

## 🔧 修复方案

### 1. 修复模型字段（assistant/models.py）

**修复前**:
```python
file_path = models.FileField(
    upload_to='ui_scripts/%Y/%m/%d/',
    validators=[FileExtensionValidator(allowed_extensions=['py', 'zip'])],
    verbose_name='脚本文件',
    help_text='单文件(.py)或ZIP包'
)

entry_point = models.CharField(
    max_length=500,
    verbose_name='执行入口路径',
    help_text='脚本执行的入口文件路径，如: main.py 或 tests/test_login.py'
)
```

**修复后**:
```python
file_path = models.FileField(
    upload_to='ui_scripts/%Y/%m/%d/',
    validators=[FileExtensionValidator(allowed_extensions=['py', 'zip'])],
    blank=True,
    null=True,
    verbose_name='脚本文件',
    help_text='单文件(.py)或ZIP包'
)

entry_point = models.CharField(
    max_length=500,
    blank=True,
    null=True,
    verbose_name='执行入口路径',
    help_text='脚本执行的入口文件路径，如: main.py 或 tests/test_login.py'
)
```

### 2. 修复序列化器验证（assistant/serialize.py）

**修复前**:
```python
def validate(self, attrs):
    """交叉字段验证"""
    script_type = attrs.get('script_type')
    file_path = attrs.get('file_path')

    if file_path:
        ext = os.path.splitext(file_path.name)[1].lower()

        if script_type == UIScriptUpload.ScriptType.LINEAR and ext == '.zip':
            pass

        if script_type == UIScriptUpload.ScriptType.POM and ext == '.py':
            pass

    return attrs
```

**修复后**:
```python
def validate(self, attrs):
    """交叉字段验证"""
    script_type = attrs.get('script_type')
    file_path = attrs.get('file_path')
    git_repo_url = attrs.get('git_repo_url')
    entry_point = attrs.get('entry_point')

    # LINEAR 类型：必须提供文件
    if script_type == UIScriptUpload.ScriptType.LINEAR:
        if not file_path:
            raise serializers.ValidationError({
                'file_path': '线性脚本必须上传文件'
            })
        # LINEAR 类型的 entry_point 可以为空，默认使用上传的文件名
        if not entry_point and file_path:
            attrs['entry_point'] = os.path.basename(file_path.name)

    # POM 类型：必须提供 entry_point，文件或 Git URL 二选一
    if script_type == UIScriptUpload.ScriptType.POM:
        if not entry_point:
            raise serializers.ValidationError({
                'entry_point': 'POM 模式必须指定执行入口点'
            })
        if not file_path and not git_repo_url:
            raise serializers.ValidationError({
                'file_path': 'POM 模式必须上传 ZIP 文件或提供 Git 仓库 URL'
            })

    return attrs
```

**同时修复 validate_file_path**:
```python
def validate_file_path(self, value):
    """验证文件类型和大小"""
    if not value:
        return value

    max_size = 50 * 1024 * 1024
    if value.size > max_size:
        raise serializers.ValidationError(
            f"文件大小不能超过50MB，当前大小: {value.size / 1024 / 1024:.2f}MB"
        )

    ext = os.path.splitext(value.name)[1].lower()
    if ext not in ['.py', '.zip']:
        raise serializers.ValidationError(
            f"不支持的文件类型: {ext}，仅支持 .py 和 .zip"
        )

    return value
```

### 3. 修复前端字段名（frontend/src/views/UiAutomationWorkbench.vue）

**修复前**:
```javascript
formData.append('file', linearConfig.value.file);
```

**修复后**:
```javascript
formData.append('file_path', linearConfig.value.file);
```

### 4. 修复前端错误处理

**修复前**:
```javascript
catch (error) {
  ElMessage.error(error.response?.data?.detail || '上传失败');
}
```

**修复后**:
```javascript
catch (error) {
  const errorMsg = error.response?.data?.file_path?.[0]
    || error.response?.data?.entry_point?.[0]
    || error.response?.data?.detail
    || '上传失败';
  ElMessage.error(errorMsg);
}
```

### 5. 修复认证问题

**修复前**:
```javascript
import axios from 'axios';
await axios.post('/api/assistant/ui-scripts/', formData);
```

**修复后**:
```javascript
import request from '@/utils/request';
await request.post('/assistant/ui-scripts/', formData);
```
