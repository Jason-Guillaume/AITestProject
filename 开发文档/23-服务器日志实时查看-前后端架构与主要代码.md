## 1. 功能范围（当前代码已实现）

- **SSH 主机配置管理（REST）**
  - 支持 Linux/Windows（Windows 侧命令为 PowerShell `Get-Content -Wait`）
  - **密码/私钥**落库前使用 **Fernet** 对称加密（密文存储）
  - 支持绑定 **Organization**，实现组织创建人 + 成员共享
- **实时日志查看（WebSocket + SSH tail）**
  - 浏览器建立 WebSocket
  - 后端消费 WS 后在服务端发起 SSH 连接并执行 `tail -f`（或 Windows 命令）
  - 后端将远端输出逐行推送到前端终端
  - 支持 `stop` 消息停止 tail
- **划选日志一键 AI 诊断（REST）**
  - 前端选中文本后调用分析接口
  - 后端复用平台 AI 配置（也支持请求体临时覆盖 api_key/base/model）
  - 诊断结果为 Markdown，前端 Drawer 渲染
- **（可选）历史检索（Loki）**
  - 配置 `SERVER_LOGS_LOKI_BASE` 后转发 `query_range`
- **审计（DB）**
  - 主机 CRUD、WS 连接/断开/stop、AI 诊断、历史检索均写审计

## 3. 数据流与关键契约

### 3.1 REST 数据流（主机配置）

1. 前端弹窗保存 → `POST /api/server-logs/hosts/`
2. 后端 `RemoteLogServerSerializer`：
   - 创建时要求 **密码/私钥至少一个**
   - `default_log_path` 做路径校验
3. 模型 `RemoteLogServer`：
   - `set_password` / `set_private_key` → Fernet 密文写库
4. 写审计：`HOST_CREATE / HOST_UPDATE / HOST_DELETE`

### 3.2 WebSocket 数据流（实时日志）

- **连接 URL（前端拼接）**：
  - `ws(s)://<前端host>/ws/server-logs/<server_id>/?token=<DRF Token>&path=<urlencoded>`
  - 代码：`frontend/src/views/server_logs/LogTerminal.vue` → `wsUrl()`

- **鉴权方式**：
  - ASGI 中间件 `execution/middleware_ws.py:TokenAuthMiddleware`
  - 从 querystring 解析 `token` → 写入 `scope["user"]`

- **Consumer 行为**（`server_logs/consumers.py:LogViewerConsumer`）：
  - 加载主机配置（含解密后的 password/private_key）
  - 校验 `path`（非法时：先 accept → 发送 `{"type":"error"}` → close）
  - 启动线程 `ssh_tail_worker(...)`，并创建队列
  - 协程任务 `_pump_queue()` 从队列读 `(line|hb|error|done)` 并推送到浏览器
  - **重要（稳定性修复）**：当队列收到 `error` 或 `done` 时，Consumer 会主动关闭 WebSocket，避免开发环境 reload 时残留任务导致 daphne 报 “took too long to shut down” 进而引发全站 500。

- **消息类型（服务端 → 前端）**：
  - `{"type":"status", "phase": "...", "message": "..."}`
  - `{"type":"log", "line": "..."}`
  - `{"type":"heartbeat"}`
  - `{"type":"error", "message": "..."}`
  - `{"type":"pong", "ts": ...}`

- **消息类型（前端 → 服务端）**：
  - `{"type":"ping","ts":...}`
  - `{"type":"stop"}`：服务端置位 stop_event，尽快停止 tail

### 3.3 SSH 执行（服务端发起）

> 重要：SSH 连接由**运行 Django/ASGI 的机器**发起，不是浏览器直连目标服务器。

- 入口：`server_logs/ssh_tail.py:ssh_tail_worker`
  - `paramiko.SSHClient.connect(...)`
  - Linux：执行 `tail -n 200 -f <path>`（路径经 `shlex.quote`）
  - Windows：执行 `powershell Get-Content -Tail 200 -Wait ...`
  - 用 `channel.settimeout(recv_timeout)` 实现读超时心跳
  - 队列背压：日志行优先 `put_nowait`，队列满则丢弃单行（避免阻塞 SSH 读线程）

## 6. 运行与联调检查点（与“需求一致性”相关）

### 6.1 必须使用 ASGI（否则 WebSocket 不工作）

开发建议：

```powershell
cd "d:\高四\AITestProduct"
.\.venv\Scripts\python.exe -m daphne -b 127.0.0.1 -p 8000 AITestProduct.asgi:application
```

### 6.2 前端代理（开发模式）

`frontend/vite.config.js` 已配置：

- `/api` → `http://127.0.0.1:8000`
- `/ws`（ws=true）→ `http://127.0.0.1:8000`

### 6.3 SSH 连接定位要点

- 目标服务器 `host:22` 对运行后端的机器可达（路由/防火墙/安全组）
- 若禁止 root 密码登录，需要使用**私钥**或改用普通用户
- Linux 日志路径必须为**绝对路径**且有读权限

### 6.4 失败路径收敛（用户体验与稳定性）

- SSH 不可达/认证失败/远程命令错误时：
  - 服务端会先推送 `{"type":"error","message":...}` 给前端
  - 随后主动关闭连接（WS close）
- 目的：
  - 前端不会“连着但没数据一直卡住”
  - 避免 dev server 自动重载时存在大量残留 WS 任务拖垮进程
