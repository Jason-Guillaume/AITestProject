# 02-目录结构与架构设计

## 0. 需求来源与开发动因

- 业务价值摘要：明确系统边界与分层职责，减少重复建设与架构漂移。
- 业务背景：模块持续增多（测试、执行、AI、环境、缺陷等），团队需要可追踪的架构地图支撑迭代。
- 现状痛点：目录命名与分层职责缺少统一说明时，容易出现重复实现、跨层调用混乱与维护成本上升。
- 建设目标：沉淀“目录-模块-职责”映射，明确边界与扩展点。
- 预期收益：降低重构与新功能接入风险，提升架构可维护性。

## 4. 请求流转过程

### 4.1 前端请求流程

```
用户操作
   ↓
Vue Router 路由匹配
   ↓
MainLayout 布局渲染
   ↓
对应 View 组件加载
   ↓
调用 api/xxx.js 中的方法
   ↓
request.js 拦截器注入 Token
   ↓
Axios 发送请求到 /api/xxx
   ↓
Vite 代理转发到 http://127.0.0.1:8000
   ↓
后端 Django 处理
   ↓
响应返回前端
   ↓
数据渲染到页面
```

### 4.2 后端请求流程

```
HTTP Request
   ↓
Django Middleware (Security, Session, CSRF等)
   ↓
主路由 AITestProduct/urls.py
   ↓
子应用路由分发 (user/project/testcase/...)
   ↓
ViewSet 视图集处理
   ↓
Serializer 数据验证与序列化
   ↓
Model ORM 查询/保存
   ↓
Database (MySQL/Redis/ChromaDB)
   ↓
响应数据序列化
   ↓
返回 JSON 响应
```

### 4.3 登录认证流程

```
POST /api/user/login/
   ↓
UserViewSet.login() 验证用户名密码
   ↓
验证成功 → 生成/获取 Token
   ↓
返回 { token: "xxx", user: {...} }
   ↓
前端存储 token 到 localStorage
   ↓
后续请求自动携带 Authorization: Token xxx
   ↓
DRF TokenAuthentication 验证 Token
   ↓
验证通过 → 执行视图逻辑
   ↓
验证失败 → 返回 401
   ↓
前端拦截器清除 token 并跳转登录页
```

## 6. 核心业务流程

### 6.1 测试用例管理流程

```
创建测试用例
   ↓
选择测试类型与模块
   ↓
填写用例基本信息(P0-P3等级)
   ↓
保存并生成唯一编号(case_number)
   ↓
(可选) 添加执行步骤
   ↓
(可选) 关联测试设计
   ↓
提交评审
   ↓
评审通过后归档
```

### 6.2 测试执行流程

```
选择测试用例
   ↓
点击执行按钮
   ↓
生成唯一 trace_id
   ↓
发送请求到被测接口
   ↓
记录请求/响应数据
   ↓
执行断言规则
   ↓
保存执行日志(api_execution_log)
   ↓
更新用例执行次数
   ↓
生成测试报告
```

### 6.3 缺陷管理流程

```
发现缺陷
   ↓
创建缺陷记录
   ↓
填写缺陷标题与内容
   ↓
选择严重程度与优先级
   ↓
指派处理人
   ↓
处理人处理缺陷
   ↓
验证修复结果
   ↓
关闭缺陷或重新打开
```

### 6.4 AI用例生成流程

```
用户输入需求描述
   ↓
调用 AI 生成接口
   ↓
LLM 生成测试用例草稿
   ↓
(可选) RAG 检索相关用例
   ↓
(可选) 用例去重
   ↓
返回生成的用例列表
   ↓
用户编辑并保存
```

## 7. 技术架构特点

### 7.1 前端技术特点

| 特性 | 说明 |
|------|------|
| 组件化 | Vue 3 Composition API，高复用性组件 |
| 响应式 | Tailwind CSS + Sass，响应式布局 |
| 状态管理 | localStorage 存储 Token 与用户信息 |
| 路由守卫 | 前端权限控制，未登录自动跳转 |
| 请求拦截 | 统一 Token 注入与 401 处理 |
| 图表可视化 | ECharts 数据可视化展示 |

### 7.2 后端技术特点

| 特性 | 说明 |
|------|------|
| RESTful API | DRF 标准化接口设计 |
| Token 认证 | 基于 TokenAuthentication 的无状态认证 |
| 软删除 | BaseModel 统一 is_deleted 字段 |
| 多表继承 | TestCase 的 API/性能/安全/UI 子类 |
| 缓存降级 | Redis 不可用时自动降级为内存缓存 |
| 流式响应 | SSE 支持 AI 用例流式生成 |
| 向量检索 | ChromaDB 支持用例 RAG 检索 |

### 7.3 安全特性

| 特性 | 说明 |
|------|------|
| 密码哈希 | Django 默认 PBKDF2 算法 |
| Token 认证 | DRF TokenAuthentication |
| CSRF 防护 | Django CSRF 中间件 |
| SQL 注入防护 | ORM 参数化查询 |
| 软删除 | 防止数据误删 |
| 审批流程 | 敏感操作需管理员审批 |
