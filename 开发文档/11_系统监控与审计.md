# 系统监控与审计（common & user）

## 功能概述
提供全局审计日志、AI 使用计量、系统消息、配额管理等监控与审计能力，帮助运维追踪关键业务操作、统计模型消耗、及时推送系统通知。

## 关键组件
| 组件 | 所在文件 | 作用 |
|------|----------|------|
| **AuditEvent** | `common/models.py` | 统一审计模型，记录用户、时间、操作对象及字段变化。
| **AiUsageEvent** | `assistant/models.py` | 记录每次调用 LLM 的模型、token 使用量、耗时，用于计费与性能分析。
| **AiPatch** | `assistant/models.py` | 保存对生成用例的补丁信息，支持回滚与审计。
| **AiCaseGenerationRun** | `assistant/models.py` | 一次 AI 用例生成的上下文（需求、模型、耗时），用于调试和统计。
| **SystemMessage** | `user/models.py` | 系统级通知（信息、警告、错误），可通过 API 查询并标记已读。
| **AiQuotaPolicy** | `user/models.py` | 为组织或用户设定每日 AI 使用配额，配合 `AiUsageEvent` 实时校验。

### 配额检查函数示例 (`assistant/views.py`或utils)
```python
def check_quota(user, requested_tokens: int):
    policy = AiQuotaPolicy.objects.filter(user=user).first()
    if not policy:
        return  # 没有配额限制
    used = AiUsageEvent.objects.filter(user=user,
                                        created_at__date=timezone.now().date()).aggregate(
        total=Sum('token_used'))['total'] or 0
    if used + requested_tokens > policy.daily_quota:
        raise PermissionDenied("AI 使用配额已耗尽")
```

### 系统消息 API 示例 (`user/views.py`)
```python
class SystemMessageListAPIView(APIView):
    def get(self, request):
        msgs = SystemMessage.objects.filter(user=request.user)
        return ApiResponse.success(SystemMessageSerializer(msgs, many=True).data)

class SystemMessageMarkReadAPIView(APIView):
    def patch(self, request, pk):
        msg = get_object_or_404(SystemMessage, pk=pk, user=request.user)
        msg.read = True
        msg.save()
        return ApiResponse.success(message="已标记已读")
```

## 为什么使用这些技术
- **统一审计模型**：所有关键业务（创建缺陷、发布计划、AI 生成）都可统一记录，满足合规审计需求。
- **AI 使用计量 + 配额**：为 SaaS 场景提供计费依据，防止模型滥用，配合 Celery 异步记录确保不阻塞业务。
- **系统消息**：提供实时通知渠道，配合前端 WebSocket 可实现弹窗提醒。
- **JSON 存储**：`JSONField` 便于 ELK/Loki 统一收集、搜索与可视化。

## 常见错误与排查
1. **审计日志未写入**：确认 `common/signals.py` 中的 `post_save`、`post_delete` 信号已在 `AssistantConfig.ready()` 或 `CommonConfig.ready()` 中导入。
2. **配额检查失效**：`AiQuotaPolicy` 必须关联到对应用户或组织，且 `daily_quota` 不为 `null`，否则视为无限制。
3. **系统消息查询为空**：前端请求未携带有效 `Authorization`，导致 `request.user` 为匿名，过滤掉消息。检查 JWT 传递。
4. **查询性能差**：`AiUsageEvent` 记录量大时，在 `user_id`、`model_type`、`created_at` 上建立复合索引，以加速配额统计。
