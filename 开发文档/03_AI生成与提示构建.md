# AI 生成与提示构建（assistant）

## 功能概述
根据业务需求或 API 规格，自动生成测试用例、补丁或安全用例。核心步骤为：Prompt 组装 → 调用大语言模型 (LLM) → 结果清洗 → 去重 → 持久化。

## 使用技术
- **LLM（OpenAI / Zhipu）**：提供自然语言或结构化 JSON 的生成能力。
- **Prompt Engineering**：在 `ai_generate_strategies.py` 中构建系统 Prompt、用户 Prompt、约束条件等，使 LLM 能够准确理解需求。
- **JSON 解析与清洗**：`api_case_generation.py` 中的 `_coerce_*`、`sanitize_ai_raw_item_for_apitestcase` 等函数统一把 LLM 输出转换为项目模型需要的结构。
- **语义去重**：`generated_case_dedup.py` 通过编辑距离、Jaccard、余弦相似度等算法消除高度相似的用例。
- **Celery**：异步执行生成任务，防止 API 调用阻塞。

## 关键代码文件及片段
### Prompt 构建 (`assistant/ai_generate_strategies.py`)
```python
class AiPromptContext:
    def __init__(self, requirement: str, api_spec: str = ""):
        self.requirement = requirement
        self.api_spec = api_spec

def build_system_prompt_for_generate(ctx: Dict[str, Any]) -> str:
    """构造系统 Prompt，明确 AI 的角色、输出格式以及约束条件。"""
    return (
        "You are an AI assistant that generates test cases. "
        f"Requirement: {ctx.get('requirement')} "
        f"API Spec: {ctx.get('api_spec', '')}"
    )
```

### 生成核心 (`assistant/api_case_generation.py`)
```python
def _coerce_headers_for_api_model(val: Any) -> Dict[str, str]:
    # 将用户提供的 Header 转换为统一的字符串字典，过滤空值
    return {k: str(v) for k, v in (val or {}).items() if v is not None}

def _coerce_body_for_api_model(val: Any) -> Union[Dict, List, str, None]:
    if isinstance(val, (dict, list)):
        return val
    return str(val) if val is not None else None

def sanitize_ai_raw_item_for_apitestcase(raw: Any) -> Dict[str, Any]:
    """把 LLM 返回的原始 JSON 转换为 `ApiTestCase` 所需结构。"""
    return {
        "name": raw.get("name"),
        "request": {
            "method": raw.get("method", "GET"),
            "url": raw.get("url"),
            "headers": _coerce_headers_for_api_model(raw.get("headers")),
            "body": _coerce_body_for_api_model(raw.get("body")),
        },
        "assertions": raw.get("assertions", []),
    }
```

### 去重实现 (`assistant/generated_case_dedup.py`)
```python
def normalized_levenshtein_similarity(s1: str, s2: str) -> float:
    """返回 0~1 之间的相似度，1 表示完全相同。"""
    distance = _levenshtein_distance(s1, s2)
    max_len = max(len(s1), len(s2))
    return 1 - distance / max_len if max_len else 1.0

def jaccard_word_similarity(s1: str, s2: str) -> float:
    set1, set2 = set(s1.split()), set(s2.split())
    intersection = len(set1 & set2)
    union = len(set1 | set2)
    return intersection / union if union else 0.0
```

## 为什么使用这些技术
- **Prompt Engineering** 让同一 LLM 可以服务多种业务场景（API、功能、安全），无需切换模型。
- **统一的 JSON 清洗** 解决了 LLM 输出不规范、缺失字段的问题，确保后端序列化不会异常。
- **语义去重** 防止生成海量相似用例，降低存储与后续执行成本。
- **Celery** 提供可靠的异步执行与重试机制，提升系统可用性。

## 常见错误与排查
1. **LLM 调用失败**：检查 `settings.py` 中的 `OPENAI_API_KEY`、`ZHIPU_API_KEY` 是否有效，网络是否被代理或防火墙阻断。
2. **返回 JSON 结构缺失关键字段**：`sanitize_ai_raw_item_for_apitestcase` 报 `KeyError`，在 Prompt 中明确要求必须返回 `name`, `url`, `method` 等字段。
3. **去重阈值设置不当**：阈值过低会误删不同用例，阈值过高会保留大量冗余，用 `similarity_threshold` 参数在 `deduplicate_generated_cases` 中调节。
4. **Celery 任务超时或未执行**：确认 Celery worker 已启动 (`celery -A AITestProduct worker -l info`)，并检查消息中间件（Redis/RabbitMQ）状态。
5. **异常未统一返回**：若未捕获 `BusinessException`，API 会返回 500，确保全局异常处理器已在 `settings.py` 中配置 `REST_FRAMEWORK['EXCEPTION_HANDLER']`。
