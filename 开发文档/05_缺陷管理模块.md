# 缺陷管理模块（defect）

## 功能概述
提供缺陷（`TestDefect`）的增删改查、批量软删/硬删、回收站管理，以及从执行结果或安全扫描报告自动生成缺陷的能力。

## 使用技术
- **Django ORM**：`TestDefect` 继承自 `BaseModel`，拥有审计字段并支持软删。
- **Django REST Framework (DRF) ViewSet**：`defect/views.py` 实现标准 CRUD、分页 (`DefectPagination`) 与自定义批量操作。
- **自定义分页**：`DefectPagination` 为缺陷列表提供页面大小控制。
- **Celery（可选）**：`DefectFromExecutionAPIView`、`DefectFromSecurityFindingAPIView` 通过异步任务写入外部系统产生的缺陷，防止阻塞主请求。

## 关键代码文件及片段
### 模型 (`defect/models.py`)
```python
class TestDefect(BaseModel):
    defect_no = models.CharField(max_length=64, unique=True)
    description = models.TextField()
    severity = models.CharField(
        max_length=20,
        choices=[("low", "Low"), ("high", "High")],
    )
    related_test_case = models.ForeignKey(
        'assistant.UITestCase', null=True, on_delete=models.SET_NULL
    )
    class Meta:
        db_table = "test_defect"
```

### 序列化器 (`defect/serialize.py`)
```python
class TestDefectSerializer(BaseModelSerializers):
    def validate_defect_no(self, value):
        if TestDefect.objects.filter(defect_no=value).exists():
            raise serializers.ValidationError("缺陷编号已存在")
        return value

    class Meta:
        model = TestDefect
        fields = "__all__"
```

### 视图集 (`defect/views.py`)
```python
class TestDefectViewSet(BaseModelViewSet):
    queryset = TestDefect.objects.all()
    serializer_class = TestDefectSerializer
    pagination_class = DefectPagination

    @action(detail=False, methods=["post"], url_path="bulk_soft_delete")
    def bulk_soft_delete(self, request):
        ids = self._parse_id_list(request.data.get("ids", []))
        TestDefect.objects.filter(id__in=ids).update(is_deleted=True)
        return ApiResponse.success(message="已软删")

    @action(detail=False, methods=["post"], url_path="bulk_hard_delete")
    def bulk_hard_delete(self, request):
        ids = self._parse_id_list(request.data.get("ids", []))
        TestDefect.objects.filter(id__in=ids).delete()
        return ApiResponse.success(message="已硬删")
```

### 自动生成缺陷接口示例 (`defect/views.py` 片段)
```python
class DefectFromExecutionAPIView(APIView):
    def post(self, request):
        serializer = TestDefectSerializer(data=request.data)
        serializer.is_valid(raise_exception=True)
        serializer.save()
        return ApiResponse.created(serializer.data)
```

## 为什么使用这些技术
- **统一基类**：`BaseModel` 为缺陷记录提供 `created_at`, `updated_at`, `is_deleted` 等审计字段，统一软删行为。
- **DRF ViewSet**：提供成熟的分页、过滤、权限检查，减少手写 API 工作量。
- **批量操作**：一次请求即可处理多条记录，提高前端交互效率。
- **Celery**：确保外部系统（如执行平台、漏洞扫描）回调时的写入不会阻塞业务请求，提升系统可靠性。

## 常见错误与排查
1. **缺陷编号冲突**：`validate_defect_no` 抛 `ValidationError`，前端应在提交前检查唯一性或捕获错误提示。
2. **外键关联失效**：`related_test_case` 指向的 UI 用例被删除后会置为 `NULL`，如果业务要求必须关联，需要在删除 UI 用例前做依赖检查。
3. **批量参数格式错误**：`_parse_id_list` 期望列表或逗号分割字符串，确保前端发送的 `ids` 符合该格式。
4. **Celery 任务未触发**：如果使用异步实现，确认相应 task 已在 `tasks.py` 中注册，并且 `celery -A AITestProduct worker -l info` 正在运行。
5. **软删未生效**：检查模型是否正确继承 `BaseModel`，以及视图中是否使用 `BaseModelViewSet.perform_destroy` 实现软删。
