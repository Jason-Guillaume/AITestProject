# 数据库结构与字段说明

本文档根据当前代码库（Django ORM 模型）整理：**关系型数据默认使用 MySQL 单一库**；另附 Redis、向量库等非表结构存储的用途说明。

**生成依据**：`AITestProduct/settings.py` 中 `DATABASES`、`INSTALLED_APPS`，以及各应用的 `models.py`（截至文档编写时的模型定义）。

---

## 1. 数据库总览

### 1.1 MySQL（Django `default`）

| 配置项 | 说明 |
|--------|------|
| 引擎 | `django.db.backends.mysql` |
| 库名 | 环境变量 `DB_NAME`，默认 `ai_test_product` |
| 主机/端口 | `DB_HOST`（默认 `127.0.0.1`）、`DB_PORT`（默认 `3306`） |
| 账号 | `DB_USER` / `DB_PASSWORD` |

**作用**：承载全部 Django 业务表、认证与权限表、REST Token、迁移记录、Session 等。

### 1.2 Redis（非「表」结构）

**作用**：缓存（如验证码）、Channels 实时通道、UI 脚本执行过程写日志等键值数据；**不以固定表结构体现在 MySQL 中**。

### 1.3 向量库 / Chroma（若启用 RAG）

**作用**：知识库语义检索；数据常位于项目 `data/chroma/` 等目录，**由向量数据库自身管理 schema**，不在本文逐字段展开。

## 2. 公共约定：`BaseModel`（抽象模型，不单独建表）

以下字段出现在所有**继承** `common.models.BaseModel` 的模型对应表中（字段名一致）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BigAutoField | 主键（Django 默认） |
| `creator_id` | 外键 → `sys_user` | 创建人；用户删除后置 `NULL` |
| `updater_id` | 外键 → `sys_user` | 最后更新人 |
| `create_time` | DateTime | 创建时间（自动） |
| `update_time` | DateTime | 更新时间（自动） |
| `is_deleted` | Boolean | 逻辑删除标记（业务表统一语义） |

下文若写「含 BaseModel 公共字段」，即上表所列（不再在每个表中重复列出）。

#### Web UI 工作台 / 自动化指挥中心（Web）— 数据库全景说明

本节专门说明 **Web 侧 UI 自动化**（前端路径如 `/automation-center/webui`、`/automation-center/assets`、左侧 Web 脚本树等）对应的 **MySQL 表**、字段语义及与接口的关系。**本节三张核心业务表均不继承 `BaseModel`**（无 `creator_id` / `org_id` 等公共字段），审计依赖 `assistant` 应用内既有审计钩子（若有）。

##### （1）功能地图：前端能力 ↔ 表

| 用户可见功能（前端） | 主要读写表 | 说明 |
|----------------------|------------|------|
| **Web UI 工作台**：导入脚本、树形浏览、在线编辑、保存工作区文件 | `ui_script_upload` | 一行通常对应一个「脚本工程」；上传 `.py` / `.zip` / `.jar` 写入 `file_path`，解压或落盘后写入 `workspace_path` |
| **Web UI 工作台**：运行脚本、执行日志、停止 | `ui_script_execution` | 每次执行一条记录；`execution_id` 与前端轮询、Redis 日志键关联 |
| **Web UI 工作台 / POM 报告**：查看 HTML 报告、报告列表与回收站 | `ui_pom_test_report` | 执行完成后从工作空间抽取报告文件归档；支持软删字段 |
| **自动化指挥中心 · Web 脚本树**（`/automation-center/web`） | `ui_script_upload`（经列表接口） | 懒加载树与 `GET /assistant/ui-scripts/` 等一致 |
| **资产中心**（`/automation-center/assets`） | **无独立业务表** | 后端 `build_asset_hub_overview()` 聚合 `ui_script_upload` + 磁盘工作区扫描生成「行」数据 |
| **脚本回收站 / 恢复 / 彻底删除** | `ui_script_upload` | `is_deleted`、`deleted_at`；恢复走专用 action |
| **批量启用、启用/停用切换** | `ui_script_upload` | 字段 `is_active` |
| **一键同步物理文件（资产中心）** | 仍以 `ui_script_upload` 与工作区为准 | 服务端重扫目录，**不写新表** |

**非 MySQL、但与 Web UI 执行强相关**：实时控制台日志、执行过程状态等多写入 **Redis**（见 §1.2），不在此逐键列举。

| `is_deleted` | `Boolean`，默认 `False` | 报告软删除（迁移 `assistant/0024_uipomtestreport_soft_delete.py`） |
| `deleted_at` | `DateTime`，可空 | 删除时间 |

**关系**：`UIScriptUpload` **1 : N** `UIScriptExecution`（`related_name='executions'`）；`UIScriptExecution` **1 : N** `UIPomTestReport`（`related_name='pom_reports'`）。删除脚本工程（级联）会带走执行与报告记录（具体是否物理删文件取决于视图与存储策略）。

##### （5）Media 目录约定（非表结构）

与 Web UI 相关的上传文件通常落在 Django `MEDIA_ROOT` 下，例如：

- `ui_scripts/年/月/日/`：原始上传文件；
- `workspaces/{脚本id}/`：解压后的工作区（逻辑路径与 `workspace_path` 对应）；
- `ui_pom_reports/{execution_id安全段}/`：POM 报告归档。

##### （6）与 §1.2 Redis 的分工

| 数据形态 | 存储 |
|----------|------|
| 脚本工程主数据、执行记录、报告元数据与文件索引 | **MySQL** 上表 |
| 单次执行的流式日志、进行中状态等高频读写 | **Redis**（键设计与 TTL 由运行时实现决定） |
